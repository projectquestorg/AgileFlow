# /habilidade:criar

Gere habilidades dinâmicas apoiadas em pesquisas que combinam documentação da web, seus documentos locais e padrões de base de código.

## Início rápido

```bash
# Interactive mode (recommended)
/agileflow:skill:create

# With skill name hint
/agileflow:skill:create supabase-swift
```

## O que faz

O comando __NOTRASLATE_16__ cria habilidades personalizadas pesquisando três fontes e construindo fluxos de trabalho apoiados por pesquisas:

1. **Documentação da Web** – Documentos oficiais, referências de API, tutoriais
2. **Documentação do projeto local** – Suas práticas, decisões de arquitetura
3. **Padrões de base de código** – Exemplos de códigos e convenções existentes

Em seguida, gera:
- **SKILL.md** - Arquivo de habilidade principal com gatilhos de ativação
- **references.md** - Links para todas as fontes de documentação
- **livro de receitas/** - Fluxos de trabalho passo a passo para diferentes casos de uso
- **.mcp.json** - Configuração opcional do servidor MCP

## Como funciona

### Fase 1: Requisitos

Você descreve qual habilidade você precisa:

```xml
Skill categories:
- Database integration (Supabase, PostgreSQL, MongoDB)
- API integration (REST, GraphQL, webhooks)
- UI/Frontend (Components, styling, design system)
- Development workflow (Testing, deployment, CI/CD)
- Custom description (describe what you need)
```

### Fase 2: Pesquisa

O comando entra no Modo Plano e pesquisa:

- **Documentos da Web**: URLs de documentação oficial, tutoriais, guias- **Documentos locais**: arquivos em seu diretório __NOTRASLATE_17__ que correspondem ao domínio da habilidade
- **Padrões de base de código**: código existente que mostra como você faz as coisas
- **Servidores MCP**: Ferramentas disponíveis para a tecnologia

Ele coleta URLs e padrões sem copiar conteúdo.

### Fase 3: Geração

Com base nos resultados da pesquisa, gera:

- **SKILL.md** com gatilhos de ativação e referências de livros de receitas
- **references.md** vinculando todas as fontes web/local/codebase
- diretório **cookbook/** com um arquivo por caso de uso
- **.mcp.json** se um servidor MCP foi encontrado

### Fase 4: Confirmação

Você vê a estrutura gerada, pode visualizar o conteúdo e aprovar a criação.

## Melhores práticas

### Habilidades de nomenclatura

Use nomes específicos e descritivos com hífens:

- ✅ __NOTRASLATE_19__ - Tecnologia e contexto claros
- ✅ __NOTRASLATE_20__ - Estrutura e recurso principal
- ✅ __NOTRASLATE_21__ - Domínio e foco
- ❌ __NOTRASLATE_22__ - Muito vago
- ❌ __NOTRASLATE_23__ - Muito genérico
- ❌ __NOTRASLATE_24__ - Use hífens, não sublinhados

### Criando habilidades eficazes

1. **Use domínios específicos** - Uma tecnologia por habilidade, não muito ampla
2. **Inclua vários casos de uso** - Adicione entradas do livro de receitas para diferentescenários
3. **Link para padrões de projeto** - O reference.md deve apontar para seu código real
4. **Adicionar solução de problemas** - Antecipe e resolva problemas comuns
5. **Mantenha o foco** - 2 a 4 entradas de livros de receitas por habilidade

### Quando criar novas habilidades

Crie uma habilidade quando:
- Você usa uma tecnologia com frequência
- Sua equipe tem padrões para alguma coisa
- Você deseja codificar as melhores práticas
- Você precisa de orientação passo a passo

Não crie se:
- O tópico é muito restrito (uma entrada no livro de receitas)
- Você raramente usa
- Documentos externos são suficientes

## Integração MCP

Se a tecnologia da sua habilidade tiver um servidor MCP disponível, o comando incluirá __NOTRASLATE_31__:

```json
{
  "mcpServers": {
    "supabase": {
      "command": "npx",
      "args": ["-y", "@supabase/mcp-server"],
      "env": {
        "SUPABASE_URL": "${SUPABASE_URL}",
        "SUPABASE_KEY": "${SUPABASE_KEY}"
      }
    }
  }
}
```

Isso permite que Claude use ferramentas MCP ao trabalhar com a habilidade. Você precisará definir variáveis ​​de ambiente para que as ferramentas funcionem.

## Tratamento de erros

### A pesquisa na Web não retorna resultados

Se a documentação não for encontrada online:
- O comando cria a habilidade com padrões genéricos
- Você pode adicionar URLs manualmente a __NOTRASLATE_32__ mais tarde
- Use __NOTRASLATE_33__ para melhorá-lo

### Servidor MCP não encontrado

Se não existir nenhum servidor MCP para sua tecnologia:
- A habilidade é criada sem __NOTRASLATE_34__
- Você pode adicionar o MCP posteriormente se um servidor ficar disponível
- São fornecidas instruções manuais de configuração do MCP

### Erros de validação

Antes de escrever, o comando validados:
- O nome da habilidade contém apenas letras, números, hifens
- A descrição tem menos de 1.024 caracteres
- SKILL.md tem menos de 500 linhas
- Todos os arquivos do livro de receitas são referenciados

Se a validação falhar, você será solicitado a corrigir antes de continuar.

## Solução de problemas

### Habilidade não funciona

Se uma habilidade recém-criada parece não ser ativada:

1. Verifique a seção "Quando usar" em SKILL.md
2. Certifique-se de que sua pergunta corresponda aos gatilhos de ativação
3. Execute __NOTRASLATE_35__ para verificar a estrutura
4. Edite com __NOTRASLATE_36__ para ajustar os gatilhos

### Documentação local ausente

Se o comando não conseguir encontrar suas práticas locais:

1. Crie documentação em __NOTRASLATE_37__ ou __NOTRASLATE_38__
2. Edite a habilidade para fazer referência ao que você cria
3. Use palavras-chave mais específicas na descrição de suas habilidades

### Entradas do livro de receitas não mostradas

Se as entradas do livro de receitas não estiverem visíveis no Código Claude:

1. Execute __NOTRASLATE_39__ para verificar se os arquivos existem
2. Verifique se SKILL.md faz referência a eles corretamente
3. Certifique-se de que os arquivos do livro de receitas sejam Markdown válidos
