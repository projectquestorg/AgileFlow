# /agileflow:context:exportar

Exporte un extracto de contexto conciso para pegarlo en herramientas web de IA.

## Inicio rápido

```bash
/agileflow:context:export
```

## Propósito

Lee `docs/context.md` y genera un extracto condensado (≤300 líneas) listo para pegar en ChatGPT, Perplexity, Gemini o Claude web.

**Este es un comando de sólo lectura**: no se escriben archivos.

## Formato de salida

El comando genera un bloque de código de rebajas para facilitar la copia:

```markdown
# Project Context Brief
Last updated: 2025-12-24

## What We're Building
AgileFlow is a comprehensive project management system...

## Current Focus
- EP-0010: Documentation System (in_progress)
- US-0042: Add Loop Mode docs (in_progress)

## Tech Stack
- Framework: Next.js 16, TypeScript
- UI: React, Tailwind CSS, shadcn/ui
- Testing: Jest, Playwright

## Key Decisions
- ADR-001: Use TypeScript for type safety (Accepted)
- ADR-002: Use React component library (Accepted)

## Feature Map
| Epic | Status | Stories |
|------|--------|---------|
| EP-0010 | in_progress | 5 total, 3 done |
| EP-0009 | complete | 7 total, 7 done |

## Next Steps
- US-0043: Visual Mode documentation (ready)
- US-0044: Session commands docs (ready)

---
**Paste this into**: ChatGPT, Perplexity, Gemini, or Claude web
**Purpose**: Load full project context for AI assistance
**Limit**: This excerpt is ≤300 lines for easy pasting
```

## Secciones incluidas

La exportación incluye sólo estas secciones (para mantenerse por debajo de las 300 líneas):

| Sección | Líneas máximas | Propósito |
|---------|-----------|---------|
| Encabezado | 2 | Título y marca de tiempo |
| Lo que estamos construyendo | 5 | Descripción general del proyecto |
| Enfoque actual | 5 | Trabajo activo |
| Pila de tecnología | 8 | Tecnologías |
| Decisiones clave | 10 | Sólo títulos ADR |
| Mapa de características | 15 | Una línea por epopeya |
| Próximos pasos | 15 | Historias listas |
| Pie de página | 3 | Pegar instrucciones |

## Secciones excluidas

Para permanecer dentro del límite de 300 líneas, se omiten estas:

- **Progreso reciente** -demasiado detallado
- **Sección de notas** - Demasiado largo
- **Descripciones detalladas de la historia** - No es necesario para el contexto
- **Contenido ADR completo** - Solo se incluyen títulos

## Contenido condensado

Al extraer, el contenido se condensa:

| Elemento | Tamaño completo | Condensado |
|---------|-----------|-----------|
| ADR | Descripción completa | Título + estado solamente |
| Historias | Título + descripción | Título + estado |
| Tecnología | Lista completa + versiones | Marco + departamentos clave |
| Épicas | Título + lista de historias | Título + recuento + estado |
| Decisiones | Contexto completo | Una sola línea |

## Requisitos

**docs/context.md debe existir** antes de exportar.

Si falta, verás:

```
Error: docs/context.md not found.

To create it, run: /agileflow:context:full

Then you can export with: /agileflow:context:export
```

## Flujo de trabajo de uso

```bash
# 1. Generate context (if needed)
/agileflow:context:full

# 2. Export concise version
/agileflow:context:export

# 3. Copy the output
# 4. Paste into ChatGPT, Perplexity, Gemini, or Claude web
# 5. Ask questions with full project context
```

## ¿Por qué 300 líneas?

Las herramientas web de IA tienen límites de ventana de contexto. Un extracto de 300 líneas:

- Se adapta cómodamente a la mayoría de las ventanas contextuales de herramientas de IA
- Proporciona suficientes detalles para una asistencia informada.
- Deja espacio para sus preguntas reales
- Es rápido de pegar y procesar

## Cuándo utilizar

- **Antes de las sesiones de IA web** - Cargar proyectocontexto ct
- **Al cambiar de herramientas de IA** - Transferir contexto entre herramientas
- **Después de las actualizaciones de contexto** - Obtenga una versión nueva y concisa
- **Para compartir en equipo** - Descripción rápida del proyecto
