# /agileflow:research:list

Zeigen Sie alle in Ihrem Projekt gespeicherten Forschungsnotizen mit einer schnellen Übersicht an.

## Schnellstart

```bash
/agileflow:research:list
```

Keine Argumente erforderlich – zeigt alle Forschungsnotizen an.

## Zweck

Zeigt Forschungsnotizen an, die in `docs/10-research/` gespeichert sind mit:

- Datum
- Thema
- Dateiname
- Einzeilige Zusammenfassung (falls verfügbar)

Dies ist ein **schreibgeschützter Befehl** – es werden keine Dateien geschrieben.

## Ausgabe

```
## Research Notes

| Date       | Topic                  | File                        |
|------------|------------------------|-----------------------------|
| 2025-01-07 | OAuth Integration       | 20250107-oauth.md           |
| 2025-01-06 | Performance Tips        | 20250106-perf.md            |
| 2025-01-05 | Database Migrations     | 20250105-migrations.md      |

Total: 3 research notes

Quick actions:
- /agileflow:research:view FILE=<filename> - Read a specific note
- /agileflow:research:analyze FILE=<filename> - Analyze for implementation
- /agileflow:research:ask TOPIC="new topic" - Create new research prompt
- /agileflow:research:import TOPIC="..." - Import external content
```

## Wenn keine Forschung vorhanden ist

Wenn keine Forschungsnotizen gefunden werden:

```
No research notes found.

Quick start:
- /agileflow:research:ask TOPIC="your topic" - Generate research prompt for ChatGPT/Claude
- /agileflow:research:import TOPIC="..." - Import external content
```

## Wie es funktioniert

1. Liest `docs/10-research/README.md` für die Indextabelle
2. Wenn README nicht vorhanden ist, werden die Dateien aus dem Verzeichnis aufgelistet
3. Wenn das Verzeichnis nicht existiert, wird eine hilfreiche Meldung angezeigt
4. Schlägt nächste Aktionen vor

## Schnelle Aktionen

Nachdem Sie die Liste angezeigt haben, können Sie:

| Aktion | Befehl |
|--------|---------|
| Eine Notiz lesen | `/agileflow:research:view FILE=<filename>` |
| Zur Umsetzung analysieren | `/agileflow:research:analyze FILE=<filename>` |
| Neue Forschung erstellen | `/agileflow:research:ask TOPIC="..."` |
| Inhalte importieren | `/agileflow:research:import TOPIC="..."` |

## Wann zu verwenden

- **Vor der Implementierung** - Vergangenheit findenRecherche zu einem Thema
- **Suche nach Codeausschnitten** - Merken Sie sich, wo Sie etwas recherchiert haben
- **Aktionselemente prüfen** - Sehen Sie, welche Aufgaben aus der Forschung hervorgegangen sind
- **Wissensaustausch im Team** - Durchsuchen Sie, was recherchiert wurde

## Verwandte Befehle

- [`/agileflow:research`](/commands/research) – Überblick über das Forschungssystem
- [`/agileflow:research:ask`](/commands/research-ask) – Rechercheaufforderungen generieren
- [`/agileflow:research:import`](/commands/research-import) - Forschungsergebnisse importieren
- [`/agileflow:research:view`](/commands/research-view) - Spezifische Forschungsnotiz lesen
- [`/agileflow:research:analyze`](/commands/research-analyze) – Auf Implementierung analysieren
