# /épico

Cree una nueva epopeya con historias secundarias opcionales para organizar historias de usuarios relacionadas en torno a una característica u objetivo más amplio.

## Inicio rápido

```bash
/agileflow:epic EPIC=<EP-ID> TITLE=<text> OWNER=<id> GOAL=<text>
```

## Parámetros

| Parámetro | Requerido | Predeterminado | Descripción |
|-----------|----------|---------|-------------|
| `EPIC` | Sí | - | Identificador épico (por ejemplo, EP-0001) |
| `TITLE` | Sí | - | Título épico que describe la característica u objetivo |
| `OWNER` | Sí | - | Nombre del propietario o ID del agente (por ejemplo, AG-UI, AG-API) |
| `GOAL` | Sí | - | Resultado o métrica que define el éxito de esta epopeya |
| `STORIES` | No | - | Lista de historias iniciales separadas por comas (formato: `US-ID\|title\|owner`) |

## Ejemplos

### Creación épica básica

```bash
/agileflow:epic EPIC=EP-0010 TITLE="User Authentication" OWNER=AG-API GOAL="Enable users to securely register and log in"
```

Crea un nuevo archivo épico en `docs/05-epics/EP-0010.md` con los detalles proporcionados y lo registra en `docs/09-agents/status.json`.

### Épico con historias iniciales

```bash
/agileflow:epic EPIC=EP-0011 TITLE="Payment Processing" OWNER=AG-API GOAL="Enable payments via Stripe" STORIES="US-0040|Stripe integration|AG-API,US-0041|Payment form|AG-UI,US-0042|Webhook handler|AG-API"
```

Crea la epopeya y genera automáticamente historias secundarias iniciales en `docs/06-stories/EP-0011/`. cada unoLa historia se crea con:
- Estado: `ready` (listo para desarrollo)
- Estimación: `0.5d` (predeterminado, se puede modificar)
- Dependencias: vacías (actualizadas manualmente según sea necesario)

## Salida

El comando crea:

1. **Archivo épico** (`docs/05-epics/EP-XXXX.md`)
   - Metadatos épicos (título, propietario, estado, objetivo)
   - Estado actual y métricas.
   - Lista de cuentos infantiles.

2. **Archivos de historias** (uno por historia, `docs/06-stories/EP-XXXX/US-XXXX-<slug>.md`)
   - Plantilla de historia con marcadores de posición de criterios de aceptación.
   - Vinculado a la epopeya de los padres.
   - Listo para una definición detallada

3. **Seguimiento de estado** (`docs/09-agents/status.json`)
   - Entrada épica con propietario y estado actual.
   - Entradas de historias vinculadas a épicas.

4. **Comunicación del agente** (`docs/09-agents/bus/log.jsonl`)
   - Mensajes de tareas que muestran las épicas creadas y las historias asignadas.

## Flujo de trabajo

1. **Planifica tu epopeya**: define el objetivo de la función y las métricas de éxito
2. **Crea la epopeya** - Ejecuta el comando `/epic` con detalles básicos
3. **Agregar historias** - Créalas ahora con el parámetro STORIESo use `/story` para agregar individualmente más tarde
4. **Desarrollar historias** - Utilice `/story-validate` para asegurarse de que cada historia esté lista antes del desarrollo.
5. **Plan de sprint**: use `/sprint` para seleccionar historias para el próximo sprint
6. **Seguimiento del progreso**: use `/board` para monitorear la finalización épica

## Integración con otros comandos

- **Después de la creación épica**: usa `/story` para agregar más historias
- **Antes de la planificación del sprint**: use `/sprint` para seleccionar historias de esta épica
- **Para seguimiento del progreso**: use `/board` o `/metrics` para ver la finalización épica
- **Para dependencias**: use `/deps` para visualizar las dependencias de la historia dentro de Epic

## Mejores prácticas

- **Mantenga las épicas enfocadas** - Una función o área de funciones por épica
- **Haga que los objetivos sean mensurables** - Utilice resultados o métricas específicas (no "mejorar la experiencia del usuario")
- **Agregue historias de forma incremental** - Comience con las historias principales y agregue otras relacionadas con el tiempo
- **Asignar propietarios apropiados** - Coincidir con el propio épicoer con agente de desarrollo primario
- **Contexto del documento** - Agregar contexto técnico en el archivo épico después de la creación
