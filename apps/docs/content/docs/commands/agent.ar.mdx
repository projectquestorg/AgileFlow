## مكونات ملف تعريف الوكيل

### المسؤوليات

وصف واضح لما يفعله الوكيل:
- مجال محدد (واجهة المستخدم، API، CI/CD)
- التقنيات المستخدمة
- التسليمات الرئيسية
- دور الفريق

### الأدوات المتاحة

ما يمكن للوكيل استخدامه:
- **قراءة** - قراءة الملفات وتحليل التعليمات البرمجية
- **الكتابة** - إنشاء ملفات جديدة
- **تحرير** - تعديل الملفات الموجودة
- **Bash** - تشغيل أوامر shell (git، npm، docker)
- **Glob** - البحث عن الملفات حسب النمط
- **Grep** - محتويات ملف البحث

### حدود النطاق

أين يمكن للوكيل العمل:
- **الدلائل المسموح بها** - يمكن لوكيل الملفات تعديلها
- **الدلائل المقيدة** - لا يمكن لوكيل الملفات لمسها
- **علامات القصة** - العمل الذي تم تعيينه لهذا الوكيل

### موجه النظام (العقد)

يجب على وكيل القواعد اتباع ما يلي:
- **قيود النطاق** - مسارات الملفات بالضبط
- **متطلبات الاختبار** - متى وكيف يتم الاختبار
- **اتفاقيات الالتزام** - تنسيق الفرع والرسالة
- **تحديثات الحالة** - كيفية تتبع التقدم
- **معايير الكود** - متطلبات الجودة

## أفضل الممارسات

### وكلاء التسمية
استخدم معرفات وصفية تشير بوضوح إلى الدور:
- ✅ جيد: AG-UI، AG-API، AG-CI، AG-DEVOPS، AG-SECURITY
- ❌ سيء: AG-001, AG-A, AG-DEV1

### تحديد النطاق

اجعل الحدود واضحة تمامًا:

**جيد** - مسارات محددة:
- __غير مترجم_13__
- __غير مترجم_14__
- __لا تترجم_15__

**سيء** - حدود غامضة:
- "أشياء الواجهة"
- "الأشياء الخلفية"

### متطلبات الاختبار

تضمين أوامر اختبار محددة:

**جيد** - أوامر واضحة:
- تشغيل `npm test -- src/components/`
- تحقق من التغطية باستخدام `npm run coverage`
- الوبر مع `npm run lint:tsx`

**سيء** - تعليمات غامضة:
- "اكتب الاختبارات"
- "تأكد من أنه يعمل"

### تعيين علامات القصة

تعيين القصص بناءً على العلامات:

| الوكيل | علامات القصة |
|-------|-----------|
| AG-UI | الواجهة الأمامية، واجهة المستخدم، تجربة المستخدم، التصميم |
| AG-API | backenد، واجهة برمجة التطبيقات، قاعدة البيانات |
| AG-CI | ci، devops، الأتمتة |

## سير العمل

1. جمع تفاصيل الوكيل:
   - المعرف الفريد (AG-DOMAIN)
   - الدور/العنوان
   - الأدوات المتاحة
   - حدود النطاق
   - علامات القصة

2. إنشاء ملف تعريف الوكيل من القالب
   - قسم المسؤوليات
   - قائمة الأدوات
   - حدود النطاق (واضحة ومحددة)
   - موجه النظام (العقد مع القواعد)

3. تحديث قائمة الوكلاء
   - إضافة إدخال إلى docs/09-agents/roster.yaml
   - معرف الخريطة → الدور والأدوات والنطاق

4. إنشاء موجه النظام
   - إخراج موجه مكثف
   - جاهز لللصق كمطالبة النظام الطرفي

5. التحقق من صحة الوكيل
   - مراجعة حدود النطاق (لا يوجد تعارض)
   - تأكيد متطلبات الاختبار
   - التحقق من تعيينات علامة القصة

## التكامل

تتكامل ملفات تعريف الوكيل مع:
- **تتبع الحالة** - تعرض القائمة الوكلاء النشطين
- **مهمة القصة** - استخدم معرف الوكيل والعلامات
- **تخطيط القدرات** - تعرف على الأشخاص المتاحين
- **عمليات التسليم** - ملفات تعريف الوكيل المرجعي في الاتصالات
- **التحقق من صحة النطاق** - منع التغييرات خارج النطاق

## يوحد ذاتها متى

- **تأهيل أعضاء الفريق الجدد** - إنشاء ملف تعريف وعقد
- **إضافة أدوار متخصصة** - عوامل الأمان والأداء وإمكانية الوصول
- **فريق إعادة الهيكلة** - تحديث حدود النطاق
- **وكلاء التدريب** - استخدم موجه النظام كمرجع
- **حل التعارضات** - يمنع النطاق الواضح التداخل
