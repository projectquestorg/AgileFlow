# /session:history

Zeigen Sie den Verlauf aller Sitzungen, einschließlich der beendeten, mit Zeit- und Aktivitätsmetriken an.

## Schnellstart

```bash
/agileflow:session:history
```

## Parameter

| Parameter | Erforderlich | Standard | Beschreibung |
|-----------|----------|---------|-------------|
| `DAYS` | Nein | `7` | Anzahl der anzuzeigenden Tage des Verlaufs |

## Beispiele

```bash
# Last 7 days (default)
/agileflow:session:history

# Last 30 days
/agileflow:session:history DAYS=30

# All history
/agileflow:session:history DAYS=all
```

## Beispielausgabe

```

  Path: /home/user/myproject
  Branch: main
  Status: active
  Created: 2025-12-20 10:00
  Duration: 6d 5h (ongoing)

Session 2 "auth"
  Path: /home/user/myproject-auth
  Branch: session-2
  Status: ended
  Created: 2025-12-24 14:00
  Ended: 2025-12-25 16:30

  Path: /home/user/myproject-3
  Branch: session-3
  Status: ended
  Created: 2025-12-26 09:00
  Ended: 2025-12-26 12:00
  Duration: 3h
  Worktree: kept at /home/user/myproject-3

─────────────────────────────────────────────────────────────────
Total sessions: 3
Active: 1    Ended: 2
Average duration: 2d 3h
```

## Verlaufsdaten

Der Sitzungsverlauf umfasst:

| Feld | Beschreibung |
|-------|-------------|
| Sitzungs-ID | Numerischer Bezeichner |
| Spitzname | Menschenfreundlicher Name, falls angegeben |
| Pfad | Verzeichnisspeicherort |
| Zweig | Git-Zweig verwendet |
| Status | aktiv, beendet oder veraltet |
| Erstellt | Wann die Sitzung gestartet wurde |
| Beendet | Wann die Sitzung beendet wurde (falls zutreffend) |
| Dauer | Wie lange war die Sitzung aktiv |
| Arbeitsbaum | Ob Arbeitsbaum beibehalten oder entfernt wurde |

## Anwendungsfälle

### Überprüfung von Arbeitsmustern

```bash
/agileflow:session:history DAYS=30
# See how many sessions you've used
# Identify if you're creating too many abandoned sessions
```

### Alte Sitzungen finden

```bash
/agileflow:session:history DAYS=all
# Find a session from weeks ago
# Check if its worktree still exists
```

### Cleanup DeEntscheidungen

```bash
/agileflow:session:history
# See which sessions have worktrees kept
# Decide which to clean up
```

## Datenspeicherung

Der Sitzungsverlauf wird in `.agileflow/sessions/registry.json` gespeichert:

```json
{
  "sessions": {
    "1": {
      "created": "2025-12-20T10:00:00Z",
      "last_active": "2025-12-26T15:00:00Z",
      "is_main": true
    },
    "2": {
      "created": "2025-12-24T14:00:00Z",
      "ended": "2025-12-25T16:30:00Z",
      "nickname": "auth",
      "worktree_removed": true
    }
  }
}
```

## Verwandte Befehle

- [`/session:status`](/commands/session-status) – Aktuelle aktive Sitzungen anzeigen
- [`/session:end`](/commands/session-end) – Sitzungen beenden und bereinigen
- [`/session:new`](/commands/session-new) – Neue Sitzung erstellen
