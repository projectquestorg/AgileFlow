# /adr

قم بإنشاء سجلات قرارات الهندسة المعمارية (ADRs) لتوثيق الاختيارات المعمارية الهامة وقرارات التصميم في مشروعك.

## بداية سريعة

| المعلمة | مطلوب | الافتراضي | الوصف |
|-----------|----------|---------|-------------|
| __غير مترجم_4__ | نعم | - | رقم ADR مكون من 4 أرقام (على سبيل المثال، 0001، 0042) |
| __غير مترجم_5__ | نعم | - | عنوان مختصر للقرار المعماري |

| __غير مترجم_7__ | نعم | - | ماذا تم اختياره ولماذا |
| __غير مترجم_8__ | نعم | - | المقايضات آثارها الإيجابية والسلبية |
| __غير مترجم_9__ | لا | - | روابط اختيارية للوثائق ذات الصلة |

عند تشغيل الأمر، سترى:

## أفضل الممارسات

### ترقيم ADR
- الرقم بالتسلسل (0001، 0002، 0003، ...)
- لا تتخطى الأرقام أو تعيد استخدامها
- استخدم الأصفار البادئة للاتساق

### كتابة ADRs جيدة
- **السياق أولاً**: اشرح المشكلة أو القيد قبل اتخاذ القرار
- **كن محددًا**: "استخدم PostgreSQL 15+ مع النسخ المتماثل غير المتزامن" وليس "استخدم قاعدة بيانات"
- **توثيق المقايضات**: قم بإدراج النتائج الإيجابية والسلبية
- **حافظ على الإيجاز**: استهدف 1-2 صفحة كحد أقصى
- **الأعمال ذات الصلة بالمراجع**: رابط للملاحم أو القصص أو غيرها من الإعلانات

### نصائح لقسم السياق
- ما المشكلة أو القيد الذي أدى إلى هذا القرار؟
- ما هي محركات الأعمال؟
- ما هي القيود التقنية الموجودة؟
- ماذا جربنا من قبل؟

### نصائح لقسم اتخاذ القرار
- الدولة بوضوح ثتم اختيار القبعة
- اشرح بإيجاز سبب فوز هذا الخيار
- اذكر البدائل التي تم أخذها بعين الاعتبار
- تضمين أرقام الإصدارات أو تفاصيل محددة

### نصائح قسم العواقب
- قم بإدراج التأثيرات الإيجابية أولاً
- ثم قم بإدراج المفاضلات والمخاطر
- كن صادقا بشأن السلبيات
- اقتراح استراتيجيات التخفيف
