# /atribuir

Atribua ou reatribua uma história a um proprietário com atualização de status opcional. Atualiza os metadados da história e transmite a tarefa por meio do barramento de mensagens.

## Início rápido

```bash
/agileflow:assign STORY=US-0001 NEW_OWNER=AG-UI
```

## Parâmetros

| Parâmetro | Obrigatório | Padrão | Descrição |
|-----------|----------|---------|------------|
| `STORY` | Sim | - | ID da história a ser atribuída (por exemplo, US-0001, US-0042) |
| `NEW_OWNER` | Sim | - | ID do agente/proprietário (por exemplo, AG-UI, AG-API, AG-CI) |
| `NEW_STATUS` | Não | - | Status de atualização: `ready`, `in-progress`, `blocked`, `in-review`, `done` |
| `NOTE` | Não | - | Contexto opcional sobre a tarefa |

## Exemplos

### Atribuir história ao proprietário

```bash
/agileflow:assign STORY=US-0042 NEW_OWNER=AG-UI
```

Atribua a história à equipe de UI sem alterar o status.

### Atribuir e começar a trabalhar

```bash
/agileflow:assign STORY=US-0042 NEW_OWNER=AG-UI NEW_STATUS=in-progress NOTE="Starting login form implementation"
```

Atribua uma história e marque imediatamente como em andamento com o contexto.

## Valores de status

Ao usar __NOTRASLATE_31__, escolha entre:

| Estado | Significado | Quando usar |
|--------|---------|---------|
| `ready` | Preparado, esperando para começar | Após a criação ou quando desbloqueado |
| `in-progress` | Atualmente em trabalho | Quando o proprietário começa a trabalhar |
| `blocked` | Esperando por dependência externa | Quando o bloqueador for identificado |
| `in-review` | Concluído, aguardando aprovação | Revisão de código ou controle de qualidade |
| `done` | Concluído e verificado | À popatodos os cheques foram aprovados |

## Segurança JSON

O comando usa manipulação segura de JSON:

- **Nunca usa echo/cat** - Sempre usa jq ou ferramenta de edição
- **Valida após atualização** - Verifica se o JSON é válido antes de terminar
- **Faz backup em caso de erro** - Pode restaurar a partir do backup se ocorrer corrupção

## Melhores práticas

1. **Atribuir imediatamente** - Não deixe histórias sem atribuir
2. **Incluir context** - Use NOTA para explicar por que a atribuição aconteceu
3. **Experiência correspondente** - Atribuir a um agente com habilidades relevantes
4. **Equilibrar a carga** – Distribua o trabalho de maneira justa entre os agentes
5. **Atualizar status** - Considere atualizar o status ao atribuir
6. **Comunique-se** – A equipe deve entender por que a reatribuição aconteceu
