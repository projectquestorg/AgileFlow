# /actualizar

Genere actualizaciones de proyectos de alto nivel para partes interesadas, ejecutivos o clientes con métricas, progreso y riesgos.

## Inicio rápido

```bash
/agileflow:update PERIOD=week AUDIENCE=exec
```

## Parámetros

| Parámetro | Requerido | Predeterminado | Descripción |
|-----------|----------|---------|-------------|
| `PERIOD` | No | semana | Marco de tiempo: semana, sprint, mes, trimestre o personalizado |
| `START_DATE` | No | - | Para período personalizado (formato AAAA-MM-DD) |
| `END_DATE` | No | - | Para período personalizado (formato AAAA-MM-DD) |
| `AUDIENCE` | No | ejecutivo | Público objetivo: ejecutivo, cliente, equipo o junta directiva |
| `FORMAT` | No | rebaja | Formato de salida: correo electrónico, rebajas, diapositivas o pdf |

## Ejemplos

### Actualización semanal ejecutiva

```bash
/agileflow:update PERIOD=week AUDIENCE=exec FORMAT=email
```

Genera una actualización concisa de una página para ejecutivos de nivel C con:
- Resumen ejecutivo y estado general.
- Logros clave
- Bloqueadores o riesgos críticos
- Velocidad y métricas.
- Sin jerga técnica

### Informe quincenal del cliente

```bash
/agileflow:update PERIOD=2weeks AUDIENCE=client FORMAT=markdown
```

Genera una feaActualización centrada en la naturaleza para clientes con:
- Qué pueden hacer los usuarios ahora
- Progreso hacia los entregables del contrato.
- Capturas de pantalla o referencias de demostración.
- Comunicación transparente del problema.
- Lenguaje de beneficio para el usuario.

### Retrospectiva de sprint por equipos

```bash
/agileflow:update PERIOD=sprint AUDIENCE=team FORMAT=markdown
```

Genera una actualización detallada para el equipo con:
- Decisiones de arquitectura y ADR.
- Logros técnicos
- Celebración de victorias
- Perspectivas retrospectivas
- Elementos de acción

### Actualización del período personalizado

```bash
/agileflow:update PERIOD=custom START_DATE=2025-12-01 END_DATE=2025-12-15 AUDIENCE=board FORMAT=pdf
```

## Salida

El comando genera una actualización completa guardada en:
```text
docs/08-project/updates/<YYYYMMDD>-update.md
```

### Actualizar estructura

**Resumen ejecutivo**
- Descripción general de 2-3 oraciones
- Logros clave
- Problemas o riesgos críticos
- Estado general (En camino / En riesgo / Bloqueado)

**Progreso este período**
- Historias completadas con conteos.
- Porcentajes de progreso de Epics
- Hitos alcanzados

**Próximas prioridades**
- Áreas de enfoque del próximo sprint
- Próximos hitos con fechas.

**Métricas y tendencias**
- Velocidad (historias/semana)
- Porcentaje de tasa de finalización
- Métricas de calidad (tcobertura est, errores)
- Tendencias (mejorando/disminuyendo)

**Bloqueadores y riesgos**
- Bloqueadores actuales con propietarios.
- Estrategias de mitigación de riesgos
- Estado de cada elemento

**Decisiones tomadas**
- Registros de decisiones de arquitectura recientes (ADR)
- Justificación de las decisiones clave

**Actualizaciones del equipo** (si corresponde)
- Cambios en la capacidad del equipo.
- Logros notables
- Indicadores de moral

## Fuentes de datos

El comando agrega datos de 10 fuentes:

1. **docs/09-agents/status.json** - Instantáneas del estado de la historia
2. **docs/05-epics/*.md** - Progreso y finalización épicos
3. **docs/06-stories/**/US-*.md** - Detalles de la historia y finalización
4. **docs/08-project/backlog.md** - Elementos del trabajo pendiente
5. **docs/08-project/roadmap.md** - Hitos de la hoja de ruta
6. **docs/08-project/milestones.md** - Objetivos de hitos
7. **docs/08-project/risks.md** - Registro de riesgos
8. **Historial de Git**: confirmaciones y relaciones públicas fusionadas
9. **docs/09-agents/bus/log.jsonl** - Actividad y eventos
10. **docs/03-decisions/adr-*.md** - Decisiones recientes

## Personalización de la audiencia

### Ejecutivoutivo (nivel C)
- Descripción general de muy alto nivel
- Enfoque en impacto empresarial
- Métricas y ROI
- Riesgos destacados tempranamente
- Máximo 1 página

### Cliente
- Lenguaje centrado en funciones
- Se enfatizan los beneficios para el usuario.
- Transparente sobre los problemas
- Capturas de pantalla/enlaces de demostración incluidos
- Progreso hacia los entregables del contrato.

### Equipo
- Más detalles técnicos incluidos.
- Decisiones de arquitectura incluidas.
- Celebración de victorias
- Perspectivas retrospectivas
- Elementos de acción para el próximo período.

### Tablero
- Visión estratégica
- Implicaciones financieras
- Posicionamiento competitivo
- Hoja de ruta a largo plazo
- Evaluación de riesgos

## Flujo de trabajo

1. Determinar el período de tiempo (predeterminado: esta semana)
2. Recopile datos de las 10 fuentes.
3. Calcular métricas y tendencias.
4. Identificar el trabajo completado y los bloqueadores.
5. Formato para el público objetivo
6. Vista previa de la actualización en Markdown
7. Pregunte: "¿Enviar actualización? (SÍ/NO/EDITAR)"
8. En caso afirmativo: guarde en docs/08-project/updates/ y, opcionalmente, envíe un correo electrónico a las partes interesadas

## Usar cuando

- **Sincronizaciones semanales del equipo** - Actualizaciones periódicas de cadencia
- **Presentación ejecutivaciones** - Informes mensuales/trimestrales
- **Comunicación con el cliente** - Actualizaciones del progreso contractual
- **Retrospectivas de Sprint** - Aprendizajes centrados en el equipo
- **Reuniones de la junta directiva** - Actualizaciones de alineación estratégica
- **Aumento del riesgo** - Destacar los bloqueadores desde el principio

## Integración

Enlace a epopeyas e historias para obtener detalles adicionales. ADR de referencia para el contexto arquitectónico. Incluya enlaces de relaciones públicas de GitHub cuando sea relevante.
