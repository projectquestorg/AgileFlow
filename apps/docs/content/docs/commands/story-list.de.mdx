# /story:list

Zeigen Sie alle User Stories mit Filteroptionen und Schnellaktionsfunktionen an.

## Schnellstart

```bash
/agileflow:story:list
```

## Parameter

| Parameter | Erforderlich | Standard | Beschreibung |
|-----------|----------|---------|-------------|
| `EPIC` | Nein | - | Nach Epic filtern (z. B. EP-0001) |
| `STATUS` | Nein | - | Nach Status filtern: `ready`, `in_progress`, `blocked`, `done` |
| `OWNER` | Nein | - | Nach Eigentümer filtern (z. B. AG-UI) |

## Beispiele

### Alle Geschichten auflisten

```bash
/agileflow:story:list
```

Zeigt alle nach Epos sortierten Storys mit ihrem Status an.

### Listen Sie Geschichten für ein bestimmtes Epos auf

```bash
/agileflow:story:list EPIC=EP-0001
```

Zeigt nur Storys im Authentifizierungs-Epos an.

### Bereite Geschichten auflisten

```bash
/agileflow:story:list STATUS=ready
```

Zeigt alle Storys an, an denen mit der Arbeit begonnen werden kann.

### In Bearbeitung befindliche Geschichten auflisten

```bash
/agileflow:story:list STATUS=in_progress
```

Zeigt aktuelle Arbeitselemente an.

### Geschichten nach Besitzer auflisten

```bash
/agileflow:story:list OWNER=AG-UI
```

Zeigt alle dem UI-Agenten zugewiesenen Storys an.

### Kombinierte Filter

```bash
/agileflow:story:list EPIC=EP-0001 STATUS=ready
```

Zeigt fertige Storys nur im Authentifizierungs-Epos an.

```bash
/agileflow:story:list STATUS=blocked OWNER=AG-API
```

Zeigt API-Storys an, die derzeit blockiert sind.

## Ausgabeformat

Stories are organized by epic with status columns:

```markdown
## Stories

### EP-0001: Authentication System
| Story | Title | Status | Owner | Estimate |
|-------|-------|--------|-------|----------|
| US-0001 | Login form | done | AG-UI | 2h |
| US-0002 | Password reset | in_progress | AG-API | 3h |
| US-0003 | Session management | ready | AG-API | 4h |

### EP-0002: User Dashboard
| Story | Title | Status | Owner | Estimate |
|-------|-------|--------|-------|----------|
| US-0004 | Dashboard layout | ready | AG-UI | 2h |
| US-0005 | Activity feed | ready | AG-UI | 3h |

---
**Summary**: 5 stories (1 done, 1 in_progress, 3 ready)
```

## Statusdefinitionen

| Status | Bedeutung | Aktion |
|--------|---------|--------|
| **bereit** | Startbereit, keine Blocker | Arbeit beginnen / Details anzeigen |
| **in_Progress** | Derzeit wird daran gearbeitet | Arbeit fortsetzen / bei Bedarf als blockiert markieren |
| **blockiert** | Warten auf Abhängigkeit oder Klärung | Blocker auflösen / Blockierung aufheben, wenn bereit |
| **erledigt** | Abgeschlossen und getestet | Zur nächsten Geschichte wechseln |

## Schnelle Aktionen

Nachdem Sie die Liste angezeigt haben, können Sie:

### Story-Details anzeigen

Wählen Sie eine Geschichte aus, um Folgendes anzuzeigen:
- Vollständige Akzeptanzkriterien
- Teststatus
- Abhängigkeiten
- Blocker (falls vorhanden)
- Fortschrittsnotizen

```bash
/agileflow:story:view STORY=US-0003
```

### Beginnen Sie mit der Arbeit an einer Geschichte

Wählen Sie eine fertige Story aus und markieren Sie sie als in_progress:

```bash
/agileflow:status US-0003 STATUS=in_progress
```

### Neue Geschichte erstellen

Fügen Sie einem Epos eine Geschichte hinzu:

```bash
/agileflow:story EPIC=EP-0001 ...
```

##Workflow-Integration

### Täglicher Standup

```bash
/agileflow:story:list STATUS=in_progress
# See what's being worked on
/agileflow:story:list STATUS=blocked
# See what's blocked
```

### Sprintplanung

```bash
/agileflow:story:list STATUS=ready
# See available work
/agileflow:epic:list
# See epic status
# Select stories for sprint
```

### Workflow entsperren

```bash
/agileflow:story:list STATUS=blocked
# See all blocked stories
# Resolve dependencies
/agileflow:story:view STORY=<blocked>
# See blocker details
/agileflow:status <story> STATUS=ready
# When unblocked, mark ready again
```

### Abschlussverfolgung

```bash
/agileflow:story:list STATUS=done
# See what was completed
/agileflow:story:list STATUS=in_progress
# See what's being worked on
/agileflow:story:list STATUS=ready
# See what's queued up
```

## Filterstrategien

### Von Epic
```bash
/agileflow:story:list EPIC=EP-0001
# Focus on one feature
```

### Nach Status
```bash
/agileflow:story:list STATUS=ready
# Find available work
```

```bash
/agileflow:story:list STATUS=blocked
# Find blockers to resolve
```

### Vom Eigentümer
```bash
/agileflow:story:list OWNER=AG-API
# See what an agent is working on
```

### Nach Status + Episch
```bash
/agileflow:story:list EPIC=EP-0001 STATUS=ready
# Ready stories in specific epic
```

### Nach Status + Eigentümer
```bash
/agileflow:story:list STATUS=in_progress OWNER=AG-UI
# Current work for specific agent
```

## Ergebnisse interpretieren

### Gesunde Story-Verbreitung

```
5 stories (1 done, 2 in_progress, 2 ready)
```

- ✅ Die Leute haben Arbeit zu erledigen
- ✅ Die Arbeit läuft im Fluss
- ✅ Backlog ist fertig

### Mögliche Probleme

**Zu viele in_progress (>3)**
```
5 stories (0 done, 4 in_progress, 1 ready)
```
- Erwägen Sie eine Reduzierung des WIP (in Arbeit)
- Konzentrieren Sie sich darauf, Geschichten zu vervollständigen

**Keine fertigen Geschichten**
```
5 stories (2 done, 2 in_progress, 0 ready)
```
- Es müssen mehr Geschichten geplant werden
– Erstellen Sie `/agileflow:story` für die nächsten Arbeitselemente

**Viele blockiert**
```
7 stories (1 done, 1 in_progress, 3 blocked, 2 ready)
```
- Blocker dringend ansprechen
- Möglicherweise müssen Abhängigkeiten entsperrt werden
