# /historia:lista

Muestre todas las historias de usuarios con opciones de filtrado y funciones de acción rápida.

## Inicio rápido

```bash
/agileflow:story:list
```

## Parámetros

| Parámetro | Requerido | Predeterminado | Descripción |
|-----------|----------|---------|-------------|
| `EPIC` | No | - | Filtrar por épica (por ejemplo, EP-0001) |
| `STATUS` | No | - | Filtrar por estado: `ready`, `in_progress`, `blocked`, `done` |
| `OWNER` | No | - | Filtrar por propietario (por ejemplo, AG-UI) |

## Ejemplos

### Listar todas las historias

```bash
/agileflow:story:list
```

Muestra todas las historias organizadas por épica con su estado.

### Lista de historias para epopeyas específicas

```bash
/agileflow:story:list EPIC=EP-0001
```

Muestra solo historias en la epopeya de Autenticación.

### Lista de historias listas

```bash
/agileflow:story:list STATUS=ready
```

Muestra todas las historias listas para comenzar a trabajar.

### Listar historias en progreso

```bash
/agileflow:story:list STATUS=in_progress
```

Muestra elementos de trabajo actuales.

### Listar historias por propietario

```bash
/agileflow:story:list OWNER=AG-UI
```

Muestra todas las historias asignadas al agente de UI.

### Filtros combinados

```bash
/agileflow:story:list EPIC=EP-0001 STATUS=ready
```

Muestra historias listas solo en la epopeya de Autenticación.

```bash
/agileflow:story:list STATUS=blocked OWNER=AG-API
```

Muestra historias de API que están actualmente bloqueadas.

## Formato de salida

Las historias están organizadas por epopeya con columnas de estado:

```markdown
## Stories

### EP-0001: Authentication System
| Story | Title | Status | Owner | Estimate |
|-------|-------|--------|-------|----------|
| US-0001 | Login form | done | AG-UI | 2h |
| US-0002 | Password reset | in_progress | AG-API | 3h |
| US-0003 | Session management | ready | AG-API | 4h |

### EP-0002: User Dashboard
| Story | Title | Status | Owner | Estimate |
|-------|-------|--------|-------|----------|
| US-0004 | Dashboard layout | ready | AG-UI | 2h |
| US-0005 | Activity feed | ready | AG-UI | 3h |

---
**Summary**: 5 stories (1 done, 1 in_progress, 3 ready)
```

## Definiciones de estado

| Estado | Significado | Acción |
|--------|---------|--------|
| **listo** | Listo para empezar, sin bloqueadores | Iniciar trabajo / ver detalles |
| **en_progreso** | Actualmente en proceso | Continuar trabajo/marcar bloqueado si es necesario |
| **bloqueado** | A la espera de dependencia o aclaración | Resolver bloqueador/desbloquear cuando esté listo |
| **hecho** | Completado y probado | Pasar a la siguiente historia |

## Acciones rápidas

Después de mostrar la lista, puede:

### Ver detalles de la historia

Seleccione una historia para ver:
- Criterios de aceptación total.
- Estado de la prueba
- Dependencias
- Bloqueadores (si los hay)
- Notas de progreso

```bash
/agileflow:story:view STORY=US-0003
```

### Comience a trabajar en una historia

Elija una historia lista y márquela como en progreso:

```bash
/agileflow:status US-0003 STATUS=in_progress
```

### Crear nueva historia

Añade una historia a una epopeya:

```bash
/agileflow:story EPIC=EP-0001 ...
```

##Integración del flujo de trabajo

### Stand-up diario

```bash
/agileflow:story:list STATUS=in_progress
# See what's being worked on
/agileflow:story:list STATUS=blocked
# See what's blocked
```

### Planificación de sprints

```bash
/agileflow:story:list STATUS=ready
# See available work
/agileflow:epic:list
# See epic status
# Select stories for sprint
```

### Desbloqueo del flujo de trabajo

```bash
/agileflow:story:list STATUS=blocked
# See all blocked stories
# Resolve dependencies
/agileflow:story:view STORY=<blocked>
# See blocker details
/agileflow:status <story> STATUS=ready
# When unblocked, mark ready again
```

### Seguimiento de finalización

```bash
/agileflow:story:list STATUS=done
# See what was completed
/agileflow:story:list STATUS=in_progress
# See what's being worked on
/agileflow:story:list STATUS=ready
# See what's queued up
```

## Estrategias de filtrado

### Por épico
```bash
/agileflow:story:list EPIC=EP-0001
# Focus on one feature
```

### Por estado
```bash
/agileflow:story:list STATUS=ready
# Find available work
```

```bash
/agileflow:story:list STATUS=blocked
# Find blockers to resolve
```

### Por propietario
```bash
/agileflow:story:list OWNER=AG-API
# See what an agent is working on
```

### Por estado + Épico
```bash
/agileflow:story:list EPIC=EP-0001 STATUS=ready
# Ready stories in specific epic
```

### Por estado + propietario
```bash
/agileflow:story:list STATUS=in_progress OWNER=AG-UI
# Current work for specific agent
```

## Interpretación de resultados

### Distribución de historias saludables

```
5 stories (1 done, 2 in_progress, 2 ready)
```

- ✅ La gente tiene trabajo que hacer
- ✅ El trabajo fluye
- ✅ El trabajo pendiente está listo

### Problemas potenciales

**Demasiados en_progreso (>3)**
```
5 stories (0 done, 4 in_progress, 1 ready)
```
- Considere reducir el WIP (trabajo en progreso)
- Centrarse en completar historias.

**No hay historias listas**
```
5 stories (2 done, 2 in_progress, 0 ready)
```
- Necesidad de planificar más historias.
- Crear `/agileflow:story` para los próximos elementos de trabajo

**Muchos bloqueados**
```
7 stories (1 done, 1 in_progress, 3 blocked, 2 ready)
```
- Abordar los bloqueadores con urgencia
- Puede que sea necesario desbloquear dependencias.
