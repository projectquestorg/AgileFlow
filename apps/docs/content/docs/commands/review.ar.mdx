# /مراجعة

قم بإجراء مراجعة شاملة للتعليمات البرمجية مدعومة بالذكاء الاصطناعي بشأن تغييرات طلبات السحب. قم بمراجعة التعليمات البرمجية لتحليل الجودة والأمان والأداء وأفضل الممارسات، وتقديم تعليقات قابلة للتنفيذ مع أمثلة محددة للتعليمات البرمجية.

## بداية سريعة

__لا تترجم_0__

الأمر سوف:
1. احصل على تغييرات التعليمات البرمجية (git diff)
2. التحليل عبر 6 فئات (الجودة، الأمان، الأداء، أفضل الممارسات، الاختبار، التوثيق)
3. قم بإنشاء تقرير مفصل بالمشكلات ذات الأولوية حسب الخطورة
4. حساب نقاط جودة الكود
5. اقتراح الإصلاحات التلقائية ومتابعة العمل

## الميزات الرئيسية

### تحليل شامل

تقوم المراجعة بفحص تغييرات التعليمات البرمجية الخاصة بك عبر 6 فئات مهمة:

| الفئة | الشيكات |
|----------|--------|
| **جودة الكود** | التعقيد، الازدواجية، التسمية، الوظيفة/طول الملف |
| **الأمن** | حقن SQL، XSS، الأسرار المشفرة، إلغاء التسلسل غير الآمن |
| **الأداء** | استعلامات N+1، وخوارزميات غير فعالة، وتسربات للذاكرة |
| **أفضل الممارسات** | معالجة الأخطاء، التسجيل، نوع الأمان، غير متزامن/awaذلك |
| **الاختبار** | الاختبارات المفقودة وجودة الاختبار وفجوات التغطية |
| **التوثيق** | JSDoc مفقود، تعليقات قديمة، تحديثات README |

## الشيكات الأمنية

تقوم المراجعة تلقائيًا بالتحقق من:

### الأسرار وبيانات الاعتماد
- مفاتيح API المشفرة
- كلمات مرور قاعدة البيانات
- رموز OAuth
- أسرار JWT
- بيانات اعتماد AWS

### نقاط الضعف الشائعة
- حقن SQL (استعلامات غير محددة)
- هجمات XSS (إخراج لم يتم الهروب منه)
- إلغاء التسلسل غير الآمن
- نقاط الضعف CSRF
- التحقق من صحة CORS مفقود

### الأنماط غير الآمنة
- التقييم () أو الوظيفة ()
- تسلسل سلسلة غير آمنة
- التحقق من صحة الإدخال مفقود
- تشفير ضعيف
- التبعيات غير الآمنة

## فحوصات الأداء

المراجعة تبحث عن:

### قاعدة البيانات
- مشاكل الاستعلام N+1
- الفهارس المفقودة
- الانضمامات غير الفعالة
- الاستفسارات غير الضرورية

### الخوارزميات
- O(n²) عندما يكون O(n) ممكنًا
- حلقات متداخلة يمكن أن تكون مسطحة
- الحسابات المتكررة
- العودية غير الأمثل

### الذاكرة
- تسرب الذاكرة
- لم يتم تحرير الأشياء الكبيرة
- المراجع الدائرية
- هياكل البيانات التي تستهلك الذاكرة

## تيستيالتغطية

مراجعة الشيكات:

### كود جديد
- هل هناك اختبار للوظائف الجديدة؟
- هل يتم تغطية حالات الحافة؟
- هل جودة الاختبار جيدة؟

### فجوات التغطية
- الخطوط المضافة بدون اختبارات
- وظائف مع عدم وجود تغطية الاختبار
- مسارات الخطأ لم يتم اختبارها

### جودة الاختبار
- هل التأكيدات ذات معنى؟
- هل الاختبارات تتحقق من السلوك؟
- هل الاختبارات قابلة للصيانة؟

## المبادئ

- **كن بناءًا** - قدم تعليقات قابلة للتنفيذ، وليس مجرد انتقادات
- **عرض الأمثلة** - قم بتضمين الكود ❌ BAD و ✅ GOOD
- **تحديد الأولويات** - المشكلات الحرجة أولاً، والأسلوب أخيرًا
- **الاحتفال** - قم بتضمين ملاحظات إيجابية
- ** أقترحالإصلاحات الأولى** - لا تكتفي بتحديد المشاكل
- **لا يوجد التزام بالقوة** - اسأل دائمًا قبل الإصلاح التلقائي
