# / تعيين

قم بتعيين أو إعادة تعيين قصة لمالك من خلال تحديث الحالة الاختياري. يقوم بتحديث بيانات تعريف القصة ويبث المهمة عبر ناقل الرسائل.

## بداية سريعة

__لا تترجم_0__

## المعلمات

| المعلمة | مطلوب | الافتراضي | الوصف |
|-----------|----------|---------|-------------|
| __غير مترجم_19__ | نعم | - | معرف القصة المطلوب تعيينه (على سبيل المثال، US-0001، US-0042) |
| __غير مترجم_20__ | نعم | - | معرف الوكيل/المالك (على سبيل المثال، AG-UI، AG-API، AG-CI) |
| __غير مترجم_21__ | لا | - | حالة التحديث: `ready`، `in-progress`، `blocked`، `in-review`، `done` |
| __غير مترجم_27__ | لا | - | سياق اختياري حول المهمة |

## الإخراج

يعرض الأمر:

1. **معاينة الفرق** - يعرض التغييرات التي سيتم إجراؤها
2. **التأكيد** - نعم/لا مطالبة للتأكيد
3. **رسالة النجاح** - تؤكد اكتمال المهمة

## سلامة JSON

يستخدم الأمر معالجة JSON الآمنة:

- **لا يستخدم echo/cat مطلقًا** - يستخدم دائمًا أداة jq أو أداة التحرير
- **التحقق من الصحة بعد التحديث** - التحقق من صحة JSON قبل الانتهاء
- **النسخ الاحتياطي عند وجود خطأ** - يمكن الاستعادة من النسخة الاحتياطية في حالة حدوث تلف

## أفضل الممارسات

1. **التعيين فورًا** - لا تترك القصص بدون تعيين
2. ** تشمل المشتركntext** - استخدم ملاحظة لشرح سبب حدوث المهمة
3. **مطابقة الخبرات** - قم بتعيين وكيل يتمتع بالمهارات ذات الصلة
4. **تحميل الرصيد** - توزيع العمل بشكل عادل بين الوكلاء
5. **تحديث الحالة** - فكر في تحديث الحالة عند التعيين
6. **التواصل** - يجب أن يفهم الفريق سبب حدوث إعادة التعيين
