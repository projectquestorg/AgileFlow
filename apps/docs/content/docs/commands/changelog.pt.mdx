# /generate-changelog

Gere automaticamente um changelog a partir de seu histórico de commits e pull requests usando commits convencionais e formato Keep a Changelog.

## Início rápido

```bash
/agileflow:generate-changelog VERSION=2.5.0 SINCE=v2.4.0
```

## Objetivo

Este comando cria changelogs profissionais ao:
- Detectando automaticamente a versão das tags git mais recentes
- Coleta de commits e PRs mesclados desde o último lançamento
- Categorizando alterações (Adicionado/Alterado/Fixo/Removido/Segurança)
- Destacando alterações importantes
- Sugestão de versões semânticas com base nas alterações
- Atualizando __NOTRASLATE_17__ com formatação adequada
- Opcionalmente, criar tags git e versões do GitHub

## Parâmetros

| Parâmetro | Obrigatório | Padrão | Descrição |
|-----------|----------|---------|------------|
| `VERSION` | Não | detecção automática | Número da versão (por exemplo, __NOTRASLATE_19__) |
| `SINCE` | Não | última etiqueta | Ponto de partida (por exemplo, __NOTRASLATE_21__) |
| `FORMAT` | Não | `keepachangelog` | Estilo de formato: `keepachangelog`, `simple`, __NOTTRANSLATE_26__ |
| `AUTO_COMMIT` | Não | `no` | Confirmar automaticamente atualizações do changelog: `yes` ou __NOTRASLATE_30__ |

## Formato do registro de alterações

O comando usa o padrão Keep a Changelog com estas seções:

### [Não lançado]
Mudanças de desenvolvimento ainda não divulgadas (fica sempre no topo)

### [VERSÃO] - DATA
Seções para cada versão:

- **Adicionado** - Nova funcionalidade
- **Alterado** - Mudanças na funcionalidade existente
- **Descontinuado** - Funcionalidade que será removida em breve
- **Removido** - Funcionalidade removida
- **Corrigido** - Correções de bugs
- **Segurança** - Correções de vulnerabilidades de segurança
