# /epic:lista

Muestra todas las epopeyas con estado, seguimiento del progreso y opciones de acción rápida.

## Inicio rápido

```bash
/agileflow:epic:list
```

## Parámetros

| Parámetro | Requerido | Predeterminado | Descripción |
|-----------|----------|---------|-------------|
| `STATUS` | No | - | Filtrar por estado: `active`, `complete`, `on-hold` |

## Ejemplos

### Listar todas las epopeyas

```bash
/agileflow:epic:list
```

Muestra todas las epopeyas con su estado y porcentaje de finalización de la historia.

### Listar solo épicas activas

```bash
/agileflow:epic:list STATUS=active
```

Filtros para mostrar solo las epopeyas actualmente en progreso.

### Listar solo épicas completadas

```bash
/agileflow:epic:list STATUS=complete
```

Muestra epopeyas que tienen todas las historias completadas.

## Formato de salida

El comando muestra epopeyas en una tabla estructurada:

```markdown
## Epics

| Epic | Title | Status | Progress | Owner |
|------|-------|--------|----------|-------|
| EP-0003 | User Dashboard | active | 2/5 (40%) | AG-UI |
| EP-0002 | Authentication | active | 4/6 (67%) | AG-API |
| EP-0001 | Core Setup | complete | 3/3 (100%) | AG-DEVOPS |

---
**Summary**: 3 epics (2 active, 1 complete)
**Stories**: 9/14 complete (64%)
```

## Definiciones de estado

| Estado | Significado | Barra de progreso | Acción |
|--------|---------|------|--------|
| **activo** | En proceso, aceptando nuevas historias | Relleno parcial | Continuar trabajando, agregar historias |
| **completo** | Todas las historias hechas | Completo (100%) | Ver resumen, posiblemente reabrir |
| **en espera** | En pausa, actualmente no trabajado en | Congelado | Reanudar o cerrar |

## Visualización del progreso

El progreso se muestra como:
- **Numerador**: Historias completadas
- **Denominador**: Total de historias en épica
- **Porcentaje**: finalización general

Ejemplo: `4/6 (67%)` = 4 historias terminadas, 2 restantes, 67% completo

## Acciones rápidas

Después de mostrar la lista, puede:

### Ver detalles épicos

Seleccione una epopeya para ver:
- Todas las historias agrupadas por estado (en progreso, listo, listo)
- Visualización del progreso
- Dueño y objetivo
- Investigaciones relacionadas y RAM

```bash
/agileflow:epic:view EPIC=EP-0003
```

### Agregar historia a Epic

Crea una nueva historia bajo una epopeya existente:

```bash
/agileflow:story EPIC=EP-0003 ...
```

### Crear nueva epopeya

Inicie una nueva característica épica:

```bash
/agileflow:epic TITLE="..." ...
```

## Integración del flujo de trabajo

### Planificación de sprints

```bash
/agileflow:epic:list STATUS=active
# See which epics are in progress
/agileflow:epic:view EPIC=EP-0003
# Review stories and their status
/agileflow:sprint
# Plan which stories to work on
```

### Planificación trimestral

```bash
/agileflow:epic:list
# See all epics and their progress
# Identify bottlenecks (epics with less than 50% progress)
# Decide which epics to prioritize next
```

### Seguimiento de finalización

```bash
/agileflow:epic:list STATUS=active
# Work on remaining stories in in-progress epics
/agileflow:epic:list STATUS=complete
# Review what was shipped
```

### Épicas en espera

```bash
/agileflow:epic:list STATUS=on-hold
# Check if any on-hold epics can be resumed
/agileflow:epic:view EPIC=EP-0002
# Understand why it's on hold
```

## Interpretación de resultados

### Alto progreso (más del 80%)

```
EP-0002 | Authentication | active | 5/6 (83%) | AG-API
```

- Epic está casi completo.
- Concéntrate en terminar la última historia.
- Planifica la próxima épica para comenzar.
- No cerrar hasta el 100%

### Progreso medio (40-80%)

```
EP-0003 | User Dashboard | active | 3/7 (43%) | AG-UI
```

- Epic está a mitad del sprint.
- Puede necesitar historias adicionales
- Comprobar si las historias están bloqueadas.
- Considere agregar recursos

### Progreso bajo (menos del 40%)

```
EP-0001 | Onboarding Flow | active | 1/8 (12%) | AG-UI
```

- La epopeya acaba de comenzar
- Aceleración en progreso
- Monitor de bloqueadores
- Es posible que desee agregar más historias.
