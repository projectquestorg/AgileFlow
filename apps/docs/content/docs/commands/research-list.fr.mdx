# /agileflow:recherche:liste

Affichez toutes les notes de recherche stockées dans votre projet avec un aperçu rapide.

## Démarrage rapide

```bash
/agileflow:research:list
```

Aucun argument nécessaire : affiche toutes les notes de recherche.

## But

Affiche les notes de recherche stockées dans `docs/10-research/` avec:

- Date
- Sujet
- Nom de fichier
- Résumé sur une ligne (si disponible)

C'est un **read-only command**-aucun fichier n'est écrit.

## Sortir

```
## Research Notes

| Date       | Topic                  | File                        |
|------------|------------------------|-----------------------------|
| 2025-01-07 | OAuth Integration       | 20250107-oauth.md           |
| 2025-01-06 | Performance Tips        | 20250106-perf.md            |
| 2025-01-05 | Database Migrations     | 20250105-migrations.md      |

Total: 3 research notes

Quick actions:
- /agileflow:research:view FILE=<filename> - Read a specific note
- /agileflow:research:analyze FILE=<filename> - Analyze for implementation
- /agileflow:research:ask TOPIC="new topic" - Create new research prompt
- /agileflow:research:import TOPIC="..." - Import external content
```

## Quand aucune recherche n’existe

Si aucune note de recherche n’est trouvée :

```
No research notes found.

Quick start:
- /agileflow:research:ask TOPIC="your topic" - Generate research prompt for ChatGPT/Claude
- /agileflow:research:import TOPIC="..." - Import external content
```

## Comment ça marche

1. Lit `docs/10-research/README.md` pour la table d'index
2. Si README n'existe pas, répertorie les fichiers du répertoire
3. Si le répertoire n'existe pas, affiche un message utile
4. Suggère les prochaines actions

## Actions rapides

Après avoir consulté la liste, vous pouvez :

| Action | Command |
|--------|---------|
| Read a note | `/agileflow:research:view FILE=<filename>` |
| Analyze for implementation | `/agileflow:research:analyze FILE=<filename>` |
| Create new research | `/agileflow:research:ask TOPIC="..."` |
| Import content | `/agileflow:research:import TOPIC="..."` |

## Quand utiliser

- **Before implementing** - Retrouver des recherches antérieures sur un sujet
- **Looking for code snippets** - Rappelez-vous où vous avez recherché quelque chose
- **Check action items** - Voir quelles tâches sont issues de la recherche
- **Team knowledge sharing** - Parcourez ce qui a été recherché
