# /session:statut

Affichez un tableau formaté de toutes les sessions enregistrées, indiquant celles qui sont actuellement actives.

## Démarrage rapide

```bash
/agileflow:session:status
```

## Exemple de sortie

```
┌─────┬──────────────────────────────┬────────────┬─────────┬────────────┐
│ ID  │ Path                         │ Branch     │ Status  │ Nickname   │
├─────┼──────────────────────────────┼────────────┼─────────┼────────────┤
│ 1   │ /home/user/myproject         │ main       │ active  │ (main)     │
│ 2   │ /home/user/myproject-auth    │ session-2  │ active  │ auth       │
│ 3   │ /home/user/myproject-3       │ session-3  │ stale   │ -          │
└─────┴──────────────────────────────┴────────────┴─────────┴────────────┘

Active: 2    Stale: 1

Current session: 1

To switch sessions, run /agileflow:session:resume
To create new session, run /agileflow:session:new
```

## Valeurs d'état

| Statut | Signification |
|--------|---------|
| `active` | La session a un processus Claude Code en cours d'exécution |
| `stale` | Le fichier de verrouillage existe mais le processus n'est plus en cours d'exécution |
| `(main)` | Le répertoire du projet d'origine (Session 1) |

## Détection de session

Les sessions sont détectées à l'aide du verrouillage basé sur PID :

1. Chaque session crée un fichier de verrouillage avec son ID de processus
2. La vérification de l'état vérifie si chaque PID est toujours en cours d'exécution
3. Les verrous périmés sont automatiquement nettoyés au prochain démarrage

## Cas d'utilisation

### Avant de commencer les travaux

Vérifiez si d'autres sessions travaillent déjà sur quelque chose :

```bash
/agileflow:session:status
# See what's active, decide if you need a new session
```

### Après le retour au travail

Découvrez quelles sessions existent et leur état :

```bash
/agileflow:session:status
# Review sessions, then:
/agileflow:session:resume  # to switch to one
```

### Conflits de débogage

Si vous soupçonnez des conflits de session :

```bash
/agileflow:session:status
# Check for multiple "active" sessions
# Stale sessions may need cleanup
```
