# /histoire:liste

Affichez toutes les user stories avec des options de filtrage et des fonctionnalités d’action rapide.

## Démarrage rapide

```bash
/agileflow:story:list
```

## Paramètres

| Parameter | Required | Default | Description |
|-----------|----------|---------|-------------|
| `EPIC` | No | - | Filter by epic (e.g., EP-0001) |
| `STATUS` | No | - | Filter by status: `ready`, `in_progress`, `blocked`, `done` |
| `OWNER` | No | - | Filter by owner (e.g., AG-UI) |

## Exemples

### Lister toutes les histoires

```bash
/agileflow:story:list
```

Affiche toutes les histoires organisées par épopée avec leur statut.

### Répertorier les histoires pour une épopée spécifique

```bash
/agileflow:story:list EPIC=EP-0001
```

Affiche uniquement les histoires dans l'épopée d'authentification.

### Liste des histoires prêtes

```bash
/agileflow:story:list STATUS=ready
```

Affiche toutes les histoires prêtes à commencer à travailler.

### Liste des histoires en cours

```bash
/agileflow:story:list STATUS=in_progress
```

Affiche les éléments de travail en cours.

### Répertorier les histoires par propriétaire

```bash
/agileflow:story:list OWNER=AG-UI
```

Affiche tous les étages attribués à l'agent d'interface utilisateur.

### Filtres combinés

```bash
/agileflow:story:list EPIC=EP-0001 STATUS=ready
```

Affiche les histoires prêtes uniquement dans l'épopée d'authentification.

```bash
/agileflow:story:list STATUS=blocked OWNER=AG-API
```

Affiche les histoires d'API actuellement bloquées.

## Format de sortie

Les histoires sont organisées par épopée avec des colonnes de statut :

```markdown
## Stories

### EP-0001: Authentication System
| Story | Title | Status | Owner | Estimate |
|-------|-------|--------|-------|----------|
| US-0001 | Login form | done | AG-UI | 2h |
| US-0002 | Password reset | in_progress | AG-API | 3h |
| US-0003 | Session management | ready | AG-API | 4h |

### EP-0002: User Dashboard
| Story | Title | Status | Owner | Estimate |
|-------|-------|--------|-------|----------|
| US-0004 | Dashboard layout | ready | AG-UI | 2h |
| US-0005 | Activity feed | ready | AG-UI | 3h |

---
**Summary**: 5 stories (1 done, 1 in_progress, 3 ready)
```

## Définitions de statut

| Status | Meaning | Action |
|--------|---------|--------|
| **ready** | Ready to start, no blockers | Start work / view details |
| **in_progress** | Currently being worked on | Continue work / mark blocked if needed |
| **blocked** | Waiting for dependency or clarification | Resolve blocker / unblock when ready |
| **done** | Completed and tested | Move to next story |

## Actions rapides

Après avoir affiché la liste, vous pouvez :

### Afficher les détails de l'histoire

Sélectionnez une histoire pour voir :
- Critères d'acceptation complets
- Statut des tests
- Dépendances
- Bloqueurs (le cas échéant)
- Notes d'avancement

```bash
/agileflow:story:view STORY=US-0003
```

### Commencez à travailler sur une histoire

Choisissez une histoire prête et marquez-la comme in_progress :

```bash
/agileflow:status US-0003 STATUS=in_progress
```

### Créer une nouvelle histoire

Ajouter une histoire à une épopée :

```bash
/agileflow:story EPIC=EP-0001 ...
```

## Intégration du flux de travail

### Standup quotidien

```bash
/agileflow:story:list STATUS=in_progress
# See what's being worked on
/agileflow:story:list STATUS=blocked
# See what's blocked
```

### Planification des sprints

```bash
/agileflow:story:list STATUS=ready
# See available work
/agileflow:epic:list
# See epic status
# Select stories for sprint
```

### Débloquer le flux de travail

```bash
/agileflow:story:list STATUS=blocked
# See all blocked stories
# Resolve dependencies
/agileflow:story:view STORY=<blocked>
# See blocker details
/agileflow:status <story> STATUS=ready
# When unblocked, mark ready again
```

### Suivi d'achèvement

```bash
/agileflow:story:list STATUS=done
# See what was completed
/agileflow:story:list STATUS=in_progress
# See what's being worked on
/agileflow:story:list STATUS=ready
# See what's queued up
```

## Stratégies de filtrage

### Par Epic
```bash
/agileflow:story:list EPIC=EP-0001
# Focus on one feature
```

### Par statut
```bash
/agileflow:story:list STATUS=ready
# Find available work
```

```bash
/agileflow:story:list STATUS=blocked
# Find blockers to resolve
```

### Par propriétaire
```bash
/agileflow:story:list OWNER=AG-API
# See what an agent is working on
```

### Par statut + épique
```bash
/agileflow:story:list EPIC=EP-0001 STATUS=ready
# Ready stories in specific epic
```

### Par statut + propriétaire
```bash
/agileflow:story:list STATUS=in_progress OWNER=AG-UI
# Current work for specific agent
```

## Interprétation des résultats

### Distribution d’histoires saines

```
5 stories (1 done, 2 in_progress, 2 ready)
```

- ✅ Les gens ont du travail à faire
- ✅ Le travail coule à flot
- ✅ Le backlog est prêt

### Problèmes potentiels

**Too many in_progress (>3)**
```
5 stories (0 done, 4 in_progress, 1 ready)
```
- Envisagez de réduire les en-cours (travail en cours)
- Concentrez-vous sur la réalisation d'histoires

**No ready stories**
```
5 stories (2 done, 2 in_progress, 0 ready)
```
- Besoin de planifier plus d'histoires
- Créer `/agileflow:story` pour les prochains éléments de travail

**Lots of blocked**
```
7 stories (1 done, 1 in_progress, 3 blocked, 2 ready)
```
- Adressez-vous de toute urgence aux bloqueurs
- Il faudra peut-être débloquer les dépendances
