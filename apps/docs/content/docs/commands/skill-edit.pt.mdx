# /habilidade:editar

Modifique uma habilidade existente em __NOTRASLATE_27__ - atualize instruções, adicione entradas de livros de receitas, altere referências ou reescreva completamente com novas pesquisas.

## Início rápido

```bash
# Interactive mode
/agileflow:skill:edit

# Edit specific skill
/agileflow:skill:edit supabase-swift
```

## O que faz

Permite modificar qualquer aspecto de uma habilidade:

- **SKILL.md** - Atualizar gatilhos e instruções de ativação
- **references.md** - Adicionar ou atualizar links de documentação
- **Entradas do livro de receitas** - Editar fluxos de trabalho de casos de uso específicos
- **Adicionar novo livro de receitas** - Criar fluxos de trabalho para novos casos de uso
- **Configuração MCP** - Atualizar configurações do servidor MCP
- **Renomear habilidade** - Alterar o nome e a pasta da habilidade

## Operações Especiais de Edição

### Reescrita completa (recomendado para alterações importantes)

Quando você seleciona "Reescrita completa":

1. **Entre no Modo de Plano** - O Comando pesquisa as melhores práticas atuais
2. **Colete novas fontes**:
   - Documentação web atualizada
   - Nova documentação local
   - Padrões atuais de base de código
3. **Mostrar plano atualizado** - Você aprova as alterações
4. **Regenerar** – Cria uma nova versão com novas pesquisas
5. **Preservar histórico** - É feito backup da versão antiga

Isso é ideal quando:
- Versão tecnológica conta com grandes atualizações
- Os padrões do seu projeto evoluíram
- Você deseja atualizar todo o conteúdo

### Adicionar nova entrada no livro de receitas

Ao selecionar "Adicionar novo livro de receitas":

1. **Nomeie o usocase** - e.g., "pagination", "bulk-operations"
2. **Choose template** - Cookbook structure is auto-filled
3. **Update SKILL.md** - References are added automatically
4. **Edit if needed** - Refine the generated content

### Update MCP Configuration

When you select "MCP configuration":

1. **View current .mcp.json**
2. **Edit server settings**
3. **Add/remove environment variables**
4. **Test configuration** - Verify syntax

## Best Practices

### When to Edit

Edit a skill when:
- Technology version updates
- Your team discovers new patterns
- New use cases emerge
- Documentation changes
- Errors need fixing

### Keeping Skills Current

1. **Regular audits** - Test skills quarterly
2. **User feedback** - Edit based on what Claude gets wrong
3. **Pattern evolution** - Update as your code patternvai evoluir
4. **Atualizações do Documentos**: mantenha os links de documentos externos atualizados

### Fazendo mudanças eficazes

1. **Pequenas mudanças focadas** - Atualize uma coisa de cada vez
2. **Sempre visualizar** - Verifique as diferenças antes de salvar
3. **Teste após edição** - Execute __NOTRASLATE_30__ após grandes alterações
4. **Documente o porquê** - Adicione comentários explicando as alterações

## Tratamento de erros

### Habilidade não encontrada

```
❌ Skill "<name>" not found.

Available skills:
  - supabase-swift
  - ui-components

Use /agileflow:skill:list to see all skills.
```

**Correção**: verifique a ortografia do nome da habilidade ou liste as habilidades disponíveis.

### Não é possível analisar o Frontmatter

```
⚠️ Could not parse SKILL.md frontmatter.

The skill file may have formatting issues.
Would you like me to try anyway, or edit manually?
```

**Correção**: Verifique se SKILL.md tem frontmatter YAML válido:
```yaml
---
name: skill-name
description: Skill purpose
---
```

### Erro de permissão de gravação

```
❌ Cannot write to skill file.

Check permissions for: .claude/skills/<skill>/
```

**Correção**: verifique as permissões do diretório:
```bash
chmod -R u+w .claude/skills/<skill>/
```

## Após a edição

Depois que as alterações forem salvas:

1. **Teste a habilidade**:
   ```bash
   /agileflow:skill:test <skill-name>
   ```

2. **Usar imediatamente** - As alterações entrarão em vigor no próximo uso

3. **Se houver erros** - Use __NOTRASLATE_31__ novamente para corrigir

4. **Permita o aprendizado** - Se ainda não o fez:
   ```bash
   /agileflow:skill:upgrade <skill-name>
   ```
