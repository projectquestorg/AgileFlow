# /retro

Gerador retrospectivo automatizado que analisa dados do projeto para revelar insights, padrões e melhorias acionáveis usando o formato Iniciar/Parar/Continuar.

## Início rápido

```bash
/agileflow:retro
```

Gera uma retrospectiva das últimas 2 semanas usando dados do projeto.

## Parâmetros

| Parâmetro | Obrigatório | Padrão | Descrição |
|-----------|----------|---------|------------|
| `TIMEFRAME` | Não | 2 semanas | Período de análise: sprint, 2 semanas, 30 dias ou 90 dias |
| `EPIC` | Não | - | Analise apenas épicos específicos (por exemplo, EP-0010) |
| `FORMAT` | Não | asci | Formato de saída: ascii, markdown ou html |
| `SAVE` | Não | verdade | Salve em docs/08-project/retrospectives/ |

## Exemplos

### Retrospectiva padrão de 2 semanas

```bash
/agileflow:retro
```

Analisa as últimas 2 semanas com saída formatada em ASCII, incluindo:
- Resumo do sprint (histórias, velocidade, tempo de ciclo)
- O que correu bem (secção Continuar)
- O que precisa ser melhorado (seções Iniciar/Parar)
- Itens acionáveis por prioridade
-Contribuições da equipe
- Previsões para o próximo sprint

### Retrospectiva dos últimos 30 dias

```bash
/agileflow:retro TIMEFRAME=30d FORMAT=markdown
```

Analisa os últimos 30 dias e salva como descontos para facilitar o compartilhamento.

## Fontes de dados

Analisa dados de 4 fontes:

1. **docs/09-agents/bus/log.jsonl** – Padrões de eventos, transições de status, bloqueadores
2. **docs/09-agents/status.json** - Estado atual, níveis de WIP, propriedade
3. **docs/06-stories/**/US-*.md** - Dados de conclusão, estimativas versus valores reais
4. **Veldados de ocidade** - Pontos concluídos, tendências de rendimento

## Estrutura Retrospectiva

### Continuar (o que deu certo)

Padrões identificados:
- Aumentos de alta velocidade
- Tempos de ciclo rápidos (menos de 2 dias)
- Zero histórias bloqueadas
- Conclusões épicas
- Boa precisão de estimativa (variação inferior a 20%)
- Carga de trabalho equilibrada do agente

### Início (Novas Práticas)

Recomendações para:
- Levantamentos diários para identificar bloqueadores mais cedo
- Dividir histórias em pedaços menores
- Terminar histórias antes de começar novas
- Verificações de dependência pré-sprint
- Blocos de tempo de revisão dedicados
- Planejando pôquer para estimativa
- Treinamento cruzado em áreas de gargalo

### Pare (coisas para eliminar)

Antipadrões detectados:
- Deixar os PRs sem serem revisados
- Iniciando histórias com dependências externas
- Estimar sem entender a complexidade
- Comprometimento excessivo no planejamento do sprint
- Troca de contexto entre histórias
- PRs grandes (>400 linhas)
- Atribuir todo o trabalho a um agente

## Insights de análise

### Detecção de padrões

O comando identificaé:
- **Bloqueadores recorrentes** - Histórias bloqueadas várias vezes
- **Padrões de dia da semana** - Quando a maior parte do trabalho é concluída
- **Padrões de transferência** - Silos de conhecimento ou propriedade pouco clara
- **Correlações de tamanho de história** - Quais tamanhos são mais eficientes
- **Tendências de velocidade** - Melhoria ou diminuição da produtividade

### Momentos de celebração

Destaques vitórias como:
- Conclusões épicas
- Marcos de velocidade (histórias de dois dígitos/semana)
- Zero bugs relatados
- Entregas rápidas (menos de 4 horas)
- 100% de conquistas de cobertura de teste

## Fluxo de trabalho

1. Carregar fontes de dados (logs de barramento, status, dados de história)
2. Analise o que deu certo (velocidade, tempo de ciclo, conclusão)
3. Analise o que precisa ser melhorado (quedas, ciclos longos, WIP)
4. Detecte padrões avançados
5. Gere itens de ação priorizados
6. Crie uma seção de celebração
7. Gere um relatório abrangente
8. Salve no diretório de retrospectivas
9. Atualize o índice com a entrada mais recente

## Use quando

- **Limites do sprint** - Após cada sprint ou iteração
- **Conclusões épicas** - Retrospectivano épico completo
- **Avaliações da equipe** - Check-ins mensais ou trimestrais
- **Melhoria de processos** - Identificação de padrões ao longo do tempo
- **Antes de planejar** - Aplicar o aprendizado no próximo sprint
- **Contexto das métricas** - Entenda o que os números significam
