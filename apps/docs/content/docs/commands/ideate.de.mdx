## Ausgabestruktur

### Ideenbericht (`output.md`)

The command generates `docs/08-project/ideation-<YYYYMMDD>.md` with:

1. **Summary**: Totale Ideen, Vertrauensverlust
2. **High-Confidence Improvements**: Ideen, die von mehr als 2 Experten vereinbart wurden (diese priorisieren)
3. **Medium-Confidence Opportunities**: Einzelner Experte mit Beweisen (eine Untersuchung lohnt sich)
4. **Summary by Category**: Tabelle der Ideenanzahl pro Domain
5. **Recommended Next Steps**: Implementierungsanleitung

### Vertrauensbewertung

| Confidence | Criteria | Action |
|------------|----------|--------|
| **HIGH** | 2+ experts suggest similar idea | Include prominently, recommend immediate action |
| **MEDIUM** | 1 expert with specific evidence (file paths, metrics) | Include as opportunity |
| **LOW** | 1 expert, vague/no evidence | Exclude from report |

## Wie es funktioniert

1. **Parse Arguments**: Bestimmt die einzusetzenden Experten und die Analysetiefe
2. **Deploy Experts in Parallel**: Startet alle Experten gleichzeitig
3. **Each Expert Generates Ideas**: 3-5 konkrete, umsetzbare Verbesserungsideen
4. **Collect Results**: Wartet auf den Abschluss aller Experten
5. **Synthesize**: Gruppiert sich √ºberschneidende Ideen und bewertet das Vertrauen
6. **Generate Output**: Erstellt einen Bericht (und optionale Storys)
