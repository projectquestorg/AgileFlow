# /modelo

Crie, gerencie e use modelos personalizados para padronizar a documentação em todo o seu projeto. Crie modelos para histórias, reuniões, relatórios de incidentes e muito mais.

## Início rápido

```bash
# List all available templates
/agileflow:template

# Create a new custom template
/agileflow:template ACTION=create TYPE=custom NAME=meeting-notes

# Use a template to generate a document
/agileflow:template ACTION=use TYPE=custom NAME=meeting-notes
```

## Parâmetros

| Parâmetro | Obrigatório | Padrão | Descrição |
|-----------|----------|---------|------------|
| `ACTION` | Não | lista | Operação: listar, criar, editar ou usar |
| `TYPE` | Para criar/editar/usar | - | Tipo de modelo: história, épico, anúncio, personalizado, etc. |
| `NAME` | Para criar/editar/usar | - | Nome do modelo (por exemplo, notas de reunião) |
| `PATH` | Opcional | automóvel | Caminho personalizado para salvar o modelo |

## Sintaxe do modelo

### Formato variável

Os modelos usam chaves duplas para variáveis:

```markdown
# {{TITLE}}

**Created by**: {{AUTHOR}}
**Date**: {{DATE}}
**Status**: {{STATUS}}
```

### Variáveis incorporadas

Disponível em todos os modelos:

| Variável | Exemplo | Descrição |
|----------|------------|-------------|
| `{{DATE}}` | 24/12/2025 | Data atual (AAAA-MM-DD) |
| `{{DATETIME}}` | 2025-12-24T10:30:00Z | Data e hora ISO 8601 |
| `{{USER}}` | Alice | Usuário git atual |
| `{{YEAR}}` | 2025 | Ano atual |

## Melhores práticas

### Quando criar modelos

Crie um modelo quando:
- Um tipo de documento é criadovárias vezes
- Uma estrutura específica deve ser seguida de forma consistente
- Certos campos devem sempre ser incluídos

### Nomenclatura de modelo
