# /session:status

Zeigt eine formatierte Tabelle aller registrierten Sitzungen an und zeigt an, welche derzeit aktiv sind.

## Schnellstart

```bash
/agileflow:session:status
```

## Beispielausgabe

```
┌─────┬──────────────────────────────┬────────────┬─────────┬────────────┐
│ ID  │ Path                         │ Branch     │ Status  │ Nickname   │
├─────┼──────────────────────────────┼────────────┼─────────┼────────────┤
│ 1   │ /home/user/myproject         │ main       │ active  │ (main)     │
│ 2   │ /home/user/myproject-auth    │ session-2  │ active  │ auth       │
│ 3   │ /home/user/myproject-3       │ session-3  │ stale   │ -          │
└─────┴──────────────────────────────┴────────────┴─────────┴────────────┘

Active: 2    Stale: 1

Current session: 1

To switch sessions, run /agileflow:session:resume
To create new session, run /agileflow:session:new
```

## Statuswerte

| Status | Bedeutung |
|--------|---------|
| `active` | Die Sitzung verfügt über einen laufenden Claude-Code-Prozess |
| `stale` | Sperrdatei existiert, aber Prozess läuft nicht mehr |
| `(main)` | Das ursprüngliche Projektverzeichnis (Sitzung 1) |

## Sitzungserkennung

Sitzungen werden mithilfe der PID-basierten Sperrung erkannt:

1. Jede Sitzung erstellt eine Sperrdatei mit ihrer Prozess-ID
2. Die Statusprüfung überprüft, ob jede PID noch läuft
3. Veraltete Sperren werden beim nächsten Start automatisch gelöscht

## Anwendungsfälle

### Vor Arbeitsbeginn

Überprüfen Sie, ob andere Sitzungen bereits an etwas arbeiten:

```bash
/agileflow:session:status
# See what's active, decide if you need a new session
```

### Nach der Rückkehr zur Arbeit

Finden Sie heraus, welche Sitzungen vorhanden sind und welchen Status sie haben:

```bash
/agileflow:session:status
# Review sessions, then:
/agileflow:session:resume  # to switch to one
```

### Konflikte debuggen

Wenn Sie Sitzungskonflikte vermuten:

```bash
/agileflow:session:status
# Check for multiple "active" sessions
# Stale sessions may need cleanup
```

## Verwandtes Kommands

- [`/session:new`](/commands/session-new) – Erstellen Sie eine neue parallele Sitzung
- [`/session:resume`](/commands/session-resume) – Wechseln Sie zu einer anderen Sitzung
- [`/session:end`](/commands/session-end) – Aktuelle Sitzung beenden
- [`/session:init`](/commands/session-init) – Sitzungssystem initialisieren
