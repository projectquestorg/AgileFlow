# /verificar

Execute os testes do seu projeto e atualize automaticamente o status do teste da história no seu sistema de rastreamento. Verify é um componente central do conjunto de sessões do AgileFlow que garante a validação do teste em todo o fluxo de trabalho de desenvolvimento.

## Início rápido

```bash
# Run all tests and update all in-progress stories
/agileflow:verify

# Run tests and update specific story
/agileflow:verify US-0042
```

O comando irá:
1. Carregue a configuração de teste do seu projeto
2. Execute o conjunto de testes
3. Analise os resultados do teste
4. Atualizar o status do teste de história (aprovado/reprovado)
5. Exiba um relatório claro com indicadores visuais

## Principais recursos

### Análise automática de teste

O Verify detecta e analisa automaticamente resultados de testes de:

| Estrutura | Exemplo de saída |
|-----------|-----------------|
| Jest/Vitest (Node.js) | `Tests: 42 passed, 42 total` |
| Pytest (Python) | `42 passed in 2.5s` |
| Carga (ferrugem) | `test result: ok. 42 passed` |
| Vá testar | `ok github.com/user/project` |

Se a análise falhar, ela retornará ao código de saída (0 = aprovação, diferente de zero = falha).

### Atualizações de status da história

Os resultados dos testes são registrados automaticamente em suas histórias:

```json
{
  "story_id": "US-0042",
  "test_status": "passing",
  "test_results": {
    "last_run": "2025-12-06T10:30:00Z",
    "command": "npm test",
    "passed": 42,
    "failed": 0,
    "exit_code": 0,
    "output_summary": "All tests passed (42/42)"
  }
}
```

### Multi-StRastreamento de memória

Sem um ID de história específico, a verificação atualiza todas as __NOTRASLATE_29__ histórias:

```bash
/agileflow:verify
# Updates test_status for:
# - US-0042: passing
# - US-0043: passing
# - US-0044: failing
```

## Pré-requisitos

A verificação requer:

- **Arnês de sessão inicializado** - Execute __NOTRASLATE_35__ primeiro
- **docs/00-meta/environment.json** - Arquivo de configuração de teste
- **docs/09-agents/status.json** - Acompanhamento de histórias
- **Testes de trabalho** - Seu projeto deve ter um conjunto de testes

### Inicializar o chicote da sessão

Se você receber um erro "Sessão de chicote não inicializada":

```bash
/agileflow:session:init
```

Isto irá:
- Detecte seu tipo de projeto
- Crie ambiente.json com configuração de teste
- Configure a verificação de teste para todas as histórias

## Integração com outros comandos

Verify é usado automaticamente por:

| Comando | Quando |
|--------|------|
| `/babysit` | Após implementar alterações de código |
| `/baseline` | Antes de criar pontos de verificação verificados |
| `/session:resume` | No início da sessão de trabalho |
| `/pr` | Pode incluir o status do teste na descrição do PR |

Você também pode executar verify manualmente a qualquer momento para verificar o status.

```text

Run /agileflow:session:init to set up test verification
```

**Correção:**

### "Testes não encontrados"

**Erro:**
__NOTRADUZIR_19__

**Correção:**
- Instale dependências: __NOTRASLATE_40__
- Verifique se o comando de teste em Environment.json corresponde à sua configuração
- Verifique PATH inclui as ferramentas necessárias

### "História não encontrada"

**Erro:**
```text
⚠️  Story US-0099 not found in status.json
Available in_progress stories:
- US-0042
- US-0043
```

**Correção:**
- Verifique se o ID da história está correto (verifique status.json)
- Execute sem ID da história para verificar todas as histórias em andamento

### "Status.json inválido"

**Erro:**
```text
❌ Cannot Update Status
File docs/09-agents/status.json is invalid or missing.
```

**Correção:**
```bash
# Re-initialize AgileFlow setup
/agileflow:setup
```

## Dicas

### Execute testes com frequência

```bash
# After every major change
/agileflow:verify

# Before committing
/agileflow:verify

# Before creating PR
/agileflow:verify

# Before baseline
/agileflow:verify
```

### Capture regressões antecipadamente

O Verify rastreia automaticamente o status do teste por história, para que você possa:
-Veja quando os testes são interrompidos (mudanças no test_status)
- Identifique qual história introduziu o fracasso
- Corrija os problemas antes que eles se acumulem

### Use com linha de base

```bash
# Ensure all tests pass
/agileflow:verify

# If all passing:
/agileflow:baseline "Milestone complete"

# If any failing:
# (Fix first, then baseline)
```
