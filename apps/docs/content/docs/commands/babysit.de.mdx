# /babysitten

Ein interaktiver Mentor, der Sie durch die Implementierung von Funktionen führt, von einfachen Anforderungen bis hin zu fertigen Pull-Anfragen. Babysit kombiniert automatisierte Kontexterfassung, intelligente Aufgabenvorschläge und praktische Codeunterstützung.

## Schnellstart

```bash
/agileflow:babysit
```

Der Befehl wird:
1. Sammeln Sie den gesamten Projektkontext (Geschichten, Epen, Tests, Dokumentation).
2. Präsentieren Sie intelligente Aufgabenvorschläge basierend auf Priorität und Bereitschaft
3. Führen Sie Sie Schritt für Schritt durch die Umsetzung
4. Aktualisieren Sie Ihren Projektstatus, während Sie arbeiten
5. Erstellen Sie PR-Beschreibungen, wenn Sie bereit sind

## Hauptmerkmale

### Intelligente Aufgabenvorschläge

Wenn du anfängst `/babysit`, analysiert es Ihr Projekt und empfiehlt:

- **READY stories** (alle AC abgeschlossen, Tests unterbrochen) – Höchste Priorität
- **Blocked stories** (mit freien Entsperrpfaden) – Hohe Priorität
- **Near-complete epics** (Über 80 % der Geschichten sind fertig) – Beenden Sie, was Sie begonnen haben
- **Roadmap priorities** - Wichtige strategische Arbeit
- **Research gaps** - Vorbereitende Arbeiten, um ein späteres Debuggen zu vermeiden

Aufgaben werden nach den Prioritäten eines echten Entwicklers geordnet, nicht nur nach Aktualität oder Reihenfolge.

### Praktische Implementierungsanleitungen

Für jede Geschichte oder jedes Feature:

```text
1. Validate story readiness (acceptance criteria, tests)
2. Read architecture context relevant to the task
3. Plan implementation in small steps
4. Apply code changes incrementally with diffs
5. Run tests after each change
6. Update project status as you progress
7. Generate PR description when complete
```

### Agent-Experten-Spawning

Bei komplexen Aufgaben erkennt Babysit automatisch die Domäne und generiert Spezialisten:

- **Database work** → AG-DATABASE-Experte
- **API endpoints** → AG-API-Experte
- **UI components** → AG-UI-Experte
- **Testing** → AG-TESTING-Experte
- **Documentation** → AG-DOKUMENTATIONSexperte

Spezialisten kümmern sich um die konzentrierte Tiefenarbeit, während der Babysitter den Gesamtablauf koordiniert.

### Fortschrittsverfolgung

Alle Arbeiten werden mit einer Todo-Liste verfolgt:

```text
1. Run context script (loads all project state)
2. Present task suggestions
3. Read task-specific documentation
4. Validate story readiness
5. Plan implementation steps
6. Apply code changes with tests
7. Update project status
8. Generate PR description
9. Mark complete
```

Sie können genau sehen, wo Sie sind und was noch übrig ist.

## Verwendung

### Einfach – Lassen Sie den Babysitter Arbeit vorschlagen

```bash
/agileflow:babysit
```

Babysit analysiert Ihr Projekt und fragt, woran Sie arbeiten möchten.

### Bestehende Arbeit fortsetzen

```bash
/agileflow:babysit
```

Wenn Sie Geschichten haben `in_progress` Status, Babysitter wird anbieten, sie fortzusetzen.

### Arbeiten Sie an einer bestimmten Geschichte

Sie können Babysit sagen: „Ich möchte an US-0042 arbeiten“ und es wird:
- Laden Sie die Story-Datei
- Bestätigen Sie, dass es bereit ist
- Planen Sie die Umsetzung
- Führt Sie durch den Aufbau

## Parameter

Babysit akzeptiert Freiform-Eingaben in natürlicher Sprache:

| Input | Effect |
|-------|--------|
| `"Add user profile page"` | Create new feature or find matching story |
| `"US-0042"` | Work on specific story |
| `"Continue"` | Resume the last story you were working on |
| `"What should I work on?"` | Get recommendations (default) |

## Autonomer Schleifenmodus

Babysit unterstützt die autonome Ausführung durch **Loop Mode**, das die Geschichten eines gesamten Epos automatisch verarbeitet.

### Schnellstart

```bash
/agileflow:babysit EPIC=EP-0042 MODE=loop MAX=20
```

### Wie es funktioniert

```text
1. Initialize loop for epic
2. Load first "ready" story
3. Work on implementation
4. When Claude stops → Stop hook runs tests
   - If tests PASS → mark story complete, load next
   - If tests FAIL → show errors, continue fixing
5. Repeat until epic complete or max iterations
```

### Parameter

| Parameter | Required | Default | Description |
|-----------|----------|---------|-------------|
| `EPIC` | Yes | - | Epic ID to process (e.g., EP-0042) |
| `MODE` | Yes | - | Must be `loop` for autonomous mode |
| `MAX` | No | 20 | Maximum iterations (safety limit) |

### Schleifensteuerung

```bash
# Check loop status
node scripts/ralph-loop.js --status

# Stop the loop manually
node scripts/ralph-loop.js --stop

# Reset loop state
node scripts/ralph-loop.js --reset
```

### Wann sollte der Loop-Modus verwendet werden?

**Good for:**
- Ein klar definiertes Epos mit klaren Geschichten verarbeiten
- Testgetriebene Entwicklung (Tests definieren „erledigt“)
- Stapelverarbeitung mehrerer Geschichten

**Not good for:**
- Sondierungsarbeit ohne klare Akzeptanzkriterien
- Geschichten, die eine menschliche Überprüfung erfordern, bevor fortgefahren wird
- Komplexe Multi-Domain-Arbeit, die Koordination erfordert

Eine umfassende Dokumentation zum Loop-Modus, einschließlich Visual Mode und Coverage Mode, finden Sie im [Funktionshandbuch für den Loop-Modus](/features/loop-mode).

### Beispielsitzung

```text
/agileflow:babysit EPIC=EP-0042 MODE=loop MAX=20

→ Ralph Loop Initialized
  Epic: EP-0042
  Stories: 5 ready, 7 total
  Starting: US-0015 "Add user authentication"

[Claude implements US-0015...]

→ Stop Hook: Running tests...
  ✓ Tests pass
  ✓ Marked US-0015 complete
  → Moving to US-0016

[Claude implements US-0016...]

→ Stop Hook: Running tests...
  ✗ 2 tests failing
  → Fix errors and continue

[Claude fixes tests...]

→ Stop Hook: Running tests...
  ✓ Tests pass
  ✓ Marked US-0016 complete
  → Moving to US-0017

[...continues until epic complete...]

→ EPIC COMPLETE!
  EP-0042 finished in 12 iterations
  5 stories completed
```

## Beispiele

### Beginnen Sie mit der neuen Implementierung

```bash
/agileflow:babysit
# Babysit suggests: "US-0042: User Login API is READY"
# You confirm → Implementation begins
# Babysit guides through: Plan → Code → Test → PR
```

### Schließe mehrere verwandte Geschichten ab

```bash
/agileflow:babysit
# Babysit shows: "EP-0005 is 80% done. 1 story left: US-0051"
# You accept → Start final story
# Babysit tracks progress
# When done → Ready for baseline or release
```

### Unbekannte Anforderungen

```bash
/agileflow:babysit
# You: "I want user authentication but not sure the best way"
# Babysit: "Should I research JWT vs sessions first? This prevents debugging later"
# You: "Yes, research"
# Babysit: Runs /agileflow:research:ask, saves findings, then plans implementation
```

## Workflow-Übersicht

```text
Start → Gather Context → Suggest Tasks → Choose Task
  ↓
Read Story & Architecture → Validate Readiness → Plan Steps
  ↓
Implement Code → Run Tests → Update Status
  ↓
Generate PR → Finish
```

## Definition von „Bereit“.

Vor der Implementierung prüft Babysit, ob die Geschichten Folgendes haben:

- ✓ Akzeptanzkriterien (Format „Gegeben/Wann/Dann“)
- ✓ Architekturkontext mit Quellenangaben
- ✓ Test-Stubs in docs/07-testing/
- ✓ Abhängigkeiten gelöst und dokumentiert
- ✓ Frühere Erkenntnisse aus verwandten Geschichten

Wenn welche fehlen, schlägt Babysit vor, diese zuerst zu vervollständigen (Geschichten bleiben „Entwurf“, bis sie fertig sind).

## Kontextladen

Wenn Sie babysit starten, wird dieses Kontextskript automatisch ausgeführt:

```bash
node .agileflow/scripts/obtain-context.js babysit
```

Dieser einzelne Befehl erfasst Folgendes:
- Git-Status, Zweig, nicht festgeschriebene Änderungen
- Alle Geschichten und Epen von status.json
- Sitzungsstatus und aktive Sitzungen
- Dokumentationsstruktur
- Forschungsnotizen (Neueste zuerst)
- Aktuelle Agentennachrichten
- Vorhandensein der Schlüsseldatei

Ein Skriptaufruf, umfassender Kontext, schneller Start.

## Wann zu verwenden

### Am besten für

- **Starting implementation** - Brauchen Sie eine Anleitung, woran Sie arbeiten sollen?
- **Picking up mid-feature** - Fahren Sie dort fort, wo Sie aufgehört haben
- **Epic completion** - Beenden Sie die letzten Geschichten in einem Feature-Set
- **Uncertain requirements** - Recherchieren Sie Fragen vor dem Codieren
- **Multi-step features** - Komplexe dateiübergreifende Arbeit
- **Testing validation** - Stellen Sie sicher, dass die Tests vor PR bestanden werden

### Nicht für

- **Quick fixes** (Tippfehler, Einzeiler) – Machen Sie sie einfach selbst
- **Pure research** - Verwenden `/agileflow:research:ask` stattdessen
- **Architectural decisions** - Verwenden `/agileflow:adr` stattdessen
- **Documentation only** - Verwenden `/agileflow:docs` stattdessen

## Ausgabestil

Babysit verwendet klare, umsetzbare Ergebnisse:

- **Headings** für Hauptabschnitte
- **Short bullets** für Optionen und Erklärungen
- **Code blocks** für Unterschiede und Beispiele
- **Questions** um Ihre Entscheidungen zu leiten (fordert Sie nie zum „Eintippen“ auf)
- **Progress updates** während Sie arbeiten

Jede Antwort endet mit einer Frage zum nächsten Schritt, sodass Sie immer die Kontrolle behalten.

## Tipps

### Holen Sie sich bessere Vorschläge

- Behalten Sie Ihr Story-Backlog in status.json
- Schreiben Sie klare Akzeptanzkriterien
- Markieren Sie Geschichten als fertig, wenn sie abgeschlossen sind
- Halten Sie die Architekturdokumente aktuell

### Erfolgreiche Umsetzungen

- Lassen Sie den Babysitter zuerst die Story-Bereitschaft überprüfen
- Lesen Sie den Abschnitt zum Architekturkontext
- In kleinen Schritten umsetzen (nicht alles auf einmal)
- Führen Sie nach jeder Änderung Tests durch
- Bitten Sie um Hilfe, wenn Sie nicht weiterkommen

### Multi-Experten-Orchestrierung

Bei komplexen Funktionen führt Babysit automatisch Folgendes durch:
- Spawn-API-Experte für Backend-Arbeit
- Spawn-UI-Experte für Frontend-Arbeit
- Führen Sie beide parallel aus, wenn sie unabhängig sind
- Koordinieren Sie die endgültige Integration

Sie müssen nicht mehrere Agenten manuell verwalten – babysit übernimmt das.

## Gemeinsame Muster

### „Ich weiß nicht, woran ich arbeiten soll“

```text
/agileflow:babysit
→ Babysit shows ranked suggestions
→ Pick the one that interests you
→ Let babysit guide the implementation
```

### „Ich möchte dieses Epos beenden“

```text
/agileflow:babysit
→ "EP-0005 is 80% done, 2 stories left"
→ Choose story
→ Complete it with babysit's guidance
→ Move to next story
→ When done: /agileflow:baseline "EP-0005 complete"
```

### „Das muss ich erst recherchieren“

```text
/agileflow:babysit
→ "Need to understand JWT patterns first?"
→ "Yes, research"
→ /agileflow:research:ask TOPIC="JWT best practices"
→ Copy research results back
→ Babysit plans implementation based on research
→ Code with confidence
```
