# Commandes

AgileFlow fournit **74 commandes slash** que vous pouvez utiliser directement dans Claude Code. Toutes les commandes suivent le modèle `/agileflow:command-name`.

## Catégories de commandes

### Planification

Commandes pour créer et organiser le travail :

| Commande | Descriptif |
|---------|-------------|
| [`/ideate`](/commands/ideate) | Générer des idées d'amélioration à l'aide d'une analyse multi-experts |
| [`/epic`](/commands/epic) | Créer une nouvelle épopée avec des histoires (sous-commandes : list, view) |
| [`/story`](/commands/story) | Créer une user story avec des critères d'acceptation (sous-commandes : list, view) |
| [`/sprint`](/commands/sprint) | Planification de sprint basée sur les données avec prévision de vitesse |
| [`/deps`](/commands/deps) | Visualisez le graphique de dépendance avec la détection du chemin critique |
| [`/auto`](/commands/auto) | Générez automatiquement des histoires à partir de PRD, de maquettes ou de spécifications |

## Modèles d'utilisation

### Utilisation de base

```bash
# Simple command
/agileflow:board

# Command with required parameters
/agileflow:status STORY=US-0001 STATUS=in-progress

# Command with optional parameters
/agileflow:metrics TIMEFRAME=30d EPIC=EP-0001 FORMAT=markdown
```

### Conventions des paramètres

- **Paramètres obligatoires** : `PARAM=<value>` - Doit être fourni
- **Paramètres facultatifs** : `[PARAM=<value>]` - Peut être omis pour les valeurs par défaut
- **ID** : utilisez des modèles tels que `EP-0001`, `US-0001`, `AG-UI`
- **Listes** : valeurs séparées par des virgules là où elles sont indiquées

### Obtenir de l'aide

```bash
# Show all available commands
/agileflow:help

# System diagnostics
/agileflow:diagnose
```
