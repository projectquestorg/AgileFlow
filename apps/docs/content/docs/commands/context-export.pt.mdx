# /agileflow:context:exportar

Exporte um trecho de contexto conciso para colar em ferramentas de IA da web.

## Início rápido

```bash
/agileflow:context:export
```

## Objetivo

Lê `docs/context.md` e gera um trecho condensado (≤300 linhas) pronto para colar no ChatGPT, Perplexity, Gemini ou Claude web.

**Este é um comando somente leitura** - nenhum arquivo é gravado.

## Formato de saída

O comando gera um bloco de código markdown para facilitar a cópia:

```markdown
# Project Context Brief
Last updated: 2025-12-24

## What We're Building
AgileFlow is a comprehensive project management system...

## Current Focus
- EP-0010: Documentation System (in_progress)
- US-0042: Add Loop Mode docs (in_progress)

## Tech Stack
- Framework: Next.js 16, TypeScript
- UI: React, Tailwind CSS, shadcn/ui
- Testing: Jest, Playwright

## Key Decisions
- ADR-001: Use TypeScript for type safety (Accepted)
- ADR-002: Use React component library (Accepted)

## Feature Map
| Epic | Status | Stories |
|------|--------|---------|
| EP-0010 | in_progress | 5 total, 3 done |
| EP-0009 | complete | 7 total, 7 done |

## Next Steps
- US-0043: Visual Mode documentation (ready)
- US-0044: Session commands docs (ready)

---
**Paste this into**: ChatGPT, Perplexity, Gemini, or Claude web
**Purpose**: Load full project context for AI assistance
**Limit**: This excerpt is ≤300 lines for easy pasting
```

## Seções incluídas

A exportação inclui apenas estas seções (para ficar abaixo de 300 linhas):

| Seção | Linhas máximas | Finalidade |
|--------|-----------|---------|
| Cabeçalho | 2 | Título e carimbo de data/hora |
| O que estamos construindo | 5 | Visão geral do projeto |
| Foco Atual | 5 | Trabalho ativo |
| Pilha de tecnologia | 8 | Tecnologias |
| Principais decisões | 10 | Apenas títulos ADR |
| Mapa de recursos | 15 | Uma linha por épico |
| Próximas etapas | 15 | Histórias prontas |
| Rodapé | 3 | Instruções para colar |

## Seções excluídas

Para permanecer dentro do limite de 300 linhas, estes são ignorados:

- **Progresso recente** -Muito detalhado
- **Seção de notas** - Muito longo
- **Descrições detalhadas da história** - Não é necessário para contextualizar
- **Conteúdo ADR completo** - Apenas títulos incluídos

## Condensação de conteúdo

Ao extrair, o conteúdo é condensado:

| Elemento | Tamanho completo | Condensado |
|--------|-----------|-----------|
| ADRs | Descrição completa | Apenas título + status |
| Histórias | Título + descrição | Título + status |
| Tecnologia | Lista completa + versões | Estrutura + dependências principais |
| Épicos | Título + lista de histórias | Título + contagem + status |
| Decisões | Contexto completo | Uma linha |

## Requisitos

**docs/context.md deve existir** antes de exportar.

Se estiver faltando, você verá:

```
Error: docs/context.md not found.

To create it, run: /agileflow:context:full

Then you can export with: /agileflow:context:export
```

## Por que 300 linhas?

As ferramentas de IA da Web têm limites de janela de contexto. Um trecho de 300 linhas:

- Adapta-se confortavelmente à maioria das janelas de contexto de ferramentas de IA
- Fornece detalhes suficientes para assistência informada
- Deixa espaço para suas perguntas reais
- É rápido para colar e processar

## Quando usar

- **Antes das sessões de IA da web** - Carregar projetocontexto do CT
- **Ao trocar de ferramentas de IA** - Transferir contexto entre ferramentas
- **Após atualizações de contexto** - Obtenha uma versão nova e concisa
- **Para compartilhamento em equipe** - Visão geral rápida do projeto
