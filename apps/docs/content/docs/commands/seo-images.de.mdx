# /seo:images

Analysieren Sie die Bildoptimierung auf einer Seite: Alternativtextqualität, explizite Größenanpassung zur CLS-Verhinderung, moderne Formate (WebP/AVIF), Lazy Loading, LCP-Bildpriorität und reaktionsfähige Bilder.

## Schnellstart

```bash
/agileflow:seo:images https://example.com                    # Image audit for homepage
/agileflow:seo:images https://example.com/products/item       # Product page images
```

| Parameter | Required | Description |

## Alt-Text-Bewertung

| Quality | Criteria | Score Impact |
|---------|----------|------------|
| **Good** | Descriptive, 10-125 chars, contextual keyword | +points |
| **Acceptable** | Present but generic ("image", "photo") | Neutral |
| **Poor** | Filename-based ("IMG_001.jpg", "screenshot") | -2 points |
| **Missing** | No alt attribute at all | -3 points (HIGH severity) |
| **Decorative** | Empty alt="" on decorative images | Correct (no deduction) |
| **Keyword-stuffed** | Excessive keywords crammed in | -2 points |

## Alt-Text-Richtlinien

Guter Alternativtext:
- Beschreibt, was das Bild zeigt
- 10–125 Zeichen empfohlen
- Enthält natürlich relevante Schlüsselwörter
- Sagt nicht „Bild von“ oder „Bild von“
- Für dekorative Bilder verwenden Sie `alt=""`

Beispiel für einen guten Alt-Text:
- „Berufstätige Frau tippt am Laptop am modernen Schreibtisch“
- „Screenshot des Google Search Console-Dashboards mit Traffic-Metriken“
- „Produktfoto: Blaue kabellose Kopfhörer mit Geräuschunterdrückung“

Beispiel für einen schlechten Alternativtext:
- „IMG_001.jpg“ ❌
- „Bild“ ❌
- „Bild von Sachen“ ❌
- „blue-headphones-wireless-noise-cancelling-buy-now“ (Keyword-Stuffing) ❌

## Empfehlungen zum Bildformat

| Format | Use When | Not When |
|--------|----------|----------|
| WebP | Modern format, wide support | Supporting very old browsers |
| AVIF | Best compression, growing support | Older browser support needed |
| JPEG | Photos, realistic images | Transparency needed |
| PNG | Transparency needed | Photos (use JPEG) |
| GIF | Animations | Use MP4 video instead |
| SVG | Icons, illustrations, logos | Photos |
| BMP/TIFF | Never | Never |

## Responsive Bilder

Für eine optimale Mobil- und Desktop-Bereitstellung:

```html
<!-- Good: Responsive with srcset and sizes -->
<img
  src="image-800.jpg"
  srcset="image-400.jpg 400w, image-800.jpg 800w, image-1200.jpg 1200w"
  sizes="(max-width: 600px) 400px, (max-width: 900px) 800px, 1200px"
  width="800"
  height="600"
  alt="Descriptive alt text"
  loading="lazy"
>

<!-- Better: Format alternatives with <picture> -->
<picture>
  <source type="image/avif" srcset="image.avif">
  <source type="image/webp" srcset="image.webp">
  <img src="image.jpg" alt="Descriptive alt text" width="800" height="600">
</picture>
```

## LCP-Bildoptimierung

Für deinen Helden/größtes Bild:

```html
<!-- Good: LCP image gets priority -->
<img
  src="hero.jpg"
  alt="Hero section describing main benefit"
  width="1200"
  height="600"
  fetchpriority="high"
  loading="eager"
/>

<!-- With preload for extra speed -->
<link rel="preload" as="image" href="hero.jpg">
<img src="hero.jpg" alt="..." width="1200" height="600">
```

## Remediation Examples

### Fix Missing Alt Text
```html
<!-- Vorher -->
<img src="hero.jpg">

<!-- Nachher -->
<img src="hero.jpg" alt="Team collaborating on project strategy in modern office">
```

### Fix Missing Width/Height
```html
<!-- Vorher (CLS-Risiko) -->
<img src="photo.jpg" alt="...">

<!-- After (verhindert Verschiebung) -->
<img src="photo.jpg" alt="..." width="800" height="600">
```

### Add Lazy Loading
```html
<!-- Vorher -->
<img src="below-fold.jpg" alt="...">

<!-- Nachher -->
<img src="below-fold.jpg" alt="..." loading="lazy">
```

### Add Format Alternatives
```html
<!-- Vorher -->
<img src="image.jpg" alt="..." width="800" height="600">

<!-- Nachher (40 % kleiner in modernen Browsern) -->
<picture>
  <source type="image/webp" srcset="image.webp">
  <img src="image.jpg" alt="..." width="800" height="600">
</picture>
```

### Add LCP Priority
```html
<!-- Vorher -->
<img src="hero.jpg" alt="..." width="1200" height="600">

<!-- Nach (schnelleres Malen) -->
<img src="hero.jpg" alt="..." width="1200" height="600" fetchpriority="high">
```

## Auswirkungen auf die Leistung

Die Optimierung von Bildern kann Folgendes verbessern:
- **Core Web Vitals**
  - LCP: Schnellere Bereitstellung von Heldenbildern
  - CLS: Verhindert Layoutverschiebungen durch Bilder ohne Größe
- **Page Speed**: 20–40 % schnelleres Laden mit modernen Formaten
- **User Experience**: Schnelleres Laden der Seite, bessere wahrgenommene Leistung
- **SEO**: Besserer Core Web Vitals-Score → potenzieller Ranking-Boost

## Zusammenfassung der Best Practices

1. **Always include alt text** - Barrierefreiheit + Bildsuche
2. **Always include width/height** - Verhindert Layoutverschiebungen (CLS)
3. **Use modern formats** - WebP für 30 % kleiner, AVIF für 50 %+
4. **Lazy load below-fold** - `loading="lazy"` verschiebt unkritische Bilder
5. **Prioritize LCP image** - `fetchpriority="high"` auf Held
6. **Make responsive** - `srcset` Und `sizes` für gerätegerechte Größen
7. **Test before publish** - Nutzen Sie PageSpeed ​​Insights, um Verbesserungen zu überprüfen
