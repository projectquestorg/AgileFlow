# /estado

Actualice el estado de la historia en el sistema de seguimiento del proyecto y transmita los cambios a los agentes a través del bus de mensajes.

## Inicio rápido

```bash
/agileflow:status STORY=US-0001 STATUS=in-progress
```

## Parámetros

| Parameter | Required | Default | Description |
|-----------|----------|---------|-------------|
| `STORY` | Yes | - | Story ID (e.g., US-0001, US-0042) |
| `STATUS` | Yes | - | New status: `ready`, `in-progress`, `blocked`, `in-review`, or `done` |
| `SUMMARY` | No | - | 1-2 line summary of progress or reason for status change |
| `PR` | No | - | Pull request URL (for review/done status) |
| `TO` | No | ALL | Target agent(s) to notify via message bus |

## Ejemplos

### Empezar a trabajar en una historia

```bash
/agileflow:status STORY=US-0042 STATUS=in-progress SUMMARY="Beginning login form implementation"
```

Esto actualiza el estado de la historia y notifica a todos los agentes que ha comenzado a trabajar.

### Marcar como bloqueado

```bash
/agileflow:status STORY=US-0045 STATUS=blocked SUMMARY="Waiting on AG-API to implement /auth/login endpoint"
```

Transmite el bloqueador para que otros miembros del equipo puedan ayudar a resolverlo.

### Pasar a revisión de código

```bash
/agileflow:status STORY=US-0038 STATUS=in-review PR=https://github.com/your-repo/pull/123 SUMMARY="Login form complete, ready for review"
```

Comparta el enlace de relaciones públicas para los revisores.

### Completa una historia

```bash
/agileflow:status STORY=US-0030 STATUS=done SUMMARY="User registration fully tested and deployed"
```

Marca la historia como terminada.

## Producción

El comando muestra:
1. **Diff Preview** - Muestra lo que cambiará en `docs/09-agents/status.json`
2. **Confirmation** - Mensaje SÍ/NO para aplicar cambios
3. **Success Message** - Confirmación de que el estado se actualizó.
4. **Message Bus Entry** - Mensaje de difusión enviado a los agentes.

Salida de ejemplo:
```text
✅ Story Status Updated
Story:    US-0042
Status:   ready → in-progress
Summary:  Beginning login form implementation
Updated:  2025-12-24T14:30:00Z

Message broadcast to: ALL agents
```

## Archivos de datos actualizados

Este comando modifica:
- `docs/09-agents/status.json` - Seguimiento del estado de la historia
- `docs/09-agents/bus/log.jsonl` - Bus de mensajes del agente (agrega evento de cambio de estado)

## Valores de estado explicados

- **ready** - La historia está preparada y esperando ser iniciada.
- **in-progress** - Actualmente se está trabajando activamente en
- **blocked** - A la espera de dependencia externa o aclaración.
- **in-review** - Código completo y pendiente de revisión/aprobación
- **done** - Terminado, probado y verificado.

## Mejores prácticas

1. **Always include a summary** - Ayuda al equipo a comprender el contexto y el progreso.
2. **Update immediately** - No espere a recibir actualizaciones de estado por lotes
3. **Link PRs for reviews** - Ayuda a los revisores a encontrar tu trabajo.
4. **Use clear language** - Explicar los bloqueadores en detalle para que otros puedan ayudar.
5. **Update when blocked** - No te sientes en silencio sobre los bloqueadores - comunícate inmediatamente
