# /adr:view

Sehen Sie sich die vollständigen ADR-Details an und ergreifen Sie kontextbezogene Maßnahmen basierend auf dem aktuellen ADR-Status.

## Schnellstart

```bash
/agileflow:adr:view NUMBER=0042
```

## Parameter

| Parameter | Required | Default | Description |
|-----------|----------|---------|-------------|
| `NUMBER` | Yes | - | ADR number (4-digit, e.g., 0042) |

## Beispiele

### Spezifische ADR anzeigen

```bash
/agileflow:adr:view NUMBER=0003
```

Zeigt den vollständigen Inhalt von ADR-0003 an und bietet kontextbezogene Aktionen.

### Interaktive Auswahl

```bash
/agileflow:adr:view
```

Ohne NUMBER-Parameter werden aktuelle ADRs angezeigt und Sie werden aufgefordert, eine auszuwählen.

## Ausgabeformat

Jede ADR-Ansicht umfasst:

### Header-Informationen

```markdown
## ADR-0042: Use PostgreSQL for Persistence

**Status**: accepted
**Date**: 2024-12-28
**Tags**: database, architecture
```

### Abschnitte

1. **Context**: Erklärt, warum diese Entscheidung notwendig war
   - Problem oder Einschränkung, die die Entscheidung ausgelöst hat
   - Geschäftstreiber und technische Anforderungen
   - Alternative Optionen, die in Betracht gezogen wurden

2. **Decision**: Was wurde ausgewählt und warum
   - Klare Aussage zur Entscheidung
   - Begründung für diese Wahl gegenüber Alternativen
   - Spezifische Versionen oder Konfigurationen, falls relevant

3. **Consequences**: Kompromisse und Auswirkungen
   - **Positive**: Vorteile und Verbesserungen
   - **Negative**: Kosten, Risiken, Lernkurve
   - Gegebenenfalls Abhilfestrategien

4. **Related**: Links zu Recherchen, Geschichten und anderen ADRs
   - Recherche, die zu dieser Entscheidung geführt hat
   - Aus diesem ADR erstellte Implementierungsgeschichten
   - Andere ADRs ersetzen diese oder werden durch diese ersetzt

## Kontextbezogene Aktionen

Die Aktionen ändern sich je nach ADR-Status:

### Für vorgeschlagene ADRs

Wenn noch ein ADR vorliegt `proposed`, du kannst:
- **Accept this ADR**: Markieren als `accepted` und tatsächlich
- **Request changes**: Ändern Sie den Vorschlag
- **Reject this ADR**: Markieren als `rejected` mit Grund

### Für akzeptierte ADRs (empfohlen)

Wenn ein ADR vorliegt `accepted`, du kannst:
- **Create implementation stories**: Entscheidungen in umsetzbare Arbeit umwandeln
- **Supersede with new ADR**: Durch aktualisierte Entscheidung ersetzen
- **Deprecate this ADR**: Als nicht mehr zutreffend markieren
- **View related research**: Siehe Forschung, die zur Entscheidung geführt hat

### Für veraltete ADRs

Wenn ein ADR vorliegt `deprecated`, du kannst:
- **View deprecation history**: Sehen Sie, warum es veraltet war
- **Reactivate this ADR**: Markieren als `accepted` erneut, wenn sich der Kontext ändert
- **View replacement ADR**: Sehen Sie, was dies ersetzt hat
- **Back to ADR list**: Zurück zur vollständigen Liste

### Für ersetzte ADRs

Wenn ein ADR vorliegt `superseded`, du kannst:
- **View replacement ADR**: Siehe das ADR, das dieses ersetzt hat
- **View decision evolution**: Verstehen Sie, wie sich diese Entscheidung entwickelt hat
- **Back to ADR list**: Zurück zur vollständigen Liste

## Typische Arbeitsabläufe

### Genehmigen Sie die vorgeschlagene Entscheidung

```
1. /agileflow:adr:view NUMBER=0045
   (See proposed architecture decision)
2. Select "Accept this ADR"
3. ADR status changes to accepted
4. Select "Create implementation stories"
5. Stories created in docs/06-stories/
```

### Akzeptierte Entscheidung umsetzen

```
1. /agileflow:adr:view NUMBER=0042
   (See accepted architecture decision)
2. Select "Create implementation stories"
3. Stories created for implementing this decision
4. Work on stories with /agileflow:babysit
```

### Verfolgen Sie die Entscheidungsentwicklung

```
1. /agileflow:adr:view NUMBER=0010
   (Old approach)
2. See "Superseded by ADR-0035"
3. Select "View replacement ADR"
4. See how decision evolved over time
```

### Überdenken Sie die veraltete Entscheidung

```
1. /agileflow:adr:view NUMBER=0015
   (Deprecated approach)
2. Check why it was deprecated
3. If context changed: "Reactivate this ADR"
4. Mark as accepted again
```

## Beispiel einer ADR-Anzeige

```markdown
## ADR-0042: Use PostgreSQL for Persistence

**Status**: accepted
**Date**: 2024-12-28
**Tags**: database, architecture

---

### Context

We need a reliable database with ACID guarantees for user data and transactions.
The application handles financial transactions requiring strong consistency.
Current SQLite implementation cannot scale to multiple servers.

---

### Decision

We will use PostgreSQL as our primary database because:
- Mature, well-tested ACID compliance
- Excellent tooling and ecosystem
- Team has existing experience
- Supports advanced features (JSONB, full-text search)

---

### Consequences

**Positive:**
- Strong data integrity guarantees
- Better performance at scale
- Rich query capabilities
- Team familiarity reduces learning curve

**Negative:**
- Requires managed database service or self-hosting
- Team needs PostgreSQL-specific training
- Migration effort from SQLite
- Slightly higher operational complexity than NoSQL

---

### Related

- [Research: Database Selection](../10-research/20241225-database-selection.md)
- [US-0050: Database migration story](../06-stories/EP-0005/US-0050.md)
- Supersedes: ADR-0010 (Use SQLite)
```

## Best Practices

### Bei der Überprüfung einer ADR

- Lesen Sie zuerst den Kontext, um das Problem zu verstehen
- Überprüfen Sie sowohl positive als auch negative Konsequenzen
- Informationen zur Implementierungsarbeit finden Sie im zugehörigen Abschnitt
- Schauen Sie sich die Entscheidungsentwicklung an (falls ersetzt)

### Bei der Entscheidung, ob eine Implementierung erfolgen soll

Stellen Sie diese Fragen:
1. Ist diese ADR in `accepted` Status?
2. Passen positive Konsequenzen zu unseren aktuellen Bedürfnissen?
3. Sind wir auf negative Folgen vorbereitet?
4. Haben wir Umsetzungskapazitäten?

### Wenn sich der ADR-Status ändert

Immer verwenden `/agileflow:adr:update` dokumentieren:
- Warum sich der Status ändert
- Was hat die Veränderung ausgelöst?
- Verweis auf Ersatz-ADR (falls ersetzt)
