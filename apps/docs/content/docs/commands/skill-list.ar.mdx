# /المهارة:قائمة

اعرض جميع المهارات المثبتة في `.claude/skills/` مع البيانات التعريفية ومعلومات الملف الخاصة بها.

## بداية سريعة

__لا تترجم_0__

يعرض جميع المهارات المثبتة مع الأوصاف وعدد الملفات وحالة MCP.

## ماذا يفعل

يقوم بمسح مهاراتك المثبتة ويعرض:

- **أسماء المهارات** - المعرف من SKILL.md
- **الأوصاف** - الغرض وحالات الاستخدام
- **عدد الملفات** - كم عدد الملفات في كل مهارة
- **إدخالات كتاب الطبخ** - تم تضمين سير عمل حالة الاستخدام
- **حالة MCP** - ما إذا كان قد تم تكوين MCP
- **توصيات العمل** - ما يجب فعله بعد ذلك

لا يحتوي هذا الأمر على معلمات.

__لا تترجم_1__

مثال على الإخراج بمهارات متعددة:

__لا تترجم_2__

## فهم الإخراج

### إدخال المهارة

تظهر كل مهارة:

1. **الاسم** - معرّف المهارة (على سبيل المثال، `supabase-swift`)
2. **الوصف** - من أهم المهارة
3. **الملفات** - تقسيم الملفات المضمنة:
   - `SKILL.md` - ملف المهارات الرئيسية (موجود دائمًا)
   - __لاRANSLATE_18__ - روابط التوثيق (موجودة عادة)
   - `cookbook/*.md` - سير عمل حالة الاستخدام
   - `.mcp.json` - تكوين خادم MCP (إن أمكن)
4. **حالة MCP** - "✓" إذا كان `.mcp.json` موجودًا وتم تكوينه

### معلومات الملف

يتضمن عدد الملفات ما يلي:

- **SKILL.md** - ملف واحد
- **references.md** - ملف واحد (إن وجد)
- **كتاب الطبخ/** - ملفات متعددة (واحد لكل حالة استخدام)
- **.mcp.json** - ملف واحد (إذا تم تكوينه)

تفصيل المثال:

## حالة MCP

يوضح مؤشر MCP ما إذا تم تكوين المهارة للوصول إلى الأداة:

- **✓ تم التهيئة** - `.mcp.json` موجود وأدوات MCP متاحة
- *لا يوجد مؤشر* - لم يتم تكوين MCP (لا يزال قابلاً للاستخدام بدون أدوات)

لإضافة MCP إلى مهارة:

__لا تترجم_10__

## أفضل الممارسات

### مهارات التنظيم

تجميع المهارات ذات الصلة حسب المجال:

- **طبقة البيانات**: `supabase-swift`، `postgresql-queries`، `mongodb-patterns`
- **طبقة واجهة برمجة التطبيقات**: `rest-api-errors`، `graphql-schema`، `webhooks`
- **طبقة واجهة المستخدم**: `react-components`، `accessibility-wcag`، `tailwind-patterns`
- **DevOps**: `github-actions`، `docker-deployment`، `kubernetes`

### عمليات التدقيق المنتظمة

قم بتشغيل `/skill:list` بشكل دوري إلى:

1. التحقق من المهارات غير المستخدمة
2. تحقق من تكوين MCP
3. تحديد المهارات التي تحتاج إلى تحديث
4. انظر إجمالي عدد المهارات

### تنظيف

احذف المهارات التي لم تعد تستخدمها:__لا تترجم_11__

وهذا يحافظ على تركيز دليل مهاراتك ويسهل إدارته.

## استكشاف الأخطاء وإصلاحها

### دليل المهارات مفقود

إذا رأيت رسالة "لم يتم تثبيت أي مهارات حتى الآن" والمهارات المتوقعة:

1. تحقق من وجود `.claude/skills/`:
   __لا تترجم_12__

2. تحقق من وجود ملفات SKILL.md في الدلائل الفرعية:
   __لا تترجم_13__

3. تأكد من أذونات القراءة المناسبة على الدليل

### معلومات المهارة غير مكتملة

إذا أظهرت المهارة بيانات تعريف غير مكتملة:

1. قد يحتوي الملف SKILL.md على مادة أمامية تالفة
2. قم بتشغيل `/agileflow:skill:test` للتشخيص
3. قم بالتحرير باستخدام `/agileflow:skill:edit` للإصلاح

### عدم ظهور MCP

إذا كنت تتوقع MCP ولكنك لا ترى "✓ تم التهيئة":

1. تحقق من وجود `.mcp.json`:
   __لا تترجم_14__

2. تحقق من أنه يحتوي على صيغة JSON صالحة
3. قم بترقية أو تعديل المهارة لإصلاح تكوين MCP

3. **اختبار إحدى المهارات**: `/agileflow:skill:test <skill-name>`
4. **تمكين التعلم**: `/agileflow:skill:upgrade <skill-name>`
