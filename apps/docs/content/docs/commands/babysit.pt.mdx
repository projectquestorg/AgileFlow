# /babá

Um mentor interativo que orienta você na implementação de recursos, desde requisitos em inglês simples até solicitações pull concluídas. Babysit combina coleta automatizada de contexto, sugestões inteligentes de tarefas e assistência prática de código.

## Início rápido

```bash
/agileflow:babysit
```

O comando irá:
1. Reúna todo o contexto do projeto (histórias, épicos, testes, documentação)
2. Apresente sugestões de tarefas inteligentes com base na prioridade e prontidão
3. Guie você passo a passo pela implementação
4. Atualize o status do seu projeto enquanto você trabalha
5. Gere descrições de PR quando estiver pronto

## Principais recursos

### Sugestões de tarefas inteligentes

Ao iniciar __NOTRASLATE_17__, ele analisa seu projeto e recomenda:

- **Histórias PRONTAS** (todos os AC concluídos, testes interrompidos) - Maior prioridade
- **Histórias bloqueadas** (com caminhos de desbloqueio claros) - Alta prioridade
- **Épicos quase completos** (mais de 80% de histórias concluídas) - Termine o que começou
- **Prioridades do roteiro** - Trabalho estratégico importante
- **Lacunas de pesquisa** - Trabalho de preparação para AVoid depuração mais tarde

As tarefas são classificadas de acordo com o que um desenvolvedor real priorizaria, não apenas por tempo recente ou ordem.

### Orientação prática de implementação

Para cada história ou recurso:

```text
1. Validate story readiness (acceptance criteria, tests)
2. Read architecture context relevant to the task
3. Plan implementation in small steps
4. Apply code changes incrementally with diffs
5. Run tests after each change
6. Update project status as you progress
7. Generate PR description when complete
```

### Geração de Agente Especialista

Para trabalhos complexos, o babysit detecta automaticamente o domínio e gera especialistas:

- **Trabalho de banco de dados** → Especialista AG-DATABASE
- **Endpoints de API** → Especialista AG-API
- **Componentes de UI** → Especialista AG-UI
- **Testes** → Especialista AG-TESTING
- **Documentação** → Especialista AG-DOCUMENTAÇÃO

Os especialistas cuidam do trabalho profundo e focado, enquanto a babá coordena o fluxo geral.

### Acompanhamento do progresso

Todo o trabalho é rastreado com uma lista de tarefas:

```text
1. Run context script (loads all project state)
2. Present task suggestions
3. Read task-specific documentation
4. Validate story readiness
5. Plan implementation steps
6. Apply code changes with tests
7. Update project status
8. Generate PR description
9. Mark complete
```

Você pode ver exatamente onde está e o que resta.

__NÃOTRADUZIR_7__

### Quando usar o modo Loop

**Bom para:**
- Processar um épico bem definido com histórias claras
- Desenvolvimento orientado a testes (os testes definem "pronto")
- Processamento em lote de várias histórias

**Não é bom para:**
- Trabalho exploratório sem critérios de aceitação claros
- Histórias que exigem revisão humana antes de prosseguir
- Trabalho complexo em vários domínios que necessita de coordenação

Para obter documentação abrangente do modo Loop, incluindo o modo visual e o modo de cobertura, consulte o [Guia de recursos do modo Loop](/features/loop-mode).

## Visão geral do fluxo de trabalho

```text
Start → Gather Context → Suggest Tasks → Choose Task
  ↓
Read Story & Architecture → Validate Readiness → Plan Steps
  ↓
Implement Code → Run Tests → Update Status
  ↓
Generate PR → Finish
```

## Definição de Pronto

Antes da implementação, a babá verifica se as histórias possuem:

- ✓ Critérios de aceitação (formato Dado/Quando/Então)
- ✓ Contexto de Arquitetura comcitações de fontes
- ✓ Esboços de teste em docs/07-testing/
- ✓ Dependências resolvidas e documentadas
- ✓ Insights anteriores de histórias relacionadas

Se alguma estiver faltando, a babá irá sugerir completá-la primeiro (as histórias ficam em “rascunho” até ficarem prontas).

## Carregamento de contexto

Quando você inicia o babysit, ele executa este script de contexto automaticamente:

Este único comando reúne:

## Quando usar

### Melhor para

- **Iniciando a implementação** - Precisa de orientação sobre o que trabalhar
- **Continuando no meio do filme** - Retome de onde você parou
- **Conclusão épica** - Conclua as histórias finais em um conjunto de recursos
- **Requisitos incertos** - Perguntas de pesquisa antes da codificação
- **Recursos de várias etapas** - Trabalho complexo em arquivos
- **Testevalidação** - Garanta que os testes sejam aprovados antes do PR

### Não para

- **Correções rápidas** (erros de digitação, frases simples) - Faça você mesmo
- **Pesquisa pura** - Use __NOTRASLATE_27__ em vez disso
- **Decisões arquitetônicas** - Use `/agileflow:adr` em vez disso
- **Somente documentação** - Use `/agileflow:docs` em vez disso

## Estilo de saída

A Babysit usa resultados claros e acionáveis:

- **Títulos** para seções principais
- **Resumos** para opções e explicações
- **Blocos de código** para diferenças e exemplos
- **Perguntas** para orientar suas decisões (nunca pede para você "digitar")
- **Atualizações de progresso** enquanto você trabalha

Cada resposta termina com uma pergunta sobre o próximo passo para que você esteja sempre no controle.

## Dicas

### Obtenha sugestões melhores

- Mantenha o backlog da sua história em status.json
- Escreva critérios de aceitação claros
- Marcar histórias como prontas quando concluídas
- Mantenha os documentos de arquitetura atualizados

### Implementações bem-sucedidas

- Deixe a babá validar primeiro a preparação da história
- Leia a seção de contexto da arquitetura
- Implementar em pequenos passos (não todos de uma vez)
- Execute testes após cada alteração
- Peça ajuda se estiver preso

### Orquestração multi-especialista

Para recursos complexos, o babysit irá automaticamente:
- Especialista em Spawn API para trabalho de back-end
- Especialista em Spawn UI para trabalho de frontend
- Execute ambos em paralelo se forem independentes
- Coordenar a integração final

Você não precisa gerenciar manualmente vários agentes - a babá cuida disso.
