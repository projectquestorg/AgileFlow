# /agileflow:recherche:importation

Importez les résultats de recherche à partir d’outils ou de contenus d’IA externes dans le dossier de recherche de votre projet avec un formatage structuré.

## Démarrage rapide

```bash
/agileflow:research:import TOPIC="OAuth 2.0 Setup"
```

Collez ensuite le contenu lorsque vous y êtes invité. Ou fournissez-le en ligne :

```bash
/agileflow:research:import TOPIC="OAuth 2.0 Setup" SOURCE="https://chatgpt.com"
```

## But

Après avoir utilisé [`/agileflow:research:demander`](/commands/research-ask) pour obtenir des réponses de l'IA externe, cette commande :

- Formate les résultats dans une démarque structurée
- Enregistre dans `docs/10-research/YYYYMMDD-topic-slug.md`
- Met à jour automatiquement l'index de recherche
- Liens optionnels vers des ADR, des Epics ou des Stories

## Paramètres

| Argument | Required | Description |
|----------|----------|-------------|
| TOPIC | Yes | Name for the research file (e.g., "OAuth 2.0 Setup") |
| CONTENT | Yes* | Research results to import (*can be pasted after command) |
| SOURCE | No | Original source URL for attribution |

## Ce qu'il importe

Fonctionne avec le contenu de :

- Réponses ChatGPT
- Conversations Web de Claude
- Résultats de la recherche sur la perplexité
- Transcriptions de vidéos YouTube
- Billets et articles de blog
- Notes de conférence
- Pages de documentation
- Notes de réunion

## Flux de travail

La commande suit un workflow structuré :

### Phase 1 : Valider les entrées

1. Vérifiez que le SUJET est fourni (demande s'il est manquant)
2. Vérifiez que le CONTENU est fourni (invite à coller s'il est manquant)

### Phase 2 : Analyser et formater

3. Extraire les principales conclusions et le résumé
4. Identifier les extraits de code (conservés exactement)
5. Générer des éléments d'action
6. Créez des suggestions d'histoires, le cas échéant

### Phase 3 : Aperçu (Diff-First)

7. Afficher le résultat formaté avant de l'enregistrer
8. Demander une confirmation

### Phase 4 : Enregistrer et indexer

9. Enregistrer dans `docs/10-research/YYYYMMDD-topic-slug.md`
10. Mise à jour `docs/10-research/README.md` indice

### Phase 5 : Analyse de l'offre (facultatif)

11. Demandez si vous souhaitez une analyse de la mise en œuvre
12. Si oui, passez en mode plan et explorez votre base de code
13. Suggérer un artefact approprié (ADR, Epic, Story, Practice doc)

## Format de sortie

La recherche importée est structurée comme suit :

```markdown
# [Topic Title]

**Import Date**: YYYY-MM-DD
**Topic**: [original topic]
**Source**: [URL or "ChatGPT/Claude/Perplexity research"]
**Content Type**: research

---

## Summary

[2-3 paragraph executive summary]

---

## Key Findings

- [Main point 1 with details]
- [Main point 2 with details]
- [Main point 3 with details]

---

## Implementation Approach

1. [Step 1]
2. [Step 2]
3. [Step 3]

---

## Code Snippets

[ALL code snippets preserved exactly as provided]

---

## Action Items

- [ ] [Action 1 - concrete next step]
- [ ] [Action 2 - concrete next step]

---

## Risks & Gotchas

- [Risk 1]
- [Risk 2]

---

## Story Suggestions

[If content suggests feature work]

---

## References

- Source: [URL]
- Import date: YYYY-MM-DD
```

## Emplacement du fichier

La recherche est enregistrée avec le préfixe de date ISO :

```
docs/10-research/YYYYMMDD-topic-slug.md

Examples:
- docs/10-research/20250101-oauth-setup.md
- docs/10-research/20241231-stripe-webhooks.md
```

## Mise à jour de l'index

Chaque importation est automatiquement mise à jour `docs/10-research/README.md`:

```markdown
| Date | Topic | File | Summary |
|------|-------|------|---------|
| 2025-01-01 | OAuth 2.0 Setup | 20250101-oauth-setup.md | OAuth flows, security, best practices |
| [older entries...] |
```

## Analyse de la mise en œuvre

Après l'enregistrement, une analyse de mise en œuvre vous est proposée :

```
Would you like me to analyze how this research could be implemented in your project?

1. Yes - Enter plan mode and explore (Recommended)
2. No - Just save the research
3. Link to existing Epic/Story
```

Si vous choisissez "Oui", la commande :

1. Passe en mode Plan
2. Rassemble le contexte de votre projet
3. Montre les avantages et les étapes de mise en œuvre
4. Recommande le type d'artefact approprié :

| Research Type | Artifact |
|---------------|----------|
| Architecture/tech decision | ADR |
| Large feature (5+ steps) | Epic + Stories |
| Single focused task | Story |
| Best practices/guidelines | Practice doc |
| Code quality improvement | Tech debt item |

## Exemple d'utilisation

### Flux de travail complet

```bash
# Step 1: Get help from ChatGPT
/agileflow:research:ask TOPIC="next-auth OAuth setup"
# Copy the generated prompt, paste into ChatGPT

# Step 2: Import the response
/agileflow:research:import TOPIC="next-auth OAuth Fix" SOURCE="https://chatgpt.com"
# Paste ChatGPT's response when prompted

# Step 3: Review and confirm
# Command shows formatted preview
# You confirm to save

# Step 4: Optionally analyze for implementation
# Choose "Yes" to enter plan mode
# Get specific recommendations for your codebase
```

### Importer la transcription YouTube

```bash
/agileflow:research:import TOPIC="React 18 Streaming SSR" SOURCE="https://youtube.com/watch?v=..."
# Paste the transcript
```

### Importer un article de blog

```bash
/agileflow:research:import TOPIC="Vercel Edge Functions" SOURCE="https://vercel.com/blog/..."
# Paste relevant sections
```

## Meilleures pratiques

1. **Always cite the source** - Inclure l'argument SOURCE
2. **Preserve code exactly** - Ne modifiez pas les exemples des réponses de l'IA
3. **Extract key findings** - Ne vous contentez pas de coller du contenu brut
4. **Generate action items** - Rendre la recherche exploitable
5. **Link to decisions** - Référence des ADR, Epics, Stories
