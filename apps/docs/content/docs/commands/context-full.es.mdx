# /agileflow:context:completo

Genere o actualice el resumen de contexto integral para las herramientas web de IA.

## Inicio rápido

```bash
/agileflow:context:full
```

## Propósito

Crea o actualiza `docs/context.md` con secciones administradas que contienen:

- Qué estamos construyendo y enfoque actual
- Mapa de características y resumen de tecnología/herramientas
- Decisiones clave (ADR) e instantánea de la arquitectura.
- Estado de pruebas y CI
- Progreso reciente (últimos 10 mensajes de bus)
- Riesgos y próximos pasos

Este archivo está diseñado para pegarse en ChatGPT, Perplexity, Gemini o Claude web para brindar a las herramientas externas de IA el contexto completo del proyecto.

## Cómo funciona

### Paso 1: Reúna las fuentes

El comando lee información de múltiples archivos de proyecto:

| Fuente | Propósito |
|--------|---------|
| `docs/09-agents/status.json` | Historia actual/estado épico |
| `docs/09-agents/bus/log.jsonl` | Últimos 10 mensajes de progreso |
| `docs/05-epics/*.md` | Detalles épicos |
| `docs/06-stories/*/*.md` | Detalles de la historia |
| `docs/03-decisions/*.md` | Registros de decisiones de arquitectura |
| `docs/02-practices/*.md` | Prácticas de desarrollo |
| __NOTANSLATE_14__ | Arquitectura del sistema |
| `package.json` | Manifiesto del proyecto y dependencias |
| `.github/workflows/*.yml` | Configuración de CI |
| `CHANGELOG.md` | Cambios recientes |

### Paso 2: Generar secciones administradas

El comando crea o actualiza estas secciones:

```markdown
<!-- MANAGED_SECTION: overview -->
## What We're Building
[Project description from package.json or README]

## Current Focus
[Active epics/stories from status.json]
<!-- END_MANAGED -->

<!-- MANAGED_SECTION: features -->
## Feature Map
| Epic | Status | Stories |
|------|--------|---------|
[One line per epic]
<!-- END_MANAGED -->

<!-- MANAGED_SECTION: tech -->
## Tech Stack
- Framework: [from package.json]
- Key dependencies: [list major deps]
- Testing: [from scripts or CI]
<!-- END_MANAGED -->

<!-- MANAGED_SECTION: decisions -->
## Key Decisions (ADRs)
[List of ADRs with titles and status]
<!-- END_MANAGED -->

<!-- MANAGED_SECTION: progress -->
## Recent Progress
[Last 10 entries from bus/log.jsonl]
<!-- END_MANAGED -->

<!-- MANAGED_SECTION: next -->
## Next Steps
[Stories in READY status]
<!-- END_MANAGED -->
```

### Paso 3: Confirmación de diferenciación primero

Antes de escribir los cambios, el comando muestra una diferencia y solicita confirmación:

```diff
- [old content]
+ [new content]
```

Los cambios solo se aplican después de que usted confirme.

## Límites de sección

Para mantener el contexto breve y legible:

| Sección | Líneas máximas |
|---------|-----------|
| descripción general | 10 |
| características | 20 |
| tecnología | 15 |
| decisiones | 15 |
| progreso | 30 |
| siguiente | 20 |

## Contenido administrado versus contenido de usuario

El comando distingue entre:

**Secciones gestionadas** (entre marcadores):
```markdown
<!-- MANAGED_SECTION: overview -->
[Content here - OK to update]
<!-- END_MANAGED -->
```

**Secciones escritas por el usuario** (sin marcadores):
```markdown
## My Custom Section
[Content here - NEVER touched]
```

El contenido del usuario fuera de los marcadores administrados siempre se conserva.

## Archivos faltantes

Si los archivos fuente no existen, el comando se adapta correctamente:

| Falta File | Acción |
|----------------------|--------|
| `docs/context.md` | Crear nuevo archivo con todas las secciones |
| `status.json` | Saltar funciones/siguientes secciones, usar marcadores de posición |
| `bus/log.jsonl` | Saltar sección de progreso |
| Archivos ADR | Saltar sección de decisiones |
| Configuración de CI | Omita los detalles de CI de la sección de tecnología |

## Salida de ejemplo

```markdown
# Project Context Brief

**Last Updated**: 2025-12-24
**Project**: AgileFlow - AI-powered project management

<!-- MANAGED_SECTION: overview -->
## What We're Building

AgileFlow is a comprehensive project management system powered by AI agents...

## Current Focus

- EP-0010: Documentation System (in_progress)
- US-0042: Add Loop Mode docs (in_progress)
<!-- END_MANAGED -->

<!-- MANAGED_SECTION: features -->
## Feature Map

| Epic | Status | Stories |
|------|--------|---------|
| EP-0010 | in_progress | 5 total, 3 done |
| EP-0009 | complete | 7 total, 7 done |
<!-- END_MANAGED -->

[...additional sections...]
```

## Cuándo utilizar

- **Después de hitos importantes** - Capture el trabajo completado
- **Antes de las sesiones de IA web** - Asegúrese de que el contexto esté actualizado
- **Después de cambios de arquitectura** - Registrar nuevas decisiones
- **Durante la planificación del sprint** - Actualizar áreas de enfoque

## Flujo de trabajo

```bash
# 1. Generate or refresh context
/agileflow:context:full

# 2. Export for web AI (read-only)
/agileflow:context:export

# 3. Copy and paste into ChatGPT/Perplexity/Gemini
```
