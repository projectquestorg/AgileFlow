# /agente

Integre novos agentes com perfis, avisos do sistema (contratos) e limites de escopo para uma colaboração eficaz em equipe.

## Início rápido

```bash
/agileflow:agent AGENT_ID=AG-UI ROLE="Frontend Developer" TOOLS="Read,Write,Bash" SCOPE="src/components/,frontend"
```

## Parâmetros

| Parâmetro | Obrigatório | Padrão | Descrição |
|-----------|----------|---------|------------|
| `AGENT_ID` | Sim | - | Identificador único do agente (por exemplo, AG-UI, AG-API) |
| `ROLE` | Sim | - | Função ou título do agente |
| `TOOLS` | Não | Ler, escrever, editar, Bash | Ferramentas disponíveis separadas por vírgula |
| `SCOPE` | Não | - | Diretórios e tags de história que o agente pode acessar |

## Exemplos

### Desenvolvedor Frontend Integrado

```bash
/agileflow:agent AGENT_ID=AG-UI ROLE="Frontend Developer" TOOLS="Read,Write,Edit,Bash,Glob" SCOPE="src/components/,src/pages/,frontend,ui"
```

Cria perfil de agente com:
- Papel e responsabilidades
- Ferramentas disponíveis (leitura, gravação, edição, Bash)
- Limites do escopo (somente src/components/ e src/pages/)
- Prompt do sistema (contrato) com regras rígidas

### Desenvolvedor de API integrado

```bash
/agileflow:agent AGENT_ID=AG-API ROLE="Backend API Developer" TOOLS="Read,Write,Edit,Bash,Grep" SCOPE="src/api/,src/services/,backend,api"
```

## Componentes do perfil do agente

### Responsabilidades

Descrição clara do que o agente faz:
- Domínio específico (UI, API, CI/CD)
- Tecnologias utilizadas
- Principais resultados
- Função da equipe

### Ferramentas Disponíveis

O que o agente pode usar:
- **Ler** - Leia arquivos e analise código
- **Escrever** - Criar novos arquivos
- **Editar** - Modificar arquivos existentes
- **Bash** - Executa comandos shell (git, npm, docker)
- **Glob** - Encontre arquivos por padrão
- **Grep** - Pesquisa o conteúdo do arquivo

### Limites do escopo

Onde o agente pode trabalhar:
- **Diretórios permitidos** - O agente de arquivos pode modificar
- **Diretórios restritos** - O agente de arquivos NÃO PODE tocar
- **Tags de história** - Que trabalho é atribuído a este agente

### Prompt do Sistema (Contrato)

Regras que o agente deve seguir:
- **Restrições de escopo** - Caminhos exatos dos arquivos
- **Requisitos de teste** - Quando/como testar
- **Convenções de commit** - Formato de ramificação e mensagem
- **Atualizações de status** - Como acompanhar o progresso
- **Padrões de código** - Requisitos de qualidade

## Melhores práticas

### Nomeando Agentes
Use IDs descritivos que indiquem claramente a função:
- ✅ Bom: AG-UI, AG-API, AG-CI, AG-DEVOPS, AG-SECURITY
- ❌ Ruim: AG-001, AG-A, AG-DEV1

### Definindo o escopo

Deixe os limites bem claros:

**Bom** - Caminhos específicos:
- `src/components/`
- `src/api/routes/`
- `.github/workflows/`

**Ruim** - Limites vagos:
- "coisas de front-end"
- "coisas de back-end"

### Requisitos de teste

Inclui comandos de teste específicos:

**Bom** - Comandos claros:
- Execute __NOTRASLATE_16__
- Verifique a cobertura com __NOTRASLATE_17__
- Lint com __NOTRASLATE_18__

**Ruim** - Instruções vagas:
- "Escrever testes"
- "Certifique-se de que funciona"

### Mapeamento de tags de história

Atribua histórias com base em tags:

| Agente | Etiquetas de história |
|-------|------------|
| AG-UI | front-end, interface do usuário, ux, design |
| AG-API | apoiard, API, banco de dados |
| AG-CI | ci, devops, automação |

## Fluxo de trabalho

1. Reúna os detalhes do agente:
   - ID exclusivo (AG-DOMAIN)
   - Função/título
   - Ferramentas disponíveis
   - Limites do escopo
   - Tags de história

2. Crie um perfil de agente a partir do modelo
   - Seção de responsabilidades
   - Lista de ferramentas
   - Limites do escopo (claros e específicos)
   - Prompt do sistema (contrato com regras)

3. Atualizar lista de agentes
   - Adicionar entrada em docs/09-agents/roster.yaml
   - ID do mapa → função, ferramentas, escopo

4. Gere prompt do sistema
   - Prompt condensado de saída
   - Pronto para colar como prompt do sistema de terminal

5. Validar agente
   - Revise os limites do escopo (sem conflitos)
   - Confirme os requisitos de teste
   - Verifique as atribuições de tags de história

## Integração

Perfis de agente integram-se com:
- **Acompanhamento de status** - Lista mostra agentes ativos
- **Atribuição de história** - Use ID e tags do agente
- **Planejamento de capacidade** - Veja quem está disponível
- **Transferências** - Perfis de agente de referência em comunicações
- **Validação de escopo** - Evitar alterações fora do escopo

## vocêse quando

- **Integração de novos membros da equipe** - Criar perfil e contrato
- **Adicionando funções especializadas** - Agentes de segurança, desempenho e acessibilidade
- **Reestruturação da equipe** - Atualização dos limites do escopo
- **Agentes de treinamento** - Use o prompt do sistema como referência
- **Resolver conflitos** - O escopo claro evita sobreposição
