# /aktualisieren

Generieren Sie Projektaktualisierungen auf hoher Ebene für Stakeholder, Führungskräfte oder Kunden mit Kennzahlen, Fortschritten und Risiken.

## Schnellstart

```bash
/agileflow:update PERIOD=week AUDIENCE=exec
```

## Parameter

| Parameter | Erforderlich | Standard | Beschreibung |
|-----------|----------|---------|-------------|
| `PERIOD` | Nein | Woche | Zeitrahmen: Woche, Sprint, Monat, Quartal oder benutzerdefiniert |
| `START_DATE` | Nein | - | Für benutzerdefinierten Zeitraum (Format JJJJ-MM-TT) |
| `END_DATE` | Nein | - | Für benutzerdefinierten Zeitraum (Format JJJJ-MM-TT) |
| `AUDIENCE` | Nein | exec | Zielgruppe: Führungskraft, Kunde, Team oder Vorstand |
| `FORMAT` | Nein | Abschlag | Ausgabeformat: E-Mail, Markdown, Folien oder PDF |

## Beispiele

### Wöchentliches Update für Führungskräfte

```bash
/agileflow:update PERIOD=week AUDIENCE=exec FORMAT=email
```

Erstellt ein prägnantes einseitiges Update für C-Level-Führungskräfte mit:
- Zusammenfassung und Gesamtstatus
- Wichtige Erfolge
- Kritische Blocker oder Risiken
- Geschwindigkeit und Metriken
- Kein Fachjargon

### Zweiwöchentlicher Kundenbericht

```bash
/agileflow:update PERIOD=2weeks AUDIENCE=client FORMAT=markdown
```

Erzeugt eine Featurorientiertes Update für Kunden mit:
- Was Benutzer jetzt tun können
- Fortschritte bei der Erbringung vertraglicher Leistungen
- Screenshots oder Demo-Referenzen
- Transparente Problemkommunikation
- Sprache mit Benutzernutzen

### Team-Sprint-Retrospektive

```bash
/agileflow:update PERIOD=sprint AUDIENCE=team FORMAT=markdown
```

Erstellt ein detailliertes Update für das Team mit:
- Architekturentscheidungen und ADRs
- Technische Errungenschaften
- Feier der Siege
- Retrospektive Einblicke
- Aktionselemente

### Benutzerdefinierte Periodenaktualisierung

```bash
/agileflow:update PERIOD=custom START_DATE=2025-12-01 END_DATE=2025-12-15 AUDIENCE=board FORMAT=pdf
```

## Ausgabe

Der Befehl generiert ein umfassendes Update, das gespeichert wird unter:
```text
docs/08-project/updates/<YYYYMMDD>-update.md
```

### Struktur aktualisieren

**Zusammenfassung**
- 2-3 sentence overview
- Wichtige Erfolge
- Kritische Probleme oder Risiken
- Gesamtstatus (Auf Kurs / In Gefahr / Blockiert)

**Fortschritt in diesem Zeitraum**
- Geschichten mit Zählungen abgeschlossen
- Epische Fortschrittsprozentsätze
- Meilensteine erreicht

**Kommende Prioritäten**
- Nächste Sprint-Schwerpunktbereiche
- Upcoming milestones with dates

**Metriken und Trends**
- Geschwindigkeit (Geschichten/Woche)
- Abschlussquote in Prozent
- Qualitätsmetriken (thöchste Abdeckung, Bugs)
- Trends (Verbesserung/Rückgang)

**Blocker und Risiken**
- Aktuelle Blocker mit Eigentümern
- Strategien zur Risikominderung
- Status jedes Artikels

**Getroffene Entscheidungen**
- Aktuelle Architekturentscheidungsaufzeichnungen (ADRs)
- Begründung für wichtige Entscheidungen

**Team-Updates** (falls zutreffend)
- Änderungen der Teamkapazität
- Bemerkenswerte Erfolge
- Moralindikatoren

## Datenquellen

Der Befehl aggregiert Daten aus 10 Quellen:

1. **docs/09-agents/status.json** – Snapshots des Story-Status
2. **docs/05-epics/*.md** – Epischer Fortschritt und Abschluss
3. **docs/06-stories/**/US-*.md** – Story-Details und Abschluss
4. **docs/08-project/backlog.md** – Backlog-Elemente
5. **docs/08-project/roadmap.md** – Roadmap-Meilensteine
6. **docs/08-project/milestones.md** – Meilensteinziele
7. **docs/08-project/risks.md** – Risikoregister
8. **Git-Verlauf** – Commits und zusammengeführte PRs
9. **docs/09-agents/bus/log.jsonl** – Aktivitäten und Ereignisse
10. **docs/03-decisions/adr-*.md** – Aktuelle Entscheidungen

## Zielgruppenanpassung

### Executiv (C-Level)
- Sehr allgemeiner Überblick
- Fokus auf geschäftliche Auswirkungen
- Kennzahlen und ROI
- Risiken frühzeitig hervorgehoben
- Maximal 1 Seite

### Kunde
- Feature-fokussierte Sprache
- Benutzervorteile hervorgehoben
- Transparent in Bezug auf Probleme
- Screenshots/Demo-Links enthalten
- Fortschritte bei der Erbringung vertraglicher Leistungen

### Team
- Weitere technische Details enthalten
- Architekturentscheidungen inklusive
- Feier der Siege
- Retrospektive Einblicke
- Aktionspunkte für die nächste Periode

### Vorstand
- Strategischer Überblick
- Finanzielle Auswirkungen
- Wettbewerbsfähige Positionierung
- Langfristige Roadmap
- Risikobewertung

## Arbeitsablauf

1. Zeitraum festlegen (Standard: diese Woche)
2. Sammeln Sie Daten aus allen 10 Quellen
3. Berechnen Sie Kennzahlen und Trends
4. Identifizieren Sie abgeschlossene Arbeiten und Blockaden
5. Format für die Zielgruppe
6. Vorschau des Updates im Markdown
7. Fragen Sie: „Update senden? (JA/NEIN/BEARBEITEN)“
8. Wenn JA: Speichern Sie es unter docs/08-project/updates/ und senden Sie optional eine E-Mail an die Beteiligten

## Verwenden Sie Wann

- **Wöchentliche Teamsynchronisierungen** - Regelmäßige Aktualisierungen der Trittfrequenz
- **Executive-Präsentationtionen** - Monatliche/vierteljährliche Berichterstattung
- **Kundenkommunikation** - Aktualisierungen des Vertragsfortschritts
- **Sprint-Retrospektiven** – Teamorientiertes Lernen
- **Vorstandssitzungen** - Aktualisierungen der strategischen Ausrichtung
- **Risikoeskalation** - Frühzeitiges Hervorheben von Blockern

## Integration

Link zu Epen und Geschichten für weitere Details. Referenz-ADRs für den architektonischen Kontext. Fügen Sie gegebenenfalls GitHub-PR-Links ein.
