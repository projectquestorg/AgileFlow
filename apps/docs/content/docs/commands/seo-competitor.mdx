# /seo:competitor

Analyze competitor pages and generate SEO-optimized comparison content with feature matrices and structured data.

## Quick Start

```bash
/agileflow:seo:competitor https://you.com https://competitor.com          # 1v1 comparison
/agileflow:seo:competitor https://you.com https://comp1.com https://comp2.com  # Multi-competitor
```

## Parameters

| Parameter | Required | Default | Description |
|-----------|----------|---------|-------------|
| `YOUR_URL` | Yes | - | Your product/service page |
| `COMPETITOR_URL` | Yes | - | Competitor page(s) to compare (1-3) |

## How It Works

1. **Fetch all pages** - Extract content, features, pricing, schema
2. **Build feature matrix** - Compare side-by-side features
3. **SEO comparison** - Analyze titles, descriptions, structure, schema
4. **Generate content** - Create draft comparison page
5. **Generate schema** - Ready-to-use JSON-LD with Product types

## Example Usage

### One Competitor
```bash
/agileflow:seo:competitor https://myproduct.com/pricing https://competitor.com/pricing
```

Compare your pricing page with one competitor.

### Multiple Competitors
```bash
/agileflow:seo:competitor https://myproduct.com https://comp1.com https://comp2.com
```

Compare against 2 competitors side-by-side.

## SEO Recommendations

### Quick Wins
1. **Add Review/Rating schema** - Competitor A shows 4.8 stars
   - Collect and add customer reviews to your page
   - Use Review schema to display in SERPs
   - Impact: +30-50% CTR from star ratings

2. **Improve title tag**
   - Current: 58 chars (good length)
   - Better: Include keyword + benefit
   - Example: "MyProduct vs Competitor A: Features, Pricing, & Comparison 2026"

3. **Add customer testimonials**
   - Competitor A has advantages from social proof
   - Add named, specific testimonials
   - Include star ratings and reviewer credentials

### Medium Priority
1. **Add embedded comparison video**
   - 156% more citations for video content
   - Show features side-by-side
   - Walk through pricing comparison

2. **Expand internal linking**
   - You: 4 links, Competitor A: 8 links
   - Add 4 more contextual links to related pages
   - Link to feature deep-dives and pricing

3. **Add FAQ schema**
   - Common questions: "Why MyProduct vs Competitor?"
   - Include detailed answers
   - FAQ schema can trigger rich results

### Long Term
1. **Ongoing competitor monitoring**
   - Track when competitors change features
   - Update your comparison regularly
   - Monitor their ranking progress

2. **Build original research**
   - Conduct surveys comparing products
   - Include statistics in comparison page
   - Link to full research methodology

3. **Case study comparison**
   - Showcase customer success stories
   - How MyProduct outperformed competitors
   - Quantified results with metrics

## Best Practices

1. **Be fair and accurate** - Correct info only, no exaggeration
2. **Update regularly** - Competitors change features; stay current
3. **Link to competitors** - Shows confidence, helps user decision
4. **Highlight unique strengths** - Don't just list features
5. **Answer key questions** - "Why choose you?" directly
6. **Include pricing openly** - Users research this most
7. **Add social proof** - Reviews, ratings, testimonials
8. **Use schema markup** - Rich results improve CTR
9. **Optimize for long tail** - "X vs Y" keywords have high intent
10. **Monitor rankings** - These pages often rank well, track position

## Comparison Page Keywords

These high-intent keywords work well for comparison pages:
- "[Your product] vs [Competitor]"
- "[Competitor] vs [Your product]"
- "[Product] comparison"
- "[Product] vs [Competitor]: which is better"
- "[Competitor] alternative"
- "Best [category]"
- "[Competitor] vs [Alternative 1] vs [Alternative 2]"

Users searching these are ready to decide - high conversion intent.
