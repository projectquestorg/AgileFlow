# /contexte

Créez, actualisez et gérez votre résumé de contexte de projet pour une utilisation dans des outils d'IA Web tels que ChatGPT, Perplexity, Gemini et Claude Web.

## Aperçu

Le `/context` La famille de commandes vous aide à maintenir et à exporter le contexte du projet pour les outils d'IA basés sur le Web. Ceci est utile lorsque vous devez :

- Générez une note contextuelle complète à coller dans les outils d'IA Web
- Exportez un extrait concis pour un chargement rapide du contexte
- Enregistrez les décisions et découvertes importantes dans votre fichier contextuel
- Tenez les outils d'IA externes informés de l'état de votre projet

## Sous-commandes

Le `/context` La commande comporte trois sous-commandes, chacune ayant un objectif spécifique :

### /agileflow:contexte:complet

**Generate or refresh the comprehensive project context brief.**

Crée ou met à jour `docs/context.md` avec des sections gérées contenant :
- Ce que nous construisons et nos priorités actuelles
- Carte des fonctionnalités et résumé des technologies/outils
- Décisions clés (ADR) et instantané de l'architecture
- Tests et statut CI
- Progrès récents
- Risques et prochaines étapes

Ce fichier est conçu pour être collé dans ChatGPT, Perplexity, Gemini ou Claude Web afin de donner aux outils d'IA externes le contexte complet du projet.

**Usage:**
```bash
/agileflow:context:full
```

**Behavior:**
- Lit existant `docs/context.md` si présent
- Rassemble les données de status.json, des ADR, des épopées, des histoires et du CHANGELOG
- Met à jour uniquement les sections gérées (entre `<!-- MANAGED_SECTION -->` marqueurs)
- Préserve tout contenu écrit par l'utilisateur en dehors des sections gérées
- Affiche la différence avant d'appliquer les modifications
- Nécessite une confirmation avant d'écrire

---

### /agileflow:contexte:export

**Export a concise context excerpt for pasting into web AI tools.**

Lit `docs/context.md` et génère un extrait condensé (≤ 300 lignes) prêt à copier et coller dans ChatGPT, Perplexity, Gemini ou Claude Web.

C'est un **read-only command** - aucun fichier n'est écrit.

**Usage:**
```bash
/agileflow:context:export
```

**Behavior:**
- Lit `docs/context.md`
- Extrait les sections clés (aperçu, objectif actuel, pile technologique, décisions, carte des fonctionnalités, prochaines étapes)
- Produit un extrait concis dans un bloc de code
- Comprend des instructions pour coller dans Web AI
- Ne nécessite pas de confirmation

---

### /agileflow:contexte:note

**Append a timestamped note to the context file.**

Ajoutez rapidement une note à `docs/context.md` dans la section "Remarques". Utile pour enregistrer les décisions, les observations ou les rappels qui doivent persister dans le contexte du projet.

**Usage:**
```bash
/agileflow:context:note NOTE="Your note here"
```

**Arguments:**

| Argument | Required | Description |
|----------|----------|-------------|
| NOTE | Yes | 1-5 line note text to append |

**Behavior:**
- Lit ou crée `docs/context.md`
- Crée une section "Notes" si elle n'existe pas
- Ajoute une note avec un horodatage ISO (par exemple, `2024-12-31T10:30:00Z`)
- Affiche la différence avant d'appliquer les modifications
- Nécessite une confirmation avant d'écrire

## Exemples

### Générer un contexte complet

```bash
/agileflow:context:full
```

Crée ou met à jour `docs/context.md` avec des informations complètes sur le projet :

```markdown
# Project Context Brief

**Last Updated**: 2025-12-24
**Project**: AgileFlow - AI-powered project management

## What We're Building

AgileFlow is a comprehensive project management system powered by AI agents. It provides automated task management, documentation synchronization, and intelligent context preservation.

## Current Focus

- Implementing context compacting for long conversations
- Improving research workflow integration
- Expanding agent capabilities

## Feature Map

- **Core**: Project setup, status tracking, agent coordination
- **Documentation**: Automatic sync, README generation, ADR management
- **Agents**: 131 specialized agents for different tasks
- **Research**: Structured research notes and integration

## Tech Stack

- **Language**: TypeScript
- **Framework**: Next.js 16 (App Router)
- **UI**: React with Tailwind CSS
- **Package Manager**: npm with legacy-peer-deps
- **Deployment**: Vercel

## Key Decisions

- ADR-0001: Use TypeScript for type safety
- ADR-0002: Use React component library
- ADR-0003: Next.js for web framework

## Architecture Snapshot

- Monorepo structure with packages/ and apps/
- CLI package with 139 commands
- Website and documentation sites
- Agent-based command coordination

## Testing & CI

- Unit tests: Jest
- E2E tests: Playwright
- CI/CD: GitHub Actions
- Pre-commit hooks for validation

## Recent Progress

- Completed context compacting feature
- Improved documentation sync workflow
- Added 5 new specialized agents

## Risks

- Token limits with large projects
- Documentation drift during rapid development

## Next Steps

- Implement automated test coverage reporting
- Add real-time collaboration features
- Expand agent capabilities for code generation
```

## Modes de sortie

### /agileflow:context:full (Créer/Mettre à jour le contexte)

**Input**: Recueille des informations auprès de :
- `status.json` - Statut actuel du travail et de l'histoire
- `bus/log.jsonl` - Messages de progression récents
- `docs/03-decisions/` - Décisions d'architecture (ADR)
- `docs/04-architecture/` - Documentation de conception du système
- `docs/05-epics/` - Feuille de route des fonctionnalités et épopées
- `CHANGELOG.md` - Historique des versions
- `package.json` - Métadonnées du projet

**Output**: Mises à jour `docs/context.md` avec des sections gérées tout en préservant le contenu personnalisé

**Key Behavior**:
- Diff-first : affiche les modifications avant d'écrire
- Sections gérées : met à jour uniquement le contenu entre `<!-- MANAGED_SECTION -->` marqueurs
- Préserve le contenu utilisateur : ne modifie jamais le contenu en dehors des sections gérées
- Nécessite une confirmation : l'utilisateur doit approuver les modifications

---

### /agileflow:context:export (Copier vers Web AI)

**Input**: Lit les existants `docs/context.md`

**Output**: Extrait concis (≤300 lignes) prêt à coller dans ChatGPT, Perplexity, Gemini ou Claude web

**Use Case**: Chargez le contexte du projet dans l'outil d'IA Web pour continuer à travailler dans une nouvelle conversation

**Key Behavior**:
- Lecture seule : aucun fichier n'est modifié
- Autonome : comprend des instructions pour le collage
- Prêt à être formaté : déjà formaté pour un copier-coller facile

---

### /agileflow:context:note (Ajouter une note horodatée)

**Input**: Argument NOTE avec une note de 1 à 5 lignes

**Output**: S'ajoute à la section "Notes" avec horodatage ISO

**Use Case**: Capture rapide d'événements, de décisions ou de découvertes importants

**Key Behavior**:
- Diff-first : affiche les modifications avant d'écrire
- Crée une section si manquante : crée automatiquement une section "Notes" si nécessaire
- Chronologique : les notes apparaissent dans l'ordre, les plus récentes en bas
- Horodatage : chaque note inclut un horodatage ISO 8601

## Meilleures pratiques

### Mises à jour régulières du contexte

Gardez le contexte à jour pour une meilleure assistance de l'IA :

1. **After major milestones**: Courir `/agileflow:context:full` pour capturer le travail terminé
2. **Before web AI sessions**: Courir `/agileflow:context:export` et collez-le dans ChatGPT
3. **During development**: Utiliser `/agileflow:context:note` pour enregistrer les décisions et les découvertes

```bash
# Recommended workflow
/agileflow:context:full        # Update full context
/agileflow:context:export      # Get concise version
# Paste into web AI for context-aware help
```

---

### Capturez des découvertes importantes

Enregistrez les informations et décisions techniques :

```bash
/agileflow:context:note NOTE="TypeScript 5.4 improves type inference - consider upgrading"
/agileflow:context:note NOTE="Stripe API changed pagination in v2.0 - update integration"
/agileflow:context:note NOTE="Team consensus: use Postgres over MongoDB for this project"
```

---

### Utilisation dans les conversations Web AI

Maximisez l’efficacité des outils d’IA Web en fournissant du contexte :

1. Exporter un contexte concis : `/agileflow:context:export`
2. Copiez la sortie
3. Collez dans ChatGPT, Perplexity ou Gemini comme premier message
4. Posez des questions de suivi avec le contexte complet du projet

Cela garantit que l'IA Web comprend votre base de code, votre architecture et votre objectif actuel.

---

### Gérer la taille du fichier de contexte

Pour les grands projets, gérez la taille du fichier context.md :

- Utiliser `/agileflow:context:export` pour vérifier la taille actuelle du contexte
- Archivez les anciennes notes si elles dépassent plus de 100 entrées
- Créez des fichiers de recherche distincts pour des analyses approfondies
- Utiliser `/agileflow:research:ask` pour des sujets de recherche complexes

---

## Flux de travail Diff-First

Toutes les commandes contextuelles qui modifient les fichiers suivent un flux de travail cohérent :

1. **Show Changes**: La commande affiche les modifications proposées sous forme de différence
2. **Confirm**: Vous examinez les modifications et approuvez ou rejetez
3. **Apply**: Les modifications ne sont écrites qu'après votre confirmation

Cela garantit que vous ne perdrez jamais d'informations importantes et que vous savez toujours ce qui est modifié.

## Remarques

- Les fichiers de contexte sont stockés dans `docs/context.md` dans le cadre de la documentation de votre projet local
- Les sections gérées utilisent des commentaires HTML comme marqueurs et ne doivent pas être modifiées manuellement
- La commande d'exportation respecte la limite de 300 lignes pour s'adapter aux fenêtres contextuelles de l'outil Web AI
- Les notes sont horodatées au format ISO 8601 pour plus de cohérence et un tri facile
