# /التسليم

قم بتوثيق عمليات تسليم العمل بين الوكلاء في قصة ما باستخدام الملخص والعناصر لتعاون الفريق بسلاسة.

## بداية سريعة

__لا تترجم_0__

## المعلمات

| المعلمة | مطلوب | الافتراضي | الوصف |
|-----------|----------|---------|-------------|
| __غير مترجم_5__ | نعم | - | معرف القصة (على سبيل المثال، US-0042) |
| __غير مترجم_6__ | نعم | - | تسليم معرف الوكيل |
| __غير مترجم_7__ | نعم | - | معرف وكيل الاستلام |
| __غير مترجم_8__ | نعم | - | ما تم إنجازه أو الوضع الحالي |
| __غير مترجم_9__ | لا | - | العوائق أو المشكلات المعروفة للتواصل |

## أمثلة

### التسليم الأساسي

__لا تترجم_1__

إنشاء مذكرة تسليم لتوثيق:
- من يقوم بتسليم العمل
- ما اكتمل
- أي حاصرات معروفة للوكيل التالي
- تاريخ ووقت التسليم

### Handoff مع حاصرات متعددة

__لا تترجم_2__

## الإخراج

يقوم الأمر بإنشاء قطعتين أثريتين:

### 1. ملاحظة التسليم

يقوم بإنشاء ملف تخفيض السعر على:
__لا تترجم_3__

محتوى المثال:
__لا تترجم_4__

###2. سجل حافلة الوكيل

يقوم بإلحاق إدخال بـ `docs/09-agents/bus/log.jsonl` بالنوع `handoff` لإنشاء سجل تدقيق لانتقالات العمل بين الوكلاء.

## سير العمل

1. تحليل جميع المدخلات (القصة، من، إلى، الملخص، الحواجز)
2. قم بإنشاء مذكرة التسليم من القالب
3. إلحاق الإدخال بسجل ناقل الوكيل
4. عرض المعاينة للتأكيد
5. انتظر موافقة نعم/لا
6. لا تكتب الملفات إلا بعد الموافقة عليها

## استخدم متى

- **تبديل الوكلاء** في قصة في منتصف المشروع
- **إيقاف العمل مؤقتًا** وتوثيق التقدم لوقت لاحق
- **تسليط الضوء على الحواجز** للوكيل التالي
- **إنشاء مسار تدقيق** لتحولات العمل
- **توثيق التبعيات** لعمليات التسليم بين الفرق
