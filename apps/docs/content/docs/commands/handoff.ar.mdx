# /تسليم

قم بتوثيق عمليات تسليم العمل بين الوكلاء في قصة ما باستخدام الملخص والعناصر لتعاون الفريق بسلاسة.

## بداية سريعة

```bash
/agileflow:handoff STORY=US-0042 FROM=agent-a TO=agent-b SUMMARY="Completed API integration" BLOCKERS="Need database schema review"
```

## حدود

| Parameter | Required | Default | Description |
|-----------|----------|---------|-------------|
| `STORY` | Yes | - | Story ID (e.g., US-0042) |
| `FROM` | Yes | - | Handing off agent ID |
| `TO` | Yes | - | Receiving agent ID |
| `SUMMARY` | Yes | - | What was accomplished or current status |
| `BLOCKERS` | No | - | Known blockers or issues to communicate |

## أمثلة

### عملية التسليم الأساسية

```bash
/agileflow:handoff STORY=US-0050 FROM=AG-API TO=AG-UI SUMMARY="Backend API complete and tested" BLOCKERS="UI styling needs brand color review"
```

إنشاء مذكرة تسليم لتوثيق:
- من يسلم العمل
- ما تم الانتهاء منه
- أي حاصرات معروفة للوكيل التالي
- تاريخ ووقت التسليم

### Handoff مع حاصرات متعددة

```bash
/agileflow:handoff STORY=US-0045 FROM=AG-CI TO=AG-DEVOPS SUMMARY="Pipeline setup complete, ready for deployment" BLOCKERS="Environment variables need to be set, Requires AWS credentials approval"
```

## الإخراج

يقوم الأمر بإنشاء قطعتين أثريتين:

### 1. مذكرة التسليم

يقوم بإنشاء ملف تخفيض السعر على:
```text
docs/09-agents/comms/<STORY>-<YYYYMMDD>-handoff.md
```

محتوى المثال:
```markdown
# Handoff: US-0042

From: AG-API
To: AG-UI
Date: 2025-12-22

## Summary
Completed API integration with authentication middleware.
All endpoints tested and passing.

## Blockers
- Database schema needs DBA review before migration
- Redis cache configuration pending DevOps approval
```

### 2. سجل حافلة الوكيل

إلحاق إدخال إلى `docs/09-agents/bus/log.jsonl` مع النوع `handoff` لإنشاء مسار تدقيق لانتقالات العمل بين الوكلاء.

## سير العمل

1. تحليل جميع المدخلات (القصة، من، إلى، الملخص، الحواجز)
2. إنشاء مذكرة تسليم من القالب
3. إلحاق الإدخال بسجل ناقل الوكيل
4. عرض المعاينة للتأكيد
5. انتظر موافقة نعم/لا
6. لا تكتب الملفات إلا بعد الموافقة عليها

## استخدم متى

- **Switching agents** في قصة منتصف المشروع
- **Pausing work** وتوثيق التقدم لوقت لاحق
- **Highlighting blockers** للوكيل القادم
- **Creating audit trail** من انتقالات العمل
- **Documenting dependencies** لعمليات التسليم بين الفرق
