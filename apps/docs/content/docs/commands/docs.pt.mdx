# /docs-sync

Sincronize sua documentação com alterações na base de código, detectando novo código e garantindo que a documentação correspondente exista.

## Início rápido

```bash
/agileflow:docs-sync
```

Para comparar com um ramo específico:

```bash
/agileflow:docs-sync BRANCH=feature-auth BASE=main
```

## Parâmetros

| Parâmetro | Obrigatório | Padrão | Descrição |
|-----------|----------|---------|------------|
| `BRANCH` | Não | Filial atual | Ramo para analisar mudanças |
| `BASE` | Não | principal/mestre | Ramificação base para comparação |
| `AUTO_CREATE` | Não | não | Criar automaticamente documentação faltante (sim/não) |

## Como funciona

### Detecção de alterações

O comando:
1. **Compara ramificações** usando `git diff` para identificar todas as alterações de código
2. **Categoriza alterações** em endpoints de API, componentes de UI, serviços, configuração e esquemas de banco de dados
3. **Mapas para documentação** locais onde os documentos deveriam existir

### Inferência Inteligente

A documentação é inferida de:
- **Tipos e interfaces TypeScript** - Assinaturas de funções e tipos de retorno
- **Comentários JSDoc** - Documentação de código existente
- **Arquivos de teste** - Exemplos de uso
- **Decoradores OpenAPI** - Especificações da API

### Análise de lacunas

Para cada arquivo alterado, o comando verifica:
- **Documentação ausente**: código recém-adicionado sem documentos
- **Documentação desatualizada**: Referências a código removido ou alterado
- ** Incompletodocumentação**: Stubs que precisam de detalhes

## Mapeamento de Documentação

As alterações de código são mapeadas para a documentação esperada:

| Localização do código | Documentação Esperada |
|--------------|----------------------|
| `src/api/` ou `src/routes/` | __NOTRASLATE_18__ ou especificação OpenAPI |
| `src/components/` ou `src/pages/` | `docs/04-architecture/components.md` ou livro de histórias |
| `src/services/` ou `src/utils/` | `docs/04-architecture/services.md` |
| `*.config.js`, `.env.example` | `docs/02-practices/configuration.md` |
| `migrations/` ou `schema/` | `docs/04-architecture/database.md` |
