# /cuidar niños

Un mentor interactivo que lo guía a través de la implementación de funciones, desde requisitos en inglés sencillo hasta solicitudes de extracción completas. Babysit combina la recopilación de contexto automatizada, sugerencias de tareas inteligentes y asistencia práctica con códigos.

## Inicio rápido

```bash
/agileflow:babysit
```

El comando:
1. Reúna todo el contexto del proyecto (historias, epopeyas, pruebas, documentación)
2. Presentar sugerencias de tareas inteligentes basadas en prioridad y preparación.
3. Guiarlo paso a paso a través de la implementación
4. Actualice el estado de su proyecto mientras trabaja
5. Genere descripciones de relaciones públicas cuando esté listo

## Características clave

### Sugerencias de tareas inteligentes

Cuando inicias `/babysit`, analiza tu proyecto y recomienda:

- **Historias LISTAS** (todos los AC completos, pruebas eliminadas) - Máxima prioridad
- **Historias bloqueadas** (con rutas de desbloqueo claras) - Alta prioridad
- **Epopeyas casi completas** (más del 80 % de las historias terminadas) - Termina lo que empezaste
- **Prioridades de la hoja de ruta** - Trabajo estratégico importante
- **Brechas de investigación** - Trabajo de preparación para avdepuración de oid más tarde

Las tareas se clasifican según lo que un desarrollador real priorizaría, no solo por antigüedad u orden.

### Guía de implementación práctica

Para cada historia o característica:

```text
1. Validate story readiness (acceptance criteria, tests)
2. Read architecture context relevant to the task
3. Plan implementation in small steps
4. Apply code changes incrementally with diffs
5. Run tests after each change
6. Update project status as you progress
7. Generate PR description when complete
```

### Generación de agentes expertos

Para trabajos complejos, babysit detecta automáticamente el dominio y genera especialistas:

- **Trabajo de base de datos** → Experto en AG-DATABASE
- **Puntos finales API** → Experto en AG-API
- **Componentes de UI** → Experto en AG-UI
- **Pruebas** → Experto en AG-TESTING
- **Documentación** → Experto AG-DOCUMENTACIÓN

Los especialistas se encargan del trabajo profundo y concentrado, mientras que el cuidado de niños coordina el flujo general.

### Seguimiento del progreso

Todo el trabajo se rastrea con una lista de tareas pendientes:

```text
1. Run context script (loads all project state)
2. Present task suggestions
3. Read task-specific documentation
4. Validate story readiness
5. Plan implementation steps
6. Apply code changes with tests
7. Update project status
8. Generate PR description
9. Mark complete
```

Puedes ver exactamente dónde estás y qué queda.

## Uso

### Básico: deja que niñera sugiera trabajo

```bash
/agileflow:babysit
```

Babysit analizará tu proyecto y te preguntará en qué te gustaría trabajar.

### Reanudar el trabajo existente

```bash
/agileflow:babysit
```

Si tiene historias en estado `in_progress`, niñera se ofrecerá a reanudarlas.

### Trabajar en especific historia

Puedes decirle al servicio de niñera "Quiero trabajar en US-0042" y éste:
- Cargar el archivo de la historia.
- Validar que esté listo
- Planificar la implementación.
- Guiarte a través de su construcción.

## Parámetros

Babysit acepta entradas de lenguaje natural de forma libre:

| Entrada | Efecto |
|-------|--------|
| `"Add user profile page"` | Crear una nueva función o encontrar una historia coincidente |
| `"US-0042"` | Trabajar en una historia específica |
| `"Continue"` | Reanuda la última historia en la que estabas trabajando |
| `"What should I work on?"` | Obtener recomendaciones (predeterminado) |

## Modo de bucle autónomo

Babysit admite la ejecución autónoma a través del **Modo de bucle**, que procesa automáticamente las historias completas de una epopeya.

### Inicio rápido

```bash
/agileflow:babysit EPIC=EP-0042 MODE=loop MAX=20
```

### Cómo funciona

```text
1. Initialize loop for epic
2. Load first "ready" story
3. Work on implementation
4. When Claude stops → Stop hook runs tests
   - If tests PASS → mark story complete, load next
   - If tests FAIL → show errors, continue fixing
5. Repeat until epic complete or max iterations
```

### Parámetros

| Parámetro | Requerido | Predeterminado | Descripción |
|-----------|----------|---------|-------------|
| `EPIC` | Sí | - | ID de Epic para procesar (por ejemplo, EP-0042) |
| `MODE` | Sí | - | Debe ser `loop` para el modo autónomo |
|`MAX` | No | 20 | Iteraciones máximas (límite de seguridad) |

### Control de bucle

```bash
# Check loop status
node scripts/ralph-loop.js --status

# Stop the loop manually
node scripts/ralph-loop.js --stop

# Reset loop state
node scripts/ralph-loop.js --reset
```

### Cuándo utilizar el modo bucle

**Bueno para:**
- Procesar una epopeya bien definida con historias claras.
- Desarrollo basado en pruebas (las pruebas definen "hecho")
- Procesamiento por lotes de varias historias.

**No es bueno para:**
- Trabajo exploratorio sin criterios claros de aceptación.
- Historias que requieren revisión humana antes de continuar.
- Trabajo complejo multidominio que necesita coordinación.

Para obtener documentación completa sobre el modo de bucle, incluido el modo visual y el modo de cobertura, consulte la [Guía de funciones del modo de bucle](/features/loop-mode).

### Sesión de ejemplo

```text
/agileflow:babysit EPIC=EP-0042 MODE=loop MAX=20

→ Ralph Loop Initialized
  Epic: EP-0042
  Stories: 5 ready, 7 total
  Starting: US-0015 "Add user authentication"

[Claude implements US-0015...]

→ Stop Hook: Running tests...
  ✓ Tests pass
  ✓ Marked US-0015 complete
  → Moving to US-0016

[Claude implements US-0016...]

→ Stop Hook: Running tests...
  ✗ 2 tests failing
  → Fix errors and continue

[Claude fixes tests...]

→ Stop Hook: Running tests...
  ✓ Tests pass
  ✓ Marked US-0016 complete
  → Moving to US-0017

[...continues until epic complete...]

→ EPIC COMPLETE!
  EP-0042 finished in 12 iterations
  5 stories completed
```

## Ejemplos

### Iniciar una nueva implementación

```bash
/agileflow:babysit
# Babysit suggests: "US-0042: User Login API is READY"
# You confirm → Implementation begins
# Babysit guides through: Plan → Code → Test → PR
```

### Completa múltiples historias relacionadas

```bash
/agileflow:babysit
# Babysit shows: "EP-0005 is 80% done. 1 story left: US-0051"
# You accept → Start final story
# Babysit tracks progress
# When done → Ready for baseline or release
```

### Requisitos desconocidos

```bash
/agileflow:babysit
# You: "I want user authentication but not sure the best way"
# Babysit: "Should I research JWT vs sessions first? This prevents debugging later"
# You: "Yes, research"
# Babysit: Runs /agileflow:research:ask, saves findings, then plans implementation
```

## Descripción general del flujo de trabajo

```text
Start → Gather Context → Suggest Tasks → Choose Task
  ↓
Read Story & Architecture → Validate Readiness → Plan Steps
  ↓
Implement Code → Run Tests → Update Status
  ↓
Generate PR → Finish
```

## Definición de listo

Antes de implementar, niñera comprueba que las historias tengan:

- ✓ Criterios de aceptación (formato dado/cuándo/entonces)
- ✓ Contexto arquitectónico concitas de fuentes
- ✓ Talones de prueba en docs/07-testing/
- ✓ Dependencias resueltas y documentadas
- ✓ Información previa de historias relacionadas

Si falta alguno, niñera sugerirá completarlos primero (las historias permanecen como "borrador" hasta que estén listas).

## Carga de contexto

Cuando inicias babysit, ejecuta este script de contexto automáticamente:

```bash
node .agileflow/scripts/obtain-context.js babysit
```

Este único comando reúne:
- Estado de Git, rama, cambios no confirmados.
- Todas las historias y epopeyas de status.json
- Estado de sesión y sesiones activas.
- Estructura de la documentación
- Notas de investigación (las más recientes primero)
- Mensajes recientes del agente
- Presencia de archivos clave

Una llamada de guión, contexto integral, inicio rápido.

## Cuándo utilizar

### Mejor para

- **Iniciando implementación** - Necesita orientación sobre en qué trabajar
- **Continuando a mitad de la función** - Reanuda desde donde lo dejaste
- **Finalización épica** - Termina las historias finales en un conjunto de funciones.
- **Requisitos inciertos** - Preguntas de investigación antes de codificar
- **Funciones de varios pasos** - Trabajo complejo entre archivos
- **Pruebavalidación**: asegúrese de que las pruebas pasen antes que PR

### No para

- **Soluciones rápidas** (errores tipográficos, frases ingeniosas) - Hágalo usted mismo
- **Investigación pura** - Utilice `/agileflow:research:ask` en su lugar
- **Decisiones arquitectónicas** - Utilice `/agileflow:adr` en su lugar
- **Solo documentación** - Utilice `/agileflow:docs` en su lugar

## Estilo de salida

Babysit utiliza resultados claros y procesables:

- **Títulos** para secciones principales
- **Viñetas cortas** para opciones y explicaciones
- **Bloques de código** para diferencias y ejemplos
- **Preguntas** para guiar tus decisiones (nunca te pide que "escribas")
- **Actualizaciones de progreso** mientras trabajas

Cada respuesta termina con una pregunta sobre el siguiente paso para que siempre tengas el control.

## Consejos

### Obtenga mejores sugerencias

- Mantenga la acumulación de historias en status.json
- Escribir criterios de aceptación claros.
- Marcar historias como listas cuando estén completas
- Mantener actualizados los documentos de arquitectura.

### Implementaciones exitosas

- Deje que la niñera valide primero la preparación para la historia
- Leer la sección de contexto de arquitectura.
- Implementar en pequeños pasos (no todos a la vez)
- Ejecutar pruebas después de cada cambio.
- Pide ayuda si estás atascado

### Orquestación multiexperto

Para funciones complejas, el servicio de niñera automáticamente:
- Experto en API de Spawn para trabajo backend.
- Experto en UI de Spawn para trabajo frontend
- Ejecutar ambos en paralelo si son independientes.
- Coordinar la integración final.

No es necesario administrar manualmente varios agentes: Babysit se encarga de ello.

```text

→ Complete it with babysit's guidance
→ Move to next story
→ When done: /agileflow:baseline "EP-0005 complete"
```

```text

→ Babysit plans implementation based on research
→ Code with confidence
```
