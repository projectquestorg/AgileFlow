# /agileflow:context:export

Exportieren Sie einen prägnanten Kontextauszug zum Einfügen in Web-KI-Tools.

## Schnellstart

```bash
/agileflow:context:export
```

## Zweck

Liest `docs/context.md` und gibt einen komprimierten Auszug (≤300 Zeilen) aus, der in ChatGPT, Perplexity, Gemini oder Claude Web eingefügt werden kann.

**Dies ist ein schreibgeschützter Befehl** – es werden keine Dateien geschrieben.

## Ausgabeformat

Der Befehl gibt einen Markdown-Codeblock zum einfachen Kopieren aus:

```markdown
# Project Context Brief
Last updated: 2025-12-24

## What We're Building
AgileFlow is a comprehensive project management system...

## Current Focus
- EP-0010: Documentation System (in_progress)
- US-0042: Add Loop Mode docs (in_progress)

## Tech Stack
- Framework: Next.js 16, TypeScript
- UI: React, Tailwind CSS, shadcn/ui
- Testing: Jest, Playwright

## Key Decisions
- ADR-001: Use TypeScript for type safety (Accepted)
- ADR-002: Use React component library (Accepted)

## Feature Map
| Epic | Status | Stories |
|------|--------|---------|
| EP-0010 | in_progress | 5 total, 3 done |
| EP-0009 | complete | 7 total, 7 done |

## Next Steps
- US-0043: Visual Mode documentation (ready)
- US-0044: Session commands docs (ready)

---
**Paste this into**: ChatGPT, Perplexity, Gemini, or Claude web
**Purpose**: Load full project context for AI assistance
**Limit**: This excerpt is ≤300 lines for easy pasting
```

## Abschnitte enthalten

Der Export umfasst nur diese Abschnitte (um unter 300 Zeilen zu bleiben):

| Abschnitt | Max-Linien | Zweck |
|---------|-----------|---------|
| Kopfzeile | 2 | Titel und Zeitstempel |
| Was wir bauen | 5 | Projektübersicht |
| Aktueller Fokus | 5 | Aktive Arbeit |
| Tech-Stack | 8 | Technologien |
| Wichtige Entscheidungen | 10 | Nur ADR-Titel |
| Feature-Map | 15 | Eine Zeile pro Epos |
| Nächste Schritte | 15 | Fertige Geschichten |
| Fußzeile | 3 | Anweisungen einfügen |

## Abschnitte ausgeschlossen

Um innerhalb der 300-Zeilen-Grenze zu bleiben, werden diese übersprungen:

- **Aktuelle Fortschritte** -Zu ausführlich
- **Notizenbereich** – Zu lang
- **Detaillierte Story-Beschreibungen** – Für den Kontext nicht erforderlich
- **Vollständiger ADR-Inhalt** – Nur Titel enthalten

## Inhaltsverdichtung

Beim Extrahieren wird der Inhalt komprimiert:

| Element | Volle Größe | Kondensiert |
|---------|-----------|-----------|
| ADRs | Vollständige Beschreibung | Nur Titel + Status |
| Geschichten | Title + description | Titel + Status |
| Technik | Vollständige Liste + Versionen | Framework + Schlüsselabfragen |
| Epen | Titel + Geschichtenliste | Titel + Anzahl + Status |
| Entscheidungen | Vollständiger Kontext | Einzeiler |

## Anforderungen

**docs/context.md muss vor dem Export vorhanden sein**.

Wenn es fehlt, sehen Sie:

```
Error: docs/context.md not found.

To create it, run: /agileflow:context:full

Then you can export with: /agileflow:context:export
```

## Nutzungsworkflow

```bash
# 1. Generate context (if needed)
/agileflow:context:full

# 2. Export concise version
/agileflow:context:export

# 3. Copy the output
# 4. Paste into ChatGPT, Perplexity, Gemini, or Claude web
# 5. Ask questions with full project context
```

## Warum 300 Zeilen?

Web-KI-Tools haben Kontextfensterbeschränkungen. Ein 300-zeiliger Auszug:

- Passt bequem in die meisten Kontextfenster von KI-Tools
- Bietet genügend Details für eine fundierte Unterstützung
- Lässt Raum für Ihre eigentlichen Fragen
- Lässt sich schnell einfügen und verarbeiten

## Wann zu verwenden

- **Vor Web-KI-Sitzungen** - Projekt ladenct-Kontext
- **Beim Wechsel von KI-Tools** - Kontext zwischen Tools übertragen
- **Nach Kontextaktualisierungen** – Erhalten Sie eine neue, prägnante Version
- **Zur Teamfreigabe** - Schneller Projektüberblick
