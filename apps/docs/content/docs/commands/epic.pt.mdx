# /épico

Crie um novo épico com histórias secundárias opcionais para organizar histórias de usuários relacionadas em torno de um recurso ou objetivo maior.

## Início rápido

```bash
/agileflow:epic EPIC=<EP-ID> TITLE=<text> OWNER=<id> GOAL=<text>
```

## Parâmetros

| Parâmetro | Obrigatório | Padrão | Descrição |
|-----------|----------|---------|------------|
| `EPIC` | Sim | - | Identificador épico (por exemplo, EP-0001) |
| `TITLE` | Sim | - | Título épico que descreve o recurso ou objetivo |
| `OWNER` | Sim | - | Nome do proprietário ou ID do agente (por exemplo, AG-UI, AG-API) |
| `GOAL` | Sim | - | Resultado ou métrica que define o sucesso deste épico |
| `STORIES` | Não | - | Lista de histórias iniciais separadas por vírgulas (formato: `US-ID\|title\|owner`) |

## Exemplos

### Criação Épica Básica

```bash
/agileflow:epic EPIC=EP-0010 TITLE="User Authentication" OWNER=AG-API GOAL="Enable users to securely register and log in"
```

Cria um novo arquivo épico em `docs/05-epics/EP-0010.md` com os detalhes fornecidos e o registra em `docs/09-agents/status.json`.

### Épico com histórias iniciais

```bash
/agileflow:epic EPIC=EP-0011 TITLE="Payment Processing" OWNER=AG-API GOAL="Enable payments via Stripe" STORIES="US-0040|Stripe integration|AG-API,US-0041|Payment form|AG-UI,US-0042|Webhook handler|AG-API"
```

Cria o épico e gera automaticamente histórias secundárias iniciais em __NOTRASLATE_11__. Cada umhistórico é criado com:
- Status: __NOTRASLATE_12__ (pronto para desenvolvimento)
- Estimativa: __NOTRASLATE_13__ (padrão, pode ser modificado)
- Dependências: vazias (atualizadas manualmente conforme necessário)

## Saída

O comando cria:

1. **Arquivo épico** (`docs/05-epics/EP-XXXX.md`)
   - Metadados épicos (título, proprietário, status, objetivo)
   - Status e métricas atuais
   - Lista de histórias infantis

2. **Arquivos de histórias** (um por história, `docs/06-stories/EP-XXXX/US-XXXX-<slug>.md`)
   - Modelo de história com espaços reservados para critérios de aceitação
   - Vinculado ao épico pai
   - Pronto para definição detalhada

3. **Rastreamento de status** (`docs/09-agents/status.json`)
   - Entrada épica com proprietário e status atual
   - Entradas de história vinculadas ao épico

4. **Comunicação do agente** (__NOTRASLATE_17__)
   - Mensagens de atribuição mostrando histórias épicas criadas e atribuídas

## Fluxo de trabalho

1. **Planeje seu épico** - Defina a meta do recurso e as métricas de sucesso
2. **Crie o épico** - Execute o comando __NOTRASLATE_18__ com detalhes básicos
3. **Adicionar histórias** - Crie-as agora com o parâmetro STORIESou use __NOTRASLATE_19__ para adicionar individualmente mais tarde
4. **Desenvolver histórias** - Use __NOTRASLATE_20__ para garantir que cada história esteja pronta antes do desenvolvimento
5. **Plano de sprint** - Use __NOTRASLATE_21__ para selecionar histórias para o próximo sprint
6. **Acompanhe o progresso** - Use __NOTRASLATE_22__ para monitorar a conclusão épica

## Integração com outros comandos

- **Após a criação épica**: Use __NOTRASLATE_23__ para adicionar mais histórias
- **Antes do planejamento do sprint**: Use __NOTRASLATE_24__ para selecionar histórias deste épico
- **Para acompanhamento do progresso**: Use `/board` ou __NOTRASLATE_26__ para ver a conclusão épica
- **Para dependências**: Use __NOTRASLATE_27__ para visualizar dependências de histórias dentro do épico

## Melhores práticas

- **Mantenha o foco nos épicos** - Um recurso ou área de recurso por épico
- **Tornar as metas mensuráveis** - Usar resultados ou métricas específicas (não "melhorar a experiência do usuário")
- **Adicione histórias de forma incremental** - Comece com histórias principais, adicione histórias relacionadas ao longo do tempo
- **Atribuir proprietários apropriados** - Combine o próprio épicoer com agente de desenvolvimento primário
- **Contexto do documento** - Adicionar contexto técnico no arquivo épico após a criação
