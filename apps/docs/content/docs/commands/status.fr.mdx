# /statut

Mettez à jour le statut de l'histoire dans le système de suivi du projet et diffusez les modifications aux agents via le bus de messages.

## Démarrage rapide

```bash
/agileflow:status STORY=US-0001 STATUS=in-progress
```

## Paramètres

| Paramètre | Obligatoire | Par défaut | Descriptif |
|-----------|----------|---------|-------------|
| `STORY` | Oui | - | ID de l'histoire (par exemple, US-0001, US-0042) |
| `STATUS` | Oui | - | Nouveau statut : `ready`, `in-progress`, `blocked`, `in-review` ou `done` |
| `SUMMARY` | Non | - | Résumé de 1 à 2 lignes des progrès ou de la raison du changement de statut |
| `PR` | Non | - | URL de demande d'extraction (pour le statut de révision/terminé) |
| `TO` | Non | TOUS | Agent(s) cible(s) à notifier via le bus de messages |

## Exemples

### Commencez à travailler sur une histoire

```bash
/agileflow:status STORY=US-0042 STATUS=in-progress SUMMARY="Beginning login form implementation"
```

Cela met à jour le statut de l'histoire et informe tous les agents que vous avez commencé à travailler.

### Marquer comme bloqué

```bash
/agileflow:status STORY=US-0045 STATUS=blocked SUMMARY="Waiting on AG-API to implement /auth/login endpoint"
```

Diffusez le bloqueur afin que les autres membres de l’équipe puissent aider à le résoudre.

### Passer à la révision du code```bash
/agileflow:status STORY=US-0038 STATUS=in-review PR=https://github.com/your-repo/pull/123 SUMMARY="Login form complete, ready for review"
```

Partagez le lien PR pour les évaluateurs.

### Terminer une histoire

```bash
/agileflow:status STORY=US-0030 STATUS=done SUMMARY="User registration fully tested and deployed"
```

Marquez l'histoire comme terminée.

## Sortie

La commande affiche :
1. **Aperçu des différences** - Affiche ce qui va changer dans `docs/09-agents/status.json`
2. **Confirmation** - Invite OUI/NON pour appliquer les modifications
3. **Message de réussite** - Confirmation que le statut a été mis à jour
4. **Message Bus Entry** - Message de diffusion envoyé aux agents

Exemple de sortie :
```text
✅ Story Status Updated
Story:    US-0042
Status:   ready → in-progress
Summary:  Beginning login form implementation
Updated:  2025-12-24T14:30:00Z

Message broadcast to: ALL agents
```

## Fichiers de données mis à jour

Cette commande modifie :
- `docs/09-agents/status.json` - Suivi du statut de l'histoire
- `docs/09-agents/bus/log.jsonl` - Bus de messages de l'agent (ajoute un événement de changement d'état)

## Valeurs d'état expliquées

- **prêt** - L'histoire est préparée et attend d'être démarrée
- **en cours** - En cours de travail actif
- **bloqué** - En attente de dépendance externe ou de clarification
- **en cours de révision** - Révision/approbation du code terminée et en attente
- **fait** - Terminé, testé et vérifié

## meilleures pratiques

1. **Toujours inclure un résumé** - Aide l'équipe à comprendre le contexte et les progrès2. **Mettre à jour immédiatement** - N'attendez pas les mises à jour de statut par lots
3. **Lier les PR pour les avis** - Aide les évaluateurs à trouver votre travail
4. **Utilisez un langage clair** - Expliquez les bloqueurs en détail pour que d'autres puissent vous aider
5. **Mettre à jour en cas de blocage** - Ne vous asseyez pas silencieusement sur les bloqueurs - communiquez immédiatement
