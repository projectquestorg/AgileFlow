# /comentários

Colete feedback de agentes e humanos sobre histórias, épicos e sprints para melhoria contínua do processo.

## Início rápido

```bash
/agileflow:feedback SCOPE=story STORY=US-0042
```

## Parâmetros

| Parâmetro | Obrigatório | Padrão | Descrição |
|-----------|----------|---------|------------|
| `SCOPE` | Não | história | Escopo do feedback: história, épico ou sprint |
| `STORY` | Condicionalmente | - | ID da história (obrigatório se SCOPE=story) |
| `EPIC` | Condicionalmente | - | Epic ID (obrigatório se SCOPE=epic) |
| `ANONYMOUS` | Não | não | Permitir feedback anônimo (sim/não) |

## Exemplos

### Feedback sobre a conclusão da história

```bash
/agileflow:feedback SCOPE=story STORY=US-0042
```

Depois de marcar uma história como concluída, colete:
- Classificação de clareza AC (1-5)
- Classificação de resolução de dependências (1-5)
- Estimativa de classificação de precisão (1-5)
- Classificação de suavidade de implementação (1-5)
- O que correu bem
- O que poderia ser melhorado

### Feedback retrospectivo épico

```bash
/agileflow:feedback SCOPE=epic EPIC=EP-0010
```

Após a conclusão épica, reúna:
- Métricas de sucesso em relação a objetivos épicos
- O que correu bem durante execution
- What didn't go well
- Surprises and learnings
- Actions for next epic

### Sprint Retrospective Feedback

```bash
/agileflow:feedback SCOPE=sprint
```

At sprint end, collect team feedback:
- Continue (keep doing)
- Stop (no longer useful)
- Start (new practices)
- Experiments to try
- Blockers removed this sprint

### Anonymous Feedback

```bash
/agileflow:feedback SCOPE=story STORY=US-0045 ANONYMOUS=yes
```

Collects feedback without attribution for sensitive topics.

## Output

### Feedback Files

Saves feedback to:
```text
docs/08-project/feedback/<YYYYMMDD>-<story-or-epic-id>.md
```

## Feedback Types

### 1. Story Completion Feedback

Ratings for:
- **AC Clarity** - Were acceptance criteria clear? (1-5)
- **Dependencies** - Were blockers resolved? (1-5)
- **Estimate Accuracy** - Was estimation accurate? (1-5)
- **Implementation** - How smooth was the process? (1-5)
- **Testing** - Were tests adequate? (1-5)
- **Documentation** - Was documentation sufficient? (1-5)

Plus open-ended:
- What went well? (2-3 bullets)
- What could be improved? (2-3 bullets)
- Bloqueadores encontrados?
- Aprendizados/insights?

### 2. Feedback de desempenho do agente

Rastreia a eficácia:
- Histórias concluídas
- Histórias bloqueadas
- Tempo médio de conclusão
- Pontos fortes observados
- Áreas para melhoria
- Recomendações

### 3. Retrospectiva épica

Após a conclusão épica:
- Métricas de sucesso versus metas
- O que correu bem?
- O que não deu certo?
- Surpresas e aprendizados
- Ações para o próximo épico

### 4. Retrospectiva do Sprint

No final do sprint:
- Continue (continue fazendo)
- Parar (não é mais útil)
- Iniciar (novas práticas)
- Experimentos para tentar
- Bloqueadores removidos
- Problemas recorrentes

## Métricas rastreadas

O sistema de feedback rastreia:

| Métrica | Alvo | Finalidade |
|--------|--------|-----|
| Pontuação de clareza da história | >4,0 | Qualidade AC |
| Precisão da estimativa | dentro de 50% | Melhoria do planejamento |
| Frequência do bloqueador | menos de 20% das histórias | Gestão de dependências |
| Média de cobertura dos testes | >85% | Qualidade do código |
| Velocidade de conclusão | Tendências | Rendimento da equipe |

## Análise e insights

### Padrão Deproteção

Detecta padrões automaticamente:
- **AC pouco claro** - Histórias com pontuação de clareza abaixo de 3 → Melhorar modelo
- **Estimativas ruins** - Grande variação → Revisar guia de estimativa
- **Bloqueadores frequentes** - Histórias frequentemente bloqueadas → Melhorar o rastreamento de dependências
- **Baixa cobertura de testes** - Testes inadequados → Aplicar os padrões mais cedo

### Melhorias acionáveis

Gera automaticamente:
- **Histórias de melhorias** para problemas recorrentes
- **Recomendações de processo** para problemas detectados
- **Reconhecimento** por vitórias e alto desempenho
- **ADRs** para aprendizados arquitetônicos

### Relatórios retrospectivos

Gera resumos abrangentes:
- Sentimento geral (melhorando/declinando/estável)
- Principais vitórias com comemoração
- Principais desafios
- Ações recomendadas
- Insights e aprendizados da equipe

## Fluxo de trabalho

1. Solicitação automática em pontos de gatilho:
   - O status da história muda para "concluído"
   - Epic atinge 100% de conclusão
   - Data de término do sprint atingida
2. Apresentar formulário de feedback com contexto pré-preenchido
3. Pergunte: "Fornecer feedback agora? (SIM/NÃO/DEPOIS)"
4. Se SIM: Colete classificações e comentários de forma interativa
5. Salve no diretório de comentários
6. Analise padrões em todos os comentários
7. Sugira histórias de melhorias para problemas

## Use quando

- **Conclusão da história** - Imediatamente após a marcação ser concluída
- **Retrospectiva épica** - Após a conclusão épica
- **Fim do sprint** - Reúna os aprendizados da equipe
- **Melhoria de processos** - Revisão mensal dos padrões de feedback
- **Avaliação de desempenho** - Avaliações trimestrais dos agentes
- **Retrospectivas da equipe** - Cerimônias formais de sprint

## Gatilhos automáticos

Solicitações de feedback aparecem:
- Quando o status da história muda para "concluído"
- Quando o épico atinge 100% de conclusão
- Nas datas de término do sprint configuradas
- Pode ser solicitado manualmente a qualquer momento

## Integração

O feedback flui para:
- **Retrospectivas** - Insights incluídos nas retrospectivas do sprint
- **Métricas** - Avaliações monitoradas ao longo do tempo
- **Criação de histórias** - Gerar automaticamente histórias de melhorias
- **ADRs** - Documentar aprendizados como decisões arquitetônicas
- **Perfis de agente** - Lista de atualizações de dados de desempenhors
