# /seo:images

Analysez l'optimisation des images sur une page : qualité du texte alternatif, dimensionnement explicite pour la prévention CLS, formats modernes (WebP/AVIF), chargement différé, priorité des images LCP et images réactives.

## Démarrage rapide

```bash
/agileflow:seo:images https://example.com                    # Image audit for homepage
/agileflow:seo:images https://example.com/products/item       # Product page images
```

## Paramètres

| Parameter | Required | Description |
|-----------|----------|-------------|
| `<URL>` | Yes | Page to analyze |

| Aspect | Weight | Impact |

| Alt text quality | 30% | Accessibility + image search rankings |

| Modern formats | 15% | Page speed + LCP |

## Notation du texte alternatif

| Quality | Criteria | Score Impact |
|---------|----------|------------|
| **Good** | Descriptive, 10-125 chars, contextual keyword | +points |
| **Acceptable** | Present but generic ("image", "photo") | Neutral |
| **Poor** | Filename-based ("IMG_001.jpg", "screenshot") | -2 points |
| **Missing** | No alt attribute at all | -3 points (HIGH severity) |
| **Decorative** | Empty alt="" on decorative images | Correct (no deduction) |
| **Keyword-stuffed** | Excessive keywords crammed in | -2 points |

## Directives relatives aux textes alternatifs

Bon texte alternatif :
- Décrit ce que montre l'image
- 10 à 125 caractères recommandés
- Inclut naturellement des mots-clés pertinents
- Ne dit pas "image de" ou "image de"
- Pour les images décoratives, utilisez `alt=""`

Exemple de bon texte alternatif :
- "Femme professionnelle tapant sur un ordinateur portable au bureau moderne"
- "Capture d'écran du tableau de bord de Google Search Console affichant les mesures de trafic"
- "Photo produit : Écouteurs sans fil bleus avec réduction de bruit"

Exemple de texte alternatif médiocre :
- "IMG_001.jpg" ❌
- "images" ❌
- "image de trucs" ❌
- "blue-headphones-wireless-noise-canceling-buy-now" (bourrage de mots clés) ❌

## Recommandations en matière de format d'image

| Format | Use When | Not When |
|--------|----------|----------|
| WebP | Modern format, wide support | Supporting very old browsers |
| AVIF | Best compression, growing support | Older browser support needed |
| JPEG | Photos, realistic images | Transparency needed |
| PNG | Transparency needed | Photos (use JPEG) |
| GIF | Animations | Use MP4 video instead |
| SVG | Icons, illustrations, logos | Photos |
| BMP/TIFF | Never | Never |

## Images réactives

Pour une diffusion optimale sur mobile et ordinateur :

```html
<!-- Good: Responsive with srcset and sizes -->
<img
  src="image-800.jpg"
  srcset="image-400.jpg 400w, image-800.jpg 800w, image-1200.jpg 1200w"
  sizes="(max-width: 600px) 400px, (max-width: 900px) 800px, 1200px"
  width="800"
  height="600"
  alt="Descriptive alt text"
  loading="lazy"
>

<!-- Better: Format alternatives with <picture> -->
<picture>
  <source type="image/avif" srcset="image.avif">
  <source type="image/webp" srcset="image.webp">
  <img src="image.jpg" alt="Descriptive alt text" width="800" height="600">
</picture>
```

## Optimisation des images LCP

Pour votre héros/image la plus grande :

```html
<!-- Good: LCP image gets priority -->
<img
  src="hero.jpg"
  alt="Hero section describing main benefit"
  width="1200"
  height="600"
  fetchpriority="high"
  loading="eager"
/>

<!-- With preload for extra speed -->
<link rel="preload" as="image" href="hero.jpg">
<img src="hero.jpg" alt="..." width="1200" height="600">
```

## Remediation Examples

### Fix Missing Alt Text
```html
<!-- Avant -->
<img src="hero.jpg">

<!-- Après -->
<img src="hero.jpg" alt="Team collaborating on project strategy in modern office">
```

### Fix Missing Width/Height
```html
<!-- Avant (risque CLS) -->
<img src="photo.jpg" alt="...">

<!-- Après (empêche le décalage) -->
<img src="photo.jpg" alt="..." width="800" height="600">
```

### Add Lazy Loading
```html
<!-- Avant -->
<img src="below-fold.jpg" alt="...">

<!-- Après -->
<img src="below-fold.jpg" alt="..." loading="lazy">
```

### Add Format Alternatives
```html
<!-- Avant -->
<img src="image.jpg" alt="..." width="800" height="600">

<!-- Après (40 % plus petit sur les navigateurs modernes) -->

  <source type="image/webp" srcset="image.webp">
  <img src="image.jpg" alt="..." width="800" height="600">
</picture>
```

### Add LCP Priority
```html
<!-- Avant -->
<img src="hero.jpg" alt="..." width="1200" height="600">

<!-- Après (peinture plus rapide) -->
<img src="hero.jpg" alt="..." width="1200" height="600" fetchpriority="high">
```

## Impact sur les performances

L'optimisation des images peut améliorer :
- **Core Web Vitals**
  - LCP : livraison plus rapide des images de héros
  - CLS : empêche le changement de mise en page des images non dimensionnées
- **Page Speed**: chargement 20 à 40 % plus rapide avec les formats modernes
- **User Experience**: Chargement des pages plus rapide, meilleures performances perçues
- **SEO**: Meilleur score Core Web Vitals → amélioration potentielle du classement

## Résumé des meilleures pratiques

1. **Always include alt text** - Accessibilité + recherche d'images
2. **Always include width/height** - Empêche le changement de disposition (CLS)
3. **Use modern formats** - WebP pour 30 % plus petit, AVIF pour 50 %+
4. **Lazy load below-fold** - `loading="lazy"` reporte les images non critiques
5. **Prioritize LCP image** - `fetchpriority="high"` sur le héros
6. **Make responsive** - `srcset` et `sizes` pour les tailles adaptées à l'appareil
7. **Test before publish** - Utilisez PageSpeed ​​Insights pour vérifier les améliorations
