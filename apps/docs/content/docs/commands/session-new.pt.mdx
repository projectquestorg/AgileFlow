# /sessão:novo

Crie uma nova sessão paralela do Claude Code com configuração automática da árvore de trabalho git. Isso permite que você trabalhe em várias tarefas simultaneamente, sem conflitos.

## Início rápido

```bash
/agileflow:session:new
```

## Como funciona

Ao executar este comando, você verá opções:

1. **Criar sessão automaticamente N** - Cria automaticamente uma sessão numerada (por exemplo, Sessão 2, Sessão 3)
2. **Nomeie esta sessão** - Dê um apelido como "auth" ou "refactor"
3. **Usar branch existente** - Anexe a um branch que você já possui

O comando então:
1. Cria uma nova árvore de trabalho git em um diretório paralelo
2. Registra a sessão no registro de sessão
3. Mostra o comando __NOTRASLATE_6__ para mudar para o novo espaço de trabalho

## Exemplo de saída

```
Creating Session 2...

✓ Session created successfully

  Session ID: 2
  Path: /home/user/myproject-2
  Branch: session-2

To start working in this session:
  cd /home/user/myproject-2 && claude
```

## Parâmetros

Este comando usa prompts interativos (AskUserQuestion) em vez de parâmetros de linha de comando.

| Opção | Descrição |
|--------|------------|
| Criar automaticamente | Usa o próximo ID de sessão disponível |
| Sessão nomeada | Cria com personalização nickname (e.g., "auth" → `/home/user/myproject-auth`) |
| Existing branch | Attaches to a branch without creating a new one |

## Why Use Parallel Sessions?

### The Problem

Running multiple Claude Code instances in the same directory causes:
- File locking conflicts (especially on macOS)
- Risk of conflicting edits to the same files
- Shared `~/.claude/` directory issues
- No visibility into what other sessions are working on

### The Solution

Git worktrees provide:
- **Isolated directories** - Each session has its own folder
- **Shared .git** - Disk efficient, no duplicate repo
- **Independent branches** - No merge conflicts during work
- **Clean separation** - Sessions can't step on each other

## Workflow

### Starting a Second Task

```bash
# In your main project
/agileflow:session:new
# Pick "Auto-create Session 2"

# Claude shows:
#   cd /home/user/myproject-2 && claude

# Open new terminal
cd /home/user/myproject-2
claude
```

### Naming Sessions by Feature

```bash
# Create session for auth work
/agileflow:session:new
# Pick "Name this session" → "auth"

# Creates: /home/user/myproject-auth on branch session-auth
```

### Using Existing Branch

```bash
# You have a feature branch already
git branch
# feature/new-api
# main

/agileflow:session:new
# Pick "Use existing branch" → "feature/new-api"

# Creates worktree attached to that branch
```

## Session Registry

Sessions are tracked in `.agileflow/sessions/registry.json`:

```json
{
  "sessions": {
    "1": {
      "path": "/home/user/myproject",
      "branch": "main",
      "is_main": true
    },
    "2": {
      "path": "/home/user/myproject-2",
      "branch": "session-2",
      "nickname": null
    }
  }
}
```

## Best Practices

- **One task per session** - Keep ssessões focadas
- **Use apelidos para trabalhos de longa duração** - Mais fácil de lembrar do que números
- **Terminar sessões quando terminar** - Use __NOTRASLATE_10__ para limpar
- **Verifique o status antes de iniciar** - Use __NOTRASLATE_11__ para ver o que está sendo executado
