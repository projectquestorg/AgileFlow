# /تحديث

قم بإنشاء تحديثات عالية المستوى للمشروع لأصحاب المصلحة أو المديرين التنفيذيين أو العملاء باستخدام المقاييس والتقدم والمخاطر.

## بداية سريعة

```bash
/agileflow:update PERIOD=week AUDIENCE=exec
```

## حدود

| Parameter | Required | Default | Description |
|-----------|----------|---------|-------------|
| `PERIOD` | No | week | Time frame: week, sprint, month, quarter, or custom |
| `START_DATE` | No | - | For custom period (YYYY-MM-DD format) |
| `END_DATE` | No | - | For custom period (YYYY-MM-DD format) |
| `AUDIENCE` | No | exec | Target audience: exec, client, team, or board |
| `FORMAT` | No | markdown | Output format: email, markdown, slides, or pdf |

## أمثلة

### التحديث الأسبوعي التنفيذي

```bash
/agileflow:update PERIOD=week AUDIENCE=exec FORMAT=email
```

يُنشئ تحديثًا موجزًا ​​من صفحة واحدة للمديرين التنفيذيين على المستوى C مع:
- الملخص التنفيذي والحالة العامة
- الإنجازات الرئيسية
- العوائق أو المخاطر الحرجة
- السرعة والمقاييس
- لا المصطلحات التقنية

### تقرير العميل نصف الأسبوعي

```bash
/agileflow:update PERIOD=2weeks AUDIENCE=client FORMAT=markdown
```

يُنشئ تحديثًا يركز على الميزات للعملاء الذين لديهم:
- ما يمكن للمستخدمين القيام به الآن
- التقدم نحو تسليمات العقد
- لقطات الشاشة أو المراجع التجريبية
- التواصل الشفاف بشأن القضايا
- لغة مفيدة للمستخدم

### فريق سبرينت بأثر رجعي

```bash
/agileflow:update PERIOD=sprint AUDIENCE=team FORMAT=markdown
```

يولد تحديثا مفصلا للفريق مع:
- قرارات الهندسة المعمارية و ADRs
- الإنجازات الفنية
- الاحتفال بالانتصارات
- رؤى بأثر رجعي
- عناصر العمل

### تحديث الفترة المخصصة

```bash
/agileflow:update PERIOD=custom START_DATE=2025-12-01 END_DATE=2025-12-15 AUDIENCE=board FORMAT=pdf
```

## الإخراج

يقوم الأمر بإنشاء تحديث شامل محفوظ في:
```text
docs/08-project/updates/<YYYYMMDD>-update.md
```

### تحديث الهيكل

**Executive Summary**
- 2-3 نظرة عامة على الجملة
- الإنجازات الرئيسية
- القضايا الحرجة أو المخاطر
- الحالة العامة (على المسار الصحيح / معرض للخطر / محظور)

**Progress This Period**
- قصص مكتملة مع التهم
- نسب التقدم الملحمة
- وصلت المعالم

**Upcoming Priorities**
- مجالات التركيز التالية للسباق
- المعالم القادمة مع التواريخ

**Metrics & Trends**
- السرعة (قصص / أسبوع)
- النسبة المئوية لمعدل الإنجاز
- مقاييس الجودة (تغطية الاختبار، الأخطاء)
- الاتجاهات (التحسن/الانخفاض)

**Blockers & Risks**
- الحاصرات الحالية مع أصحابها
- استراتيجيات تخفيف المخاطر
- حالة كل عنصر

**Decisions Made**
- سجلات القرارات المعمارية الحديثة (ADRs)
- الأساس المنطقي للاختيارات الرئيسية

**Team Updates** (إن أمكن)
- تغييرات في قدرات الفريق
- إنجازات ملحوظة
- المؤشرات المعنوية

## تخصيص الجمهور

### تنفيذي (المستوى C)
- نظرة عامة رفيعة المستوى للغاية
- التركيز على تأثير الأعمال
- المقاييس وعائد الاستثمار
- تم تسليط الضوء على المخاطر في وقت مبكر
- الحد الأقصى 1 صفحة

### عميل
- لغة تركز على الميزات
- وشدد على فوائد المستخدم
- الشفافية في التعامل مع القضايا
- تم تضمين لقطات الشاشة/الروابط التجريبية
- التقدم نحو تسليمات العقد

### فريق
- تم تضمين المزيد من التفاصيل الفنية
- وشملت القرارات المعمارية
- الاحتفال بالانتصارات
- رؤى بأثر رجعي
- بنود العمل للفترة القادمة

### سبورة
- نظرة استراتيجية
- الآثار المالية
- تحديد المواقع التنافسية
- خارطة طريق طويلة المدى
- تقييم المخاطر

## سير العمل

1. تحديد الفترة الزمنية (الافتراضي: هذا الأسبوع)
2. جمع البيانات من جميع المصادر العشرة
3. حساب المقاييس والاتجاهات
4. تحديد العمل المكتمل والحاصرات
5. تنسيق للجمهور المستهدف
6. معاينة التحديث في تخفيض السعر
7. اسأل: "إرسال التحديث؟ (نعم/لا/تحرير)"
8. إذا كانت الإجابة بنعم: احفظ في docs/08-project/updates/ وأرسل بريدًا إلكترونيًا اختياريًا إلى أصحاب المصلحة

## استخدم متى

- **Weekly team syncs** - تحديثات إيقاع منتظمة
- **Executive presentations** - التقارير الشهرية / ربع السنوية
- **Client communication** - تحديثات التقدم التعاقدي
- **Sprint retrospectives** - التعلم الذي يركز على الفريق
- **Board meetings** - تحديثات التوافق الاستراتيجي
- **Risk escalation** - تسليط الضوء على حاصرات في وقت مبكر

## اندماج

رابط للملاحم والقصص لمزيد من التفاصيل. ADRs المرجعية للسياق المعماري. قم بتضمين روابط GitHub PR حيثما كان ذلك مناسبًا.
