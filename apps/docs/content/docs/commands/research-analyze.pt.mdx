# /pesquisa:analisar

Revise as notas de pesquisa existentes e analise como elas poderiam ser implementadas em seu projeto com o contexto específico do projeto.

## Início rápido

```bash
/agileflow:research:analyze FILE=20250106-caching-strategy.md
```

## Parâmetros

| Parâmetro | Obrigatório | Padrão | Descrição |
|-----------|----------|---------|------------|
| `FILE` | Não | - | Nome do arquivo da nota de pesquisa (será solicitado se não for fornecido) |

## Exemplos

### Analise pesquisas específicas

```bash
/agileflow:research:analyze FILE=20250106-nextjs-upgrade.md
```

Analisa a pesquisa de atualização do Next.js e mostra como implementá-la em seu projeto.

### Seleção Interativa

```bash
/agileflow:research:analyze
```

Sem o parâmetro FILE, lista as pesquisas disponíveis e solicita que você selecione uma.

## Melhores práticas

### Quando usar /research:analyze

**Use-o quando:**
- Você importou pesquisas anteriormente e agora precisa de implementação
- Você deseja entender o impacto específico do projeto
- Você está decidindo se vai resolver isso agora ou mais tarde
- Vocêquer transformar pesquisa em trabalho rastreável

**Não use quando:**
- Apenas lendo pesquisas para obter informações (use __NOTRASLATE_17__)
- Não está pronto para implementar (apenas revise, não crie artefatos)
- Analisando novas pesquisas (use __NOTRASLATE_18__ primeiro)

### Obtendo melhores recomendações

Antes de analisar a pesquisa:
- Certifique-se de que sua base de código esteja atualizada (último commit)
- Tenha os arquivos principais organizados (README, documentos principais)
- Entenda sua pilha de tecnologia (documentado em CLAUDE.md)

A análise lê seu código real para fazer recomendações específicas.

### Escolhendo o tipo de artefato

**Escolha ADR se:**
- Esta é uma decisão arquitetônica única
- Várias opções de trade-off foram avaliadas
- A equipe precisa entender o porquê

**Escolha Épico + Histórias se:**
- O recurso abrange mais de 5 etapas de implementação
- Vários arquivos e domínios afetados
- 3+ dias de esforço

**Escolha História se:**
- Tarefa de implementação única e focada
- 1-3 arquivos afetados
- Pode ser feito em 1-2 horas

**Escolha Documento prático se:**
- Esta é a orientação/melhor ppráticas
- Aplica-se a muitas tarefas futuras
- Não rastreado como trabalho
