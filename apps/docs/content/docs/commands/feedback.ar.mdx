# /تعليق

اجمع التعليقات من الوكلاء والبشر حول القصص والملاحم والسباقات السريعة من أجل التحسين المستمر للعملية.

## بداية سريعة

```bash
/agileflow:feedback SCOPE=story STORY=US-0042
```

## حدود

| Parameter | Required | Default | Description |
|-----------|----------|---------|-------------|
| `SCOPE` | No | story | Feedback scope: story, epic, or sprint |
| `STORY` | Conditionally | - | Story ID (required if SCOPE=story) |
| `EPIC` | Conditionally | - | Epic ID (required if SCOPE=epic) |
| `ANONYMOUS` | No | no | Allow anonymous feedback (yes/no) |

## أمثلة

### ردود الفعل على إكمال القصة

```bash
/agileflow:feedback SCOPE=story STORY=US-0042
```

بعد وضع علامة "تمام" على القصة، اجمع:
- تصنيف وضوح التيار المتردد (1-5)
- تصنيف دقة التبعية (1-5)
- تقدير معدل الدقة (1-5)
- تصنيف سهولة التنفيذ (1-5)
- ما سار على ما يرام
- ما يمكن تحسينه

### ردود فعل ملحمية بأثر رجعي

```bash
/agileflow:feedback SCOPE=epic EPIC=EP-0010
```

بعد الانتهاء الملحمي، اجمع:
- مقاييس النجاح ضد الأهداف الملحمية
- ما سار بشكل جيد أثناء التنفيذ
- ما لم يسير على ما يرام
- مفاجآت ودروس
- الإجراءات للملحمة القادمة

### سبرينت ردود الفعل بأثر رجعي

```bash
/agileflow:feedback SCOPE=sprint
```

في نهاية السباق، اجمع تعليقات الفريق:
- استمر (استمر في العمل)
- توقف (لم يعد مفيدًا)
- ابدأ (ممارسات جديدة)
- تجارب للمحاولة
- قام الحاصرون بإزالة هذا السباق

### ردود فعل مجهولة

```bash
/agileflow:feedback SCOPE=story STORY=US-0045 ANONYMOUS=yes
```

يجمع التعليقات دون إسناد للمواضيع الحساسة.

## الإخراج

### ملفات ردود الفعل

يحفظ التعليقات في:
```text
docs/08-project/feedback/<YYYYMMDD>-<story-or-epic-id>.md
```

### مثال على تعليقات القصة

```markdown
## Story Feedback: US-0042

**Completed by**: AG-API
**Date**: 2025-12-22

### Ratings (1-5)
- AC clarity: 5 (crystal clear)
- Dependencies resolved: 4 (one minor blocker)
- Estimate accuracy: 5 (spot on)
- Implementation smoothness: 4 (smooth)

### What Went Well
- Clear acceptance criteria with examples
- All tests passed on first run
- Good documentation

### What Could Be Improved
- Database schema migration took longer than expected

### Blockers Encountered
- None significant

### Learnings/Insights
- JSON schema validation saved hours of debugging
```

## أنواع ردود الفعل

### 1. ردود الفعل على إكمال القصة

تقييمات ل:
- **AC Clarity** - هل كانت معايير القبول واضحة؟ (1-5)
- **Dependencies** - هل تم حل الحاصرات؟ (1-5)
- **Estimate Accuracy** - هل كان التقدير دقيقا؟ (1-5)
- **Implementation** - ما مدى سلاسة العملية؟ (1-5)
- **Testing** - هل كانت الاختبارات كافية؟ (1-5)
- **Documentation** - هل كان التوثيق كافيا؟ (1-5)

بالإضافة إلى مفتوحة:
- ما الذي سار على ما يرام؟ (2-3 رصاصات)
- ما الذي يمكن تحسينه؟ (2-3 رصاصات)
- واجهت حاصرات؟
- التعلم/الرؤى؟

### 2. ملاحظات أداء الوكيل

فعالية المسارات:
- اكتملت القصص
- قصص محظورة
- متوسط ​​وقت الانتهاء
- لوحظت نقاط القوة
- مجالات التحسين
- التوصيات

### 3. ملحمة بأثر رجعي

بعد الانتهاء الملحمي:
- مقاييس النجاح مقابل الأهداف
- ما الذي سار على ما يرام؟
- ما الذي لم يسير على ما يرام؟
- مفاجآت ودروس
- الإجراءات للملحمة القادمة

### 4. سبرينت بأثر رجعي

في نهاية السباق:
- استمر (استمر في العمل)
- توقف (لم يعد مفيدًا)
- ابدأ (ممارسات جديدة)
- تجارب للمحاولة
- تمت إزالة الحواجز
- القضايا المتكررة

## تتبع المقاييس

يتتبع نظام ردود الفعل:

| Metric | Target | Purpose |
|--------|--------|---------|
| Story clarity score | >4.0 | AC quality |
| Estimate accuracy | within 50% | Planning improvement |
| Blocker frequency | under 20% of stories | Dependency management |
| Test coverage average | >85% | Code quality |
| Completion velocity | Trending up | Team throughput |

## التحليل والرؤى

### كشف النمط

أنماط الاكتشاف التلقائي:
- **Unclear AC** - قصص ذات درجات وضوح أقل من 3 → تحسين القالب
- **Poor estimates** - تباين كبير → مراجعة دليل التقدير
- **Frequent blockers** - القصص محظورة غالبًا → تحسين تتبع التبعية
- **Low test coverage** - الاختبارات غير كافية → تطبيق المعايير في وقت مبكر

### تحسينات قابلة للتنفيذ

الإنشاء التلقائي:
- **Improvement stories** للمشاكل المتكررة
- **Process recommendations** للمشاكل المكتشفة
- **Recognition** لتحقيق الانتصارات والأداء العالي
- **ADRs** للتعلم المعماري

### تقارير بأثر رجعي

يولد ملخصات شاملة:
- الشعور العام (تحسن/انخفاض/مستقر)
- أعلى الانتصارات مع الاحتفال
- أعلى التحديات
- الإجراءات الموصى بها
- رؤى الفريق والتعلم

## سير العمل

1. المطالبة التلقائية عند نقاط الزناد:
   - تتغير حالة القصة إلى "تم"
   - الملحمة تصل إلى 100٪ من الاكتمال
   - تم الوصول إلى تاريخ انتهاء السباق
2. تقديم نموذج الملاحظات مع سياق مملوء مسبقًا
3. اسأل: "هل تريد تقديم تعليقات الآن؟ (نعم/لا/لاحقًا)"
4. إذا كانت الإجابة بنعم: اجمع التقييمات والتعليقات بشكل تفاعلي
5. حفظ في دليل ردود الفعل
6. تحليل الأنماط عبر جميع ردود الفعل
7. اقتراح قصص التحسين للقضايا

## استخدم متى

- **Story completion** - مباشرة بعد وضع علامة الانتهاء
- **Epic retrospective** - بعد الانتهاء الملحمي
- **Sprint end** - جمع تعلم الفريق
- **Process improvement** - مراجعة شهرية لأنماط ردود الفعل
- **Performance review** - تقييمات الوكلاء ربع السنوية
- **Team retrospectives** - مراسم العدو الرسمية

## المشغلات التلقائية

تظهر مطالبات التعليقات:
- عندما تتغير حالة القصة إلى "مكتملة"
- عندما تصل الملحمة إلى اكتمال 100%
- في تواريخ انتهاء السباق التي تم تكوينها
- يمكن طلبها يدويًا في أي وقت

## اندماج

تتدفق ردود الفعل إلى:
- **Retrospectives** - الرؤى المضمنة في سباقات الرجعية
- **Metrics** - Ratings tracked over time
- **Story creation** - إنشاء قصص التحسين تلقائيًا
- **ADRs** - توثيق الدروس كقرارات معمارية
- **Agent profiles** - قوائم تحديثات بيانات الأداء
