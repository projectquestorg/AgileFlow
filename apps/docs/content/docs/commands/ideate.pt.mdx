# /idear

Gere ideias de melhoria implantando vários especialistas de domínio em paralelo para analisar sua base de código de diferentes perspectivas (segurança, desempenho, qualidade de código, UX, testes, API/Arquitetura).

## Início rápido

```bash
/agileflow:ideate SCOPE=all DEPTH=quick OUTPUT=report
```

## Parâmetros

| Parâmetro | Obrigatório | Padrão | Descrição |
|-----------|----------|---------|------------|
| `SCOPE` | Não | `all` | Quais domínios analisar: `all`, `security`, `perf`, `code`, `ux` |
| `DEPTH` | Não | `quick` | Profundidade de análise: `quick` (3 ideias), `deep` (5 ideias) |
| `OUTPUT` | Não | `report` | Tipo de saída: `report` (apenas idealização), `stories` (criar histórias), `both` |

## Exemplos

### Analise todos os domínios - Escopo rápido

```bash
/agileflow:ideate SCOPE=all DEPTH=quick OUTPUT=report
```

Implanta 6 especialistas (segurança, desempenho, refatoração, interface do usuário, testes, API) para gerar 3 ideias de melhoria cada. Réresulta em relatório de ideação com ideias de confiança alta/média.

### Análise profunda de segurança

```bash
/agileflow:ideate SCOPE=security DEPTH=deep OUTPUT=stories
```

Foco na segurança com análise profunda (5 ideias por especialista). Cria histórias para itens de alta confiança imediatamente.

### Análise de desempenho com relatórios e histórias

__NOTRADUZIR_3__

Analisa desempenho com especialistas (desempenho, banco de dados, API), gera relatório e cria histórias de implementação.

## Mapeamento de escopo

| ESCOPO | Especialistas destacados | Melhor para |
|-------|------------------|----------|
| `all` | Segurança, Desempenho, Refatoração, UI, Testes, API | Análise abrangente (6 especialistas) |
| `security` | Segurança, API, Testes | Revisão de segurança (3 especialistas) |
| `perf` | Desempenho, banco de dados, API | Otimização de desempenho (3 especialistas) |
| `code` | Refatorar, testar, API | Qualidade do código (3 especialistas) |
| `ux` | UI, Acessibilidade, API | Experiência do usuário (3 especialistas) |

## Estrutura de saída

### Relatório de Ideação (`output.md`)

O comando gera __NOTRASLATE_31__ com:

1. **Resumo**: total de ideias, quebra de confiança
2. **Melhorias de alta confiança**: ideias acordadas por mais de 2 especialistas (priorize-as)
3. **Oportunidades de confiança média**: Especialista único com evidências (vale a pena explorar)
4. **Resumo por categoria**: tabela de contagens de ideias por domínio
5. **Próximas etapas recomendadas**: Orientação para implementação

### Pontuação de confiança

| Confiança | Critérios | Ação |
|------------|----------|--------|
| **ALTO** | Mais de 2 especialistas sugerem ideia semelhante | Incluir com destaque, recomendar ação imediata |
| **MÉDIO** | 1 especialista com evidências específicas (caminhos de arquivos, métricas) | Incluir como oportunidade |
| **BAIXO** | 1 perito, vago/sem provas | Excluir do relatório |

## Como funciona

1. **Analisar argumentos**: determina quais especialistas implantar e analisar a profundidade
2. **Implantar especialistas em paralelo**: lança todos os especialistas simultaneamente
3. **Cada especialista gera ideias**: 3 a 5 ideias de melhoria específicas e acionáveis
4. **Coletar resultados**: aguarda a conclusão de todos os especialistas
5. **Sintetizar**: agrupa ideias sobrepostas e avalia confiança
6. **Gerar saída**: cria relatório (e histórias opcionais)

## Melhores práticas

### Aproveite ao máximo /ideate

**Faça isto:**
- Use __NOTRASLATE_35__ trimestralmente para análise abrangente
- Use ESCOPO específico para áreas de melhoria focadas
- Use __NOTRASLATE_36__ ao investigar um domínio específico
- Revise primeiro as ideias de ALTA confiança
- Crie histórias para melhorias de ALTA confiança

**Não faça isso:**
- Ignore ideias de confiança MÉDIA com evidências específicas
- Tente implementar tudo de uma vez
- Use sem executar o git para obter o contexto do código mais recente
- Pule a seção “por que” ao decidir implementar

### Interpretando Resultados

**Ideias de alta confiança** (2+ especialistas):
- Forte sinal de oportunidade real de melhoria
- Recomendado para ação imediata
- Geralmente requerem 1-3 dias de esforço

**Ideias de confiança média** (especialista único):
- Valioso, mas menos certo
- Vale a pena discutir com a equipe
- Muitas vezes bons candidatos para o próximo sprint

**Ideias Vagas**:
- Excluído automaticamente do relatório
- Muito genérico, sem caminhos de arquivo ou métricas
- Se estiver interessado, faça primeiro uma pesquisa focada
