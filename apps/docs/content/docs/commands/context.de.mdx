# /Kontext

Erstellen, aktualisieren und verwalten Sie Ihre Projektkontextbeschreibung zur Verwendung in Web-KI-Tools wie ChatGPT, Perplexity, Gemini und Claude Web.

## Überblick

Der `/context` Mit der Befehlsfamilie können Sie den Projektkontext für webbasierte KI-Tools verwalten und exportieren. Dies ist nützlich, wenn Sie Folgendes benötigen:

- Generieren Sie eine umfassende Kontextbeschreibung zum Einfügen in Web-KI-Tools
- Exportieren Sie einen prägnanten Auszug zum schnellen Laden des Kontexts
- Halten Sie wichtige Entscheidungen und Entdeckungen in Ihrer Kontextdatei fest
- Halten Sie externe KI-Tools über Ihren Projektstatus auf dem Laufenden

## Unterbefehle

Der `/context` Der Befehl verfügt über drei Unterbefehle, von denen jeder einen bestimmten Zweck hat:

### /agileflow:context:full

**Generate or refresh the comprehensive project context brief.**

Erstellt oder aktualisiert `docs/context.md` mit verwalteten Abschnitten, die Folgendes enthalten:
- Was wir aufbauen und aktueller Schwerpunkt
- Funktionskarte und Zusammenfassung der Technik/Werkzeuge
- Wichtige Entscheidungen (ADRs) und Architektur-Snapshot
- Tests und CI-Status
- Aktuelle Fortschritte
- Risiken und nächste Schritte

Diese Datei kann in ChatGPT, Perplexity, Gemini oder Claude Web eingefügt werden, um externen KI-Tools den vollständigen Projektkontext bereitzustellen.

**Usage:**
```bash
/agileflow:context:full
```

**Behavior:**
- Liest vorhanden `docs/context.md` falls vorhanden
- Sammelt Daten aus status.json, ADRs, Epics, Stories und CHANGELOG
- Aktualisiert nur verwaltete Abschnitte (zwischen `<!-- MANAGED_SECTION -->` Markierungen)
- Behält alle vom Benutzer geschriebenen Inhalte außerhalb der verwalteten Abschnitte bei
- Zeigt Unterschiede vor dem Anwenden von Änderungen an
- Erfordert eine Bestätigung vor dem Schreiben

---

### /agileflow:context:export

**Export a concise context excerpt for pasting into web AI tools.**

Liest `docs/context.md` und gibt einen komprimierten Auszug (≤300 Zeilen) aus, der zum Kopieren und Einfügen in ChatGPT, Perplexity, Gemini oder Claude Web bereit ist.

Das ist ein **read-only command** - Es werden keine Dateien geschrieben.

**Usage:**
```bash
/agileflow:context:export
```

**Behavior:**
- Liest `docs/context.md`
- Extrahiert wichtige Abschnitte (Übersicht, aktueller Fokus, Tech-Stack, Entscheidungen, Feature-Map, nächste Schritte)
- Gibt einen prägnanten Auszug in einem Codeblock aus
- Enthält Anweisungen zum Einfügen in Web-KI
- Erfordert keine Bestätigung

---

### /agileflow:context:note

**Append a timestamped note to the context file.**

Fügen Sie schnell eine Notiz hinzu `docs/context.md` im Abschnitt „Notizen“. Nützlich zum Aufzeichnen von Entscheidungen, Beobachtungen oder Erinnerungen, die im Projektkontext bestehen bleiben sollen.

**Usage:**
```bash
/agileflow:context:note NOTE="Your note here"
```

**Arguments:**

| Argument | Required | Description |
|----------|----------|-------------|
| NOTE | Yes | 1-5 line note text to append |

**Behavior:**
- Liest oder erstellt `docs/context.md`
- Erstellt einen Abschnitt „Notizen“, falls dieser nicht vorhanden ist
- Fügt eine Notiz mit ISO-Zeitstempel hinzu (z. B. `2024-12-31T10:30:00Z`)
- Zeigt Unterschiede vor dem Anwenden von Änderungen an
- Erfordert eine Bestätigung vor dem Schreiben

## Beispiele

### Vollständigen Kontext generieren

```bash
/agileflow:context:full
```

Erstellt oder aktualisiert `docs/context.md` mit ausführlichen Projektinformationen:

```markdown

**Last Updated**: 2025-12-24
**Project**: AgileFlow - AI-powered project management

## What We're Building

AgileFlow is a comprehensive project management system powered by AI agents. It provides automated task management, documentation synchronization, and intelligent context preservation.

## Current Focus

- Implementing context compacting for long conversations
- Improving research workflow integration
- Expanding agent capabilities

## Feature Map

- **Core**: Project setup, status tracking, agent coordination
- **Documentation**: Automatic sync, README generation, ADR management
- **Agents**: 83 specialized agents for different tasks
- **Research**: Structured research notes and integration

## Tech Stack

- **Language**: TypeScript
- **Framework**: Next.js 16 (App Router)
- **UI**: React with Tailwind CSS
- **Package Manager**: npm with legacy-peer-deps
- **Deployment**: Vercel

## Key Decisions

- ADR-0001: Use TypeScript for type safety
- ADR-0002: Use React component library
- ADR-0003: Next.js for web framework

## Architecture Snapshot

- Monorepo structure with packages/ and apps/
- CLI package with 98 commands
- Website and documentation sites
- Agent-based command coordination

- E2E tests: Playwright

- Pre-commit hooks for validation

- Completed context compacting feature
- Improved documentation sync workflow

## Risks

- Token limits with large projects
- Documentation drift during rapid development

## Next Steps

- Implement automated test coverage reporting
- Add real-time collaboration features
- Expand agent capabilities for code generation
```

## Ausgabemodi

### /agileflow:context:full (Kontext erstellen/aktualisieren)

**Input**: Sammelt Informationen von:
- `status.json` - Aktueller Arbeits- und Story-Status
- `bus/log.jsonl` - Aktuelle Fortschrittsmeldungen
- `docs/03-decisions/` - Architekturentscheidungen (ADRs)
- `docs/04-architecture/` - Dokumentation des Systemdesigns
- `docs/05-epics/` - Feature-Roadmap und Epics
- `CHANGELOG.md` - Veröffentlichungsverlauf
- `package.json` - Projektmetadaten

**Output**: Aktualisierungen `docs/context.md` mit verwalteten Abschnitten unter Beibehaltung benutzerdefinierter Inhalte

**Key Behavior**:
- Diff-first: Zeigt Änderungen vor dem Schreiben an
- Verwaltete Abschnitte: Aktualisiert nur Inhalte zwischen `<!-- MANAGED_SECTION -->` Markierungen
- Behält Benutzerinhalte bei: Ändert niemals Inhalte außerhalb verwalteter Abschnitte
- Erfordert Bestätigung: Der Benutzer muss Änderungen genehmigen

---

### /agileflow:context:export (In Web AI kopieren)

**Input**: Liest vorhanden `docs/context.md`

**Output**: Prägnanter Auszug (≤300 Zeilen), bereit zum Einfügen in ChatGPT, Perplexity, Gemini oder Claude Web

**Use Case**: Laden Sie den Projektkontext in das Web-KI-Tool, um die Arbeit in einem neuen Gespräch fortzusetzen

**Key Behavior**:
- Schreibgeschützt: Es werden keine Dateien geändert
- In sich geschlossen: Enthält Anweisungen zum Einfügen
- Formatbereit: Bereits formatiert für einfaches Kopieren und Einfügen

---

### /agileflow:context:note (Zeitgestempelte Notiz anhängen)

**Input**: NOTE-Argument mit 1-5-zeiliger Notiz

**Output**: Wird mit ISO-Zeitstempel an den Abschnitt „Notizen“ angehängt

**Use Case**: Schnelle Erfassung wichtiger Ereignisse, Entscheidungen oder Entdeckungen

**Key Behavior**:
- Diff-first: Zeigt Änderungen vor dem Schreiben an
- Erstellt einen Abschnitt, falls er fehlt: Erstellt bei Bedarf automatisch den Abschnitt „Notizen“.
- Chronologisch: Die Notizen werden in der Reihenfolge angezeigt, wobei die neuesten unten stehen
- Mit Zeitstempel: Jede Notiz enthält einen ISO 8601-Zeitstempel

## Best Practices

### Regelmäßige Kontextaktualisierungen

Halten Sie den Kontext für eine bessere KI-Unterstützung aktuell:

1. **After major milestones**: Laufen `/agileflow:context:full` um abgeschlossene Arbeiten zu erfassen
2. **Before web AI sessions**: Laufen `/agileflow:context:export` und in ChatGPT einfügen
3. **During development**: Verwenden `/agileflow:context:note` Entscheidungen und Entdeckungen aufzuzeichnen

```bash
# Recommended workflow
/agileflow:context:full        # Update full context
/agileflow:context:export      # Get concise version
# Paste into web AI for context-aware help
```

---

### Erfassen Sie wichtige Entdeckungen

Technische Erkenntnisse und Entscheidungen aufzeichnen:

```bash
/agileflow:context:note NOTE="TypeScript 5.4 improves type inference - consider upgrading"
/agileflow:context:note NOTE="Stripe API changed pagination in v2.0 - update integration"
/agileflow:context:note NOTE="Team consensus: use Postgres over MongoDB for this project"
```

---

### Verwendung in Web-KI-Gesprächen

Maximieren Sie die Effektivität von Web-KI-Tools durch die Bereitstellung von Kontext:

1. Prägnanten Kontext exportieren: `/agileflow:context:export`
2. Kopieren Sie die Ausgabe
3. Als erste Nachricht in ChatGPT, Perplexity oder Gemini einfügen
4. Stellen Sie Folgefragen mit vollständigem Projektkontext

Dadurch wird sichergestellt, dass die Web-KI Ihre Codebasis, Architektur und Ihren aktuellen Fokus versteht.

---

### Verwalten Sie die Größe der Kontextdatei

Verwalten Sie bei großen Projekten die Dateigröße von context.md:

- Verwenden `/agileflow:context:export` um die aktuelle Kontextgröße zu überprüfen
- Archivieren Sie alte Notizen, wenn sie mehr als 100 Einträge umfassen
- Erstellen Sie separate Forschungsdateien für tiefe Einblicke
- Verwenden `/agileflow:research:ask` für komplexe Forschungsthemen

---

## Diff-First-Workflow

Alle Kontextbefehle, die Dateien ändern, folgen einem konsistenten Arbeitsablauf:

1. **Show Changes**: Der Befehl zeigt vorgeschlagene Änderungen als Diff an
2. **Confirm**: Sie überprüfen Änderungen und genehmigen oder lehnen sie ab
3. **Apply**: Änderungen werden erst nach Ihrer Bestätigung geschrieben

Dadurch wird sichergestellt, dass Sie keine wichtigen Informationen verlieren und immer wissen, was geändert wird.

## Notizen

- Kontextdateien werden in gespeichert `docs/context.md` als Teil Ihrer lokalen Projektdokumentation
- Verwaltete Abschnitte verwenden HTML-Kommentare als Markierungen und sollten nicht manuell bearbeitet werden
- Der Exportbefehl berücksichtigt die 300-Zeilen-Grenze, um in die Kontextfenster des Web-KI-Tools zu passen
- Notizen werden im ISO 8601-Format mit einem Zeitstempel versehen, um Konsistenz und einfache Sortierung zu gewährleisten
