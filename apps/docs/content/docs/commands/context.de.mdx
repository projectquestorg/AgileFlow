# /context

Erstellen, aktualisieren und verwalten Sie Ihre Projektkontextbeschreibung zur Verwendung in Web-KI-Tools wie ChatGPT, Perplexity, Gemini und Claude Web.

## Übersicht

Die Befehlsfamilie `/context` hilft Ihnen, den Projektkontext für webbasierte KI-Tools zu verwalten und zu exportieren. Dies ist nützlich, wenn Sie Folgendes benötigen:

- Generieren Sie eine umfassende Kontextbeschreibung zum Einfügen in Web-KI-Tools
- Exportieren Sie einen prägnanten Auszug zum schnellen Laden des Kontexts
- Notieren Sie wichtige Entscheidungen und Entdeckungen in Ihrer Kontextdatei
- Halten Sie externe KI-Tools über Ihren Projektstatus auf dem Laufenden

## Unterbefehle

Der Befehl `/context` verfügt über drei Unterbefehle mit jeweils einem bestimmten Zweck:

### /agileflow:context:full

**Generieren oder aktualisieren Sie die umfassende Projektkontextbeschreibung.**

Erstellt oder aktualisiert `docs/context.md` mit verwalteten Abschnitten, die Folgendes enthalten:
- Was wir aufbauen und aktueller Schwerpunkt
- Funktionskarte und Zusammenfassung der Technik/Werkzeuge
- Schlüsselentscheidungen (ADRs) und Architektur-Snapshot
- Tests und CI-Status
- Aktuelle Fortschritte
- Risiken und Next Schritte

Diese Datei kann in ChatGPT, Perplexity, Gemini oder Claude Web eingefügt werden, um externen KI-Tools den vollständigen Projektkontext bereitzustellen.

**Verwendung:**
```bash
/agileflow:context:full
```

**Verhalten:**
– Liest vorhandenes `docs/context.md`, falls vorhanden
- Sammelt Daten aus status.json, ADRs, Epics, Stories und CHANGELOG
– Aktualisiert nur verwaltete Abschnitte (zwischen `<!-- MANAGED_SECTION -->` Markierungen)
- Behält alle vom Benutzer geschriebenen Inhalte außerhalb der verwalteten Abschnitte bei
- Zeigt Unterschiede an, bevor Änderungen übernommen werden
- Erfordert eine Bestätigung vor dem Schreiben

---

### /agileflow:context:export

**Exportieren Sie einen prägnanten Kontextauszug zum Einfügen in Web-KI-Tools.**

Liest `docs/context.md` und gibt einen komprimierten Auszug (≤300 Zeilen) aus, der zum Kopieren und Einfügen in ChatGPT, Perplexity, Gemini oder Claude Web bereit ist.

Dies ist ein **schreibgeschützter Befehl** – es werden keine Dateien geschrieben.

**Verwendung:**
```bash
/agileflow:context:export
```

**Verhalten:**
- Liest `docs/context.md`
- Extrahiert wichtige Abschnitte (Übersicht, aktueller Fokus, Tech-Stack, Entscheidungen, Feature-Map, nächste Schritte)
- Ausgänge konzse Auszug in einem Codeblock
- Enthält Anweisungen zum Einfügen in die Web-KI
- Erfordert keine Bestätigung

---

### /agileflow:context:note

**Fügen Sie der Kontextdatei eine zeitgestempelte Notiz hinzu.**

Fügen Sie im Abschnitt „Notizen“ schnell eine Notiz zu `docs/context.md` hinzu. Nützlich zum Aufzeichnen von Entscheidungen, Beobachtungen oder Erinnerungen, die im Projektkontext bestehen bleiben sollen.

**Verwendung:**
```bash
/agileflow:context:note NOTE="Your note here"
```

**Argumente:**

| Argument | Erforderlich | Beschreibung |
|----------|----------|-------------|
| HINWEIS | Ja | 1-5-zeiliger Notiztext zum Anhängen |

**Verhalten:**
- Liest oder erstellt `docs/context.md`
- Erstellt einen Abschnitt „Notizen“, falls dieser nicht vorhanden ist
– Fügt eine Notiz mit ISO-Zeitstempel hinzu (z. B. `2024-12-31T10:30:00Z`)
- Zeigt Unterschiede an, bevor Änderungen übernommen werden
- Erfordert eine Bestätigung vor dem Schreiben

## Ausgabemodi

### /agileflow:context:full (Kontext erstellen/aktualisieren)

**Eingabe**: Sammelt Informationen von:
- `status.json` – Aktueller Arbeits- und Story-Status
– `bus/log.jsonl` – Aktuelle Fortschrittsmeldungen
- `docs/03-decisions/` - Architekturentscheidungen (ADRs)
- `docs/04-architecture/` – Systemdesigndokumentation
– `docs/05-epics/` – Feature-Roadmap und Epics
– `CHANGELOG.md` – Veröffentlichungsverlauf
- `package.json` – Projektmetadaten

**Ausgabe**: Aktualisiert `docs/context.md` mit verwalteten Abschnitten, während presBereitstellung benutzerdefinierter Inhalte

**Schlüsselverhalten**:
- Diff-first: Zeigt Änderungen vor dem Schreiben an
- Verwaltete Abschnitte: Aktualisiert nur Inhalte zwischen `<!-- MANAGED_SECTION -->`-Markierungen
- Behält Benutzerinhalte bei: Ändert niemals Inhalte außerhalb verwalteter Abschnitte
- Erfordert Bestätigung: Der Benutzer muss Änderungen genehmigen

---

### /agileflow:context:export (In Web AI kopieren)

**Eingabe**: Liest vorhandene `docs/context.md`

**Ausgabe**: Prägnanter Auszug (≤300 Zeilen), bereit zum Einfügen in ChatGPT, Perplexity, Gemini oder Claude Web

**Anwendungsfall**: Laden Sie den Projektkontext in das Web-KI-Tool, um die Arbeit in einem neuen Gespräch fortzusetzen

**Schlüsselverhalten**:
- Schreibgeschützt: Es werden keine Dateien geändert
- In sich geschlossen: Enthält Anweisungen zum Einfügen
- Formatbereit: Bereits formatiert für einfaches Kopieren und Einfügen

---

### /agileflow:context:note (Zeitgestempelte Notiz anhängen)

**Eingabe**: NOTE-Argument mit 1-5-zeiliger Notiz

**Ausgabe**: Wird mit ISO-Zeitstempel an den Abschnitt „Notizen“ angehängt

**Anwendungsfall**: Schnelle Erfassung wichtiger Ereignisse, Entscheidungen oder Entdeckungen

**Schlüsselverhalten**:
- Diff-first: Zeigt Änderungen vor dem Schreiben an
- Erstellt einen Abschnitt, falls er fehlt: Erstellt bei Bedarf automatisch den Abschnitt „Notizen“.
- Chronologisch: Notizen werden in der Reihenfolge angezeigt, wobei die neuesten unten stehen
- Zeitstempel: Jede Notiz enthält einen ISO 8601-Zeitstempel

## Best Practices

### Regelmäßige Kontextaktualisierungen

Halten Sie den Kontext für eine bessere KI-Unterstützung aktuell:

1. **Nach wichtigen Meilensteinen**: Führen Sie `/agileflow:context:full` aus, um abgeschlossene Arbeiten zu erfassen
2. **Vor Web-KI-Sitzungen**: Führen Sie `/agileflow:context:export` aus und fügen Sie es in ChatGPT ein
3. **Während der Entwicklung**: Verwenden Sie `/agileflow:context:note`, um Entscheidungen und Entdeckungen aufzuzeichnen

```bash
# Recommended workflow
/agileflow:context:full        # Update full context
/agileflow:context:export      # Get concise version
# Paste into web AI for context-aware help
```

---

### Erfassen Sie wichtige Entdeckungen

Technische Erkenntnisse und Entscheidungen aufzeichnen:

```bash
/agileflow:context:note NOTE="TypeScript 5.4 improves type inference - consider upgrading"
/agileflow:context:note NOTE="Stripe API changed pagination in v2.0 - update integration"
/agileflow:context:note NOTE="Team consensus: use Postgres over MongoDB for this project"
```

---

### Verwendung in Web-KI-Gesprächen

Maximieren Sie die Effektivität von Web-KI-Tools durch die Bereitstellung von Kontext:

1. Prägnanten Kontext exportieren: `/agileflow:context:export`
2. Kopieren Sie die Ausgabe
3. Fügen Sie sie als erste Nachricht in ChatGPT, Perplexity oder Gemini ein
4. Stellen Sie Folgefragen mit vollständigem Projektkontext

Dies gewährleistetDie Web-KI versteht Ihre Codebasis, Architektur und Ihren aktuellen Fokus.

---

### Größe der Kontextdatei verwalten

Verwalten Sie bei großen Projekten die Dateigröße von context.md:

– Verwenden Sie `/agileflow:context:export`, um die aktuelle Kontextgröße zu überprüfen
- Archivieren Sie alte Notizen, wenn sie mehr als 100 Einträge umfassen
- Erstellen Sie separate Forschungsdateien für tiefe Einblicke
- Verwenden Sie `/agileflow:research:ask` für komplexe Forschungsthemen

---

## Diff-First-Workflow

Alle Kontextbefehle, die Dateien ändern, folgen einem konsistenten Arbeitsablauf:

1. **Änderungen anzeigen**: Der Befehl zeigt vorgeschlagene Änderungen als Diff an
2. **Bestätigen**: Sie überprüfen Änderungen und genehmigen oder lehnen sie ab
3. **Anwenden**: Änderungen werden erst nach Ihrer Bestätigung vorgenommen

Dadurch wird sichergestellt, dass Sie keine wichtigen Informationen verlieren und immer wissen, was geändert wird.

## Notizen

- Kontextdateien werden in `docs/context.md` als Teil Ihrer lokalen Projektdokumentation gespeichert
- Verwaltete Abschnitte verwenden HTML-Kommentare als Markierungen und sollten nicht manuell bearbeitet werden
– Der Exportbefehl berücksichtigt die 300-Zeilen-Grenze, um in die Kontextfenster des Web-KI-Tools zu passen
- Notizen werden im ISO 8601-Format mit einem Zeitstempel versehen, um Konsistenz und einfache Sortierung zu gewährleisten
