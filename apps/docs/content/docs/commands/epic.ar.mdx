# / ملحمة

أنشئ ملحمة جديدة تحتوي على قصص أطفال اختيارية لتنظيم قصص المستخدمين ذات الصلة حول ميزة أو هدف أكبر.

## بداية سريعة

__لا تترجم_0__

## المعلمات

| المعلمة | مطلوب | الافتراضي | الوصف |
|-----------|----------|---------|-------------|
| __غير مترجم_3__ | نعم | - | المعرف الملحمي (على سبيل المثال، EP-0001) |
| __غير مترجم_4__ | نعم | - | عنوان ملحمي يصف الميزة أو الهدف |
| __غير مترجم_5__ | نعم | - | اسم المالك أو معرف الوكيل (على سبيل المثال، AG-UI، AG-API) |
| __غير مترجم_6__ | نعم | - | النتيجة أو المقياس الذي يحدد نجاح هذه الملحمة |
| __غير مترجم_7__ | لا | - | قائمة مفصولة بفواصل للقصص الأولية (التنسيق: `US-ID\|title\|owner`) |

## أمثلة

### الخلق الملحمي الأساسي

__لا تترجم_1__

ينشئ ملفًا ملحميًا جديدًا في `docs/05-epics/EP-0010.md` بالتفاصيل المقدمة ويسجله في `docs/09-agents/status.json`.

### ملحمة مع قصص أولية

__لا تترجم_2__

ينشئ الملحمة وينشئ قصص الأطفال الأولية تلقائيًا ضمن `docs/06-stories/EP-0011/`. كل قتم إنشاء توري باستخدام:
- الحالة: `ready` (جاهز للتطوير)
- التقدير: `0.5d` (افتراضي، يمكن تعديله)
- التبعيات: فارغة (يتم تحديثها يدويًا حسب الحاجة)

## الإخراج

يقوم الأمر بإنشاء:

1. **ملف ملحمي** (`docs/05-epics/EP-XXXX.md`)
   - البيانات الوصفية الملحمية (العنوان، المالك، الحالة، الهدف)
   - الوضع الحالي والمقاييس
   - قائمة قصص الأطفال

2. **ملفات القصة** (ملف واحد لكل قصة، `docs/06-stories/EP-XXXX/US-XXXX-<slug>.md`)
   - قالب القصة مع العناصر النائبة لمعايير القبول
   - مرتبطة بملحمة الوالدين
   - جاهز للتعريف التفصيلي

3. **تتبع الحالة** (`docs/09-agents/status.json`)
   - دخول ملحمي مع المالك والوضع الحالي
   - إدخالات القصة مرتبطة بالملحمة

4. **اتصال الوكيل** (`docs/09-agents/bus/log.jsonl`)
   - رسائل مهمة تظهر الملحمة التي تم إنشاؤها والقصص المخصصة

## سير العمل

1. **خطِّط لملحمتك** - حدد هدف الميزة ومقاييس النجاح
2. **إنشاء الملحمة** - قم بتشغيل أمر `/epic` بالتفاصيل الأساسية
3. **أضف قصصًا** - قم بإنشائها الآن باستخدام معلمة STORIESأو استخدم `/story` للإضافة بشكل فردي لاحقًا
4. **تطوير القصص** - استخدم `/story-validate` للتأكد من أن كل قصة جاهزة قبل التطوير
5. **خطة السباق السريع** - استخدم `/sprint` لتحديد القصص للسباق القادم
6. **تتبع التقدم** - استخدم `/board` لمراقبة اكتمال الملحمة

## التكامل مع الأوامر الأخرى

- **بعد إنشاء الملحمة**: استخدم `/story` لإضافة المزيد من القصص
- **قبل التخطيط للسباق السريع**: استخدم `/sprint` لتحديد قصص من هذه الملحمة
- **لتتبع التقدم**: استخدم `/board` أو `/metrics` لرؤية الإنجاز المذهل
- **بالنسبة للتبعيات**: استخدم `/deps` لتصور تبعيات القصة داخل الملحمة

## أفضل الممارسات

- **حافظ على تركيز الملاحم** - ميزة واحدة أو منطقة مميزة لكل ملحمة
- **اجعل الأهداف قابلة للقياس** - استخدم نتائج أو مقاييس محددة (وليس "تحسين تجربة المستخدم")
- **أضف قصصًا تدريجيًا** - ابدأ بالقصص الأساسية، وأضف القصص ذات الصلة بمرور الوقت
- **تعيين المالكين المناسبين** - قم بمطابقة الملحمة الخاصة بكإيه مع وكيل التطوير الأساسي
- **سياق المستند** - أضف سياقًا تقنيًا إلى الملف الملحمي بعد إنشائه

## الأوامر ذات الصلة

- [`/story`](/commands/story) - أنشئ قصصًا فردية داخل ملحمة
- [`/sprint`](/commands/sprint) - خطط لسباقات السرعة مع قصص ملحمية محددة
- [`/deps`](/commands/deps) - تصور تبعيات القصة داخل الملحمة
- [`/board`](/commands/board) - تتبع التقدم الملحوظ بصريًا
- [`/metrics`](/commands/metrics) - عرض مقاييس الإكمال والسرعة الرائعة
