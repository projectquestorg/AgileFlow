# /الجلسة:جديدة

قم بإنشاء جلسة Claude Code متوازية جديدة باستخدام الإعداد التلقائي لشجرة عمل git. يتيح لك ذلك العمل على مهام متعددة في وقت واحد دون تعارضات.

## بداية سريعة

__لا تترجم_0__

## كيف يعمل

عند تشغيل هذا الأمر، ستظهر لك الخيارات:

1. **إنشاء جلسة N تلقائيًا** - إنشاء جلسة مرقمة تلقائيًا (على سبيل المثال، الجلسة 2، الجلسة 3)
2. **قم بتسمية هذه الجلسة** - أعط لقبًا مثل "auth" أو "refactor"
3. **استخدم الفرع الموجود** - أرفقه بفرع لديك بالفعل

الأمر إذن:
1. إنشاء شجرة عمل git جديدة في دليل موازي
2. يسجل الجلسة في سجل الجلسة
3. يظهر لك الأمر `cd` للتبديل إلى مساحة العمل الجديدة

## لماذا نستخدم الجلسات الموازية؟

### المشكلة

يؤدي تشغيل مثيلات Claude Code المتعددة في نفس الدليل إلى ما يلي:
- تعارضات قفل الملفات (خاصة على نظام التشغيل macOS)
- خطر التعديلات المتعارضة على نفس الملفات
- مشكلات دليل `~/.claude/` المشتركة
- لا توجد رؤية لما تعمل عليه الجلسات الأخرى

### الحل

توفر أشجار عمل Git ما يلي:
- **أدلة معزولة** - لكل جلسة مجلد خاص بها
- **Shared .git** - فعال على القرص، ولا يوجد نسخ مكرر
- **فروع مستقلة** - لا يوجد تعارضات دمج أثناء العمل
- **الفصل النظيف** - لا يمكن للجلسات أن تدوس على بعضها البعض

## أفضل الممارسات

- **مهمة واحدة لكل جلسة** - احتفظ بالملفاتركزت الجلسات
- **استخدم الألقاب للعمل طويل الأمد** - أسهل في التذكر من الأرقام
- **إنهاء الجلسات عند الانتهاء** - استخدم `/session:end` للتنظيف
- **التحقق من الحالة قبل البدء** - استخدم `/session:status` لمعرفة ما يتم تشغيله
