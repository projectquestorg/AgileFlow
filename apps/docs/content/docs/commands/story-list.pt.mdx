# /história:lista

Exiba todas as histórias de usuários com opções de filtragem e recursos de ação rápida.

## Início rápido

```bash
/agileflow:story:list
```

## Parâmetros

| Parameter | Required | Default | Description |
|-----------|----------|---------|-------------|
| `EPIC` | No | - | Filter by epic (e.g., EP-0001) |
| `STATUS` | No | - | Filter by status: `ready`, `in_progress`, `blocked`, `done` |
| `OWNER` | No | - | Filter by owner (e.g., AG-UI) |

## Exemplos

### Listar todas as histórias

```bash
/agileflow:story:list
```

Mostra todas as histórias organizadas por épico com seu status.

### Listar histórias para épicos específicos

```bash
/agileflow:story:list EPIC=EP-0001
```

Mostra apenas histórias no épico de autenticação.

### Listar histórias prontas

```bash
/agileflow:story:list STATUS=ready
```

Mostra todas as histórias prontas para começar a trabalhar.

### Listar histórias em andamento

```bash
/agileflow:story:list STATUS=in_progress
```

Mostra os itens de trabalho atuais.

### Listar histórias por proprietário

```bash
/agileflow:story:list OWNER=AG-UI
```

Mostra todas as histórias atribuídas ao agente de UI.

### Filtros Combinados

```bash
/agileflow:story:list EPIC=EP-0001 STATUS=ready
```

Mostra histórias prontas apenas no épico de autenticação.

```bash
/agileflow:story:list STATUS=blocked OWNER=AG-API
```

Mostra histórias de API que estão bloqueadas no momento.

## Formato de saída

As histórias são organizadas por épico com colunas de status:

```markdown
## Stories

### EP-0001: Authentication System
| Story | Title | Status | Owner | Estimate |
|-------|-------|--------|-------|----------|
| US-0001 | Login form | done | AG-UI | 2h |
| US-0002 | Password reset | in_progress | AG-API | 3h |
| US-0003 | Session management | ready | AG-API | 4h |

### EP-0002: User Dashboard
| Story | Title | Status | Owner | Estimate |
|-------|-------|--------|-------|----------|
| US-0004 | Dashboard layout | ready | AG-UI | 2h |
| US-0005 | Activity feed | ready | AG-UI | 3h |

---
**Summary**: 5 stories (1 done, 1 in_progress, 3 ready)
```

## Definições de status

| Status | Meaning | Action |
|--------|---------|--------|
| **ready** | Ready to start, no blockers | Start work / view details |
| **in_progress** | Currently being worked on | Continue work / mark blocked if needed |
| **blocked** | Waiting for dependency or clarification | Resolve blocker / unblock when ready |
| **done** | Completed and tested | Move to next story |

## Ações rápidas

Depois de exibir a lista, você pode:

### Ver detalhes da história

Selecione uma história para ver:
- Critérios de aceitação completos
- Status do teste
- Dependências
- Bloqueadores (se houver)
- Notas de progresso

```bash
/agileflow:story:view STORY=US-0003
```

### Comece a trabalhar em uma história

Escolha uma história pronta e marque-a como in_progress:

```bash
/agileflow:status US-0003 STATUS=in_progress
```

### Criar nova história

Adicione uma história a um épico:

```bash
/agileflow:story EPIC=EP-0001 ...
```

## Integração de fluxo de trabalho

### Levantamento Diário

```bash
/agileflow:story:list STATUS=in_progress
# See what's being worked on
/agileflow:story:list STATUS=blocked
# See what's blocked
```

### Planejamento de Sprints

```bash
/agileflow:story:list STATUS=ready
# See available work
/agileflow:epic:list
# See epic status
# Select stories for sprint
```

### Desbloqueando fluxo de trabalho

```bash
/agileflow:story:list STATUS=blocked
# See all blocked stories
# Resolve dependencies
/agileflow:story:view STORY=<blocked>
# See blocker details
/agileflow:status <story> STATUS=ready
# When unblocked, mark ready again
```

### Acompanhamento de conclusão

```bash
/agileflow:story:list STATUS=done
# See what was completed
/agileflow:story:list STATUS=in_progress
# See what's being worked on
/agileflow:story:list STATUS=ready
# See what's queued up
```

## Estratégias de filtragem

### Por épico
```bash
/agileflow:story:list EPIC=EP-0001
# Focus on one feature
```

### Por status
```bash
/agileflow:story:list STATUS=ready
# Find available work
```

```bash
/agileflow:story:list STATUS=blocked
# Find blockers to resolve
```

### Por proprietário
```bash
/agileflow:story:list OWNER=AG-API
# See what an agent is working on
```

### Por status + épico
```bash
/agileflow:story:list EPIC=EP-0001 STATUS=ready
# Ready stories in specific epic
```

### Por Status + Proprietário
```bash
/agileflow:story:list STATUS=in_progress OWNER=AG-UI
# Current work for specific agent
```

## Interpretando Resultados

### Distribuição saudável de histórias

```
5 stories (1 done, 2 in_progress, 2 ready)
```

- ✅ As pessoas têm trabalho a fazer
- ✅ O trabalho está fluindo
- ✅ O backlog está pronto

### Problemas potenciais

**Too many in_progress (>3)**
```
5 stories (0 done, 4 in_progress, 1 ready)
```
- Considere reduzir o WIP (trabalho em andamento)
- Concentre-se em completar histórias

**No ready stories**
```
5 stories (2 done, 2 in_progress, 0 ready)
```
- Precisa planejar mais histórias
- Criar `/agileflow:story` para os próximos itens de trabalho

**Lots of blocked**
```
7 stories (1 done, 1 in_progress, 3 blocked, 2 ready)
```
- Bloqueadores de endereços com urgência
- Pode ser necessário desbloquear dependências
