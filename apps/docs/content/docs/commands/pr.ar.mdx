# /العلاقات العامة

أنشئ أوصاف طلب سحب كاملة وجاهزة لللصق مباشرةً من قصص المستخدم وأدلة الاختبار. يضمن أمر PR أن طلبات السحب الخاصة بك موثقة جيدًا، وارتباطها بالعمل ذي الصلة، وتتضمن جميع التفاصيل الضرورية للمراجعين.

## بداية سريعة

__لا تترجم_0__

الأمر سوف:
1. قم بتحميل القصة من `docs/06-stories/US-0042.md`
2. استخراج معايير القبول والتبعيات والسياق
3. بناء وصف كامل للعلاقات العامة مع جميع الأقسام
4. علامة تجارية جاهزة للصق الإخراج لـ GitHub/GitLab
5. اقترح رسالة التزام تقليدية لدمج الاسكواش

## الميزات الرئيسية

### سياق القصة التلقائي

يتضمن وصف العلاقات العامة تلقائيًا ما يلي:

- **مرجع ملحمي** - ما هي الميزة التي ينتمي إليها هذا؟
- **ملخص القصة** - ما تم تنفيذه
- **التبعيات** - قصص أخرى يعتمد عليها هذا
- **معايير القبول** - كل AC مع حالات مربع الاختيار
- **أدلة الاختبار** - نتائج الاختبار والتحقق
- **تقييم المخاطر** - التأثيرات المحتملة وخطة التراجع
- **أصحاب الكود** - From CODEOWNERS file

### Checklist Management

Acceptance criteria become a PR checklist:

```markdown
## Checklist

- [x] AC1: User can login with email and password
- [x] AC2: Login errors show validation feedback
- [ ] AC3: 2FA support (scope creep - skipped)
- [x] AC4: Session persists across page reloads
```

### Test Evidence Integration

Include test results and verification directly:

```markdown
## Test Evidence

### Unit Tests
- ✅ 42 tests passing, 0 failing
- Coverage: 87% (auth module)

### Integration Tests
- ✅ Login flow end-to-end verified
- ✅ Session persistence confirmed
- ✅ Tested with Chrome, Firefox, Safari

### Manual Testing
- ✅ Tested on desktop and mobile
- ✅ Accessibility check passed (WCAG AA)
```

### Conventional Commits

Suggests proper commit messages for squash merges:

```text
feat(auth): implement user login with JWT tokens

Implements US-0042: User Login API
- Email/password authentication
- JWT token generation and validation
- Session persistence across reloads
- Comprehensive test coverage

Closes #42
```

## Usage

### Basic PR description

```bash
/agileflow:pr STORY=US-0042 TITLE="Add user login API"
```

Generates PR with:
- Story summary from US-0042.md
- All acceptance criteria
- Linked issues (story + dependencies)
- Risk assessment
- Code owners

### PR with test evidence

```bash
/agileflow:pr STORY=US-0042 TITLE="Add user login" TEST_EVIDENCE="All tests passing (42/42), E2E verified"
```

Adds test results section to PR.

### PR with acceptance criteria status

```bash
/agileflow:pr STORY=US-0042 TITLE="Login API" AC_CHECKED="✓ Email auth, ✓ JWT generation, ✗ 2FA"
```

Shows which AC were completed vs skipped.

### PR with additional notes

```bash
/agileflow:pr STORY=US-0042 TITLE="Login API" NOTES="Coordinated with frontend team on API contract"
```

Adds custom notes to the description.

## Parameters

| Parameter | Required | Default | Description |
|-----------|----------|---------|-------------|
| `STORY` | Yes | - | Story ID (e.g., US-0042) |
| `TITLE` | Yes | - | Short PR title (1-5 كلمات) |
| __غير مترجم_30__ | لا | "" | نتائج الاختبار، التغطية، ملاحظات الاختبار اليدوي |
| __غير مترجم_31__ | لا | - | حالة معايير القبول (✓/✗ لكل منها) |
| __غير مترجم_32__ | لا | "" | سياق إضافي أو ملاحظات تنسيقية |

## أقسام القالب

يتضمن كل وصف للعلاقات العامة ما يلي:

1. **ملخص** - ما تم بناؤه
2. ** المشكلات المرتبطة ** - القصة + التبعيات
3. **قائمة المراجعة** - حالة معايير القبول
4. **أدلة الاختبار** - كيف يتم ذلككما تم التحقق منها
5. **لقطات الشاشة/صور GIF** - دليل مرئي (إن أمكن)
6. **تقييم المخاطر** - التأثيرات المحتملة + خطة التراجع
7. **مالكو الكود** - من يجب عليه المراجعة
