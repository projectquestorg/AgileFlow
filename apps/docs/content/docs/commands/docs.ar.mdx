# / مستندات-مزامنة

قم بمزامنة وثائقك مع تغييرات قاعدة التعليمات البرمجية عن طريق اكتشاف التعليمات البرمجية الجديدة والتأكد من وجود الوثائق المقابلة.

## بداية سريعة

__لا تترجم_0__

للمقارنة مع فرع معين:

__لا تترجم_1__

## أمثلة

### تحليل الفرع الحالي

__لا تترجم_2__

يقوم بتحليل فرعك الحالي مقابل `main` والتقارير:
- نقاط نهاية API جديدة بدون وثائق
- مكونات واجهة مستخدم جديدة بدون وثائق
- تغييرات التكوين غير موثقة
- الوثائق التي عفا عليها الزمن

### مقارنة فرع الميزات

__لا تترجم_3__

قارن `feature-oauth` بـ `main` للعثور على جميع الفجوات في التوثيق.

### إنشاء المستندات المفقودة تلقائيًا

__لا تترجم_4__

يقوم تلقائيًا بإنشاء وثائق كعب الروتين لجميع العناصر المفقودة وعمليات التنفيذ مع الرسالة `docs: sync with codebase changes`.

## الإخراج

يقوم الأمر بإنشاء تقرير شامل:

__لا تترجم_5__

## كيف يعمل

### كشف التغيير

الأمر:
1. **مقارنة الفروع** باستخدام `git diff` لتحديد كافة تغييرات التعليمات البرمجية
2. **يصنف التغييرات** إلى نقاط نهاية واجهة برمجة التطبيقات، ومكونات واجهة المستخدم، والخدمات، والتكوين، ومخططات قاعدة البيانات
3. **خرائط للوثائق** المواقع التي يجب أن توجد فيها المستندات

### الاستدلال الذكي

يتم استنتاج التوثيق من:
- **أنواع وواجهات TypeScript** - تواقيع الوظائف وأنواع الإرجاع
- **تعليقات JSDoc** - وثائق التعليمات البرمجية الموجودة
- **ملفات الاختبار** - أمثلة الاستخدام
- **ديكورات OpenAPI** - مواصفات API

### تحليل الفجوة

لكل ملف تم تغييره، يتحقق الأمر مما يلي:
- **الوثائق المفقودة**: كود مضاف حديثًا بدون مستندات
- **الوثائق القديمة**: مراجع للتعليمات البرمجية التي تمت إزالتها أو تغييرها
- **غير مكتملوثائق te **: بذرة تحتاج إلى تفاصيل

## رسم الخرائط التوثيقية

يتم تعيين تغييرات التعليمات البرمجية إلى الوثائق المتوقعة:

| موقع الكود | التوثيق المنتظر |
|--------------|---------------------|
| `src/api/` أو `src/routes/` | `docs/04-architecture/api.md` أو مواصفات OpenAPI |
| `src/components/` أو `src/pages/` | `docs/04-architecture/components.md` أو القصص المصورة |
| `src/services/` أو `src/utils/` | __غير مترجم_24__ |
| __غير مترجم_25__، __غير مترجم_26__ | __غير مترجم_27__ |
| `migrations/` أو `schema/` | __غير مترجم_30__ |

## أفضل الممارسات

### قبل تنفيذ التعليمات البرمجية
قم بتشغيل `docs-sync` لتحديد فجوات التوثيق مبكرًا:

__لا تترجم_6__

### عند إضافة نقاط نهاية جديدة
سيقوم الأمر بوضع علامة عليها تلقائيًا، مما يسهل تذكر الوثائق:

__لا تترجم_7__

يؤدي هذا إلى إنشاء بذرة تحمل علامة "TODO: أضف تفاصيل" للمراجعة اليدوية.

### أقسام التوثيق التي يجب الحفاظ عليها
احتفظ بهذه المزامنة مع الكود:
- مثال لطلب/استجابة APIق
- تعريفات دعامة المكونات
- خيارات التكوين
- تغييرات مخطط قاعدة البيانات
- نقاط نهاية API المهملة
