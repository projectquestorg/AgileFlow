# /épique

Créez une nouvelle épopée avec des histoires enfants facultatives pour organiser les histoires utilisateur associées autour d'une fonctionnalité ou d'un objectif plus large.

## Démarrage rapide

```bash
/agileflow:epic EPIC=<EP-ID> TITLE=<text> OWNER=<id> GOAL=<text>
```

## Paramètres

| Paramètre | Obligatoire | Par défaut | Descriptif |
|-----------|----------|---------|-------------|
| `EPIC` | Oui | - | Identifiant épique (par exemple, EP-0001) |
| `TITLE` | Oui | - | Titre épique décrivant la fonctionnalité ou l'objectif |
| `OWNER` | Oui | - | Nom du propriétaire ou identifiant de l'agent (par exemple, AG-UI, AG-API) |
| `GOAL` | Oui | - | Résultat ou mesure qui définit le succès de cette épopée |
| `STORIES` | Non | - | Liste des histoires initiales séparées par des virgules (format : `US-ID\|title\|owner`) |

## Exemples

### Création épique de base

```bash
/agileflow:epic EPIC=EP-0010 TITLE="User Authentication" OWNER=AG-API GOAL="Enable users to securely register and log in"
```

Crée un nouveau fichier épique dans `docs/05-epics/EP-0010.md` avec les détails fournis et l'enregistre dans `docs/09-agents/status.json`.

### Épique avec des histoires initiales

```bash
/agileflow:epic EPIC=EP-0011 TITLE="Payment Processing" OWNER=AG-API GOAL="Enable payments via Stripe" STORIES="US-0040|Stripe integration|AG-API,US-0041|Payment form|AG-UI,US-0042|Webhook handler|AG-API"
```

Crée l'épopée et génère automatiquement les histoires enfants initiales sous `docs/06-stories/EP-0011/`. Chaque sL'histoire est créée avec :
- Statut : `ready` (prêt pour le développement)
- Estimation : `0.5d` (par défaut, peut être modifié)
- Dépendances : vides (mises à jour manuellement si nécessaire)

## Sortie

La commande crée :

1. **Fichier épique** (`docs/05-epics/EP-XXXX.md`)
   - Métadonnées épiques (titre, propriétaire, statut, objectif)
   - Statut actuel et mesures
   - Liste d'histoires pour enfants

2. **Fichiers d'histoire** (un par histoire, `docs/06-stories/EP-XXXX/US-XXXX-<slug>.md`)
   - Modèle d'histoire avec espaces réservés aux critères d'acceptation
   - Lié à l'épopée parent
   - Prêt pour une définition détaillée

3. **Suivi du statut** (`docs/09-agents/status.json`)
   - Entrée épique avec propriétaire et statut actuel
   - Entrées d'histoire liées à l'épopée

4. **Communication avec les agents** (`docs/09-agents/bus/log.jsonl`)
   - Messages d'affectation montrant l'épopée créée et les histoires attribuées

## Flux de travail

1. **Planifiez votre épopée** - Définissez l'objectif de la fonctionnalité et les indicateurs de réussite
2. **Créez l'épopée** - Exécutez la commande `/epic` avec les détails de base
3. **Ajouter des histoires** - Créez-les maintenant avec le paramètre STORIESou utilisez `/story` pour ajouter individuellement plus tard
4. **Développer des histoires** - Utilisez `/story-validate` pour vous assurer que chaque histoire est prête avant le développement
5. **Plan de sprint** - Utilisez `/sprint` pour sélectionner des histoires pour le prochain sprint
6. **Suivez la progression** - Utilisez `/board` pour surveiller l'achèvement épique

## Intégration avec d'autres commandes

- **Après la création épique** : utilisez `/story` pour ajouter plus d'histoires
- **Avant la planification du sprint** : utilisez `/sprint` pour sélectionner des histoires de cette épopée
- **Pour le suivi des progrès** : utilisez `/board` ou `/metrics` pour voir l'achèvement épique
- **Pour les dépendances** : utilisez `/deps` pour visualiser les dépendances des histoires dans Epic

## meilleures pratiques

- **Gardez les épopées concentrées** - Une fonctionnalité ou un domaine de fonctionnalités par épopée
- **Rendre les objectifs mesurables** - Utiliser des résultats ou des mesures spécifiques (et non "améliorer l'expérience utilisateur")
- **Ajouter des histoires progressivement** - Commencez par les histoires principales, ajoutez celles associées au fil du temps
- **Attribuer les propriétaires appropriés** - Faire correspondre les propriétaires épiqueseuh avec l'agent de développement principal
- **Contexte du document** - Ajouter un contexte technique dans le fichier épique après la création

## Commandes associées

- [`/story`](/commands/story) - Créez des histoires individuelles au sein d'une épopée
- [`/sprint`](/commands/sprint) - Planifiez des sprints avec des histoires spécifiques à l'épopée
- [`/deps`](/commands/deps) - Visualisez les dépendances des histoires au sein d'Epic
- [`/board`](/commands/board) - Suivez visuellement les progrès épiques
- [`/metrics`](/commands/metrics) - Afficher les mesures d'achèvement et de vitesse épiques
