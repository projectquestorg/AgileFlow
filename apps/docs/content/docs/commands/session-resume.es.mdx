# /sesión:reanudar

Cambie a una sesión de Claude Code diferente. Muestra las sesiones disponibles y proporciona el comando para cambiar.

## Inicio rápido

```bash
/agileflow:session:resume
```

## Cómo funciona

1. Enumera todas las sesiones registradas con estado.
2. Presenta un selector interactivo para seleccionar una sesión.
3. Muestra el comando de terminal para cambiar

## Ejemplo de interacción

```
Which session do you want to resume?

○ Session 1 (main) - /home/user/myproject [active]
○ Session 2 "auth" - /home/user/myproject-auth [active]
○ Session 3 - /home/user/myproject-3 [stale]
```

Después de seleccionar la Sesión 2:

```
To resume Session 2 "auth":

  cd /home/user/myproject-auth && claude --resume

Note: Use a new terminal window/tab for best experience.
```

## Estados de sesión

| Estado | ¿Se puede reanudar? | Notas |
|-------|-------------|-------|
| `active` | Sí | Allí se está ejecutando otra instancia de Claude |
| `stale` | Sí | La sesión anterior finalizó, el directorio aún existe |
| `current` | No | Ya estás en esta sesión |

## La bandera --resume

La bandera `claude --resume` le dice a Claude Code que:
- Cargar historial de conversaciones de la sesión anterior
- Restaurar el contexto sobre lo que estabas trabajando.
- Continúa donde lo dejaste

Sin `--resume`, Claude comienza con un contexto nuevo.

## Ejemplos de flujo de trabajo

### CambiarIntercambiar tareas

```bash
# Currently in Session 1 working on feature A
/agileflow:session:resume
# Select Session 2 "auth"

# In new terminal:
cd /home/user/myproject-auth && claude --resume
# Continue auth work with full context
```

### Regresando después del descanso

```bash
# Start Claude Code
claude

# See: "2 other active sessions"
/agileflow:session:resume
# Pick the session you want to continue
```

### Limpiar antes de reanudar

Si una sesión se muestra obsoleta:

```bash
/agileflow:session:status
# Session 3 shows "stale"

/agileflow:session:end
# Select Session 3 to clean up

/agileflow:session:resume
# Now switch to a clean session
```

## Consejos para terminales

**Mejores prácticas**: abra una nueva pestaña/ventana de terminal para cada sesión.

```bash
# Tab 1: Session 1 (main project)
cd /home/user/myproject && claude

# Tab 2: Session 2 (auth feature)
cd /home/user/myproject-auth && claude --resume

# Tab 3: Session 3 (API work)
cd /home/user/myproject-3 && claude --resume
```
