# /Vorlage

## Schnellstart

```bash
# List all available templates
/agileflow:template

# Create a new custom template
/agileflow:template ACTION=create TYPE=custom NAME=meeting-notes

# Use a template to generate a document
/agileflow:template ACTION=use TYPE=custom NAME=meeting-notes
```

## Parameter

| Parameter | Required | Default | Description |
|-----------|----------|---------|-------------|
| `ACTION` | No | list | Operation: list, create, edit, or use |
| `TYPE` | For create/edit/use | - | Template type: story, epic, adr, custom, etc. |
| `NAME` | For create/edit/use | - | Template name (e.g., meeting-notes) |
| `PATH` | Optional | auto | Custom path to save template |

## Best Practices

### Wann Vorlagen erstellt werden sollten

Erstellen Sie eine Vorlage, wenn:
- Ein Dokumenttyp wird mehrfach erstellt
- Eine bestimmte Struktur sollte konsequent befolgt werden
- Bestimmte Felder sollten immer enthalten sein

### Benennung der Vorlage

Verwenden Sie aussagekr√§ftige Kebab-Namen:

```text
Good:
- meeting-notes
- sprint-retrospective
- incident-report
- design-proposal

Avoid:
- template1
- doc
- new-file
```

### Variablenbenennung

Verwenden Sie UPPERCASE_WITH_UNDERSCORES:

```text
Good:
{{STORY_ID}}, {{OWNER}}, {{DUE_DATE}}

Avoid:
{{story_id}}, {{owner}}, {{due-date}}, {{Author}}
```
