## Schnellstart


## Parameter

| Parameter | Erforderlich | Standard | Beschreibung |
|-----------|----------|---------|-------------|
| `ACTION` | Nein | Liste | Vorgang: Auflisten, erstellen, bearbeiten oder verwenden |
| `TYPE` | Zum Erstellen/Bearbeiten/Verwenden | - | Vorlagentyp: Story, Epos, ADR, Benutzerdefiniert usw. |
| `NAME` | Zum Erstellen/Bearbeiten/Verwenden | - | Vorlagenname (z. B. Besprechungsnotizen) |
| `PATH` | Optional | auto | Benutzerdefinierter Pfad zum Speichern der Vorlage |

## Best Practices

### Wann Vorlagen erstellt werden sollten

Erstellen Sie eine Vorlage, wenn:
- Ein Dokumenttyp wird erstelltmehrfach
- Eine bestimmte Struktur sollte konsequent befolgt werden
- Bestimmte Felder sollten immer enthalten sein

### Vorlagenbenennung

Verwenden Sie aussagekr√§ftige Kebab-Namen:

```text
Good:
- meeting-notes
- sprint-retrospective
- incident-report
- design-proposal

Avoid:
- template1
- doc
- new-file
```

### Variablenbenennung

Verwenden Sie UPPERCASE_WITH_UNDERSCORES:

```text
Good:
{{STORY_ID}}, {{OWNER}}, {{DUE_DATE}}

Avoid:
{{story_id}}, {{owner}}, {{due-date}}, {{Author}}
```
