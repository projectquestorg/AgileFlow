# /agileflow:research

Umfassendes Recherchesystem zum Generieren detaillierter Untersuchungsaufforderungen, zum Importieren von Ergebnissen und zum Verwalten von Forschungsnotizen.

## Schnellstart

Das Forschungssystem bietet vier Spezialbefehle:

```bash
/agileflow:research:ask TOPIC="your research topic"       # Generate research prompt
/agileflow:research:import TOPIC="title"                  # Import external results
/agileflow:research:list                                   # View all research notes
/agileflow:research:view FILE=<filename>                  # Read specific note
```

## Übersicht

Das Forschungssystem ermöglicht:

1. **Generieren Sie detaillierte Forschungsaufforderungen** für ChatGPT, Claude Web, Perplexity oder Gemini
2. **Importieren Sie Ergebnisse** von externen KI-Tools in Ihr Projekt
3. **Durchsuchen und referenzieren** frühere Forschungsergebnisse
4. **Befunde organisieren** mit strukturierten Vorlagen und Indizes

Forschung ist unerlässlich für:
- Behebung schwieriger Probleme mit externer KI-Expertise
- Untersuchung von Technologien vor Architekturentscheidungen
- Dokumentation des Lernens aus externen Quellen
- Aufbau einer Wissensdatenbank für Ihr Team

## Unterbefehle

### /agileflow:research:ask – Forschungsaufforderung generieren

Generieren Sie eine umfassende, kontextreiche Rechercheaufforderung für Web-KI-Tools.

**Zweck**: Erstellt detaillierte Eingabeaufforderungen mit mehr als 200 Zeilen, die Sie einbeziehenr-Codebasiskontext, genaue Fehlermeldungen, was Sie versucht haben und spezifische Fragen.

**Befehl**:
```bash
/agileflow:research:ask TOPIC="your topic" [DETAILS="constraints"] [ERROR="error message"]
```

**Parameter**:

| Argument | Erforderlich | Beschreibung |
|----------|----------|-------------|
| THEMA | Ja | Was Sie recherchieren (z. B. „OAuth 2.0 mit Google“) |
| DETAILS | Nein | Einschränkungen, Fristen oder spezifische Anforderungen |
| FEHLER | Nein | Genaue Fehlermeldung bei Fehlerbehebung |

**Hauptmerkmale**:

- **Erzeugt mindestens 200+ Zeilen** - Detailliert und umfassend
- **Enthält tatsächliche Codeausschnitte** – 50+ Zeilen aus Ihrer Codebasis
- **Bettet genaue Fehler ein** - Vollständige Stack-Traces zur Fehlerbehebung
- **Listet Lösungsversuche auf** - Zeigt an, was Sie bereits versucht haben
- **Stellt spezifische Fragen** – 3+ gezielte Fragen an die KI
- **Nur Ausgabe** – Keine Dateien geschrieben, Sie kopieren sie in ein externes KI-Tool

**Was generiert wird**:

Die Eingabeaufforderung enthält die folgenden obligatorischen Abschnitte:

- **Projektkontext** – Framework, Abhängigkeiten, relevante Dateien
- **Aktuelle Implementierung** – Zeile 50+s tatsächlichen Code aus Ihrer Codebasis
- **Das Problem** – Genaue Fehlermeldungen, Stack-Traces, erwartetes vs. tatsächliches Verhalten
- **Was wir bereits ausprobiert haben** – 2+ Ansätze mit Ergebnissen und Analyse
- **Spezifische Fragen** – 3+ gezielte Fragen zur Beantwortung
- **Was ich brauche** – Ursachenanalyse, Schritt-für-Schritt-Lösung, Codebeispiele, Fallstricke
- **Umgebungsdetails** – Versionen und Systeminformationen
- **Referenzen** – Quelllinks, wenn Dokumentation zitiert wird

**Qualitätsvalidierung**:

Vor der Ausgabe überprüft der Befehl Folgendes:
- Länge ≥ 200 Zeilen
- Codefragmente ≥ 50 Zeilen
- Wörtliche Fehlermeldungen (falls zutreffend)
- ≥ 2 ausprobierte Ansätze aufgelistet
- ≥ 3 spezifische Fragen enthalten

**Beispiel**:

```bash
# Research OAuth issue
/agileflow:research:ask TOPIC="next-auth Google OAuth unauthorized_client" ERROR="[auth][error] CallbackRouteError: unauthorized_client"

# Output: 250+ line prompt with code, error details, what was tried, and specific questions
# Copy output → ChatGPT/Claude web → Get expert analysis
# Then use /agileflow:research:import to save the response
```

**Arbeitsablauf**:

```
1. /agileflow:research:ask TOPIC="..."
   ↓ (command generates prompt)
2. Copy prompt output
3. Paste into ChatGPT/Claude web/Perplexity
4. Get expert analysis
5. /agileflow:research:import TOPIC="..." CONTENT="paste response"
   ↓ (command saves and structures results)
6. Results saved to docs/10-research/YYYYMMDD-topic-slug.md
```

**Integration mit Babysitter**:

Wenn `/babysit` feststellt, dass Sie nicht weiterkommen, schlägt es Folgendes vor:
```
Looks like you're stuck on this problem.
Try: /agileflow:research:ask TOPIC="describe your problem"
```

### /agileflow:research:import – Forschungsergebnisse importieren

Importieren Sie Forschungsergebnisse von externen KI-Tools mit struct in Ihren ForschungsordnerFormatierung festgelegt.

**Zweck**: Übernimmt Ergebnisse aus ChatGPT, Claude Web, Perplexity, YouTube-Transkripten oder Blog-Beiträgen und speichert sie als organisierte Markdown-Dateien.

**Befehl**:
```bash
/agileflow:research:import TOPIC="topic" [CONTENT="results"] [SOURCE="url"]
```

**Parameter**:

| Argument | Erforderlich | Beschreibung |
|----------|----------|-------------|
| THEMA | Ja | Name für die Forschungsdatei (z. B. „OAuth 2.0 Setup“) |
| INHALT | Ja* | Zu importierende Forschungsergebnisse (*können nach dem Befehl eingefügt werden) |
| QUELLE | Nein | Ursprüngliche Quell-URL zur Namensnennung |

**Hauptmerkmale**:

- **Behält ALLE Codeschnipsel bei** - Genau wie bereitgestellt
- **Generiert umsetzbare Elemente** - Konkrete nächste Schritte, keine vagen Vorschläge
- **Diff-First-Vorschau** – Formatiertes Ergebnis vor dem Speichern anzeigen
- **Aktualisiert den Forschungsindex** – Wird automatisch zu docs/10-research/README.md hinzugefügt
- **Links zu Entscheidungen** – Kann auf ADRs, Epics oder Stories verweisen

**Was erstellt wird**:

Strukturierte Forschungsdatei mit Abschnitten:

- **Zusammenfassung** – 2-3 Absatz TL;DR
- **Wichtige Erkenntnisse** - Wichtigste Erkenntnisse als Bullet Punkte
- **Implementierungsansatz** - Schritt-für-Schritt-Plan, falls zutreffend
- **Code-Snippets** – Der gesamte Code bleibt genau wie bereitgestellt erhalten
- **Aktionselemente** - Konkrete nächste Schritte (Kontrollkästchen)
- **Risiken und Fallstricke** – Warnungen und Randfälle
- **Story-Vorschläge** – Mögliche Features, wenn die Recherche auf Arbeit schließen lässt
- **Referenz zum Rohinhalt** – Der Originalinhalt wurde zu Referenzzwecken ausgeblendet

**Speicherort der Datei**:
```
docs/10-research/YYYYMMDD-topic-slug.md
```

**Indexaktualisierung**:

Forschungseintrag zu `docs/10-research/README.md` hinzugefügt:
```markdown
| Date | Topic | File | Summary |
|------|-------|------|---------|
| 2025-01-01 | OAuth 2.0 Setup | 20250101-oauth-setup.md | OAuth flows, security, best practices |
```

**Beispiel**:

```bash
# After getting ChatGPT response about OAuth
/agileflow:research:import TOPIC="OAuth 2.0 Setup" SOURCE="https://chatgpt.com"

# Prompts: "Paste the ChatGPT response"
# You paste content
# Command shows preview
# You confirm
# Results saved to docs/10-research/20250101-oauth-setup.md
```

**Funktioniert mit**:

- ChatGPT-Antworten
- Claude Web-Gespräche
- Ergebnisse der Ratlosigkeitsforschung
- YouTube-Videotranskripte
- Auszüge aus Blogbeiträgen
- Gesprächsnotizen zur Konferenz
- Dokumentationsseiten
- Besprechungsnotizen

### /agileflow:research:list – Index der Forschungsnotizen anzeigen

Zeigen Sie alle Forschungsnotizen in Ihrem Projekt mit einer schnellen Übersicht an.

**Zweck**: Zeigt in `docs/10-research/` gespeicherte Forschungsnotizen mit Datum, Thema, Dateiname und Zusammenfassung an.

**Befehl**:
__NICHTÜBERSETZEN_9__

**Parameter**: Keine

**Hauptmerkmale**:

- **Schreibgeschützt** – Keine Dateien geschrieben
- **Zeigt alle Notizen an** - Datum, Thema, Dateiname
- **Sortiert nach Datum** – Neueste zuerst
- **Schnelle Aktionshinweise** - Vorschläge für die nächsten Befehle

**Ausgabebeispiel**:

```
## Research Notes

| Date | Topic | File |
|------|-------|------|
| 2025-01-01 | OAuth 2.0 Setup | 20250101-oauth-setup.md |
| 2024-12-31 | TypeScript Performance | 20241231-typescript-perf.md |
| 2024-12-28 | Mermaid Diagrams | 20241228-mermaid.md |

Total: 3 research notes

Quick actions:
- /agileflow:research:view FILE=20250101-oauth-setup.md - Read a note
- /agileflow:research:ask TOPIC="new topic" - Create new research
- /agileflow:research:import - Import external content
```

**Wann zu verwenden**:

- Suchen Sie vor der Implementierung nach früheren Forschungsergebnissen
- Suchen Sie nach Codeausschnitten, die Sie recherchiert haben
- Überprüfen Sie Aktionselemente aus früheren Untersuchungen

### /agileflow:research:view – Spezifische Forschungsnotiz lesen

Zeigen Sie den vollständigen Inhalt einer Forschungsnotiz aus Ihrem Forschungsordner an.

**Purpose**: Read and review research notes to find code snippets, action items, and findings.

**Befehl**:
```bash
/agileflow:research:view FILE=<filename>
```

**Parameter**:

| Argument | Erforderlich | Beschreibung |
|----------|----------|-------------|
| DATEI | Ja | Dateiname der Forschungsnotiz (z. B. „20250101-oauth-setup.md“) |

**Hauptmerkmale**:

- **Schreibgeschützt** – Keine Dateien geschrieben
- **Vollständiger Inhalt** – Die gesamte Datei wird angezeigt, nicht abgeschnitten
- **Alles finden** - Codeausschnitte, actionsgegenstände, Erkenntnisse

**Beispiel**:

```bash
# View OAuth research
/agileflow:research:view FILE=20250101-oauth-setup.md

# Output: Full content of the file
```

**Ausgabe beinhaltet**:

- Vollständiger Inhalt der Forschungsdatei
- Alle Codeausschnitte aus der Recherche
- Aktionselemente mit Kontrollkästchen
- Key findings and gotchas
- Risiken und Überlegungen
- Referenzen und Quellen

## Workflow-Beispiele

### Vollständiger Forschungsworkflow

**Szenario**: Sie stecken bei der Authentifizierung fest und benötigen Expertenhilfe.

```bash
# Step 1: Generate detailed research prompt
/agileflow:research:ask TOPIC="next-auth Google OAuth unauthorized_client error" \
  ERROR="[auth][error] CallbackRouteError: unauthorized_client"

# Output: 250+ line detailed prompt with your code, error, what you tried, questions
# → Copy the prompt

# Step 2: Paste into ChatGPT/Perplexity
# → Get expert analysis back

# Step 3: Import the results
/agileflow:research:import TOPIC="next-auth OAuth Fix" SOURCE="https://chatgpt.com"

# Prompts for content → paste ChatGPT response
# → Shows preview of formatted file
# → Saves to docs/10-research/YYYYMMDD-nextauth-oauth-fix.md

# Step 4: Reference in your work
# - Create ADR referencing the research
# - Create story based on action items
# - Check action items when implementing

# Step 5: Review research when implementing
/agileflow:research:list                    # See all research
/agileflow:research:view FILE=<filename>    # Review details
```

### Nachforschungen vor der Architekturentscheidung

```bash
# Step 1: Research database options
/agileflow:research:ask TOPIC="PostgreSQL vs MongoDB for real-time collaboration"

# Step 2: Get expert comparison, use cases, tradeoffs
# → Copy response from ChatGPT/Perplexity

# Step 3: Import results
/agileflow:research:import TOPIC="Database Selection Analysis"

# Step 4: Create ADR referencing research
# /agileflow:adr NEW
# Mention: "See research in docs/10-research/"

# Step 5: Team reviews research before decision
/agileflow:research:view FILE=20250101-database-selection.md
```

### Von externen Inhalten lernen

```bash
# You find a great article/YouTube video about your tech

# Step 1: Import the content
/agileflow:research:import TOPIC="Next.js App Router Advanced Patterns" \
  SOURCE="https://example.com/article"

# Copy/paste key sections → saves to research folder

# Step 2: Reference from architecture docs
# docs/04-architecture/... → "See research: [link]"

# Step 3: Create stories from action items
# "Per research, we should implement error boundaries"
```

## Best Practices

### Generierung von Forschungsaufforderungen

1. **Geben Sie das THEMA genau an** – Sagen Sie nicht nur „Hilfe bei der Authentifizierung“, sondern „nächste Authentifizierung 5.0“.mit Google OAuth“
2. **Fehlermeldungen einschließen** – Vollständige Stack-Traces helfen enorm
3. **Dokumentieren Sie, was Sie versucht haben** – Zeigt der KI an, dass Sie Ihre Hausaufgaben gemacht haben
4. **Stellen Sie konkrete Fragen** – „Warum passiert X?“ nicht „Wie behebe ich das?“

**Gute THEMEN-Beispiele:**
- „next-auth Google OAuth unautorisierter_Client-Fehler mit Next.js 14 App Router“
- „TypeScript-Kompilierungsfehler im strengen Modus im Monorepo-Setup“
– „Stripe-Webhook-Signaturvalidierung schlägt in der Entwicklung fehl“

**Schlechte THEMA-Beispiele:**
- „Hilfe bei der Authentifizierung“
- „Warum ist mein Code kaputt?“
- „Fehler in meiner App“

### Forschungsimport

1. **Geben Sie immer die Quelle an** – Woher haben Sie das?
2. **Code-Snippets genau beibehalten** – Beispiele nicht ändern
3. **Wichtige Erkenntnisse extrahieren** – Fügen Sie nicht nur Rohinhalte ein
4. **Aktionselemente generieren** – Was sollten Sie damit tun?
5. **Link zu Entscheidungen** – Verweisen Sie auf ADRs, Epics, Stories, falls relevant

### Forschung verwalten

1. **Überprüfung vor der Implementierung** – Folgen Sie der Forschung nicht blind
2. **Prüfungk Aktionselemente** – Verwenden Sie Kontrollkästchen, um zu verfolgen, was Sie getan haben
3. **Dateien aktuell halten** – Aktualisieren Sie, wenn Sie mehr erfahren
4. **Referenz in Entscheidungen** – Link von ADRs und Geschichten
5. **Archivieren Sie alte Forschungsergebnisse** – Verschieben Sie veraltete Ergebnisse in das Archivverzeichnis

## Wann man Recherche nutzen sollte

### Vor Architekturentscheidungen
Informieren Sie sich über Tools und Frameworks, bevor Sie sie empfehlen:

```bash
/agileflow:research:ask TOPIC="PostgreSQL vs MongoDB for real-time sync"
# → Get expert comparison
/agileflow:research:import  # Import results
# → Create ADR referencing the research
```

### Feature-Untersuchung
Erfahren Sie mehr über die Implementierung komplexer Funktionen:

```bash
/agileflow:research:ask TOPIC="Next.js App Router middleware patterns"
/agileflow:research:ask TOPIC="OAuth 2.0 implementation with security best practices"
/agileflow:research:ask TOPIC="Real-time WebSocket architecture in Node.js"
```

### Fehlerbehebung bei schwierigen Problemen
Holen Sie sich professionelle Hilfe, wenn Sie nicht weiterkommen:

```bash
/agileflow:research:ask TOPIC="TypeScript strict mode in monorepo" \
  ERROR="error TS1149: File is a CommonJS module..."
```

### Lernen und Training
Dokumentieren Sie Erkenntnisse aus Kursen, Tutorials oder Dokumentationen:

```bash
/agileflow:research:import TOPIC="React 18 Suspense and Streaming" \
  SOURCE="https://react.dev/reference/react/Suspense"
```

### Technologiebewertung
Vergleichen Sie Optionen vor der Einführung:

```bash
/agileflow:research:ask TOPIC="GraphQL vs tRPC for type-safe APIs"
/agileflow:research:ask TOPIC="SvelteKit vs Next.js for new project"
```

---## Fehlerbehebung

### Forschungsaufforderung zu kurz

Wenn Sie weniger als 200 Zeilen lang eine Eingabeaufforderung erhalten, wird der Befehl neu generiert. Stellen Sie sicher:
- THEMA ist spezifisch (einschließlich Framework-Versionen, Fehlertypen)
- FEHLER beinhaltet vollständigen Stack-Trace zur Fehlerbehebung
– Der Befehl liest Ihre tatsächliche Codebasis

### Import aktualisiert den Index nicht

Der Index (`docs/10-research/README.md`) wird automatisch erstellt und aktualisiert. Falls es fehlt:

```bash
/agileflow:research:list  # This will create the index if missing
```

### Forschungsdatei kann nicht gefunden werden

Verwenden Sie `/agileflow:research:list`, um alle verfügbaren Dateien anzuzeigen:

```bash
/agileflow:research:list
# Shows all files with correct filenames for /agileflow:research:view
```

### Forschungsordner existiert nicht

Der erste Recherche-Import- oder Listenbefehl erstellt automatisch das Verzeichnis `docs/10-research/`. Keine manuelle Einrichtung erforderlich.
