# /readme-sync

Mantenha os arquivos README.md da pasta atualizados, gerando automaticamente uma seção de conteúdo que corresponda aos arquivos reais da pasta.

## Início rápido

```bash
/agileflow:readme-sync FOLDER=docs/02-practices
```

Para sincronizar todas as pastas de documentação de uma só vez:

```bash
/agileflow:readme-sync FOLDER=all
```

## Parâmetros

| Parâmetro | Obrigatório | Padrão | Descrição |
|-----------|----------|---------|------------|
| `FOLDER` | Não | - | Caminho para a pasta (por exemplo, `docs/02-practices`) ou `all` para sincronizar documentos inteiros/ |

## Exemplos

### Sincronizar pasta única

```bash
/agileflow:readme-sync FOLDER=docs/02-practices

## Como funciona

### Etapa 1: listar o conteúdo
O comando verifica a pasta em busca de:
- Todas as remarcações e outros arquivos
- Subdiretórios e seus conteúdos

### Etapa 2: extrair descrições
Para cada arquivo, ele extrai:
- **Arquivos Markdown**: O primeiro título (# Heading) ou primeira frase
- **Outros arquivos**: primeira linha ou descrição baseada no nome do arquivo

### Etapa 3: Criar seção de conteúdo
Gera um formato de lista com marcadores limpo com:
- Nomes de arquivos em negrito
- Descrições de uma frase
- Estrutura aninhada para subdiretórios

```

## What It Updates

The command **only updates the `## Contents` section**:

- Removes old file listings
- Adds all current files with descriptions
- Maintains all other README sections unchanged
- Preserves custom notes and links
- Keeps header, intro, and footer sections intact

This means you can safely use the command without losing custom documentation.

## File Description Extraction

The command intelligently extracts descriptions:

```markdown
# Testing Strategy

Guidelines for writing and running tests...
```

Extracts as: "Testing Strategy – Guidelines for writing and running tests"

```markdown
<!-- This file documents git workflow -->
Guidelines for branching and committing
```

Extracts the first meaningful line as description.
