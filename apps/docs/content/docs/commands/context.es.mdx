# /contexto

Cree, actualice y administre el resumen del contexto de su proyecto para usarlo en herramientas web de inteligencia artificial como ChatGPT, Perplexity, Gemini y Claude web.

## Descripción general

El `/context` La familia de comandos lo ayuda a mantener y exportar el contexto del proyecto para herramientas de inteligencia artificial basadas en web. Esto es útil cuando necesitas:

- Genere un resumen de contexto completo para pegarlo en herramientas web de IA
- Exporte un extracto conciso para una carga rápida del contexto
- Registre decisiones y descubrimientos importantes en su archivo contextual
- Mantenga informadas las herramientas externas de IA sobre el estado de su proyecto

## Subcomandos

El `/context` El comando tiene tres subcomandos, cada uno con un propósito específico:

### /agileflow:context:completo

**Generate or refresh the comprehensive project context brief.**

Crea o actualiza `docs/context.md` con secciones administradas que contienen:
- Qué estamos construyendo y enfoque actual
- Mapa de características y resumen de tecnología/herramientas
- Decisiones clave (ADR) e instantánea de la arquitectura
- Estado de pruebas y CI
- Progreso reciente
- Riesgos y próximos pasos

Este archivo está diseñado para pegarse en ChatGPT, Perplexity, Gemini o Claude web para brindar a las herramientas externas de IA el contexto completo del proyecto.

**Usage:**
```bash
/agileflow:context:full
```

**Behavior:**
- Lee existente `docs/context.md` si está presente
- Recopila datos de status.json, ADR, epopeyas, historias y CHANGELOG
- Actualiza solo las secciones administradas (entre `<!-- MANAGED_SECTION -->` marcadores)
- Conserva cualquier contenido escrito por el usuario fuera de las secciones administradas
- Muestra diferencias antes de aplicar cambios.
- Requiere confirmación antes de escribir.

---

### /agileflow:context:exportar

**Export a concise context excerpt for pasting into web AI tools.**

Lee `docs/context.md` y genera un extracto condensado (≤300 líneas) listo para copiar y pegar en ChatGPT, Perplexity, Gemini o Claude web.

Este es un **read-only command** - no se escriben archivos.

**Usage:**
```bash
/agileflow:context:export
```

**Behavior:**
- Lee `docs/context.md`
- Extrae secciones clave (descripción general, enfoque actual, pila tecnológica, decisiones, mapa de características, próximos pasos)
- Genera un extracto conciso en un bloque de código
- Incluye instrucciones para pegar en web AI
- No requiere confirmación

---

### /agileflow:context:nota

**Append a timestamped note to the context file.**

Añade rápidamente una nota a `docs/context.md` en la sección "Notas". Útil para registrar decisiones, observaciones o recordatorios que deben persistir en el contexto del proyecto.

**Usage:**
```bash
/agileflow:context:note NOTE="Your note here"
```

**Arguments:**

| Argument | Required | Description |
|----------|----------|-------------|
| NOTE | Yes | 1-5 line note text to append |

**Behavior:**
- Lee o crea `docs/context.md`
- Crea una sección de "Notas" si no existe
- Añade nota con marca de tiempo ISO (por ejemplo, `2024-12-31T10:30:00Z`)
- Muestra diferencias antes de aplicar cambios.
- Requiere confirmación antes de escribir.

## Ejemplos

### Generar contexto completo

```bash
/agileflow:context:full
```

Crea o actualiza `docs/context.md` con información completa del proyecto:

```markdown
# Project Context Brief

**Last Updated**: 2025-12-24
**Project**: AgileFlow - AI-powered project management

## What We're Building

AgileFlow is a comprehensive project management system powered by AI agents. It provides automated task management, documentation synchronization, and intelligent context preservation.

## Current Focus

- Implementing context compacting for long conversations
- Improving research workflow integration
- Expanding agent capabilities

## Feature Map

- **Core**: Project setup, status tracking, agent coordination
- **Documentation**: Automatic sync, README generation, ADR management
- **Agents**: 46 specialized agents for different tasks
- **Research**: Structured research notes and integration

## Tech Stack

- **Language**: TypeScript
- **Framework**: Next.js 16 (App Router)
- **UI**: React with Tailwind CSS
- **Package Manager**: npm with legacy-peer-deps
- **Deployment**: Vercel

## Key Decisions

- ADR-0001: Use TypeScript for type safety
- ADR-0002: Use React component library
- ADR-0003: Next.js for web framework

## Architecture Snapshot

- Monorepo structure with packages/ and apps/
- CLI package with 85 commands
- Website and documentation sites
- Agent-based command coordination

## Testing & CI

- Unit tests: Jest
- E2E tests: Playwright
- CI/CD: GitHub Actions
- Pre-commit hooks for validation

## Recent Progress

- Completed context compacting feature
- Improved documentation sync workflow
- Added 5 new specialized agents

## Risks

- Token limits with large projects
- Documentation drift during rapid development

## Next Steps

- Implement automated test coverage reporting
- Add real-time collaboration features
- Expand agent capabilities for code generation
```

## Modos de salida

### /agileflow:context:full (Crear/Actualizar contexto)

**Input**: Recopila información de:
- `status.json` - Trabajo actual y estado de la historia.
- `bus/log.jsonl` - Mensajes de progreso recientes
- `docs/03-decisions/` - Decisiones de arquitectura (ADR)
- `docs/04-architecture/` - Documentación de diseño del sistema.
- `docs/05-epics/` - Hoja de ruta de funciones y epopeyas.
- `CHANGELOG.md` - Historial de lanzamientos
- `package.json` - Metadatos del proyecto

**Output**: Actualizaciones `docs/context.md` con secciones administradas conservando el contenido personalizado

**Key Behavior**:
- Diff-first: muestra los cambios antes de escribir
- Secciones administradas: solo actualiza el contenido entre `<!-- MANAGED_SECTION -->` marcadores
- Conserva el contenido del usuario: nunca modifica el contenido fuera de las secciones administradas
- Requiere confirmación: el usuario debe aprobar los cambios.

---

### /agileflow:context:export (Copiar a Web AI)

**Input**: Lee existente `docs/context.md`

**Output**: Extracto conciso (≤300 líneas) listo para pegar en ChatGPT, Perplexity, Gemini o Claude web

**Use Case**: Cargue el contexto del proyecto en la herramienta web de IA para continuar trabajando en una nueva conversación.

**Key Behavior**:
- Sólo lectura: no se modifican archivos
- Autónomo: Incluye instrucciones para pegar
- Listo para formatear: ya formateado para copiar y pegar fácilmente

---

### /agileflow:context:note (Agregar nota con marca de tiempo)

**Input**: argumento NOTA con nota de 1 a 5 líneas

**Output**: Se agrega a la sección "Notas" con marca de tiempo ISO

**Use Case**: Captura rápida de eventos, decisiones o descubrimientos importantes

**Key Behavior**:
- Diff-first: muestra los cambios antes de escribir
- Crea una sección si falta: crea automáticamente la sección "Notas" si es necesario
- Cronológico: las notas aparecen en orden con las más recientes en la parte inferior.
- Marca de tiempo: cada nota incluye la marca de tiempo ISO 8601.

## Mejores prácticas

### Actualizaciones periódicas del contexto

Mantenga el contexto actualizado para una mejor asistencia de IA:

1. **After major milestones**: Correr `/agileflow:context:full` para capturar el trabajo completado
2. **Before web AI sessions**: Correr `/agileflow:context:export` y pegar en ChatGPT
3. **During development**: Usar `/agileflow:context:note` para registrar decisiones y descubrimientos

```bash
# Recommended workflow
/agileflow:context:full        # Update full context
/agileflow:context:export      # Get concise version
# Paste into web AI for context-aware help
```

---

### Capture descubrimientos importantes

Registre ideas y decisiones técnicas:

```bash
/agileflow:context:note NOTE="TypeScript 5.4 improves type inference - consider upgrading"
/agileflow:context:note NOTE="Stripe API changed pagination in v2.0 - update integration"
/agileflow:context:note NOTE="Team consensus: use Postgres over MongoDB for this project"
```

---

### Uso en conversaciones web de IA

Maximice la eficacia de las herramientas web de IA proporcionando contexto:

1. Export concise context: `/agileflow:context:export`
2. Copiar la salida
3. Péguelo en ChatGPT, Perplexity o Gemini como su primer mensaje
4. Haga preguntas de seguimiento con el contexto completo del proyecto.

Esto garantiza que la IA web comprenda su código base, su arquitectura y su enfoque actual.

---

### Administrar el tamaño del archivo de contexto

Para proyectos grandes, administre el tamaño del archivo context.md:

- Usar `/agileflow:context:export` para comprobar el tamaño del contexto actual
- Archive notas antiguas si superan las 100 entradas
- Cree archivos de investigación separados para inmersiones profundas
- Usar `/agileflow:research:ask` para temas de investigación complejos

---

## Flujo de trabajo de diferenciación primero

Todos los comandos contextuales que modifican archivos siguen un flujo de trabajo consistente:

1. **Show Changes**: El comando muestra los cambios propuestos como una diferencia.
2. **Confirm**: revisas los cambios y los apruebas o rechazas
3. **Apply**: Los cambios se escriben solo después de su confirmación.

Esto garantiza que nunca perderá información importante y siempre sabrá qué se está modificando.

## Notas

- Los archivos de contexto se almacenan en `docs/context.md` como parte de la documentación de su proyecto local
- Las secciones administradas utilizan comentarios HTML como marcadores y no deben editarse manualmente
- El comando de exportación respeta el límite de 300 líneas para caber en las ventanas contextuales de la herramienta web de IA
- Las notas tienen una marca de tiempo en formato ISO 8601 para mayor coherencia y fácil clasificación.
