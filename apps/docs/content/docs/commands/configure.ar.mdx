## الميزات المتوفرة

- **sessionstart**: عرض الترحيب مع حالة المشروع عند بدء الجلسة
- **مضغوط مسبقًا**: يحافظ على السياق عند ضغط المحادثات
- **stop**: تحذير بشأن تغييرات git غير الملتزم بها قبل إنهاء الجلسة
- **الأرشفة**: أرشفة القصص القديمة المكتملة تلقائيًا (أيام قابلة للتكوين)
- **خط الحالة**: شريط الحالة المخصص الذي يعرض القصة الحالية والتقدم

## ما الذي سيتم تكوينه

### نظام الخطافات
تكوين الخطافات في `.claude/settings.json` لـ:
- SessionStart: يعرض معلومات المشروع عند بدء الجلسة
- UserPromptSubmit: يقوم بتشغيل التحقق من الصحة قبل الالتزامات
- التوقف: يحذر من التغييرات غير الملتزم بها عند إنهاء الجلسة

### نظام الأرشفة
التحديثات `docs/00-meta/agileflow-metadata.json` لتمكين الأرشفة التلقائية للقصص المكتملة الأقدم من الحد المحدد.

### خط الحالة
يقوم بتكوين شريط حالة Claude Code المخصص الذي يظهر:
- فرع بوابة الحالي
- معرف القصة النشطة
- عدد WIP (العمل قيد التقدم).
- نسبة الإنجاز

## الإخراج

بعد التكوين، سترى:

__لا تترجم_8__

قم دائمًا بإعادة تشغيل Claude Code بعد تغييرات التكوين حتى تدخل حيز التنفيذ.

##ملاحظات هامة

- **ترحيل التنسيق**: إذا رأيت تحذيرات `⚠️ INVALID FORMAT`، فقم بتشغيل `--migrate` أولاً قبل الخيارات الأخرى
- **تم إنشاء نسخة احتياطية**: يقوم خيار الترحيل تلقائيًا بإنشاء نسخة احتياطية على `.claude/settings.json.backup`
- **مطلوب إعادة التشغيل**: يجب عليك إنهاء Claude Code بالكامل وإعادة تشغيله حتى يتم تطبيق التغييرات
- **تم تحديث البيانات الوصفية**: يتم تتبع تكوين الميزة في `docs/00-meta/agileflow-metadata.json`
