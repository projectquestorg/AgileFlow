# /agileflow:pesquisa

Sistema de pesquisa abrangente para gerar solicitações de investigação detalhadas, importar resultados e gerenciar notas de pesquisa.

## Início rápido

O sistema de pesquisa fornece quatro comandos especializados:

```bash
/agileflow:research:ask TOPIC="your research topic"       # Generate research prompt
/agileflow:research:import TOPIC="title"                  # Import external results
/agileflow:research:list                                   # View all research notes
/agileflow:research:view FILE=<filename>                  # Read specific note
```

## Visão geral

O sistema de pesquisa permite:

1. **Gere solicitações de pesquisa detalhadas** para ChatGPT, Claude web, Perplexity ou Gemini
2. **Importe resultados** de ferramentas externas de IA para o seu projeto
3. **Navegar e consultar** resultados de pesquisas anteriores
4. **Organize as descobertas** com modelos e índices estruturados

A pesquisa é essencial para:
- Solução de problemas difíceis com experiência externa em IA
- Investigar tecnologias antes das decisões de arquitetura
- Documentar a aprendizagem de fontes externas
- Construindo base de conhecimento para sua equipe

## Subcomandos

### /agileflow:research:ask - Gerar prompt de pesquisa

Gere um prompt de pesquisa abrangente e rico em contexto para ferramentas de IA da web.

**Objetivo**: Cria prompts detalhados com mais de 200 linhas que incluem vocêr contexto da base de código, mensagens de erro exatas, o que você tentou e perguntas específicas.

**Comando**:
```bash
/agileflow:research:ask TOPIC="your topic" [DETAILS="constraints"] [ERROR="error message"]
```

**Parâmetros**:

| Argumento | Obrigatório | Descrição |
|----------|----------|------------|
| TÓPICO | Sim | O que você está pesquisando (por exemplo, "OAuth 2.0 com Google") |
| DETALHES | Não | Restrições, prazos ou requisitos específicos |
| ERRO | Não | Mensagem de erro exata se solução de problemas |

**Principais recursos**:

- **Gera mais de 200 linhas no mínimo** - Detalhado e abrangente
- **Inclui trechos de código reais** - Mais de 50 linhas da sua base de código
- **Incorpora erros exatos** - Rastreamentos completos da pilha em caso de solução de problemas
- **Lista de soluções tentadas** - Mostra o que você já tentou
- **Faz perguntas específicas** - Mais de 3 perguntas direcionadas para a IA
- **Somente saída** - Nenhum arquivo gravado, você copia para uma ferramenta de IA externa

**O que é gerado**:

O prompt inclui estas seções obrigatórias:

- **Contexto do Projeto** - Estrutura, dependências, arquivos relevantes
- **Implementação atual** - mais de 50 linhass de código real da sua base de código
- **O problema** - Mensagens de erro exatas, rastreamentos de pilha, comportamento esperado versus comportamento real
- **O que já tentamos** - 2+ abordagens com resultados e análises
- **Perguntas Específicas** - 3+ perguntas direcionadas para responder
- **O que eu preciso** - Análise de causa raiz, solução passo a passo, exemplos de código, pegadinhas
- **Detalhes do ambiente** - Versões e informações do sistema
- **Referências** - Links de fontes se estiver citando documentação

**Validação de qualidade**:

Antes da saída, o comando verifica:
- Comprimento ≥ 200 linhas
- Trechos de código ≥ 50 linhas
- Mensagens de erro literalmente (se aplicável)
- ≥ 2 abordagens testadas listadas
- ≥ 3 questões específicas incluídas

**Exemplo**:

```bash
# Research OAuth issue
/agileflow:research:ask TOPIC="next-auth Google OAuth unauthorized_client" ERROR="[auth][error] CallbackRouteError: unauthorized_client"

# Output: 250+ line prompt with code, error details, what was tried, and specific questions
# Copy output → ChatGPT/Claude web → Get expert analysis
# Then use /agileflow:research:import to save the response
```

### /agileflow:research:import - Importar resultados de pesquisa

Importe resultados de pesquisa de ferramentas externas de IA para sua pasta de pesquisa com structformatação adequada.

**Objetivo**: obtém resultados do ChatGPT, Claude web, Perplexity, transcrições do YouTube ou postagens de blog e os salva como arquivos markdown organizados.

**Comando**:
__NOTRADUZIR_5__

**Parâmetros**:

| Argumento | Obrigatório | Descrição |
|----------|----------|------------|
| TÓPICO | Sim | Nome do arquivo de pesquisa (por exemplo, "Configuração do OAuth 2.0") |
| CONTEÚDO | Sim* | Resultados de pesquisa para importar (*podem ser colados após o comando) |
| FONTE | Não | URL da fonte original para atribuição |

**Principais recursos**:

- **Preserva TODOS os trechos de código** - Exatamente como fornecido
- **Gera itens acionáveis** - Próximas etapas concretas, não sugestões vagas
- **Diff-first preview** - Mostrar resultado formatado antes de salvar
- **Atualiza o índice de pesquisa** - Adiciona automaticamente a docs/10-research/README.md
- **Links para decisões** - Pode fazer referência a ADRs, Epics ou Stories

**O que é criado**:

Arquivo de pesquisa estruturado com seções:

- **Resumo** - 2-3 parágrafos TL;DR
- **Principais conclusões** - Principais conclusões como bullepontos t
- **Abordagem de implementação** - Plano passo a passo, se aplicável
- **Snippets de código** - Todo o código preservado exatamente como fornecido
- **Itens de ação** - Próximas etapas concretas (caixas de seleção)
- **Riscos e pegadinhas** - Avisos e casos extremos
- **Sugestões de histórias** - Recursos potenciais se a pesquisa sugerir trabalho
- **Referência de conteúdo bruto** - Conteúdo original recolhido para referência

### /agileflow:research:view - Leia a nota de pesquisa específica

Exiba o conteúdo completo de uma nota de pesquisa da sua pasta de pesquisa.

**Objetivo**: ler e revisar notas de pesquisa para encontrar trechos de código, itens de ação e descobertas.

**Comando**:
__NOTRADUZIR_11__

**Parâmetros**:

| Argumento | Obrigatório | Descrição |
|----------|----------|------------|
| ARQUIVO | Sim | Nome do arquivo da nota de pesquisa (por exemplo, "20250101-oauth-setup.md") |

**Principais recursos**:

- **Somente leitura** - Nenhum arquivo gravado
- **Conteúdo completo** - Arquivo inteiro exibido, não truncado
- **Encontre qualquer coisa** - Trechos de código, acitens de ação, descobertas

**Exemplo**:

```bash
# View OAuth research
/agileflow:research:view FILE=20250101-oauth-setup.md

# Output: Full content of the file
```

**Saída inclui**:

- Conteúdo completo do arquivo de pesquisa
- Todos os trechos de código da pesquisa
- Itens de ação com caixas de seleção
- Principais descobertas e dicas
- Riscos e considerações
- Referências e fontes

## Melhores práticas

### Geração de prompt de pesquisa

1. **Seja específico com o TOPIC** - Não diga apenas "ajuda com autenticação", diga "next-auth 5.0com o Google OAuth"
2. **Incluir mensagens de ERRO** - Rastreamentos completos de pilha ajudam tremendamente
3. **Documente o que você tentou** - Mostra à IA que você fez o dever de casa
4. **Faça perguntas específicas** - "Por que X está acontecendo?" não "Como faço para corrigir isso?"

**Bons exemplos de TÓPICOS:**
- "erro next-auth Google OAuth não autorizado_client com Next.js 14 App Router"
- "Erros de compilação em modo estrito TypeScript na configuração do monorepo"
- "Falha na validação da assinatura do stripe webhook no desenvolvimento"

**Exemplos de TÓPICO ruins:**
- "ajuda com autenticação"
- "por que meu código está quebrado"
- "erro no meu aplicativo"

### Importação de Pesquisa

1. **Sempre cite a fonte** - De onde você tirou isso?
2. **Preservar trechos de código com exatidão** – Não modificar exemplos
3. **Extraia as principais descobertas** - Não cole apenas conteúdo bruto
4. **Gerar itens de ação** – O que você deve fazer com isso?
5. **Link para decisões** - ADRs de referência, épicos, histórias, se relevante

### Gerenciando Pesquisa

1. **Revise antes de implementar** - Não siga cegamente a pesquisa
2. **Verificark itens de ação** - Use caixas de seleção para rastrear o que você fez
3. **Mantenha os arquivos atualizados** - Atualize se souber mais
4. **Referência em decisões** - Link de ADRs e stories
5. **Arquivar pesquisas antigas** - Mover descobertas desatualizadas para o diretório de arquivo

## Quando usar a pesquisa

### Antes das decisões de arquitetura
Pesquise ferramentas e estruturas antes de recomendá-las:

```bash
/agileflow:research:ask TOPIC="PostgreSQL vs MongoDB for real-time sync"
# → Get expert comparison
/agileflow:research:import  # Import results
# → Create ADR referencing the research
```

### Investigação de recursos
Aprofunde-se em como implementar recursos complexos:

```bash
/agileflow:research:ask TOPIC="Next.js App Router middleware patterns"
/agileflow:research:ask TOPIC="OAuth 2.0 implementation with security best practices"
/agileflow:research:ask TOPIC="Real-time WebSocket architecture in Node.js"
```

### Solução de problemas difíceis
Obtenha ajuda especializada quando tiver problemas:

```bash
/agileflow:research:ask TOPIC="TypeScript strict mode in monorepo" \
  ERROR="error TS1149: File is a CommonJS module..."
```

### Aprendizagem e treinamento
Documente o que aprendeu em cursos, tutoriais ou documentação:

```bash
/agileflow:research:import TOPIC="React 18 Suspense and Streaming" \
  SOURCE="https://react.dev/reference/react/Suspense"
```

### Avaliação de Tecnologia
Compare as opções antes da adoção:

```bash
/agileflow:research:ask TOPIC="GraphQL vs tRPC for type-safe APIs"
/agileflow:research:ask TOPIC="SvelteKit vs Next.js for new project"
```

## Troubleshooting

### Research Prompt Too Short

If you get a prompt under 200 lines, the command will regenerate. Make sure:
- TOPIC is specific (include framework versions, error types)
- ERROR includes full stack trace if troubleshooting
- Command reads your actual codebase

### Import Not Updating Index

The index (`docs/10-research/README.md`) is automatically created and updated. If it's missing:

```bash
/agileflow:research:list  # This will create the index if missing
```

### Can't Find Research File

Use `/agileflow:research:list` to see all available files:

```bash
/agileflow:research:list
# Shows all files with correct filenames for /agileflow:research:view
```

### Research Folder Doesn't Exist

First research import or list command creates the `docs/10-research/` directory automatically. No manual setup needed.
