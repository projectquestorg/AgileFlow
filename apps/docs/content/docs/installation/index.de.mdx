# Installation

AgileFlow wird als npm-Paket verteilt, das direkt in Ihrem Projekt installiert wird und Slash-Befehle, Agenten und Projektstruktur hinzufügt.

## Voraussetzungen

- **Node.js** 18,0 oder höher
- **Claude Code**, **Cursor**, **Windsurf**, oder **OpenAI Codex CLI**
- Ein Git-Repository für Ihr Projekt

## Schnelle Installation

```bash
npx agileflow@latest setup
```

Das ist es! Der `npx` Der Befehl ruft immer die neueste Version ab, sodass Sie sich keine Sorgen über veraltete Installationen machen müssen.

### Aktualisierungen

```bash
npx agileflow@latest update
```

## Nach dem Setup

```bash
/agileflow:help              # View all commands
/agileflow:configure         # Configure hooks, status line, etc.
/agileflow:babysit           # Interactive mentor for implementation
```

### Installierte Befehle

Alle Befehle werden verfügbar als `/agileflow:command-name` in Claude Code:

| Command | Description |
|---------|-------------|
| `/agileflow:epic` | Create a new epic |
| `/agileflow:story` | Create a user story |
| `/agileflow:status` | Update story status |
| `/agileflow:board` | View kanban board |
| `/agileflow:babysit` | Start guided implementation |

Siehe die [vollständige Befehlsreferenz](/commands) für alle 72 Befehle.

## Überprüfen Sie die Installation

Überprüfen Sie nach der Installation, ob alles richtig eingerichtet ist:

```bash
# In Claude Code, run:
/agileflow:help
```

Sie sollten das AgileFlow-Hilfemenü mit allen verfügbaren Befehlen sehen.

## IDE-Unterstützung

AgileFlow funktioniert mit mehreren KI-gestützten IDEs:

| IDE | Status | Config Location |
|-----|--------|-----------------|
| Claude Code | Supported | `.claude/commands/agileflow/` |
| Cursor | Supported | `.cursor/commands/agileflow/` |
| Windsurf | Supported | `.windsurf/workflows/agileflow/` |
| OpenAI Codex CLI | Supported | `.codex/skills/` and `~/.codex/prompts/` |

### IDE-spezifisches Setup

```bash
# Claude Code (default)
agileflow setup

# Cursor
agileflow setup --ide cursor

# Windsurf
agileflow setup --ide windsurf

# OpenAI Codex CLI
agileflow setup --ide codex
```

## Konfiguration

Nach der Installation können Sie das AgileFlow-Verhalten anpassen:

```bash
/agileflow:configure
```

Mit diesem interaktiven Befehl können Sie:

- Funktionen aktivieren/deaktivieren
- Legen Sie Standardbesitzer für Storys fest
- Konfigurieren Sie Hooks und Automatisierung
- Passen Sie die Statuszeilenanzeige an

Sehen [Konfiguration](/configuration) für alle Optionen.

## Aktualisierung

So aktualisieren Sie AgileFlow auf die neueste Version:

```bash
npx agileflow@latest setup
```

Ihre bestehenden Projektdaten (Epics, Stories, Status) bleiben erhalten.

## Deinstallation

So entfernen Sie AgileFlow aus Ihrem Projekt:

```bash
# Remove AgileFlow directories
rm -rf .agileflow
rm -rf .claude/commands/agileflow

# Optionally remove docs (contains your project data)
rm -rf docs
```

## Fehlerbehebung

### Befehle werden nicht angezeigt

Stellen Sie sicher, dass Sie Claude Code in Ihrem Projektstammverzeichnis ausführen, in dem AgileFlow installiert wurde.

### Berechtigungsfehler

Stellen Sie sicher, dass Sie Schreibzugriff auf das Projektverzeichnis haben:

```bash
chmod -R u+w .agileflow .claude docs
```

### Probleme mit der Knotenversion

AgileFlow erfordert Node.js 18+:

```bash
node --version  # Should be v18.0.0 or higher
```

## Nächste Schritte

Nachdem AgileFlow nun installiert ist:

1. **Create your first epic**: `/agileflow:epic EPIC=EP-0001 TITLE="My Feature"`
2. **Break it into stories**: `/agileflow:story EPIC=EP-0001 STORY=US-0001 TITLE="First task"`
3. **Start working**: `/agileflow:babysit`

Verwenden `/agileflow:help` um alle verfügbaren Befehle anzuzeigen.
