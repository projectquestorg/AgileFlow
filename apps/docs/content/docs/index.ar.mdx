# التدفق الرشيق

AgileFlow هو **إطار عمل لإدارة المشاريع مدعوم بالذكاء الاصطناعي** مصمم خصيصًا لـ [Claude Code](https://claude.ai/code). فهو يوفر سير عمل رشيق منظم ووكلاء أذكياء وإدارة مشروع المستندات كتعليمات برمجية مباشرة في بيئة التطوير الخاصة بك.

## لماذا AgileFlow؟

أدوات إدارة المشاريع التقليدية تخلق احتكاكًا بين التخطيط والتنفيذ. يعمل AgileFlow على القضاء على هذه الفجوة عن طريق دمج إدارة المشروع مباشرة في سير عمل البرمجة بمساعدة الذكاء الاصطناعي.

- **لا يوجد تبديل للسياق** - قم بإدارة الملاحم والقصص والحالة دون مغادرة جهازك
- ** سير عمل الذكاء الاصطناعي الأصلي ** - مصمم خصيصًا لإمكانيات Claude Code
- **Docs-as-code** - جميع عناصر المشروع موجودة في مستودعك
- **وكلاء أذكياء** - وكلاء ذكاء اصطناعي متخصصون في مجالات مختلفة

## الميزات الرئيسية

### 74 أمر القطع

يوفر AgileFlow مكتبة أوامر شاملة لكل جانب من جوانب إدارة المشروع:

| الفئة | الأوامر |
|----------|---------|
| **التخطيط** | __لاترجمةTE_2__، `/story`، `/sprint`، `/deps` |
| **التتبع** | `/status`، `/board`، ​​`/blockers`، `/metrics` |
| **تنمية** | `/babysit`, `/verify`, `/review`, `/pr` |
| **العمليات** | `/ci`, `/deploy`, `/changelog`, `/packages` |
| **أبحاث** | `/research:ask`, `/research:import`, `/context:full`, `/context:export`, `/multi-expert` |

### 30 وكيلًا متخصصًا

كل وكيل هو خبير في مجاله، وعلى استعداد للمساعدة في مهام محددة:

- **المرشد** - إرشادات التنفيذ الشاملة
- **مخطط ملحمي** - قم بتقسيم الميزات إلى قصص
- **قاعدة البيانات** - تصميم المخطط والترحيل
- **api** - خدمات الواجهة الخلفية وطبقة البيانات
- **ui** - مكونات الواجهة الأمامية وتصميمها
- **الأمان** - تحليل نقاط الضعف والامتثال
- **الاختبار** - استراتيجية الاختبار والتغطية
- **devops** - CI/CD والنشر

## كيف يعمل

1. **تثبيت AgileFlow** - يضيف أمر واحد جميع الإمكانات إلى مشروعك
2. **التخطيط باستخدام الذكاء الاصطناعي** - استخدم الوكلاء لتقسيم الميزات إلى قصص قابلة للتنفيذ
3. **تتبع التقدم** - تحديثات الحالة في الوقت الفعلي في `status.json`
4. **التعاون** - يتواصل الوكلاء عبر ناقل الرسائل
5. **السفينة** - إنشاء العلاقات العامة وسجل التغيير تلقائيًاupdates

## Philosophy

AgileFlow is built on these principles:

- **Convention over configuration** - Sensible defaults that just work
- **Transparency** - All data is plain text, version-controlled
- **Composability** - Use only what you need
- **AI-first** - Designed for human-AI collaboration
