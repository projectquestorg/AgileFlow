# Agentes

AgileFlow incluye **111 specialized agents**, cada uno experto en su campo. Los agentes pueden ser generados por el `/babysit` mentor o invocado directamente usando la herramienta Tarea.

## Cómo trabajan los agentes

1. **Expertise-First**: Cada agente carga su archivo de experiencia para comprender los patrones de su código base
2. **Context-Aware**: Los agentes leen los documentos del proyecto, status.json y cambios recientes.
3. **Context Preservation**: Los agentes mejorados utilizan la configuración compact_context para preservar el conocimiento crítico durante la compactación de la conversación.
4. **Collaborative**: Los agentes se comunican a través del bus de mensajes y actualizan el estado de la historia.
5. **Self-Improving**: Los agentes actualizan su experiencia después de completar el trabajo.

## Característica de contexto compacto

Para mantener el enfoque durante conversaciones largas, los agentes de AgileFlow utilizan **compact_context** configuración para preservar información crítica cuando el contexto se compacta automáticamente. Esto garantiza que los agentes no pierdan reglas, estados y prioridades importantes en mitad de la conversación.

### Cómo funciona

Cada agente tiene un `compact_context` Configuración con tres componentes:

**Priority Level**: Determina con qué agresividad se preserva el contexto del agente.
- `critical` - Siempre preservado (agentes de mayor prioridad como mentor, api, ui)
- `high` - Conservado en la mayoría de compactaciones (agentes importantes)
- `medium` - Conservado cuando sea posible (agentes estándar)
- `low` - Primero en ser eliminado (raro, usado para agentes asesores)

**Preserve Rules**: Reglas básicas de comportamiento que nunca deben perderse
- Identidad y rol del agente
- Restricciones críticas y reglas de seguridad.
- Procedimientos que se deben seguir (por ejemplo, "Cargar experiencia primero")
- Requisitos de gestión estatal.
- Protocolos de coordinación

**State Fields**: Estado dinámico que cambia durante el trabajo.
- Historia actual en implementación (EE.UU.-#### ID)
- Progreso de la tarea y porcentaje de finalización
- Bloqueadores y dependencias conocidos
- Próximas acciones priorizadas
- Estado de prueba/compilación

### Configuración de ejemplo

```yaml
compact_context:
  priority: critical
  preserve_rules:
    - "LOAD EXPERTISE FIRST: Read packages/cli/src/core/experts/api/expertise.yaml"
    - "VERIFY TEST BASELINE: Check test_status before starting"
    - "PRIORITIZE AG-UI BLOCKERS: Unblock UI stories waiting on API endpoints"
    - "DIFF-FIRST APPROACH: Show edits with confirmation before applying"
  state_fields:
    - current_story
    - endpoints_implemented
    - blocked_ui_stories
    - test_status_baseline
```

Cuando Claude Code comprime el contexto, preserva estas reglas y campos de estado para todos los agentes, asegurando la continuidad incluso después de múltiples rondas de compactación de conversaciones.

### Distribución de prioridad del agente

- **Critical** (4 agentes): mentor, api
- **High** (12 agentes): interfaz de usuario, base de datos, ci, devops, pruebas, seguridad, documentación, rendimiento, consulta de código base
- **Medium** (30 agentes): migración de datos, control de calidad, accesibilidad, monitoreo, investigación, adr-writer, análisis, integraciones, dispositivos móviles, diseño, cumplimiento, refactor, producto, readme-updater
- **Low** (10 agentes): planificador épico, orquestrador

Este enfoque escalonado garantiza que los agentes más críticos siempre mantengan el foco mientras que otros degradan elegantemente el contexto cuando es necesario.

## Agentes reproductores

### Vía /babysit (recomendado)

El `/babysit` El mentor detecta automáticamente qué agente se necesita:

```bash
/agileflow:babysit
# Then describe your task - babysit will spawn the right agent
```

### Desove directo

También puedes generar agentes directamente usando la herramienta Tarea:

```text
Task(
  description: "Design user sessions schema",
  prompt: "Create a sessions table to track user logins with proper indexes",
  subagent_type: "agileflow-database"
)
```

### Agentes paralelos

Para funciones de pila completa, genere varios agentes simultáneamente:

```text
# API work
Task(subagent_type: "agileflow-api", run_in_background: true, ...)

# UI work (runs in parallel)
Task(subagent_type: "agileflow-ui", run_in_background: true, ...)
```

## Comunicación del agente

Los agentes se coordinan a través de `docs/09-agents/`:

  </Folder>

### Actualizaciones de estado

Actualización de agentes `status.json` mientras trabajan:

```json
{
  "stories": {
    "US-0042": {
      "status": "in-progress",
      "owner": "AG-DATABASE",
      "summary": "Created sessions table with indexes"
    }
  }
}
```

### Autobús de mensajes

Los agentes registran mensajes de coordinación:

```json
{"timestamp": "2025-12-24T10:00:00Z", "type": "assign", "story": "US-0042", "owner": "AG-DATABASE"}
{"timestamp": "2025-12-24T11:00:00Z", "type": "handoff", "story": "US-0042", "from": "AG-DATABASE", "to": "AG-API"}
```

## Mejores prácticas

1. **Let babysit choose**: El mentor sabe qué agente encaja mejor
2. **Provide context**: informa a los agentes sobre los patrones existentes en tu código base
3. **Review updates**: Verifique los cambios que hacen los agentes en status.json
4. **Use multi-expert for uncertainty**: Cuando necesite mucha confianza, implemente varios expertos
