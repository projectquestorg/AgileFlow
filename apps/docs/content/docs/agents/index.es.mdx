# Agentes

AgileFlow incluye **30 agentes especializados**, cada uno de ellos experto en su dominio. Los agentes pueden ser generados por el mentor `/babysit` o invocados directamente usando la herramienta Tarea.

## Cómo trabajan los agentes

1. **Experiencia primero**: cada agente carga su archivo de experiencia para comprender los patrones de su código base.
2. **Context-Aware**: los agentes leen los documentos del proyecto, status.json y cambios recientes.
3. **Preservación del contexto**: los agentes mejorados utilizan la configuración compact_context para preservar el conocimiento crítico durante la compactación de la conversación.
4. **Colaborativo**: los agentes se comunican a través del bus de mensajes y actualizan el estado de la historia.
5. **Superación personal**: los agentes actualizan su experiencia después de completar el trabajo

## Función de contexto compacto

Para mantener el enfoque durante conversaciones largas, los agentes de AgileFlow utilizan la configuración **compact_context** para preservar información crítica cuando el contexto se compacta automáticamente. Esto garantiza que los agentes no pierdan reglas, estados y prioridades importantes en mitad de la conversación.

### Cómo funciona

cada unoEl agente tiene una configuración `compact_context` con tres componentes:

**Nivel de prioridad**: determina con qué agresividad se preserva el contexto del agente.
- `critical` - Siempre preservado (agentes de mayor prioridad como mentor, api, ui)
- `high` - Preservado en la mayoría de compactaciones (agentes importantes)
- `medium` - Conservado cuando sea posible (agentes estándar)
- `low` - Primero en ser eliminado (raro, usado para agentes asesores)

**Preservar reglas**: reglas de comportamiento fundamentales que nunca deben perderse
- Identidad y rol del agente.
- Restricciones críticas y reglas de seguridad.
- Procedimientos obligatorios (por ejemplo, "Cargar experiencia primero")
- Requisitos de gestión estatal.
- Protocolos de coordinación

**Campos de estado**: estado dinámico que cambia durante el trabajo
- Historia actual en implementación (US-#### ID)
- Progreso de la tarea y porcentaje de finalización.
- Bloqueadores y dependencias conocidos.
- Próximas acciones priorizadas
- Estado de prueba/construcción

### Ejemplo de configuración

```yaml
compact_context:
  priority: critical
  preserve_rules:
    - "LOAD EXPERTISE FIRST: Read packages/cli/src/core/experts/api/expertise.yaml"
    - "VERIFY TEST BASELINE: Check test_status before starting"
    - "PRIORITIZE AG-UI BLOCKERS: Unblock UI stories waiting on API endpoints"
    - "DIFF-FIRST APPROACH: Show edits with confirmation before applying"
  state_fields:
    - current_story
    - endpoints_implemented
    - blocked_ui_stories
    - test_status_baseline
```

Cuando Claude Code se comprime contexto, preserva estas reglas y campos de estado para todos los agentes, asegurando la continuidad incluso después de múltiples rondas de compactación de la conversación.

### Distribución de prioridad del agente

- **Crítico** (2 agentes): mentor, api
- **Alto** (9 agentes): interfaz de usuario, base de datos, ci, devops, pruebas, seguridad, documentación, rendimiento, consulta de código base
- **Medio** (15 agentes): migración de datos, control de calidad, accesibilidad, monitoreo, investigación, adr-writer, análisis, integraciones, dispositivos móviles, diseño, cumplimiento, refactor, producto, actualizador de archivos Léame
- **Bajo** (2 agentes): planificador épico, orquestador

Este enfoque escalonado garantiza que los agentes más críticos siempre mantengan el foco mientras que otros degradan elegantemente el contexto cuando es necesario.

## Agentes reproductores

### Vía /babysit (recomendado)

El mentor `/babysit` detecta automáticamente qué agente se necesita:

```bash
/agileflow:babysit
# Then describe your task - babysit will spawn the right agent
```

### Desove directo

También puedes spawn agentes directamente usando la herramienta Tarea:

```text
Task(
  description: "Design user sessions schema",
  prompt: "Create a sessions table to track user logins with proper indexes",
  subagent_type: "agileflow-database"
)
```

### Agentes paralelos

Para funciones de pila completa, genere varios agentes simultáneamente:

```text
# API work
Task(subagent_type: "agileflow-api", run_in_background: true, ...)

# UI work (runs in parallel)
Task(subagent_type: "agileflow-ui", run_in_background: true, ...)
```

### Actualizaciones de estado

Los agentes actualizan `status.json` mientras trabajan:

```json
{
  "stories": {
    "US-0042": {
      "status": "in-progress",
      "owner": "AG-DATABASE",
      "summary": "Created sessions table with indexes"
    }
  }
}
```

### Autobús de mensajes

Los agentes registran mensajes de coordinación:

```json
{"timestamp": "2025-12-24T10:00:00Z", "type": "assign", "story": "US-0042", "owner": "AG-DATABASE"}
{"timestamp": "2025-12-24T11:00:00Z", "type": "handoff", "story": "US-0042", "from": "AG-DATABASE", "to": "AG-API"}
```

## Mejores prácticas

1. **Deja que el niñero elija**: el mentor sabe qué agente se adapta mejor
2. **Proporcione contexto**: informe a los agentes sobre los patrones existentes en su código base
3. **Revisar actualizaciones**: Verifique los cambios que hacen los agentes en status.json
4. **Utilice varios expertos en caso de incertidumbre**: cuando necesite un alto nivel de confianza, implemente varios expertos
