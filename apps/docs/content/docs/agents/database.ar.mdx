# وكيل قاعدة البيانات

وكيل قاعدة البيانات (AG-DATABASE) هو متخصصك في تصميم المخطط وتحسين الاستعلام والترحيل وإدارة قاعدة البيانات والموثوقية والأداء.

## القدرات

- تصميم المخطط والعلاقات
- تحسين الاستعلام ومنع مشاكل N+1
- كتابة والتحقق من الترحيلات
- إعداد النسخ الاحتياطية والاسترجاع
- مراقبة الأداء والنمو
- معالجة البيانات الكبيرة والأرشيفة
- الحفاظ على سلامة البيانات والقيود

## متى تستخدم

استخدم وكيل قاعدة البيانات عندما:

- تصميم مخطط قاعدة البيانات لميزة جديدة
- تحسين الاستعلامات البطيئة
- كتابة ترحيلات قاعدة البيانات
- إعداد النسخ الاحتياطية والاسترجاع
- تحجيم قاعدة البيانات لبيانات متزايدة
- إصلاح مشاكل الأداء
- التخطيط للأرشيفة والحذف

## مثال

```bash
/agileflow:babysit
> "تصميم قاعدة بيانات للمستخدمين والتعليقات"

# AG-DATABASE ستقوم بـ:
# 1. تصميم جداول المستخدمين والتعليقات
# 2. إضافة الفهارس للأداء
# 3. تعريف القيود والعلاقات
# 4. كتابة ترحيل قاعدة البيانات
# 5. توثيق المخطط
```

## السلوكيات الرئيسية

- **تحسين أولاً**: فهارس صحيحة ولا مشاكل N+1
- **سلامة البيانات**: قيود وعلاقات قوية
- **الترحيل الآمن**: بدون فقدان البيانات
- **الأداء**: استعلامات سريعة وفعالة
- **الموثوقية**: نسخ احتياطية منتظمة

## الأدوات المتاحة

Read, Write, Edit, Bash, Glob, Grep
