# Agent d'assurance qualité

L'agent QA (AG-QA) est un spécialiste de l'assurance qualité qui garantit que les fonctionnalités répondent aux normes de qualité avant leur publication. Cet agent se concentre sur la stratégie de test, les tests de régression, les mesures de qualité et la préparation à la publication, en complément de l'agent de test qui conçoit les implémentations de tests.

## Capacités

- **Développement d'une stratégie de test** : créez des plans de test complets avec la portée, les objectifs de couverture et les critères de réussite
- **Tests de régression** : planifiez et gérez les suites de tests de régression pour les versions
- **Metriques de qualité** : Définir et suivre les KPI (couverture du code, taux d'échappement des bogues, densité des défauts)
- **Préparation à la version** : créer et valider des listes de contrôle de version et des procédures d'approbation
- **Gestion des cas de test** : organisez et gérez les cas de tests manuels
- **Triage des bogues** : évaluez la gravité et priorisez la résolution des problèmes
- **Planification UAT** : coordonner les tests d'acceptation des utilisateurs avec les parties prenantes
- **Quality Gates** : définir des points de contrôle de qualité automatisés dans le pipeline CI/CD

## Quand l'utiliser

Utilisez l'agent d'assurance qualitét quand :

- Planification d'une nouvelle version et besoin de critères de qualité
- Nécessité de définir une stratégie de tests avec des objectifs de couverture
- Création de cas de tests pour les workflows utilisateurs critiques
- Mise en place de tests de régression pour les releases
- Trier les bugs et évaluer leur gravité
- Planification des tests d'acceptation utilisateur (UAT)
- Nécessité d'établir des portes de qualité et des critères de sortie
- Création de listes de contrôle de préparation à la publication

## Comment ça marche

1. **Chargement des connaissances** : l'agent lit l'expertise et les normes de qualité du projet
2. **Analyse des fonctionnalités** : l'agent examine l'histoire pour les besoins de test
3. **Stratégie de test** : l'agent conçoit un plan de test avec des objectifs de portée et de couverture
4. **Planification des tests** : l'agent crée des cas de test et une portée de régression
5. **Quality Gates** : l'agent définit les mesures et les seuils de qualité
6. **Préparation à la publication** : l'agent crée des procédures d'approbation et des listes de contrôle
7. **Coordination** : l'agent s'aligne sur les équipes de test et de développement
8. **Vérification** : L'agent s'assure que les critères de qualité sont respectés
9. **Documentation** : l'agent met à jour le statut.jfils et rapports de qualité

## Comportements clés

- **La qualité d'abord** : ne publiez jamais sans répondre aux critères de qualité
- **Test Early** : les tests ont lieu tout au long du développement, pas seulement à la fin
- **Basé sur les risques** : les zones à risque plus élevé font l'objet de davantage d'attention lors des tests.
- **Metrics-Driven** : décisions de qualité étayées par des données et des KPI
- **Critères clairs** : Critères de sortie documentés et mesurables
- **Axé sur la régression** : les tests de régression automatisés évitent les bogues connus
- **User Sign-Off** : UAT valide les attentes des utilisateurs avant la publication

## Cadre de mesures de qualité

**Mesures de couverture** :
- Couverture du code : >80% unité, >60% intégration, >30% E2E
- Couverture des fonctionnalités : % de fonctionnalités testées
- Couverture des exigences : % des exigences testées

**Mesures de bogues** :
- Bugs trouvés : nombre total
- Taux d'évasion de bogues : % de bogues trouvés en production par rapport aux tests (target : moins de 2 %)
- Densité de défauts : Bugs pour 1000 lignes de code (cible : inférieure à 2,5)

**KPI de qualité** :
- Taux de réussite aux tests : >95 %
- Temps moyen de résolution : délai moyen entre le rapport et la résolution
- Taux de régression : % de régressions dans les sorties

## Liste de contrôle de préparation à la publication

**Doit avoir avant la sortie** :
- [ ] Révision du code terminée (couverture à 100 %)
- [ ] Tous les tests automatisés réussissent
- [ ] Bugs critiques résolus
- [ ] Référence de performance atteinte
- [ ] Accessibilité vérifiée (WCAG AA)
- [ ] Examen de sécurité réussi
- [ ] Documentation mise à jour
- [ ] Approbation UAT obtenue

**Aurait dû avant la sortie** :
- [ ] Bugs hautement prioritaires résolus
- [ ] Performances optimisées
- [ ] Test de charge terminé
- [ ] Mise en place du monitoring

**Agréable à avoir avant la sortie** :
- [ ] Bugs de priorité moyenne résolus
- [ ] Benchmarks de performance publiés
- [ ] Communication utilisateur rédigée

## Niveaux de gravité des bogues

| Gravité | Descriptif | Exemple | Priorité |
|--------------|-------------|---------|----------|
| Critique | Fonctionnalité inutilisable, ouiperte de ta, faille de sécurité | Les utilisateurs ne peuvent pas se connecter | Réparer immédiatement |
| Élevé | Fonctionnalité considérablement altérée | Le paiement échoue 50 % du temps | Réparer dès que possible |
| Moyen | La fonctionnalité fonctionne avec des limitations | Email retardé de 2 heures | Calendrier proche futur |
| Faible | Problème mineur, solution de contournement simple | Texte du bouton mal aligné | Carnet de commandes |

## Gestion des cas de tests

**Modèle de cas de test** :
```text
Test Case ID: TC-001
Title: User can log in with valid credentials
Feature: Authentication

Preconditions:
- User has registered account
- Login page accessible

Test Steps:
1. Navigate to login page
2. Enter email address
3. Enter password
4. Click "Sign In"

Expected Result:
- Dashboard loads in <2 seconds
- User data displayed correctly
- Welcome message shown

Status: ☐ Pass ☐ Fail ☐ Blocked
```

## Stratégie de tests de régression

**Quels tests de régression** :
- Flux de travail des utilisateurs de base (connexion, inscription, fonctionnalités critiques)
- Fonctionnalités modifiées (affectées par cette version)
- Fonctionnalités associées (dépendances)
- Chemins critiques (fonctionnalités très utilisées)
- Domaines sensibles aux performances
- Fonctionnalités sensibles à la sécurité

**Exemple de portée de régression** :
```text
Feature: User Login
├── Happy path: Valid credentials
├── Invalid credentials: Wrong password
├── Inactive account: Suspended user
├── Password reset flow
├── Login rate limiting
├── Session management
└── Related features:
    ├── Signup (account creation)
    ├── Password change
    └── Session timeout
```

## Tests d'acceptation des utilisateurs (UAT)

**Préparation UAT** :
- Identifier les parties prenantes et les utilisateurs
- Créer des scénarios de tests à partir de user stories
- Préparer un environnement de test de type production
- Préparer des données de test réalistes
- Planifier des sessions UAT

**Approbation UAT** :
```text
UAT Sign-Off Form

Project: [Project Name]
Version: [Version]
Date: [Date]

Tested By: [Name, Title]
Approved By: [Name, Title]

Overall Result: ☐ Approved ☐ With Conditions ☐ Rejected

Issues Found: [List]

Signature: ________________
```

## QualitéPortes

**Code Quality Gate** (avant la fusion) :
- Révision du code approuvée
- Le peluchage est passé
- Vérification de type réussie
- Réussite des tests unitaires (couverture >80%)
- Aucune faille de sécurité

**Integration Quality Gate** (avant la préparation) :
- Réussite des tests d'intégration
- Réussite des tests E2E
- Performance dans les limites de la référence
- Migrations de bases de données validées
- Contrats API validés

**Publier Quality Gate** (avant production) :
- Tests de régression réussis
- Approbation UAT obtenue
- Bugs critiques résolus
- Critères de performance respectés
- Examen de sécurité approuvé

## Principes de qualité

- **La qualité n'est pas négociable** : ne publiez jamais en dessous des normes de qualité
- **Mesurez tout** : décisions de qualité étayées par des métriques et des données
- **Tester tôt** : déplacer les tests vers le développement
- **Approche basée sur les risques** : tester de manière plus approfondie les zones à haut risque
- **Amélioration continue** : suivez les métriques et améliorez-les au fil du temps
