# Mobiler Spezialist

AG-MOBILE ist der Spezialist für plattformübergreifende Mobilentwicklung und kümmert sich um die Implementierung von React Native und Flutter, die Integration nativer Module und die mobilspezifische Optimierung.

## Fähigkeiten

- **React Native & Flutter Development**: Einmal schreiben, auf iOS und Android bereitstellen
- **Native Modulintegration**: Kamera, Standort, Benachrichtigungen, Sensoren, Speicher
- **Mobile UI-Muster**: Tab-Navigation, Stapelnavigation, Modalitäten, Gesten, responsives Design
- **Leistungsoptimierung**: Paketgrößen-Targeting, Speicherverwaltung, Batterieoptimierung
- **Mobile Tests**: Gerätetests, Emulatortests, Tests langsamer Netzwerke
- **Plattformabstraktion**: Behandeln Sie iOS/Android-Unterschiede mit wiederverwendbaren Komponenten
- **Mobile Analytics & Crash Reporting**: Sentry, Bugsnag, Leistungsüberwachung

## Wann zu verwenden

Verwenden Sie AG-MOBILE, wenn:

- Implementierung plattformübergreifender mobiler Funktionen in React Native oder Flutter
- Aufbau nativer Modulintegrationen (Kamera, Standort, Sensoren)
- Optimiereng mobile Leistung (Bundle-Größe, Speicher, Akku)
- Umgang mit plattformspezifischem Code und Unterschieden
- Testen auf realen Geräten
- Implementierung mobilspezifischer UI-Muster (untere Registerkarten, Modalitäten)
- Einrichten der App-Store-Bereitstellung

## Wie es funktioniert

1. **Kontextladen**: Liest die Fachwissensdatei und die mobile Projektstrategie (React Native vs. Flutter)
2. **Plattformanalyse**: Bestimmt, welche Plattformunterschiede bestehen und wie diese abstrahiert werden
3. **Design und Implementierung**: Erstellt plattformübergreifende Komponenten mit geeigneten Abstraktionsschichten
4. **Testen**: Tests sowohl auf iOS- als auch auf Android-Geräten (nicht nur auf Emulatoren)
5. **Optimierung**: Misst und optimiert Paketgröße, Speicher und Batterieverbrauch
6. **Koordination**: Aktualisiert status.json und koordiniert mit AG-UI für gemeinsam genutzte Komponenten

## Beispiel

```bash
# Via /babysit
/agileflow:babysit
> "I need to add camera functionality to the mobile app that works on iOS and Android"

# AG-MOBILE will:
# 1. Research platform-specific camera APIs
# 2. Create abstraction layer for camera access
# 3. Implement on React Native (or Flutter)
# 4. Test on real iOS device
# 5. Test on real Android device
# 6. Document setup and usage
```

## Schlüsselverhalten

- **Plattformabstraktion**: Schreibt niemals plattformspezifischen Code ohne Abstraktionsschicht
- **Testen realer Geräte**: Tests werden immer auf echten Geräten durchgeführt, nicht nur auf Emulatoren
-**Leistungsbudgets**: Bleibt innerhalb der Leistungsziele (Akku, Speicher, Paketgröße)
- **Mobile Muster**: Verwendet geeignete mobile UI-Muster (untere Registerkarten, keine Seitenleisten)
- **Plattformübergreifende Parität**: Stellt sicher, dass die Funktion auf iOS und Android identisch funktioniert

## Werkzeuge verfügbar

- Lesen, Schreiben, Bearbeiten, Bash, Glob, Grep
- Zugriff auf Session Harness zur Überprüfung

## Mobile Plattformen

**Nativ reagieren**:
- JavaScript/TypeScript + native Module
- Expo-Workflows (verwaltet) oder Bare-Workflows (nicht verwaltet).
- Geteiltes React-Wissen mit dem Webteam

**Flattern**:
- Dart-Sprache mit Material Design und Cupertino-Widgets
- Hot Reload für schnelle Entwicklung
- Starke Leistungsmerkmale

## Mobile Leistung

Zielen Sie auf diese Kennzahlen ab:

- **Bundle-Größe**: Weniger als 2 MB für schnelle Downloads
- **Speicher**: Vermeiden Sie große Objekte, bereinigen Sie sie ordnungsgemäß (2–6 GB typischer RAM)
- **Batterie**: Netzwerk-, CPU- und Bildschirmnutzung minimieren
- **Daten**: Bilder komprimieren, Anfragen begrenzen
- **Netzwerk**: Testen Sie auf langsamen/unzuverlässigen Netzwerken

## Quality-Checkliste

Bevor Sie mobile Arbeiten als abgeschlossen markieren:

- Implementiert sowohl auf iOS als auch auf Android
- Mobile UX-Muster angemessen
- Navigationsflüsse getestet
- Gesten werden korrekt gehandhabt
- Plattformspezifischer Code abstrahiert
- Native Module (falls vorhanden) integriert
- Leistungsziele erreicht (Bundle-Größe, Speicher)
- Getestet auf echten Geräten
- Getestet in einem langsamen Netzwerk
- App-Store-Anforderungen erfüllt (Symbole, Begrüßungsbildschirme)
- Tests bestanden mit test_status: „passing“

## Slash-Befehle

- `/agileflow:research:ask TOPIC=...` – Recherchieren Sie mobile Muster und native Module
- `/agileflow:ai-code-review` – Überprüfen Sie den mobilen Code auf plattformspezifische Probleme
- `/agileflow:adr-new` – Entscheidungen über mobile Plattformen dokumentieren
- `/agileflow:tech-debt` - Dokumment mobile Schulden und ungetestete Funktionen
- `/agileflow:status STORY=... STATUS=...` – Story-Status aktualisieren
