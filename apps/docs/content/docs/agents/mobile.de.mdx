# Mobiler Spezialist

AG-MOBILE ist der Spezialist für plattformübergreifende Mobilentwicklung und kümmert sich um die Implementierung von React Native und Flutter, die Integration nativer Module und die mobilspezifische Optimierung.

## Fähigkeiten

- **React Native & Flutter Development**: Einmal schreiben, auf iOS und Android bereitstellen
- **Native Module Integration**: Kamera, Standort, Benachrichtigungen, Sensoren, Speicher
- **Mobile UI Patterns**: Tab-Navigation, Stapelnavigation, Modalitäten, Gesten, responsives Design
- **Performance Optimization**: Bündelgrößen-Targeting, Speicherverwaltung, Batterieoptimierung
- **Mobile Testing**: Gerätetests, Emulatortests, langsame Netzwerktests
- **Platform Abstraction**: Behandeln Sie iOS/Android-Unterschiede mit wiederverwendbaren Komponenten
- **Mobile Analytics & Crash Reporting**: Sentry, Bugsnag, Leistungsüberwachung

## Wann zu verwenden

Verwenden Sie AG-MOBILE, wenn:

- Implementierung plattformübergreifender mobiler Funktionen in React Native oder Flutter
- Aufbau nativer Modulintegrationen (Kamera, Standort, Sensoren)
- Optimierung der mobilen Leistung (Bundle-Größe, Speicher, Akku)
- Umgang mit plattformspezifischem Code und Unterschieden
- Testen auf realen Geräten
- Implementierung mobilspezifischer UI-Muster (untere Registerkarten, Modalitäten)
- Einrichten der App-Store-Bereitstellung

## Wie es funktioniert

1. **Context Loading**: Liest Fachwissensdatei und mobile Projektstrategie (React Native vs Flutter)
2. **Platform Analysis**: Bestimmt, welche Plattformunterschiede bestehen und wie diese abstrahiert werden
3. **Design & Implementation**: Erstellt plattformübergreifende Komponenten mit geeigneten Abstraktionsschichten
4. **Testing**: Tests sowohl auf iOS- als auch auf Android-Geräten (nicht nur auf Emulatoren)
5. **Optimization**: Misst und optimiert Bündelgröße, Speicher und Batterieverbrauch
6. **Coordination**: Aktualisiert status.json und stimmt sich mit AG-UI für gemeinsam genutzte Komponenten ab

## Beispiel

```bash
# Via /babysit
/agileflow:babysit
> "I need to add camera functionality to the mobile app that works on iOS and Android"

# AG-MOBILE will:
# 1. Research platform-specific camera APIs
# 2. Create abstraction layer for camera access
# 3. Implement on React Native (or Flutter)
# 4. Test on real iOS device
# 5. Test on real Android device
# 6. Document setup and usage
```

## Schlüsselverhalten

- **Platform Abstraction**: Schreibt niemals plattformspezifischen Code ohne Abstraktionsschicht
- **Real Device Testing**: Immer auf echten Geräten testen, nicht nur auf Emulatoren
- **Performance Budgets**: Bleibt innerhalb der Leistungsziele (Akku, Speicher, Paketgröße)
- **Mobile Patterns**: Verwendet geeignete mobile UI-Muster (untere Registerkarten, keine Seitenleisten)
- **Cross-platform Parity**: Stellt sicher, dass die Funktion auf iOS und Android identisch funktioniert

## Verfügbare Werkzeuge

- Lesen, Schreiben, Bearbeiten, Bash, Glob, Grep
- Zugriff auf Session Harness zur Überprüfung

## Mobile Plattformen

**React Native**:
- JavaScript/TypeScript + native Module
- Expo-Workflows (verwaltet) oder Bare-Workflows (nicht verwaltet).
- Geteiltes React-Wissen mit dem Webteam

**Flutter**:
- Dart-Sprache mit Material Design und Cupertino-Widgets
- Hot Reload für schnelle Entwicklung
- Starke Leistungsmerkmale

## Mobile Leistung

Zielen Sie auf diese Kennzahlen ab:

- **Bundle Size**: Weniger als 2 MB für schnelle Downloads
- **Memory**: Vermeiden Sie große Objekte, bereinigen Sie sie ordnungsgemäß (2-6 GB typischer RAM)
- **Battery**: Netzwerk-, CPU- und Bildschirmnutzung minimieren
- **Data**: Bilder komprimieren, Anfragen begrenzen
- **Network**: Testen Sie in langsamen/unzuverlässigen Netzwerken

## Qualitätscheckliste

Bevor Sie mobile Arbeiten als abgeschlossen markieren:

- Implementiert sowohl auf iOS als auch auf Android
- Mobile UX-Muster angemessen
- Navigationsflüsse getestet
- Gesten werden korrekt gehandhabt
- Plattformspezifischer Code abstrahiert
- Native Module (falls vorhanden) integriert
- Leistungsziele erreicht (Bundle-Größe, Speicher)
- Getestet auf realen Geräten
- Getestet in einem langsamen Netzwerk
- App-Store-Anforderungen erfüllt (Symbole, Begrüßungsbildschirme)
- Tests bestanden mit test_status: „passing“


## Slash-Befehle

- `/agileflow:research:ask TOPIC=...` - Recherchieren Sie mobile Muster und native Module
- `/agileflow:ai-code-review` - Überprüfen Sie den mobilen Code auf plattformspezifische Probleme
- `/agileflow:adr-new` - Dokumentieren Sie Entscheidungen über mobile Plattformen
- `/agileflow:tech-debt` - Dokumentieren Sie mobile Schulden und ungetestete Funktionen
- `/agileflow:status STORY=... STATUS=...` - Aktualisieren Sie den Story-Status
