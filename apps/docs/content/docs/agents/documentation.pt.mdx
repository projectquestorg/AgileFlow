# Agente de Documentação

O Agente de Documentação (AG-DOCUMENTATION) é o seu especialista em redação técnica, documentação de API, guias do usuário e manutenção da documentação atualizada com sua base de código. Este agente garante que toda a documentação permaneça atualizada, clara e útil para desenvolvedores novos e experientes.

## Capacidades

- **Documentação da API**: especificações OpenAPI/Swagger, descrições de endpoint, exemplos de solicitação/resposta e códigos de erro
- **Guias do usuário e tutoriais**: guias de introdução, instruções passo a passo, práticas recomendadas e solução de problemas
- **Guias do desenvolvedor**: configuração do projeto, fluxo de trabalho de desenvolvimento, visão geral da arquitetura e padrões de extensão
- **Gerenciamento README**: arquivos README raiz, específicos do módulo e específicos do recurso
- **Manutenção do registro de alterações**: notas de versão e controle de alterações voltado para o usuário
- **Auditorias de documentação**: identifique conteúdo desatualizado, links quebrados e exemplos ausentes
- **Documentos gerados automaticamente**: suporte para OpenAPI Generator, TypeDoc/JSDoce diagramas de arquitetura

## Quando usar

Use este agente quando precisar:

- Escreva ou atualize a documentação da API com exemplos de trabalho
- Crie guias do usuário para novos recursos (instruções passo a passo)
- Atualize os arquivos README para refletir o estado atual do projeto
- Escreva guias de integração para desenvolvedores
- Crie guias de solução de problemas e perguntas frequentes para problemas comuns
- Documentar decisões de arquitetura e design de sistema
- Manter changelog com notas de lançamento
- Corrigir links de documentação quebrados
- Documentação de auditoria para obsolescência e precisão
- Keep documentation in sync with code changes

## Como funciona

1. **Carregamento de contexto**: o agente lê o arquivo de expertise (padrões de documentação, localizações de arquivos, convenções) e analisa a estrutura do projeto (CLAUDE.md, status.json, versões recentes)

2. **Auditoria de documentação**: o agente identifica lacunas na documentação, seções desatualizadas, links quebrados e exemplos ausentes examinando documentos/estrutura e comparando com o código

3. **Planejamento**: Agente propõe melhoria na documentaçãoentes, sugere novos guias, identifica conteúdo obsoleto e prioriza o trabalho

4. **Escrita**: o agente cria/atualiza a documentação com clareza priorizada em vez de brevidade, inclui exemplos de trabalho (prontos para copiar e colar) e mantém uma formatação consistente

5. **Análise de qualidade**: o agente verifica se a documentação está atualizada com o código, se todas as etapas foram explicadas, se os exemplos funcionam e se a navegação entre os documentos está clara

6. **Coordenação**: o agente atualiza o status.json, anexa mensagens de conclusão e publica atualizações no barramento do agente

```bash
# Via /babysit
/agileflow:babysit

```

O Agente de Documentação:

2. Atualize a documentação da API com exemplos de solicitação/resposta
3. Adicione notas de versão ao changelog destacando as alterações voltadas para o usuário
4. Atualize o README se necessário
5. Verifique se todos os links funcionam

## Principais comportamentos

- **Prioridade da documentação**: Clareza em vez de brevidade - explica tudo para novos usuários
- **Sempre inclua exemplos**: Documentação sem exemplos é inútil; todos os exemplos são copy-paste pronto
- **Manter atualizado**: a documentação deve permanecer atualizada com o código; documentos obsoletos são piores do que nenhum documento
- **Incluir solução de problemas**: solucione problemas comuns que os usuários encontrarão
- **Alterações que quebram documentos**: Críticas para usuários que atualizam versões
- **Verificação de links**: corrige proativamente links de documentação quebrados
- **Consistência**: mantém formatação, tom e estrutura consistentes em todos os documentos
- **Preservação de contexto**: usa compact_context (prioridade: alta) para manter o foco durante longas conversas, preservando o rastreamento da cobertura da documentação e a validação de links por meio da compactação de contexto

## Configuração de contexto compacto

O agente de documentação usa compact_context de **alta prioridade** para garantir que a qualidade da documentação permaneça em foco:

```yaml
compact_context:
  priority: high
  preserve_rules:
    - "LOAD EXPERTISE FIRST: Always read packages/cli/src/core/experts/documentation/expertise.yaml"
    - "NEVER LET DOCS LAG BEHIND CODE: Stale docs are worse than no docs (misinformation)"
    - "ALWAYS INCLUDE EXAMPLES: Documentation without examples is useless"
    - "KEEP LINKS CURRENT: Run link checker, fix broken references"
    - "DOCUMENT BREAKING CHANGES: Critical for users upgrading versions"
    - "CLARITY OVER BREVITY: Explain everything for new users"
  state_fields:
    - current_story
    - documentation_coverage
    - broken_links_count
    - outdated_sections
```

Isso garante que as regras críticas da documentação (manter os documentos atualizados, incluindo exemplos, integridade da documentação) e o estado atual (quais documentos estão desatualizados, quais links estão quebrados) persistam por meio da compactação do contexto.

## Lista de verificação de qualidade

Antes de documentar:
- A documentação está atualizada com o código
- Todos os novos recursos documentados com exemplos
- A documentação da API inclui exemplos de trabalho
- Links não estão quebrados
- A formatação é consistente entre arquivos
- Os exemplos estão funcionando e prontos para copiar e colar
- A seção de solução de problemas aborda problemas comuns
- A navegação entre documentos é clara
- README é preciso e útil
- Nenhuma informação obsoleta permanece

## Padrões Comuns

**Documentação como código**
- Toda documentação versionada com código em git
- A fonte da verdade é a base de código viva, não documentos separados
- Mudanças nas atualizações da documentação do gatilho de código

**Estrutura de diretório numerada**
- Documentação organizada em docs/ com diretórios numerados (00-meta até 10-research)
- Estrutura consistente em todos os projetos

**Manutenção da documentação**
- Agende auditorias trimestrais para encontrar informações desatualizadas
- Após cada lançamento, atualize a documentação
- Após cada alteração da API, atualize os documentos da API
- Corrija links quebrados de forma proativa

## Mensagens de Coordenação

O Agente de Documentação se comunica com outros agentes através de mensagens de barramento:

```jsonl
{"ts":"2025-10-21T10:00:00Z","from":"AG-DOCUMENTATION","type":"status","text":"API docs updated for v2.17.0"}
{"ts":"2025-10-21T10:05:00Z","from":"AG-DOCUMENTATION","type":"question","text":"UI component API changed - need updated documentation for props"}
```

Quando outros agentes concluem o trabalho:
- AG-API conclui endpoint → Agente de documentação atualiza documentos da API
- Componente de lançamentos AG-UI → Componente de documentos do Agente de Documentação proPS e uso
- O lançamento acontece → O Agente de Documentação atualiza o changelog e as notas de lançamento
- Decisão de arquitetura tomada → Agente de Documentação atualiza documentação ADR
