# وكلاء

يتضمن AgileFlow **30 وكيلًا متخصصًا**، كل منهم خبير في مجاله. يمكن إنشاء الوكلاء بواسطة معلم `/babysit` أو استدعاؤهم مباشرة باستخدام أداة المهام.

## كيف يعمل الوكلاء

1. **الخبرة أولاً**: يقوم كل وكيل بتحميل ملف خبرته لفهم أنماط قاعدة التعليمات البرمجية الخاصة بك
2. **مدرك للسياق**: يقرأ الوكلاء مستندات المشروع وstatus.json والتغييرات الأخيرة
3. **الحفاظ على السياق**: يستخدم الوكلاء المحسّنون تكوين Compact_context للحفاظ على المعرفة الهامة أثناء ضغط المحادثة
4. **تعاوني**: يتواصل الوكلاء عبر ناقل الرسائل ويحدثون حالة القصة
5. **تحسين الذات**: يقوم الوكلاء بتحديث خبراتهم بعد إكمال العمل

## ميزة السياق المضغوط

للحفاظ على التركيز أثناء المحادثات الطويلة، يستخدم وكلاء AgileFlow تكوين **compact_context** للحفاظ على المعلومات الهامة عندما يتم ضغط السياق تلقائيًا. وهذا يضمن عدم فقدان الوكلاء للقواعد والحالة والأولويات المهمة في منتصف المحادثة.

### كيف يعمل

كللدى الوكيل تكوين `compact_context` بثلاثة مكونات:

**مستوى الأولوية**: يحدد مدى قوة الحفاظ على سياق الوكيل
- `critical` - يتم الاحتفاظ به دائمًا (الوكلاء ذوو الأولوية القصوى مثل المرشد، وapi، وui)
- `high` - محفوظ في معظم عمليات الضغط (العوامل المهمة)
- `medium` - يتم الاحتفاظ بها عندما يكون ذلك ممكنًا (الوكلاء القياسيون)
- `low` - أول ما تتم إزالته (نادر، يستخدم للوكلاء الاستشاريين)

**قواعد الحفاظ**: القواعد السلوكية الأساسية التي لا يجب فقدانها أبدًا
- هوية الوكيل ودوره
- القيود الحرجة وقواعد السلامة
- الإجراءات التي يجب اتباعها (على سبيل المثال، "تحميل الخبرة أولاً")
- متطلبات إدارة الدولة
- بروتوكولات التنسيق

**حقول الحالة**: الحالة الديناميكية التي تتغير أثناء العمل
- القصة الحالية قيد التنفيذ (US-#### ID)
- تقدم المهمة ونسبة الإنجاز
- الحاصرات والتبعيات المعروفة
- الإجراءات ذات الأولوية التالية
- اختبار/حالة البناء

### مثال للتكوين

__لا تترجم_0__

عندما يقوم كلود كود بضغط conالنص، فإنه يحافظ على هذه القواعد وحقول الحالة لجميع الوكلاء، مما يضمن الاستمرارية حتى بعد عدة جولات من ضغط المحادثة.

### توزيع أولوية الوكيل

- **حرج** (وكلاء 2): المرشد، API
- **عالية** (9 وكلاء): واجهة المستخدم، وقاعدة البيانات، وCI، وdevops، والاختبار، والأمان، والوثائق، والأداء، واستعلام قاعدة التعليمات البرمجية
- **متوسط** (15 وكيل): نقل البيانات، ضمان الجودة، إمكانية الوصول، المراقبة، البحث، كاتب الإعلانات، التحليلات، عمليات التكامل، الهاتف المحمول، التصميم، الامتثال، إعادة البناء، المنتج، محدث الملف التمهيدي
- **منخفض** (عميلان): مخطط ملحمي، ومنسق

يضمن هذا النهج المتدرج أن يحافظ الوكلاء الأكثر أهمية دائمًا على التركيز بينما يقوم الآخرون بتدهور السياق برشاقة عند الحاجة.
