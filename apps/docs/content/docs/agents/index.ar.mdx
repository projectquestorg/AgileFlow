# الوكلاء

يتضمن AgileFlow **111 specialized agents**، كل خبير في مجاله. يمكن إنتاج الوكلاء بواسطة `/babysit` معلمه أو استدعاؤه مباشرة باستخدام أداة المهام.

## كيف يعمل الوكلاء

1. **Expertise-First**: يقوم كل وكيل بتحميل ملف خبرته لفهم أنماط قاعدة التعليمات البرمجية الخاصة بك
2. **Context-Aware**: يقوم الوكلاء بقراءة مستندات المشروع وstatus.json والتغييرات الأخيرة
3. **Context Preservation**: يستخدم الوكلاء المحسنون تكوين Compact_context للحفاظ على المعرفة الهامة أثناء ضغط المحادثة
4. **Collaborative**: يتواصل الوكلاء عبر ناقل الرسائل ويحدثون حالة القصة
5. **Self-Improving**: يقوم الوكلاء بتحديث خبراتهم بعد الانتهاء من العمل

## ميزة السياق المضغوط

للحفاظ على التركيز أثناء المحادثات الطويلة، يستخدم وكلاء AgileFlow **compact_context** التكوين للحفاظ على المعلومات الهامة عندما يتم ضغط السياق تلقائيًا. وهذا يضمن عدم فقدان الوكلاء للقواعد والحالة والأولويات المهمة في منتصف المحادثة.

### كيف يعمل

كل وكيل لديه `compact_context` التكوين مع ثلاثة مكونات:

**Priority Level**: يحدد مدى قوة الحفاظ على سياق الوكيل
- `critical` - محفوظ دائمًا (الوكلاء ذوو الأولوية القصوى مثل معلمه وواجهة برمجة التطبيقات وواجهة المستخدم)
- `high` - محفوظ في معظم الدمك (عوامل مهمة)
- `medium` - يتم الحفاظ عليها عندما يكون ذلك ممكنًا (العوامل القياسية)
- `low` - أول ما يتم إزالته (نادر، يستخدم للوكلاء الاستشاريين)

**Preserve Rules**: القواعد السلوكية الأساسية التي لا يجب فقدانها أبدًا
- هوية الوكيل ودوره
- القيود الحرجة وقواعد السلامة
- الإجراءات التي يجب اتباعها (على سبيل المثال، "تحميل الخبرة أولاً")
- متطلبات إدارة الدولة
- بروتوكولات التنسيق

**State Fields**: الحالة الديناميكية التي تتغير أثناء العمل
- القصة الحالية قيد التنفيذ (US-#### ID)
- تقدم المهمة ونسبة الإنجاز
- الحاصرات والتبعيات المعروفة
- الإجراءات ذات الأولوية التالية
- حالة الاختبار/البناء

### تكوين المثال

```yaml
compact_context:
  priority: critical
  preserve_rules:
    - "LOAD EXPERTISE FIRST: Read packages/cli/src/core/experts/api/expertise.yaml"
    - "VERIFY TEST BASELINE: Check test_status before starting"
    - "PRIORITIZE AG-UI BLOCKERS: Unblock UI stories waiting on API endpoints"
    - "DIFF-FIRST APPROACH: Show edits with confirmation before applying"
  state_fields:
    - current_story
    - endpoints_implemented
    - blocked_ui_stories
    - test_status_baseline
```

عندما يقوم Claude Code بضغط السياق، فإنه يحافظ على هذه القواعد وحقول الحالة لجميع الوكلاء، مما يضمن الاستمرارية حتى بعد عدة جولات من ضغط المحادثة.

### توزيع أولوية الوكيل

- **Critical** (4 وكلاء): معلمه، API
- **High** (12 وكيل): واجهة المستخدم، قاعدة البيانات، CI، Devops، الاختبار، الأمان، التوثيق، الأداء، الاستعلام عن قاعدة التعليمات البرمجية
- **Medium** (30 وكيل): نقل البيانات، ضمان الجودة، إمكانية الوصول، المراقبة، البحث، كاتب الإعلانات، التحليلات، التكامل، الهاتف المحمول، التصميم، الامتثال، إعادة البناء، المنتج، محدث الملف التمهيدي
- **Low** (10 وكلاء): مخطط الملحمة، المنسق

يضمن هذا النهج المتدرج أن يحافظ الوكلاء الأكثر أهمية دائمًا على التركيز بينما يقوم الآخرون بتدهور السياق برشاقة عند الحاجة.

## وكلاء التفريخ

### عبر/مجالسة الأطفال (مستحسن)

ال `/babysit` يكتشف المرشد تلقائيًا الوكيل المطلوب:

```bash
/agileflow:babysit
# Then describe your task - babysit will spawn the right agent
```

### التبويض المباشر

يمكنك أيضًا إنشاء الوكلاء مباشرةً باستخدام أداة المهام:

```text
Task(
  description: "Design user sessions schema",
  prompt: "Create a sessions table to track user logins with proper indexes",
  subagent_type: "agileflow-database"
)
```

### وكلاء موازية

للحصول على ميزات المكدس الكامل، قم بإنشاء العديد من الوكلاء في وقت واحد:

```text
# API work
Task(subagent_type: "agileflow-api", run_in_background: true, ...)

# UI work (runs in parallel)
Task(subagent_type: "agileflow-ui", run_in_background: true, ...)
```

### تحديثات الحالة

تحديث الوكلاء `status.json` كما يعملون:

```json
{
  "stories": {
    "US-0042": {
      "status": "in-progress",
      "owner": "AG-DATABASE",
      "summary": "Created sessions table with indexes"
    }
  }
}
```

### حافلة الرسائل

رسائل تنسيق تسجيل الوكلاء:

```json
{"timestamp": "2025-12-24T10:00:00Z", "type": "assign", "story": "US-0042", "owner": "AG-DATABASE"}
{"timestamp": "2025-12-24T11:00:00Z", "type": "handoff", "story": "US-0042", "from": "AG-DATABASE", "to": "AG-API"}
```

## أفضل الممارسات

1. **Let babysit choose**: المرشد يعرف أي وكيل يناسبه أكثر
2. **Provide context**: أخبر الوكلاء عن الأنماط الموجودة في قاعدة التعليمات البرمجية الخاصة بك
3. **Review updates**: تحقق من التغييرات التي أجراها الوكلاء على Status.json
4. **Use multi-expert for uncertainty**: عندما تحتاج إلى ثقة عالية، قم بنشر العديد من الخبراء
