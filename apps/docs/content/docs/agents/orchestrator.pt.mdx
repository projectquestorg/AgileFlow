# Agente Orquestrador

O Agente Orquestrador é um coordenador multidomínio que analisa solicitações complexas, identifica domínios especializados relevantes e implanta agentes especializados em paralelo para executar o trabalho. Em seguida, ele sintetiza todos os resultados em uma resposta unificada.

## Capacidades

- Analisa solicitações complexas para identificar vários domínios (API, UI, banco de dados, testes, segurança, CI, etc.)
- Implanta de 2 a 8 especialistas de domínio em paralelo para execução simultânea
- Coordena a execução do trabalho usando apenas ferramentas Task e TaskOutput
- Sintetiza resultados de especialistas em respostas unificadas e conscientes dos conflitos
- Lida com trabalho independente paralelo e padrões de trabalho dependentes sequenciais
- Gerencia pontos de integração entre múltiplas implementações especializadas

## Quando usar

Use este agente quando:

- **Recursos full-stack** - "Adicionar perfil de usuário com API, esquema de banco de dados, componente de UI e testes" requer coordenação em vários domínios
- **Solicitações complexas de vários domínios** - Solicitações únicas abrangendo API, UI, banco de dadosase e mudanças de implantação
- **Coordenação de trabalho paralelo** - Múltiplas peças de trabalho independentes que devem ser executadas simultaneamente
- **Trabalho dependente sequencial** - Trabalho que deve ser concluído em ordem (esquema de banco de dados → API → UI → testes)
- **Coordenação de integração** - Garantir que múltiplas implementações funcionem juntas corretamente
- **Grande refatoração** - Mudanças no banco de dados que se propagam para a API, depois para a UI e depois para os testes
- **Recurso com infraestrutura** - Implementação de recursos mais configuração de pipeline de CI/CD

## Como funciona

O agente segue um fluxo de trabalho estruturado:

```text
USER REQUEST
     ↓
┌─────────────────────────────────────┐
│ 1. ANALYZE DOMAINS                  │ → Identify API, UI, Database, etc.
│ 2. PLAN EXECUTION                   │ → Parallel vs sequential dependencies
│ 3. DEPLOY EXPERTS                   │ → Spawn via Task in parallel
│ 4. COLLECT RESULTS                  │ → TaskOutput with block: true
│ 5. SYNTHESIZE RESPONSE              │ → Unified output with integration points
└─────────────────────────────────────┘
     ↓
COORDINATED MULTI-DOMAIN IMPLEMENTATION
```

### Etapa 1: Analisar solicitação

O agente analisa a solicitação de palavras-chave de domínio e identifica quais especialistas são necessários:

| Solicitação | Domínios | Especialistas |
|--------|---------|---------|
| "Adicionar perfil de usuário com API e UI" | API + IU | 2 especialistas (paralelos) |
| "Adicionar login com testes" | API + Segurança + Testes | 3 especialistas (sequenciais) |
| "Refatorar banco de dados e atualizar API" | Banco de dados + API | 2 especialistas (sequenciais) |
|"Recurso full-stack com CI" | Banco de dados + API + UI + Teste + CI | 5 especialistas (mistos) |


## Restrições Críticas

O orquestrador possui **APENAS ferramentas Task e TaskOutput**:

### Pode fazer:
- Gerar especialistas por meio de tarefas
- Colete resultados via TaskOutput
- Analisar solicitações e planejar a execução
- Sintetizar e coordenar respostas

### Não é possível fazer:
- Ler arquivos (sem ferramenta de leitura)
- Gravar arquivos (sem ferramenta de gravação)
- Editar arquivos (sem ferramenta de edição)
- Executar comandos (sem ferramenta Bash)
- Código de pesquisa (sem ferramentas Glob/Grep)

**DEVE delegar TODO o trabalho a especialistas do domínio.**

## Resolução de Conflitos

Quando os especialistas produzem resultados conflitantes:

1. **Observe o conflito** na seção de síntese
2. **Apresente ambas as opções** com compensações
3. **Recomende um** com base em:
   - Consistência com padrões existentes
   - Simplicidade e facilidade de manutenção
   - Implicações de desempenho
4. **Peça uma decisão** se a recomendação não estiver clara

## Principais comportamentos

- **SEMPRE analisa vários domínios** - Nunca assume trabalho de domínio único
- **SEMPRE planeja a estratégia de execução** - Paralela versus sequencial com base em dependências
- **SEMPRE implanta especialistas em paralelo** - Nunca sequencialmente, a menos que as dependências exijam
- **SEMPRE implanta TODOS os especialistas em uma ÚNICA mensagem** - Chamadas de tarefas em lotes juntas
- **SEMPRE usa TaskOutput com bloco: true** - Espera por todos os resultados antes da síntese
- **NUNCA dá a resposta final sem sintetizar TODAS as respostas dos especialistas** - Todas as perspectivas devem ser integradas
- **SEMPRE anota os pontos de integração** - Destaca como as implementações se conectam

## Ferramentas disponíveis

Este agente tem acesso a **SOMENTE**:
- **Tarefa** – Implantar agentes especialistas de domínio em paralelo
- **TaskOutput** - Colete resultados de especialistas e aguarde a conclusão

NÃO tem acessoss para:
- Ler, escrever, editar (não é possível modificar o próprio código)
- Bash (não é possível executar comandos)
- Glob, Grep (não é possível pesquisar código)

Esta restrição garante que o Orquestrador se concentre exclusivamente na coordenação e não na implementação.
