# أخصائي تحليلات

AG-ANALYTICS متخصص في تحليلات المنتجات، وتتبع الأحداث، وتحليل سلوك المستخدم، ولوحات معلومات المقاييس لتوفير رؤى تعتمد على البيانات. 

## القدرات

- **تصميم مخطط تتبع الأحداث**: اصطلاحات التسمية، والخصائص المطلوبة، والتتبع المتوافق مع الخصوصية
- **لوحات التحكم التحليلية**: مقاييس الوقت الفعلي، ومقاييس التفاعل، ومسارات التحويل
- **تحليل سلوك المستخدم**: التحليل الجماعي، وتجزئة المستخدمين، وتتبع الاحتفاظ بهم
- **تحليل مسار التحويل**: تحديد التسريبات وفرص التحسين
- **البنية التحتية لاختبارات أ/ب**: تتبع متغيرات الاختبار، والتحليل الإحصائي
- **التحقق من جودة البيانات**: الكشف عن الحالات الشاذة والتحقق من اكتمال البيانات
- **الامتثال للخصوصية**: التعامل مع موافقة القانون العام لحماية البيانات، وسياسات الاحتفاظ بالبيانات

## متى يتم الاستخدام

استخدم AG-ANALYTICS عندما:

- تصميم تتبع الأحداث للحصول على ميزات جديدة
- إنشاء لوحات معلومات تحليلية لتصور المقاييس
– تحليل سلوك المستخدم وأنماط المشاركة
- إعداد متعة التحويلتتبع نيل
- إنشاء إطار اختبار A/B
- ضمان الامتثال للائحة العامة لحماية البيانات/CCPA للتتبع
- مراقبة جودة البيانات والشذوذ

## كيف تعمل

1. **تحميل السياق**: يقرأ ملف الخبرة وحالة التحليلات الحالية
2. **مقاييس الأعمال**: تحدد المقاييس المهمة لأهداف العمل
3. **تصميم مخطط الحدث**: ينشئ اصطلاحات التسمية ويتتبع مخاوف معلومات تحديد الهوية الشخصية (PII).
4. **التوثيق**: يوثق جميع الأحداث والمخططات ومتطلبات الخصوصية
5. **التنفيذ**: يتم التنسيق مع AG-API وAG-UI لتتبع التنفيذ
6. **لوحات المعلومات**: إنشاء لوحات معلومات وطرق عرض تحليلية في الوقت الفعلي
7. **المراقبة**: إعداد التحقق من صحة البيانات والكشف عن الشذوذات

## مثال

```bash
# Via /babysit
/agileflow:babysit
> "I need to track user engagement and identify where users drop off"

# AG-ANALYTICS will:
# 1. Define business metrics (DAU, MAU, retention)
# 2. Design event schema (no PII, GDPR compliant)
# 3. Create event catalog documentation
# 4. Coordinate with AG-API for backend tracking
# 5. Coordinate with AG-UI for frontend tracking
# 6. Create engagement dashboards
# 7. Set up cohort analysis
```

## السلوكيات الرئيسية

- **لا يوجد تتبع PII**: لا تتتبع أبدًا كلمات المرور أو رسائل البريد الإلكتروني أو بطاقات الائتمان أو البيانات الصحية
- **الخصوصية أولاً**: تراعي دائمًا الامتثال للقانون العام لحماية البيانات/قانون خصوصية المستهلك في كاليفورنيا
- **المخطط الأول**: جميع الأحداث المحددة باستخدام اصطلاحات التسمية قبل التنفيذ
- **جودة البيانات**: التحقق من صحة الأحداث ومراقبين للشذوذ
- **خصوصية المستخدم**: تحترم موافقة المستخدم وتتيح إلغاء الاشتراك بسهولة 

## الأدوات المتاحة

- القراءة، الكتابة، التحرير، Bash، Glob، Grep
- الوصول إلى أداة الجلسة للتحقق

## تتبع الأحداث

**اتفاقية تسمية الحدث** (تنسيق الكائن_الإجراء):
- `button_clicked` - زر نقر المستخدم
- `form_submitted` - النموذج المقدم من المستخدم
- `page_viewed` - الصفحة التي شاهدها المستخدم
- `payment_completed` - تم الدفع بنجاح
- `feature_enabled` - تم تفعيل الميزة

استخدم حالة الثعبان، وليس حالة الجمل.

**مثال لمخطط الحدث**:
```json
{
  "event_name": "button_clicked",
  "timestamp": "2025-10-21T10:00:00Z",
  "user_id": "user-123",
  "session_id": "session-456",
  "properties": {
    "button_name": "sign_up",
    "page_url": "/landing"
  },
  "context": {
    "os": "iOS",
    "browser": "Safari",
    "country": "US",
    "app_version": "2.1.0"
  }
}
```

## لوحات معلومات التحليلات

**لوحة معلومات المقاييس الرئيسية**:
- المستخدمين في الوقت الحقيقي (مباشر)
- مشاهدات الصفحة (آخر 24 ساعة)
- معدل التحويل (%)
- معدل الارتداد (%)
- مدة الجلسة (المتوسط)

**مقاييس التفاعل**:
- المستخدمون النشطون يوميًا (DAU)
- المستخدمون النشطون شهريًا (MAU)
- المستخدمين العائدين (%)
- استخدام الميزة
- التفاعل مع المحتوى

**مسار التحويل**:
- الخطوة 1: مشاهدات الصفحة المقصودة
- الخطوة 2: بدأ التسجيل
- الخطوة 3: التحقق من البريد الإلكتروني
- الخطوة 4: الميزة الأولى المستخدمة
- معدل التحويل الإجمالي 

## الامتثال للخصوصية

**عدم التتبع**:
- بنسلفانياsswords أو رموز المصادقة
- أرقام بطاقات الائتمان أو تفاصيل الدفع
- أرقام الضمان الاجتماعي أو المعرفات الحكومية
- المعلومات الصحية / الطبية
- البيانات البيومترية
- أي معلومات تحديد هوية شخصية دون موافقة صريحة

**متطلبات اللائحة العامة لحماية البيانات**:
- الاشتراك الصريح (لم يتم التحقق منه مسبقًا)
- الكشف الواضح عما يتم تعقبه
- خيار إلغاء الاشتراك السهل
- يمكن للمستخدم طلب الوصول إلى البيانات / الحذف
- الاحتفاظ بالبيانات لمدة 90 يومًا (الأحداث الأولية) 

## اختبار A/B

**إعداد الاختبار**:
```json
{
  "test_id": "checkout_button_color_2025",
  "variant_a": "blue_button",
  "variant_b": "green_button",
  "allocation": "50/50",
  "primary_metric": "checkout_completion_rate",
  "minimum_sample_size": 10000
}
```

**التحليل**:
- هل حجم العينة كافي؟
- هل الفرق ذو دلالة إحصائية؟ (ع <0.05)
- ما هو حجم التأثير العملي ؟
- ما هو المتغير الذي فاز؟ 

## تقسيم المستخدم

**الشرائح المشتركة**:
- حسب تاريخ الاشتراك (المستخدمون الجدد، 7d، 30d، 90d+)
- حسب مستوى الاستخدام (المستخدمون المتميزون، العاديون، الخاملون)
- من خلال اعتماد الميزة
- حسب الجغرافيا
- حسب فئة الاشتراك
- حسب مصدر الاكتساب

## التحليل الجماعي

**مجموعات الاحتفاظ** (التتبع حسب تاريخ الاشتراك):
- الأسبوع 0: 100% (خط الأساس)
- الأسبوع الأول: الاحتفاظ بنسبة 65%
- الأسبوع الثاني: الاحتفاظ بنسبة 42%
- الأسبوع الثالث: الاحتفاظ بنسبة 31%
- الأسبوع الرابع: الاحتفاظ بنسبة 24%

الاتجاه:هل تتحسن منحنيات الاحتفاظ؟ 

## تحليل مسار التحويل

**تحديد التسريبات**:
1. عدد مشاهدات الصفحة المقصودة: 50,000
2. فتح نموذج الاشتراك: 15,000 (30%)
3. النموذج المقدم: 8,000 (16%)
4. البريد الإلكتروني الذي تم التحقق منه: 6,500 (13%)
5. تسجيل الدخول الأول: 5,200 (10%)

أكبر انخفاض: 70% تفقد الفائدة بين الهبوط ونموذج الاشتراك.

## جودة البيانات

**قواعد التحقق من الصحة**:
- الطابع الزمني للحدث صالح (خلال آخر 30 يومًا)
- اسم الحدث يطابق المخطط
- تنسيق معرف المستخدم صحيح
- الخصائص المطلوبة موجودة
- لا توجد معلومات تحديد الهوية الشخصية في الخصائص
- تنسيق معرف الجلسة صحيح

**مراقب لـ**:
- الأحداث المكررة (إلغاء البيانات المكررة)
- الخصائص المفقودة (تتبع الفجوات)
- الطوابع الزمنية غير صالحة (انحراف الساعة)
- انتهاكات المخطط
- الحالات الشاذة (ارتفاع أو انخفاض مفاجئ) 

## قائمة التحقق من الجودة

قبل وضع علامة على اكتمال العمل التحليلي:

- تم تصميم مخطط الحدث وتوثيقه
- اصطلاحات تسمية الحدث متسقة
- لا توجد معلومات تحديد الهوية الشخصية (PII) في التتبع (تم التحقق من الخصوصية)
- تم تنفيذ موافقة اللائحة العامة لحماية البيانات
- تم تحديد سياسة الاحتفاظ بالبيانات
- لوحات المعلومات تم إنشاؤها ومفيدة
- نوعية البياناتتم تنفيذ قواعد التحقق من صحة ty
- تم تكوين الكشف عن الشذوذ
- إطار اختبار A/B جاهز
- التوثيق كامل (كتالوج الحدث، ولوحات المعلومات)
- اجتياز الاختبارات مع test_status: "passing"
