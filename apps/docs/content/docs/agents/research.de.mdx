# Forschungsagent

Der Research Agent (RESEARCH) ist Ihr Spezialist für technische Forschung, Wissensmanagement und die Erstellung umfassender Forschungsanreize. Dieser Agent sammelt Informationen aus maßgeblichen Quellen, fasst Ergebnisse zusammen und verwaltet einen durchsuchbaren Forschungsindex zur späteren Bezugnahme.

## Fähigkeiten

- **Web-Recherche**: Durchsuchen Sie offizielle Dokumentationen und maßgebliche Quellen nach technischen Informationen
- **ChatGPT-Eingabeaufforderungserstellung**: Erstellen Sie umfassende Recherche-Eingabeaufforderungen, in denen Sie Implementierungspläne, Codeausschnitte, ADR-Entwürfe, Story-Aufschlüsselungen und mehr anfordern
- **Kuratierung von Forschungsnotizen**: Speichern Sie strukturierte Forschungsergebnisse mit einheitlicher Formatierung in docs/10-research/
- **Wartung des Forschungsindex**: Halten Sie docs/10-research/README.md zuerst mit der neuesten Forschung auf dem neuesten Stand
- **Erkennung veralteter Forschungsergebnisse**: Identifizieren Sie Forschungsergebnisse, die älter als 90 Tage sind und möglicherweise aktualisiert werden müssen
- **Identifizierung von Forschungslücken**: Markieren Sie fehlende Forschungsergebnisse für technische Entscheidungen und ADRs
- **Agentenkoordination**: NeinInformieren Sie anfragende Agenten über Busnachrichten, wenn die Recherche abgeschlossen ist

## Wann zu verwenden

Verwenden Sie diesen Agenten, wenn Sie Folgendes benötigen:

- Recherchieren Sie ein technisches Thema oder eine Technologieauswahl
- Kompromisse zwischen verschiedenen Ansätzen verstehen
- Erstellen Sie eine umfassende ChatGPT-Eingabeaufforderung für eine tiefergehende Analyse
- Dokumentieren Sie die Ergebnisse in strukturiertem Format zum späteren Nachschlagen
- Aktualisieren Sie veraltete Forschungsergebnisse (>90 Tage alt)
- Finden Sie heraus, welche Forschungsergebnisse bereits vorhanden sind, bevor Sie Entscheidungen treffen
- Unterstützen Sie die ADR-Erstellung mit technischer Hintergrundrecherche
- Holen Sie sich Implementierungsanleitungen für einen Feature-Ansatz

## Wie es funktioniert

### Web-Recherche-Workflow

1. **Verständnis**: Der Agent klärt die Forschungsfrage und die Einschränkungen
2. **Suche**: Agent durchsucht offizielle Dokumentation und maßgebliche Quellen (WebSearch, WebFetch)
3. **Sammeln**: Der Agent sammelt wichtige Erkenntnisse – Ansätze, Kompromisse, Best Practices, Benchmarks
4. **Synthese**: Der Agent organisiert Ergebnisse in einem strukturierten Format
5. **Speichern**: Agent speichert Recherchen in `docs/10-research/<YYYYMMDD>-<slug>.md`
6. **Indizierung**: Agent aktualisiert docs/10-research/README.md mit Eintrag in der chronologischen Tabelle
7. **Koordination**: Der Agent sendet eine Busnachricht, um den anfragenden Agenten zu benachrichtigen

### ChatGPT Research Prompt Workflow

1. **Kontextladen**: Der Agent liest CLAUDE.md, context.md, status.json, verwandte ADRs und vorhandene Recherchen, um Duplikate zu vermeiden
2. **Verständnis**: Der Agent klärt das Forschungsthema und spezifische Fragen, die beantwortet werden müssen
3. **Aufforderungserstellung**: Der Agent erstellt eine umfassende Eingabeaufforderung mit der Anforderung:
   - TL;DR-Zusammenfassung
   - Schritt-für-Schritt-Implementierungsplan mit Dateipfaden
   - Minimale ausführbare Codefragmente (Kopieren und Einfügen möglich)
   - Konfiguration und Umgebungseinrichtung
   - Fehlerbehandlungsmuster
   - Beobachtbarkeits- und Analyse-Hooks
   - Testbeispiele (Einheit, Integration, E2E)
   - Checkliste für manuelle Tests
   - Sicherheits- und Datenschutzaspekte
   - ADR-Entwurf mit Optionen und Vor-/Nachteilen
   - Story-Aufschlüsselung mit Akzeptanzkriterien „Gegeben/Wann/Dann“.
   - Rollback-Plan und Risiken
   - PR-Body-Templgegessen
   - Nur offizielle Dokumentationsquellen (zitiert mit Titel, URL, Datum)

4. **Ausgabe**: Der Agent stellt eine Eingabeaufforderung als Codeblock zum einfachen Kopieren und Einfügen bereit
5. **Verarbeitung**: Der Benutzer fügt ChatGPT-Ergebnisse zurück in Claude ein
6. **Speichern**: Agent speichert Ergebnisse unter `docs/10-research/<YYYYMMDD>-<slug>.md`
7. **Indexierung**: Agent aktualisiert den Forschungsindex
8. **Benachrichtigung**: Der Agent benachrichtigt den anfragenden Agenten per Busnachricht

## Beispiel

```bash
# Via /babysit
/agileflow:babysit
> "Need to research JWT vs OAuth2 for authentication"
```

Der Forschungsagent würde:
1. Durchsuchen Sie maßgebliche Quellen (JWT RFC, OAuth2-Spezifikation, security.com-Artikel)
2. Sammeln Sie wichtige Erkenntnisse (zustandslos vs. zustandsbehaftet, Aktualisierungstoken, Delegation, Anwendungsfälle)
3. Erstellen Sie eine Forschungsnotiz, in der Ansätze verglichen werden
4. Speichern Sie es unter docs/10-research/20251021-jwt-vs-oauth2.md
5. Forschungsindex aktualisieren
6. Post-Bus-Nachricht: „Recherche abgeschlossen: JWT vs. OAuth2 in docs/10-research/20251021-jwt-vs-oauth2.md gespeichert“

## Schlüsselverhalten

- **Nur offizielle Quellen**: Die Forschung zitiert offizielle Dokumentation, keine zufälligen Blogbeiträge
- **Umfassende Eingabeaufforderungen**: ChatGPT-Eingabeaufforderungen sind ausführlichgh, um Implementierungsdetails, Tests, Sicherheitsüberlegungen und ADR-Entwürfe anzufordern
- **Strukturierte Formatierung**: Die gesamte Forschung folgt einer einheitlichen Struktur (Zusammenfassung, Ergebnisse, empfohlener Ansatz, Kompromisse, Quellen).
- **Veraltete Erkennung**: Identifiziert Forschungsergebnisse, die >90 Tage alt sind, und markiert sie für eine mögliche Aktualisierung, wenn sich die Technologie geändert hat
- **Keine Duplizierung**: Überprüft vorhandene Recherchen, bevor mit neuen Recherchen begonnen wird
- **Agentenkoordination**: Benachrichtigt anfragende Agenten sofort, wenn die Recherche abgeschlossen ist
- **Clear Sourcing**: Bei allen Recherchen werden Quellen mit Titel, URL und Abrufdatum zitiert

## Werkzeuge verfügbar

Dieser Agent hat Zugriff auf:
- **Lesen**: Durchsuchen Sie Forschungsdateien und verstehen Sie vorhandene Ergebnisse
- **Schreiben**: Erstellen Sie neue Forschungsnotizen
- **Bearbeiten**: Vorhandene Forschungsnotizen aktualisieren
- **Glob**: Finden Sie Forschungsdateien nach Muster
- **Grep**: Forschungsinhalte durchsuchen
- **WebSearch**: Durchsuchen Sie das Internet nach technischen Informationen
- **WebFetch**: Webseiten abrufen und analysieren

## Qualitätscheckliste

Vor dem Speichern der Recherche:
- Das Datum ist aktuell (Format JJJJ-MM-TT)
- Die Zusammenfassung ist prägnant (2-3 Sätze)
- Mindestens 3 wichtige Ergebnisse aufgeführt
- Empfohlener Ansatz klar dargelegt
- Zitierte Quellen mit URLs und Abrufdaten
- Verwandte ADRs/Geschichten/Epen verlinkt
- Index aktualisiert mit neuestem Eintrag oben
- Alle externen Links führen zur offiziellen Dokumentation

## Forschungslücken identifizieren

Der Research Agent weist auf Forschungslücken hin, wenn:
- Technologiewahl noch nicht erforscht (vor ADR erforderlich)
- Gehen Sie auf die in den Geschichtennotizen erwähnte Unsicherheit ein
- Mehrere Ansätze diskutiert, ohne klaren Gewinner
- ADR existiert, aber es mangelt an unterstützender Forschung
- Forschung ist veraltet (>90 Tage und Technologie hat sich geändert)
– Doppelte Recherche erforderlich (technische Versionen aktualisiert)

## Koordination mit anderen Agenten

Forschungsanfragen kommen üblicherweise von:
- **AG-UI**: Design sSysteme, Komponentenmuster, Barrierefreiheitstechniken
- **AG-API**: API-Architekturen, Datenbankdesigns, Authentifizierungsmuster
- **AG-CI**: Test-Frameworks, CI-Plattformen, Code-Qualitätstools
- **AG-DEVOPS**: Bereitstellungsstrategien, Container-Orchestrierung, Überwachung
- **ADR-WRITER**: Technische Alternativen vor der Entscheidungsfindung
- **EPIC-PLANNER**: Technologie-Stack-Recherche vor der Planung von Epics

Wenn eine Recherche angefordert wird:
1. Überprüfen Sie docs/09-agents/bus/log.jsonl auf eine Forschungsanforderungsnachricht
2. Recherche abschließen (Web- oder ChatGPT-Eingabeaufforderung)
3. Ergebnisse unter docs/10-research/ speichern
4. Forschungsindex aktualisieren
5. Senden Sie eine Busnachricht, um den Antragsteller zu benachrichtigen

## ChatGPT-Eingabeaufforderungsvorlage

Berücksichtigen Sie bei der Erstellung von Rechercheaufforderungen Folgendes:

```text
Context: [Project overview from context.md]
Current priority: [From status.json]
Related ADRs: [List relevant decisions]

Research topic: [Specific question]

Please provide:
1. TL;DR (2-3 sentences)
2. Step-by-step implementation plan with exact file paths
3. Minimal runnable code snippets (copy-paste ready)
4. Configuration (env vars, config files, CLI flags)
5. Error handling patterns
6. Observability (logging, metrics, tracing)
7. Tests: unit, integration, E2E examples
8. Manual testing checklist
9. Security checklist (auth, input validation, secrets, etc.)
10. Privacy considerations (PII, GDPR, data retention)
11. ADR draft with context, options, recommendation, consequences
12. Story breakdown (3-6 stories with Given/When/Then AC)
13. Rollback plan
14. Risks and gotchas
15. PR body template
16. Sources: official docs/repos only with title, URL, date

Final checklist:
- [ ] Ready to paste back to Claude for implementation
- [ ] All code snippets tested conceptually
- [ ] Risks clearly flagged
```

## Gemeinsame Muster

**Zwei Forschungsworkflows**
- **Web-Recherche**: Direkte Suche und Synthese (schneller für einfache Themen)
- **ChatGPT-Eingabeaufforderung**: Erstellen Sie eine umfassende Eingabeaufforderung für die KI-Analyse (besser für komplexe Entscheidungen)

**Recherche vor Entscheidungen**
- Recherchieren Sie immer, bevor Sie ADRs erstellen (ADR-WRITER fordert Recherche an)
- Recherchieren Sie immer, bevor Sie Epics planen (EPIC-PLANNER fordert Recherche an)
- Markieren Sie Unsicherheiten in Geschichten und Recherchen vor der Umsetzung

**Veraltetes Forschungsmanagement**
- Vierteljährliche Prüfung von docs/10-research/ auf alte Forschung
- Markieren Sie es als „Ersetzt“, wenn neue Forschungsergebnisse im Widerspruch zu alten stehen
- Löschen Sie niemals Forschungsergebnisse (behalten Sie den historischen Kontext bei)
- Aktualisieren Sie, wenn sich die technischen Versionen geändert haben
