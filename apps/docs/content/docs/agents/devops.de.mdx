# AG-DEVOPS

Der DevOps & Automation Agent ist auf Infrastrukturautomatisierung, Abhängigkeitsmanagement und Bereitstellungspipelines spezialisiert. AG-DEVOPS stellt sicher, dass Projekte eine hohe Codequalität beibehalten, Sicherheitslücken verwalten und zuverlässig bereitgestellt werden.

## Fähigkeiten

- **Dependency Management**: Sicherheitsüberprüfungen, Versionsverfolgung, Schwachstellenbehebungen, Lizenzeinhaltung
- **Deployment Automation**: Einrichtung der CI/CD-Pipeline, Staging-/Produktionskonfiguration, Bereitstellungen ohne Ausfallzeiten
- **Testing Infrastructure**: Test-Framework-Setup, Optimierung, Leistungstests
- **Code Quality Automation**: Linting, Formatierung, Code-Review-Bots, Coverage-Tracking
- **Impact Analysis**: Abhängigkeitsbäume, Bewertung des Explosionsradius, Bewertung der Änderungsauswirkungen
- **Technical Debt Management**: Schuldenbewertung, Priorisierung, Reduzierungsverfolgung
- **Documentation Synchronization**: API-Dokumente, README-Dateien, Änderungsprotokollpflege
- **Changelog Generation**: Semantische Versionierung, automatisierte Versionshinweise
- **Stakeholder Reporting**: Statusaktualisierungen, Metriken, Fortschrittsverfolgung

## Wann zu verwenden

Verwenden Sie AG-DEVOPS, wenn Sie Folgendes benötigen:

- Aktualisieren Sie Abhängigkeiten oder führen Sie Sicherheitsüberprüfungen durch
- CI/CD-Pipelines einrichten oder optimieren
- Konfigurieren Sie die Testinfrastruktur und -abdeckung
- Automatisieren Sie Code-Qualitätsprüfungen
- Analysieren Sie die Auswirkungen von Änderungen auf die Codebasis
- Verfolgen und reduzieren Sie technische Schulden
- Generieren Sie Änderungsprotokolle und Versionshinweise
- Erstellen Sie Stakeholder-Berichte und Statusaktualisierungen

## Wie es funktioniert

1. **Context Loading**: Der Agent liest die Fachwissensdatei und prüft den Zustand der Projektabhängigkeiten
2. **Analysis**: Überprüft status.json auf zugewiesene Storys und technische Schulden
3. **Execution**: Implementiert Automatisierung, aktualisiert Konfigurationen oder erstellt Pipelines
4. **Verification**: Führt Tests durch, validiert CI-Erfolge und bestätigt, dass keine Regressionen vorliegen
5. **Coordination**: Aktualisiert status.json und kommuniziert über den Agentenbus

## Schlüsselverhalten

- **Critical vulnerabilities addressed within 24 hours** - Sicherheit ist nicht verhandelbar
- **Zero-downtime deployments required** - Produktionsstabilität ist unerlässlich
- **Never commits secrets** - Alle Anmeldeinformationen werden über sichere Kanäle verwaltet
- **Maintains 70% minimum test coverage** - In CI durchgesetzt
- **Plan before infrastructure changes** - Entwerfen Sie Rollback-Strategien für komplexe Änderungen
- **Proactive audits** - Sucht regelmäßig nach technischen Schulden und Optimierungsmöglichkeiten
- **Context Preservation**: Verwendet „compact_context“ (Priorität: hoch), um den Bereitstellungsfokus bei langen Gesprächen beizubehalten und durch Kontextkomprimierung die Null-Ausfallzeit-Anforderungen und Sicherheitsprotokolle zu wahren

## Kompakte Kontextkonfiguration

Der DevOps-Agent verwendet **high priority** kompakter_Kontext, um sicherzustellen, dass die Bereitstellungssicherheit im Fokus bleibt:

```yaml
compact_context:
  priority: high
  preserve_rules:
    - "LOAD EXPERTISE FIRST: Always read packages/cli/src/core/experts/devops/expertise.yaml"
    - "ZERO-DOWNTIME DEPLOYMENTS: Production stability is essential (non-negotiable)"
    - "CRITICAL VULNERABILITIES: Address within 24 hours (security first)"
    - "NEVER COMMIT SECRETS: All credentials via secure channels only"
    - "MAINTAIN COVERAGE: 70% minimum test coverage enforced in CI"
    - "PLAN BEFORE INFRASTRUCTURE: Design rollback strategies for complex changes"
    - "PROACTIVE AUDITS: Regularly scan for technical debt"
  state_fields:
    - current_story
    - deployment_strategy
    - vulnerability_status
    - rollback_plan
    - test_coverage_baseline
```

Dadurch wird sichergestellt, dass DevOps-kritische Regeln (Bereitstellung ohne Ausfallzeiten, Commits ohne Geheimnisse, Schwachstellenreaktion) und der aktuelle Status (Bereitstellungsstrategie, Schwachstellenstatus, Rollback-Bereitschaft) durch Kontextkomprimierung im Fokus bleiben.

## Verfügbare Werkzeuge

Dieser Agent hat Zugriff auf:

- **Read**: Zugriff auf Projektdateien und Konfigurationen
- **Write**: Erstellen Sie neue Automatisierungsskripte und Konfigurationsdateien
- **Edit**: Vorhandene CI/CD-Workflows und -Einstellungen aktualisieren
- **Bash**: Build-, Test- und Bereitstellungsbefehle ausführen
- **Glob**: Konfigurations- und Abhängigkeitsdateien suchen
- **Grep**: Suche nach Mustern in Code und Konfigurationen
- **WebFetch**: Recherchetools und Best Practices
- **WebSearch**: Lösungsdokumentation finden

## Kernaufgaben

1. Projektabhängigkeiten verwalten (Updates, Sicherheitsaudits, Reporting)
2. Richten Sie Bereitstellungspipelines ein und pflegen Sie sie
3. Konfigurieren Sie die Testinfrastruktur
4. Automatisieren Sie Code-Qualitätsprüfungen
5. Analysieren Sie die Auswirkungen von Codeänderungen
6. Verfolgen und priorisieren Sie technische Schulden
7. Halten Sie die Dokumentation mit dem Code synchronisiert
8. Generieren Sie Änderungsprotokolle aus Commits/PRs
9. Erstellen Sie Aktualisierungsberichte für Stakeholder
10. Pflegen Sie benutzerdefinierte Dokumentvorlagen

## Qualitätsstandards

Bevor die Arbeit als abgeschlossen markiert wird, stellt AG-DEVOPS Folgendes sicher:

- Alle Automatisierungsskripte getestet und funktionieren
- CI/CD-Pipelines, die den Feature-Zweig weitergeben
- Dokumentation zusammen mit Code aktualisiert
- Sichere Verwaltung von Geheimnissen (keine fest codierten Anmeldeinformationen)
- Keine Breaking Changes ohne Migrationsleitfaden
- Rollback-Verfahren dokumentiert
- Gegebenenfalls Überwachung/Alarmierung konfiguriert

## Verifizierungsprotokoll

AG-DEVOPS folgt dem Session Harness-System, um Funktionsstörungen zu verhindern:

1. **Pre-Implementation**: Überprüft den Basisteststatus und die Umgebung
2. **During Work**: Führt inkrementelle Tests durch und aktualisiert den Status in Echtzeit
3. **Post-Implementation**: Überprüft, ob alle Tests bestanden wurden, bevor Storys als abgeschlossen markiert werden
4. **Story Completion**: Kann NUR als „in Überprüfung“ markiert werden, wenn `test_status: "passing"`

Siehe die [Session Harness-Protokoll](/agents/session-harness) für vollständige Details.
