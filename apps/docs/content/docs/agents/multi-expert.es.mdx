# Agente multiexperto

El agente multiexperto es un orquestador que analiza preguntas complejas mediante el despliegue de entre 3 y 5 expertos en dominios especializados en paralelo y luego sintetiza sus hallazgos en un análisis integral con puntaje de confianza.

## Capacidades

- Genera múltiples expertos en dominios especializados en paralelo para un análisis integral
- Detecta dominios relevantes a partir de preguntas (seguridad, API, UI, base de datos, pruebas, rendimiento, CI, DevOps)
- Sintetiza los hallazgos de expertos con puntuación de confianza (Alta/Media/Baja)
- Identifica acuerdos entre expertos (alta confianza) y desacuerdos (necesita revisión)
- Extrae conocimientos únicos de expertos individuales
- Proporciona acciones recomendadas priorizadas.

## Cuándo utilizar

Utilice este agente cuando:

- **Preguntas complejas que abarcan varios dominios** - "¿Es segura nuestra implementación de autenticación?" necesita seguridad + API + pruebas + perspectivas de la base de datos
- **Revisiones de seguridad** - Analizar código desde múltiples ángulos (seguridad, arquitectura, pruebas)
- **Arquitectorevisiones de arquitectura**: obtenga comentarios de expertos en el campo sobre las decisiones de diseño
- **Revisiones de relaciones públicas** - Revisión de código desde múltiples perspectivas para mayor confianza en la corrección
- **Análisis de flujo de un extremo a otro** - Seguimiento de solicitudes a través de múltiples capas con validación de expertos
- **Depuración de problemas difíciles** - Múltiples causas posibles que necesitan un análisis de expertos
- **Preguntas sobre "mejores prácticas"** - Necesita validación en varios dominios

## Cómo funciona

El agente sigue un flujo de trabajo estructurado:

```text
USER QUESTION
      ↓
┌─────────────────┐
│ 1. ANALYZE      │ → Identify 3-5 relevant domains
│ 2. DEPLOY       │ → Spawn experts IN PARALLEL
│ 3. COLLECT      │ → Wait for all results
│ 4. SYNTHESIZE   │ → Combine with confidence scoring
│ 5. REPORT       │ → High-confidence answer
└─────────────────┘
```

### Paso 1: Análisis de dominio

El agente analiza la pregunta en busca de palabras clave de dominio y selecciona de 3 a 5 expertos:
- **1 experto PRIMARIO** - Dominio más relevante
- **2-4 expertos de APOYO** - Proporcionar perspectivas adicionales

### Paso 2: Implementación en paralelo

Todos los expertos se implementan simultáneamente mediante ejecución en segundo plano para mayor eficiencia:

```bash
Task(
  description: "Security analysis",
  prompt: "FIRST: Read packages/cli/src/core/experts/security/expertise.yaml\n\nQUESTION: ...",
  subagent_type: "agileflow-security",
  run_in_background: true
)
```

### Paso 3: recopilar resultados

Utiliza `TaskOutput` con `block: true` para esperar todas las respuestas de los expertos.

### Paso 4: Sintetizar con confianza Puntuación

analiza all respuestas y hallazgos de los grupos:

- **ACUERDO (2+ expertos)** → Alta confianza
- **INFORMACIÓN ÚNICA (1 experto)** → Hallazgo notable
- **DESACUERDO** → Marcado para revisión humana

### Paso 5: Informe

Análisis estructurado de resultados con secciones:
- **Hallazgos clave (alta confianza)** - Acuerdos entre más de 2 expertos
- **Perspectivas únicas** - Hallazgos notables de expertos individuales
- **Desacuerdos (Necesita revisión)** - Opiniones contradictorias
- **Acciones recomendadas** - Próximos pasos priorizados

## Puntuación de confianza

| Nivel | Criterios |
|-------|----------|
| **Alto** | Más de 3 expertos están de acuerdo con la evidencia específica citada |
| **Medio** | 2 expertos coinciden con algunas evidencias |
| **Bajo** | 1 solo experto, necesita verificación |

## Detección de dominio

El agente asigna palabras clave de preguntas a expertos en el dominio:

| Palabras clave | Experto |
|----------|--------|
| base de datos, esquema, tabla, SQL, consulta | base de datos de flujo ágil |
| API, punto final, REST, ruta, controlador | api de flujo ágil |
| componente, interfaz de usuario, interfaz, estilo | interfaz de usuario ágil |
| test, especificación, cobertura, simulacro | pruebas de flujo ágil |
| seguridad, autenticación, JWT, vulnerabilidad | seguridad de flujo ágil |
| rendimiento, caché, optimizar, lento | rendimiento de flujo ágil |
| CI, flujo de trabajo, canalización, construcción | agileflow-ci |
| implementar, infraestructura, Docker | ágiles-devops |

## Ejemplo

**Pregunta del usuario**: "¿Es segura nuestra implementación de autenticación?"

**Respuesta del agente**:

```text
Analyzing for relevant domains...

Detected domains:
- Security (primary) - auth security analysis
- API (supporting) - implementation review
- Testing (supporting) - coverage verification
- Database (supporting) - credential storage

Deploying 4 experts in parallel...

## Multi-Expert Analysis: Authentication Security

**Confidence**: High (4 experts, strong agreement)

### Key Findings (High Confidence)
1. JWT tokens never expire - CRITICAL - Agreed by: Security, API
2. Auth middleware inconsistent - HIGH - Agreed by: Security, API

### Unique Insights
- Database: Password hashing is correct (bcrypt with proper salt rounds)
- Testing: Only 40% coverage on auth routes, missing integration tests

### Disagreements (Needs Review)
None - all experts aligned on security stance

### Recommended Actions
1. Add JWT expiration and refresh token rotation (CRITICAL)
2. Audit all routes for consistent middleware application (HIGH)
3. Increase test coverage to 80% for auth routes (MEDIUM)
4. Add integration tests for auth flow (MEDIUM)
```

## Comportamientos clave

- **SIEMPRE implementa expertos en paralelo** - Nunca de forma secuencial (eficiencia)
- **SIEMPRE implementa TODOS los expertos en un ÚNICO mensaje** - Llamadas de tareas por lotes juntas
- **NUNCA da una respuesta final sin sintetizar TODAS las respuestas de los expertos** - Todas las perspectivas importan
- **SIEMPRE incluye lectura experta en indicaciones de expertos** - `"FIRST: Read packages/cli/src/core/experts/{domain}/expertise.yaml"`
- **Implementa entre 3 y 5 expertos en total** - Ni menos ni más (síntesis óptima)
- **SIEMPRE usa TaskOutput con bloque: verdadero** - Para recopilar todos los resultados

## Herramientas disponibles

Este agente tiene acceso a:
- **Leer**: acceda a los archivos de experiencia del agente
- **Escribir** - Crear undocumentos de análisis
- **Editar** - Actualizar análisis existentes
- **Bash** - Ejecutar comandos de verificación
- **Glob** - Busca archivos relevantes
- **Grep** - Código de búsqueda y documentación
- **Tarea** - Implementar agentes expertos en paralelo
- **TaskOutput** - Recopila resultados de expertos

## Cuando NOT para usar

- **Preguntas sencillas de un solo dominio** - En su lugar, utilice el experto en dominio directo
- **Preguntas que requieren autoridad para tomar decisiones** - Multi-Expert solo proporciona análisis
- **Información confidencial** - Documente a los expertos involucrados antes de la implementación
