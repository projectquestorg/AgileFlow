# Agente Multi-Especialista

O Agente Multi-Especialista é um orquestrador que analisa questões complexas, implantando de 3 a 5 especialistas de domínio especializados em paralelo e, em seguida, sintetiza suas descobertas em uma análise abrangente e com pontuação de confiança.

## Capacidades

- Gera vários especialistas de domínio especializados em paralelo para análises abrangentes
- Detecta domínios relevantes a partir de questões (segurança, API, UI, banco de dados, testes, desempenho, CI, DevOps)
- Sintetiza descobertas de especialistas com pontuação de confiança (Alta/Média/Baixa)
- Identifica acordos entre especialistas (alta confiança) e divergências (precisa de revisão)
- Extrai insights exclusivos de especialistas individuais
- Fornece ações recomendadas priorizadas

## Quando usar

Use este agente quando:

- **Perguntas complexas que abrangem vários domínios** - "Nossa implementação de autenticação é segura?" precisa de segurança + API + testes + perspectivas de banco de dados
- **Revisões de segurança** - Analise o código de vários ângulos (segurança, arquitetura, testes)
- **Arquitetoavaliações de natureza** - Obtenha feedback de especialistas do domínio sobre decisões de design
- **Revisões de RP** - Revisão de código multiperspectiva para confiança na correção
- **Análise de fluxo ponta a ponta** - Rastreie solicitações através de múltiplas camadas com validação especializada
- **Depuração de problemas difíceis** - Múltiplas causas possíveis que precisam de análise especializada
- **Perguntas de "práticas recomendadas"** - Precisa de validação em vários domínios

## Como funciona

O agente segue um fluxo de trabalho estruturado:

```text
USER QUESTION
      ↓
┌─────────────────┐
│ 1. ANALYZE      │ → Identify 3-5 relevant domains
│ 2. DEPLOY       │ → Spawn experts IN PARALLEL
│ 3. COLLECT      │ → Wait for all results
│ 4. SYNTHESIZE   │ → Combine with confidence scoring
│ 5. REPORT       │ → High-confidence answer
└─────────────────┘
```

### Etapa 1: Análise de Domínio

O agente analisa a pergunta em busca de palavras-chave de domínio e seleciona de 3 a 5 especialistas:
- **1 especialista PRIMÁRIO** - Domínio mais relevante
- **2-4 especialistas em APOIO** - Fornecem perspectivas adicionais

### Etapa 2: implantar em paralelo

Todos os especialistas são implantados simultaneamente usando execução em segundo plano para maior eficiência:

```bash
Task(
  description: "Security analysis",
  prompt: "FIRST: Read packages/cli/src/core/experts/security/expertise.yaml\n\nQUESTION: ...",
  subagent_type: "agileflow-security",
  run_in_background: true
)
```

### Etapa 3: coletar resultados

Usa `TaskOutput` com `block: true` para aguardar todas as respostas dos especialistas.

### Etapa 4: Sintetizar com pontuação de confiança

Analisa tudol respostas e resultados dos grupos:

- **ACORDO (2+ especialistas)** → Alta confiança
- **INSIGHT ÚNICO (1 especialista)** → Descoberta notável
- **DESACORDO** → Sinalizado para revisão humana

### Etapa 5: Relatório

Produz análise estruturada com seções:
- **Principais conclusões (alta confiança)** - Acordos entre mais de 2 especialistas
- **Insights exclusivos** - Descobertas notáveis de especialistas individuais
- **Discordâncias (revisão necessária)** - Opiniões conflitantes
- **Ações recomendadas** - Próximas etapas priorizadas

## Pontuação de confiança

| Nível | Critérios |
|-------|----------|
| **Alto** | Mais de 3 especialistas concordam com evidências específicas citadas |
| **Médio** | 2 especialistas concordam com algumas evidências |
| **Baixo** | Apenas 1 perito, necessita de verificação |

## Detecção de domínio

O agente mapeia palavras-chave de perguntas para especialistas do domínio:

| Palavras-chave | Especialista |
|----------|--------|
| banco de dados, esquema, tabela, SQL, consulta | banco de dados ágilflow |
| API, endpoint, REST, rota, controlador | API Agileflow |
| componente, UI, frontend, estilo | ágilflow-ui |
| vocêst, especificação, cobertura, simulação | teste ágil de fluxo |
| segurança, autenticação, JWT, vulnerabilidade | ágilflow-segurança |
| desempenho, cache, otimização, lentidão | desempenho ágil do fluxo |
| CI, fluxo de trabalho, pipeline, construção | ágilflow-ci |
| implantar, infraestrutura, Docker | ágilflow-devops |

## Exemplo

**Pergunta do usuário**: "Nossa implementação de autenticação é segura?"

**Resposta do agente**:

```text
Analyzing for relevant domains...

Detected domains:
- Security (primary) - auth security analysis
- API (supporting) - implementation review
- Testing (supporting) - coverage verification
- Database (supporting) - credential storage

Deploying 4 experts in parallel...

## Multi-Expert Analysis: Authentication Security

**Confidence**: High (4 experts, strong agreement)

### Key Findings (High Confidence)
1. JWT tokens never expire - CRITICAL - Agreed by: Security, API
2. Auth middleware inconsistent - HIGH - Agreed by: Security, API

### Unique Insights
- Database: Password hashing is correct (bcrypt with proper salt rounds)
- Testing: Only 40% coverage on auth routes, missing integration tests

### Disagreements (Needs Review)
None - all experts aligned on security stance

### Recommended Actions
1. Add JWT expiration and refresh token rotation (CRITICAL)
2. Audit all routes for consistent middleware application (HIGH)
3. Increase test coverage to 80% for auth routes (MEDIUM)
4. Add integration tests for auth flow (MEDIUM)
```

## Principais comportamentos

- **SEMPRE implanta especialistas em paralelo** - Nunca sequencialmente (eficiência)
- **SEMPRE implanta TODOS os especialistas em uma ÚNICA mensagem** - Chamadas de tarefas em lote juntas
- **NUNCA dá a resposta final sem sintetizar TODAS as respostas dos especialistas** - Todas as perspectivas são importantes
- **SEMPRE inclui leitura especializada em instruções de especialistas** - `"FIRST: Read packages/cli/src/core/experts/{domain}/expertise.yaml"`
- **Implanta de 3 a 5 especialistas no total** - Nem menos, nem mais (síntese ideal)
- **SEMPRE usa TaskOutput com bloco: true** - Para coletar todos os resultados

## Ferramentas disponíveis

Este agente tem acesso a:
- **Ler** - Acesse arquivos de experiência do agente
- **Escrever** - Criar umdocumentos de análise
- **Editar** - Atualizar análises existentes
- **Bash** - Execute comandos de verificação
- **Glob** - Pesquise arquivos relevantes
- **Grep** - Pesquisa de código e documentação
- **Tarefa** – Implantar agentes especializados em paralelo
- **TaskOutput** - Colete resultados de especialistas

- **Modelo**: Claude Opus 4.5 (Soneto)
- **Raciocínio**: análise completa e complexa com múltiplas perspectivas
