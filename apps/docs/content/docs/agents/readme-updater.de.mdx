# README-Updater-Agent

Der README Updater Agent (AG-README-UPDATER) ist Ihr Spezialist für die Prüfung und Aktualisierung von README.md-Dateien in Ihrem Projekt. Dieser Agent stellt sicher, dass die Ordnerdokumentation aktuell bleibt, Inhalte genau beschreibt und eine klare Navigation zwischen verwandten Abschnitten ermöglicht.

## Fähigkeiten

- **README Auditing**: Scannen Sie Ordner und identifizieren Sie Dokumentationslücken, veraltete Informationen und fehlende Dateibeschreibungen
- **Content Documentation**: Dokumentieren Sie alle Dateien und Ordner in einem Ordner mit klaren Beschreibungen
- **Navigation Links**: Stellen Sie eine klare Navigation zwischen verwandten Ordnern und übergeordneten/untergeordneten Verzeichnissen sicher
- **Structure Standardization**: Wenden Sie ein einheitliches README-Format für alle Projektordner an
- **Broken Link Detection**: Identifizieren und beheben Sie Links, die auf nicht vorhandene Dateien oder Ordner verweisen
- **Folder-Specific Guidance**: Zweck und Konventionen für jeden Ordnertyp verstehen (00-Meta bis 10-Recherche)
- **Parallel Execution**: Arbeiten Sie unabhängig am zugewiesenen Ordner, wenn er mit dem Befehl /readme-sync erzeugt wird

## Wann zu verwenden

Verwenden Sie diesen Agenten, wenn Sie Folgendes benötigen:

- Prüfen Sie alle READMEs in Ihrem Projekt (via `/agileflow:readme-sync`)
- Aktualisieren Sie die README-Datei eines bestimmten Ordners, um den aktuellen Inhalt wiederzugeben
- Fügen Sie Dokumentation zu einem Ordner hinzu, in dem README.md fehlt
- Sorgen Sie für eine konsistente Navigation und ordnerübergreifende Verlinkung
- Beheben Sie defekte Dokumentationslinks
- Aktualisieren Sie die README-Datei, nachdem Sie Dateien zu einem Ordner hinzugefügt/entfernt haben
- Erstellen Sie Dokumentation für neue Projektordner
- Standardisieren Sie das README-Format teamübergreifend

## Wie es funktioniert

1. **Context Loading**: Agent liest die Fachwissensdatei und scannt den zugewiesenen Ordner (via `/readme-sync`)

2. **Audit**: Der Agent listet alle Dateien/Ordner im Zielordner auf und liest die aktuelle README-Datei (falls vorhanden).

3. **Gap Analysis**: Agent identifiziert:
   - Dateien, die nicht in der README-Datei dokumentiert sind
   - In README erwähnte Dateien, die nicht mehr existieren
   - Veraltete Beschreibungen oder Ordnerzweck
   - Defekte Navigationslinks
   - Es fehlt eine Anleitung zur Verwendung

4. **Planning**: Agent schlägt Verbesserungen vor:
   - Bessere Dokumentation der Ordnerstruktur
   - Aktualisierte Dateibeschreibungen
   - Navigationslinks korrigiert oder hinzugefügt
   - Neue Abschnitte (Verwendung, Bekannte Probleme, Nächste Schritte)

5. **Updating**: Der Agent verwendet das Bearbeitungstool (falls README vorhanden ist) oder das Schreibtool (falls eine neue Datei erstellt wird), um Verbesserungen anzuwenden

6. **Reporting**: Der Agent fasst zusammen, was hinzugefügt/aktualisiert/entfernt wurde

## Beispiel

```bash
# Single folder update
/agileflow:readme-updater
> "Update README for docs/02-practices/ folder"

# Parallel update of all folders
/agileflow:readme-sync
> "Audit and update all README.md files across docs/"
```

Der README-Updater-Agent würde:
1. Durchsuchen Sie docs/02-practices/ nach allen Dateien
2. Lesen Sie die aktuelle README.md
3. Identifizieren Sie fehlende Dateien (testing.md, security.md usw.)
4. Identifizieren Sie defekte Links
5. Aktualisieren Sie die README-Datei mit der vollständigen Dateiliste
6. Fügen Sie einen klaren Abschnitt „Verwendung“ hinzu
7. Korrigieren Sie Navigationslinks zu übergeordneten/verwandten Ordnern
8. Bericht: „Dokumente/02-practices/README.md aktualisiert – 3 Abschnitte hinzugefügt, 2 Links korrigiert, alle Dateien dokumentiert“

## Schlüsselverhalten

- **Complete File List**: Jede Datei im Ordner sollte in der README-Datei dokumentiert sein
- **Clear Descriptions**: Jede Dateibeschreibung erläutert den Zweck und wann sie verwendet werden soll
- **Navigation First**: READMEs bieten klare Pfade zu übergeordneten, verwandten und untergeordneten Ordnern
- **Current Information**: Wird automatisch aktualisiert, wenn sich der Ordnerinhalt ändert
- **Consistent Format**: Verwendet die Standard-README-Vorlage für alle Ordner
- **No Broken Links**: Alle ordnerübergreifenden Links werden überprüft
- **Helpful to New Users**: READMEs gehen davon aus, dass der Leser die Projektstruktur nicht kennt

## Verfügbare Werkzeuge

Dieser Agent hat Zugriff auf:
- **Bash**: Dateien/Ordner entdecken (Befehle ls, find, wc)
- **Read**: Lesen Sie die aktuelle README-Datei und verstehen Sie die Struktur
- **Edit**: Vorhandene README.md-Dateien aktualisieren
- **Write**: Neue README.md-Dateien erstellen
- **Glob**: Dateien nach Muster suchen
- **Grep**: Dokumentationsinhalt durchsuchen

## Ordnerspezifische Zwecke

- **Purpose**: AgileFlow-Systemdokumentation und Einrichtungshandbücher

## Audit-Checkliste

Überprüfen Sie vor Abschluss der README-Aktualisierung Folgendes:
- Zweck des Ordners klar erklärt (1-2 Sätze)
- Alle wichtigen Dateien mit Beschreibungen aufgelistet
- Navigationslinks sind aktuell und funktionieren
- Offene Fragen dokumentiert
- Nächste Schritte/TODOs aufgelistet
- Links zu verwandten Ordnern
- Keine defekten Referenzen
- Konsistente Formatierung mit anderen READMEs
- Aktualisiert mit aktuellen Ordnerinhalten
- Hilfreich für jemanden, der neu im Projekt ist

## Update-Prozess

**Step 1: Read Current README**
- If README.md exists, read it completely
- Verstehen Sie, was derzeit dokumentiert ist
- Beachten Sie, was veraltet ist oder fehlt

**Step 2: Scan Folder Contents**
- Listen Sie alle Dateien und Ordner auf
- Struktur und Organisation verstehen
- Identifizieren Sie neue Dateien, die nicht in der README-Datei enthalten sind
- Identifizieren Sie die erwähnten Dateien, die nicht mehr vorhanden sind

**Step 3: Identify Gaps**
- Was ist im Ordner, aber nicht dokumentiert?
- Welche Dokumentation ist veraltet?
- Sind Links noch gültig?
- Ist der Ordnerzweck noch korrekt?

**Step 4: Plan Updates**
- Bei Bedarf neu organisieren (bessere Struktur?)
- Beschreibungen aktualisieren
- Fehlende Dateien hinzufügen
- Entfernen Sie veraltete Inhalte
- Defekte Links reparieren

**Step 5: Apply Updates**
- Schreiben Sie README.md mit Verbesserungen neu
- Standardformat verwenden (siehe Vorlage oben)
- Bleiben Sie konsistent mit anderen READMEs
- Stellen Sie sicher, dass alle Links funktionieren

**Step 6: Report**
- Zusammenfassung dessen, was aktualisiert wurde
- Zur Dokumentation hinzugefügte Dateien
- Dateien aus der Dokumentation entfernt
- Strukturelle Verbesserungen vorgenommen

## Parallele Ausführung

Wann `/agileflow:readme-sync` läuft:
- 11 Agenten erscheinen gleichzeitig (einer pro Ordner)
- Jeder Agent arbeitet unabhängig an seinem Ordner
- Alle Updates erfolgen parallel
- Die Ergebnisse werden gesammelt und zurückgemeldet

Jeder Agent:
- Wartet NICHT auf andere Agenten
- Konzentriert sich nur auf den zugewiesenen Ordner
- Arbeitet auf Hochtouren
- Lässt sich selbstständig abschließen

## Gemeinsame Muster

**Navigation Web**
- Jede README-Datei enthält Links zu Eltern, Geschwistern und Kindern
- Leser können ganze Dokumente/Strukturen durchsuchen
- Keine Sackgassen oder unterbrochenen Wege

**Descriptive Filenames**
- Dateibeschreibungen erklären den ZWECK, nicht nur WAS
- „Codierungsstandards für UI-Komponenten“ vs. nur „UI-Standards“
- „Begründung der Test-Framework-Auswahl“ vs. nur „testing-adr“

**Clear Structure**
- Der Abschnitt „Inhalt“ listet alle Dateien auf
- Im Abschnitt „Schlüsseldateien“ werden die wichtigsten erläutert
- Der Abschnitt „Verwendung“ enthält eine Schritt-für-Schritt-Anleitung
- Der Abschnitt „Standards“ dokumentiert Konventionen

**Maintenance Trigger**
- Nach dem Hinzufügen von Dateien zum Ordner
- Nach dem Verschieben/Umbenennen von Dateien
- Nach dem Löschen/Archivieren von Dateien
- Bevor der Ordner als „vollständig“ markiert wird
- Bei vierteljährlichen Dokumentationsprüfungen

**Consistency Rules**
- Gleiche Header in allen READMEs
- Gleiche Formatierung für Links und Code
- Gleiche Benennung für Navigationsabschnitte
- Links zum übergeordneten Element verwenden immer ../README.md
- Links zu Geschwistern verwenden ../sibling-folder/README.md
