# وكيل تحديث التمهيدي

وكيل تحديث README (AG-README-UPDATER) هو المتخصص في تدقيق وتحديث ملفات README.md عبر مشروعك. يضمن هذا الوكيل بقاء وثائق المجلد محدثة، ويصف المحتويات بدقة، ويوفر تنقلًا واضحًا بين الأقسام ذات الصلة.

## القدرات

- **تدقيق README**: فحص المجلدات وتحديد فجوات الوثائق والمعلومات القديمة وأوصاف الملفات المفقودة
- **توثيق المحتوى**: قم بتوثيق جميع الملفات والمجلدات في مجلد بأوصاف واضحة
- **روابط التنقل**: تأكد من التنقل الواضح بين المجلدات ذات الصلة وأدلة الأصل/الفرع
- **توحيد الهيكل**: قم بتطبيق تنسيق README المتسق عبر جميع مجلدات المشروع
- **اكتشاف الروابط المعطلة**: تحديد الروابط التي تشير إلى ملفات أو مجلدات غير موجودة وإصلاحها
- **إرشادات خاصة بالمجلدات**: فهم الغرض والاصطلاحات لكل نوع مجلد (00-meta حتى 10-research)
- **التنفيذ الموازي**: العمل بشكل مستقل على الملف المعينder عندما يتم إنتاجه بواسطة الأمر /readme-sync

## متى يستخدم

استخدم هذا الوكيل عندما تحتاج إلى:

- قم بمراجعة كافة المستندات التمهيدية عبر مشروعك (عبر `/agileflow:readme-sync`)
- قم بتحديث الملف التمهيدي الخاص بمجلد معين ليعكس المحتويات الحالية
- أضف وثائق إلى مجلد يفتقد README.md
- ضمان التنقل المتسق والربط بين المجلدات
- إصلاح روابط الوثائق المعطلة
- قم بتحديث الملف التمهيدي (README) بعد إضافة/إزالة الملفات من المجلد
- إنشاء وثائق لمجلدات المشروع الجديدة
- توحيد تنسيق README عبر الفرق

## كيف يعمل

1. **تحميل السياق**: يقرأ الوكيل ملف الخبرة ويفحص المجلد المخصص (عبر `/readme-sync`)

2. **التدقيق**: يسرد الوكيل كافة الملفات/المجلدات الموجودة في المجلد الهدف ويقرأ الملف التمهيدي الحالي (إذا كان موجودًا)

3. **تحليل الفجوة**: يحدد الوكيل ما يلي:
   - الملفات غير موثقة في README
   - الملفات المذكورة في الملف README لم تعد موجودة
   - الأوصاف القديمة أو غرض المجلد
   - روابط التنقل المعطلة
   - إرشادات "كيفية الاستخدام" مفقودة

4. **التخطيط**: يقترح الوكيلالتحسينات:
   - توثيق أفضل لبنية المجلد
   - تحديث أوصاف الملفات
   - روابط التنقل الثابتة أو المضافة
   - أقسام جديدة (كيفية الاستخدام، المشكلات المعروفة، الخطوات التالية)

5. **التحديث**: يستخدم الوكيل أداة التحرير (في حالة وجود ملف README) أو أداة الكتابة (في حالة إنشاء ملف جديد) لتطبيق التحسينات

6. **التقارير**: يلخص الوكيل ما تمت إضافته/تحديثه/إزالته

8. التقرير: "تم تحديث docs/02-practices/README.md - تمت إضافة 3 أقسام وإصلاح رابطين وتوثيق جميع الملفات"

## السلوكيات الرئيسية

- **قائمة الملفات الكاملة**: يجب توثيق كل ملف في المجلد في ملف README
- **أوصاف واضحة**: يشرح كل وصف ملف الغرض منه ومتى يتم استخدامه
- ** الملاحةأولاً**: توفر ملفات README مسارات واضحة للمجلدات الأصلية والمجلدات ذات الصلة والفرعية
- **المعلومات الحالية**: يتم تحديثها تلقائيًا عند تغيير محتويات المجلد
- **تنسيق متسق**: يستخدم قالب README القياسي في كافة المجلدات
- **لا توجد روابط معطلة**: تم التحقق من جميع الروابط المشتركة بين المجلدات
- **مفيدة للمستخدمين الجدد**: تفترض ملفات README أن القارئ لا يعرف بنية المشروع

- **Bash**: اكتشاف الملفات/المجلدات (أوامر ls وfind وwc)

## قائمة التدقيق

قبل إكمال تحديث الملف التمهيدي (README)، تحقق مما يلي:
- شرح الغرض من المجلد بوضوح (1-2 جمل)
- جميع الملفات الرئيسية المدرجة مع الأوصاف
- روابط التصفح الحالية والعاملة
- الأسئلة المفتوحة موثقة
- الخطوات التالية/المهام المدرجة
- روابط للمجلدات ذات الصلة
- لا توجد مراجع مكسورة
- التنسيق المتسق مع ملفات README الأخرى
- ما يصل إلى موعد مع محتويات المجلد الحالي
- مفيد لشخص جديد في المشروع

## عملية التحديث

**الخطوة 1: قراءة الملف التمهيدي الحالي**
- إذا كان الملف README.md موجودًا، فاقرأه بالكامل
- فهم ما تم توثيقه حاليا
- لاحظ ما هو قديم أو مفقود

**الخطوة 2: مسح محتويات المجلد**
- قائمة جميع fالملفات والمجلدات
- فهم الهيكل والتنظيم
- تحديد الملفات الجديدة غير الموجودة في README
- تحديد الملفات المذكورة التي لم تعد موجودة

**الخطوة 3: تحديد الفجوات**
- ما الذي يوجد في المجلد ولكن غير موثق؟
- ما هي الوثائق التي عفا عليها الزمن؟
- هل الروابط لا تزال صالحة؟
- هل لا يزال غرض المجلد دقيقًا؟

**الخطوة 4: تحديثات الخطة**
- إعادة التنظيم إذا لزم الأمر (هيكل أفضل؟)
- تحديث الأوصاف
- إضافة الملفات المفقودة
- إزالة المحتوى القديم
- إصلاح الروابط المعطلة

**الخطوة 5: تطبيق التحديثات**
- إعادة كتابة README.md مع التحسينات
- استخدم التنسيق القياسي (انظر القالب أعلاه)
- حافظ على الاتساق مع ملفات README الأخرى
- التأكد من عمل جميع الروابط

**الخطوة 6: الإبلاغ**
- ملخص ما تم تحديثه
- الملفات المضافة إلى الوثائق
- تمت إزالة الملفات من الوثائق
- التحسينات الهيكلية التي تم إجراؤها

## الأنماط الشائعة

** شبكة الملاحة **
- يرتبط كل ملف README بالوالدين والأشقاء والأطفال
- يمكن للقراء اجتياز المستندات/البنية بأكملها
- لا توجد طرق مسدودة أو مسارات مقطوعة

**أسماء الملفات الوصفية**
- أوصاف الملف تشرح الغرض، وليس فقط ماذا
- "معايير الترميز لمكونات واجهة المستخدم" مقابل "معايير واجهة المستخدم" فقط
- "الأساس المنطقي لاختيار إطار الاختبار" مقابل "test-adr" فقط

**هيكل واضح**
- قسم المحتويات يسرد كافة الملفات
- قسم الملفات الرئيسية يشرح أهمها
- يقدم قسم كيفية الاستخدام إرشادات خطوة بخطوة
- قسم وثائق الاتفاقيات القياسية

** مشغل الصيانة **
- بعد إضافة الملفات إلى المجلد
- بعد نقل/إعادة تسمية الملفات
- بعد حذف/أرشفة الملفات
- قبل وضع علامة على المجلد بأنه "مكتمل"
-خلال عمليات تدقيق الوثائق الفصلية

**قواعد الاتساق**
- نفس الرؤوس في كافة ملفات README
- نفس التنسيق للروابط والتعليمات البرمجية
- نفس التسمية لأقسام التنقل
- وصلات إلى الوالدين دائمااستخدم ../README.md
- روابط للأشقاء تستخدم ../sibling-folder/README.md
