# Agente de investigación

El Agente de Investigación (RESEARCH) es su especialista en investigación técnica, gestión del conocimiento y creación de propuestas de investigación integrales. Este agente recopila información de fuentes autorizadas, sintetiza los hallazgos y mantiene un índice de búsqueda de investigaciones para referencia futura.

## Capacidades

- **Investigación web**: busque documentación oficial y fuentes autorizadas para obtener información técnica
- **Creación de mensajes de ChatGPT**: cree mensajes de investigación integrales solicitando planes de implementación, fragmentos de código, borradores de ADR, desgloses de historias y más.
- **Curación de notas de investigación**: guarde los resultados de la investigación estructurados en docs/10-research/ con un formato coherente
- **Mantenimiento del índice de investigación**: mantenga docs/10-research/README.md actualizado primero con las investigaciones más recientes
- **Detección de investigaciones obsoletas**: identifique investigaciones con más de 90 días de antigüedad que puedan necesitar actualización
- **Identificación de brechas de investigación**: marcar investigaciones faltantes para decisiones técnicas y ADR
- **Coordinación de agentes**: Noificar los agentes solicitantes cuando se complete la investigación a través de mensajes de bus

## Cuándo utilizar

Utilice este agente cuando necesite:

- Investigar un tema técnico o una elección tecnológica.
- Comprender las compensaciones entre diferentes enfoques.
- Cree un mensaje ChatGPT completo para un análisis más profundo
- Documentar los hallazgos en formato estructurado para referencia futura.
- Actualizar resultados de investigaciones obsoletos (más de 90 días)
- Encontrar qué investigaciones ya existen antes de tomar decisiones.
- Apoyar la creación de ADR con investigación de antecedentes técnicos.
- Obtenga orientación de implementación para un enfoque de funciones

## Cómo funciona

### Flujo de trabajo de investigación web

1. **Comprensión**: el agente aclara la pregunta de investigación y las limitaciones.
2. **Buscar**: el agente busca documentación oficial y fuentes autorizadas (WebSearch, WebFetch)
3. **Recopilación**: el agente recopila hallazgos clave: enfoques, compensaciones, mejores prácticas y puntos de referencia.
4. **Síntesis**: el agente organiza los hallazgos en un formato estructurado.
5. **Guardando**: el agente guarda la investigación en `docs/10-research/<YYYYMMDD>-<slug>.md`
6. **Indexación**: el agente actualiza docs/10-research/README.md con una entrada en la tabla cronológica
7. **Coordinación**: el agente publica un mensaje de autobús para notificar al agente solicitante

### Flujo de trabajo de solicitud de investigación de ChatGPT

1. **Carga de contexto**: el agente lee CLAUDE.md, context.md, status.json, ADR relacionados e investigaciones existentes para evitar duplicaciones.
2. **Comprensión**: el agente aclara el tema de investigación y las preguntas específicas que necesitan respuestas.
3. **Creación de avisos**: el agente crea un aviso completo solicitando:
   - TL; resumen DR
   - Plan de implementación paso a paso con rutas de archivos
   - Fragmentos de código ejecutables mínimos (listo para copiar y pegar)
   - Configuración y configuración del entorno.
   - Patrones de manejo de errores
   - Ganchos de observabilidad y análisis.
   - Ejemplos de prueba (unitario, integración, E2E)
   - Lista de verificación de pruebas manuales
   - Consideraciones de seguridad y privacidad.
   - Borrador de ADR con opciones y pros/contras
   - Desglose de la historia con criterios de aceptación Dado/Cuándo/Entonces
   - Plan de reversión y riesgos.
   - Plantilla del cuerpo PRcomió
   - Fuentes de documentación oficiales únicamente (citadas con título, URL y fecha)

4. **Salida**: El agente proporciona un mensaje como bloque de código para copiar y pegar fácilmente
5. **Procesando**: el usuario pega los resultados de ChatGPT a Claude
6. **Guardando**: el agente guarda los resultados en `docs/10-research/<YYYYMMDD>-<slug>.md`
7. **Indexación**: el agente actualiza el índice de investigación
8. **Notificación**: el agente notifica al agente solicitante mediante un mensaje de bus

## Ejemplo

```bash
# Via /babysit
/agileflow:babysit
> "Need to research JWT vs OAuth2 for authentication"
```

El agente de investigación:
1. Busque fuentes autorizadas (JWT RFC, especificaciones OAuth2, artículos de security.com)
2. Reúna los hallazgos clave (sin estado versus con estado, tokens de actualización, delegación, casos de uso)
3. Cree una nota de investigación que compare enfoques.
4. Guardar en docs/10-research/20251021-jwt-vs-oauth2.md
5. Actualizar el índice de investigación
6. Publicar mensaje de bus: "Investigación completa: JWT vs OAuth2 guardado en docs/10-research/20251021-jwt-vs-oauth2.md"

## Comportamientos clave

- **Solo fuentes oficiales**: la investigación cita documentación oficial, no publicaciones de blog aleatorias
- **Indicaciones completas**: las indicaciones de ChatGPT son completasgh, solicitando detalles de implementación, pruebas, consideraciones de seguridad y borradores de ADR
- **Formato estructurado**: toda la investigación sigue una estructura coherente (resumen, hallazgos, enfoque recomendado, compensaciones, fuentes)
- **Detección obsoleta**: identifica investigaciones con más de 90 días de antigüedad y señala una posible actualización cuando la tecnología ha cambiado
- **Sin duplicación**: verifica la investigación existente antes de comenzar una nueva investigación.
- **Coordinación de agentes**: notifica a los agentes solicitantes inmediatamente cuando se completa la investigación.
- **Borrar abastecimiento**: todas las investigaciones citan fuentes con título, URL y fecha de recuperación.

## Herramientas disponibles

Este agente tiene acceso a:
- **Leer**: explore archivos de investigación y comprenda los hallazgos existentes
- **Escribir**: crear nuevas notas de investigación
- **Editar**: actualizar las notas de investigación existentes
- **Glob**: busque archivos de investigación por patrón
- **Grep**: buscar contenido de investigación
- **WebSearch**: busca en la web información técnica
- **WebFetch**: busca y analiza páginas web

## Formato del índice de investigación

docs/10-research/README.md mantiene una tabla cronológica (la más reciente primero):

```markdown
| Date       | Topic                        | Path                        | Summary                     |
|------------|------------------------------|-----------------------------|-----------------------------|
| 2025-10-16 | JWT authentication patterns  | 20251016-jwt-auth.md        | Compared JWT, session, OAuth2 |
| 2025-10-15 | React state management       | 20251015-react-state.md     | Redux vs Zustand vs Context |
```

## Lista de verificación de calidad

Antes de guardar la investigación:
- La fecha es actual (formato AAAA-MM-DD)
- El resumen es conciso (2-3 oraciones)
- Se enumeran al menos 3 hallazgos clave
- Enfoque recomendado expresado claramente
- Fuentes citadas con URL y fechas de recuperación.
- ADR/historias/épicas relacionadas vinculadas
- Índice actualizado con la entrada más reciente en la parte superior
- Todos los enlaces externos son a documentación oficial.

## Identificar lagunas en la investigación

El Agente de Investigación señala lagunas en la investigación cuando:
- La elección de tecnología aún no se ha investigado (se necesita antes de la ADR)
- Incertidumbre de aproximación mencionada en las notas de la historia.
- Se discuten múltiples enfoques sin un ganador claro
- La ADR existe pero carece de investigaciones que la respalden.
- La investigación está obsoleta (más de 90 días y la tecnología ha cambiado)
- Se necesita investigación duplicada (versiones tecnológicas actualizadas)

## Coordinación con Otros Agentes

Las solicitudes de investigación comúnmente provienen de:
- **AG-UI**: Diseño sSistemas, patrones de componentes, técnicas de accesibilidad.
- **AG-API**: arquitecturas API, diseños de bases de datos, patrones de autenticación
- **AG-CI**: marcos de prueba, plataformas de CI, herramientas de calidad de código
- **AG-DEVOPS**: estrategias de implementación, orquestación de contenedores, monitoreo
- **ADR-WRITER**: Alternativas técnicas antes de tomar decisiones
- **EPIC-PLANNER**: investigación de la pila de tecnología antes de planificar epopeyas

Cuando se solicita investigación:
1. Consulte docs/09-agents/bus/log.jsonl para ver el mensaje de solicitud de investigación.
2. Completar la investigación (mensaje web o ChatGPT)
3. Guarde los resultados en docs/10-research/
4. Actualizar el índice de investigación.
5. Publicar mensaje de bus para notificar al solicitante

## Plantilla de mensaje de ChatGPT

Al crear un mensaje de investigación, incluya:

```text
Context: [Project overview from context.md]
Current priority: [From status.json]
Related ADRs: [List relevant decisions]

Research topic: [Specific question]

Please provide:
1. TL;DR (2-3 sentences)
2. Step-by-step implementation plan with exact file paths
3. Minimal runnable code snippets (copy-paste ready)
4. Configuration (env vars, config files, CLI flags)
5. Error handling patterns
6. Observability (logging, metrics, tracing)
7. Tests: unit, integration, E2E examples
8. Manual testing checklist
9. Security checklist (auth, input validation, secrets, etc.)
10. Privacy considerations (PII, GDPR, data retention)
11. ADR draft with context, options, recommendation, consequences
12. Story breakdown (3-6 stories with Given/When/Then AC)
13. Rollback plan
14. Risks and gotchas
15. PR body template
16. Sources: official docs/repos only with title, URL, date

Final checklist:
- [ ] Ready to paste back to Claude for implementation
- [ ] All code snippets tested conceptually
- [ ] Risks clearly flagged
```

## Patrones comunes

**Dos flujos de trabajo de investigación**
- **Investigación web**: búsqueda directa y síntesis (más rápida para temas sencillos)
- **Mensaje de ChatGPT**: cree un mensaje completo para que la IA lo analice (mejor para decisiones complejas)

**Investigar antes de tomar decisiones**
- Siempre investigue antes de crear ADR (ADR-WRITER solicita investigación)
- Siempre investiga antes de planificar épicas (EPIC-PLANNER solicita investigación)
- Señalar la incertidumbre en las historias y la investigación antes de la implementación.

**Gestión de investigación obsoleta**
- Auditoría trimestral de docs/10-research/ para investigaciones antiguas
- Marcar como "Reemplazado" si una nueva investigación contradice la anterior
- Nunca elimine la investigación (consérvela para el contexto histórico)
- Actualizar si las versiones técnicas han cambiado.
