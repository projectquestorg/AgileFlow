# Spécialiste mobile

AG-MOBILE est le spécialiste du développement mobile multiplateforme, gérant la mise en œuvre de React Native et Flutter, l'intégration de modules natifs et l'optimisation spécifique au mobile.

## Capacités

- **React Native & Flutter Development** : écrivez une fois, déployez sur iOS et Android
- **Intégration du module natif** : caméra, localisation, notifications, capteurs, stockage
- **Modèles d'interface utilisateur mobile** : navigation par onglets, navigation par pile, modaux, gestes, conception réactive
- **Optimisation des performances** : ciblage de la taille du bundle, gestion de la mémoire, optimisation de la batterie
- **Tests mobiles** : tests d'appareils, tests d'émulateurs, tests de réseau lents
- **Platform Abstraction** : gérez les différences iOS/Android avec des composants réutilisables
- **Mobile Analytics & Crash Reporting** : Sentry, Bugsnag, surveillance des performances

## Quand l'utiliser

Utilisez AG-MOBILE lorsque :

- Implémentation de fonctionnalités mobiles multiplateformes dans React Native ou Flutter
- Construire des intégrations de modules natifs (caméra, localisation, capteurs)
- Optimisationg performances mobiles (taille du bundle, mémoire, batterie)
- Gestion du code et des différences spécifiques à la plateforme
- Tests sur des appareils réels
- Implémentation de modèles d'interface utilisateur spécifiques aux mobiles (onglets inférieurs, modaux)
- Mise en place du déploiement de l'App Store

## Comment ça marche

1. **Chargement du contexte** : lit le fichier d'expertise et la stratégie mobile du projet (React Native vs Flutter)
2. **Analyse de plate-forme** : détermine les différences de plate-forme existantes et comment les résumer
3. **Conception et mise en œuvre** : crée des composants multiplateformes avec des couches d'abstraction appropriées
4. **Tests** : tests sur les appareils iOS et Android (pas seulement sur les émulateurs)
5. **Optimisation** : mesure et optimise la taille du bundle, la mémoire et l'utilisation de la batterie.
6. **Coordination** : met à jour status.json et se coordonne avec AG-UI sur les composants partagés

## Exemple

```bash
# Via /babysit
/agileflow:babysit
> "I need to add camera functionality to the mobile app that works on iOS and Android"

# AG-MOBILE will:
# 1. Research platform-specific camera APIs
# 2. Create abstraction layer for camera access
# 3. Implement on React Native (or Flutter)
# 4. Test on real iOS device
# 5. Test on real Android device
# 6. Document setup and usage
```

## Comportements clés

- **Platform Abstraction** : n'écrit jamais de code spécifique à la plate-forme sans couche d'abstraction
- **Tests d'appareils réels** : teste toujours sur de vrais appareils, pas seulement sur des émulateurs
-**Budgets de performances** : reste dans les limites des objectifs de performances (batterie, mémoire, taille du bundle)
- **Modèles mobiles** : utilise des modèles d'interface utilisateur mobile appropriés (onglets inférieurs, pas barres latérales)
- **Parité multiplateforme** : garantit que la fonctionnalité fonctionne de manière identique sur iOS et Android

## Plateformes mobiles

**Réagir natif** :
- JavaScript/TypeScript + modules natifs
- Workflows expo (gérés) ou nus (non gérés)
- Partage des connaissances React avec l'équipe web

**Flottement** :
- Langage Dart avec les widgets Material Design et Cupertino
- Rechargement à chaud pour un développement rapide
- Fortes caractéristiques de performance

## Performances mobiles

Ciblez ces métriques :

- **Taille du bundle** : moins de 2 Mo pour des téléchargements rapides
- **Mémoire** : évitez les objets volumineux, nettoyez correctement (2 à 6 Go de RAM typique)
- **Batterie** : minimisez l'utilisation du réseau, du processeur et de l'écran
- **Données** : compresser les images, limiter les requêtes
- **Réseau** : test sur des réseaux lents/peu fiables

## QualitéListe de contrôle de la ville

Avant de marquer le travail mobile terminé :

- Implémenté sur iOS et Android
- Modèles UX mobiles appropriés
- Flux de navigation testés
- Gestes gérés correctement
- Code spécifique à la plate-forme extrait
- Modules natifs (le cas échéant) intégrés
- Objectifs de performances atteints (taille du bundle, mémoire)
- Testé sur des appareils réels
- Testé sur réseau lent
- Exigences de l'App Store satisfaites (icônes, écrans de démarrage)
- Tests réussis avec test_status : "passing"
