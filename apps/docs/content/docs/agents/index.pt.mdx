# Agentes

O AgileFlow inclui **30 agentes especializados**, cada um especialista em seu domínio. Os agentes podem ser gerados pelo mentor __NOTRASLATE_6__ ou invocados diretamente usando a ferramenta Tarefa.

## Como funcionam os agentes

1. **Expertise First**: cada agente carrega seu arquivo de expertise para entender seus padrões de base de código
2. **Contextualizado**: os agentes leem documentos do projeto, status.json e alterações recentes
3. **Preservação de contexto**: agentes aprimorados usam a configuração compact_context para preservar o conhecimento crítico durante a compactação da conversa
4. **Colaborativo**: os agentes se comunicam por meio do barramento de mensagens e atualizam o status da história
5. **Autoaperfeiçoamento**: os agentes atualizam seus conhecimentos após concluir o trabalho

## Recurso de contexto compacto

Para manter o foco durante longas conversas, os agentes AgileFlow usam a configuração **compact_context** para preservar informações críticas quando o contexto é compactado automaticamente. Isso garante que os agentes não percam regras, estados e prioridades importantes no meio da conversa.

### Como funciona

Cadaagente tem uma configuração __NOTRASLATE_7__ com três componentes:

**Nível de prioridade**: determina quão agressivamente o contexto do agente é preservado
- `critical` - Sempre preservado (agentes de maior prioridade como mentor, api, ui)
- `high` - Preservado na maioria das compactações (agentes importantes)
- __NOTRASLATE_10__ - Preservado quando possível (agentes padrão)
- __NOTRASLATE_11__ - Primeiro a ser removido (raro, usado para agentes consultivos)

**Preservar regras**: regras comportamentais básicas que nunca devem ser perdidas
- Identidade e função do agente
- Restrições críticas e regras de segurança
- Procedimentos obrigatórios (por exemplo, "Carregar experiência primeiro")
- Requisitos de gestão do estado
- Protocolos de coordenação

**Campos de estado**: estado dinâmico que muda durante o trabalho
- História atual sendo implementada (US-#### ID)
- Progresso da tarefa e porcentagem de conclusão
- Bloqueadores e dependências conhecidos
- Próximas ações priorizadas
- Status de teste/construção

### Exemplo de configuração

```yaml
compact_context:
  priority: critical
  preserve_rules:
    - "LOAD EXPERTISE FIRST: Read packages/cli/src/core/experts/api/expertise.yaml"
    - "VERIFY TEST BASELINE: Check test_status before starting"
    - "PRIORITIZE AG-UI BLOCKERS: Unblock UI stories waiting on API endpoints"
    - "DIFF-FIRST APPROACH: Show edits with confirmation before applying"
  state_fields:
    - current_story
    - endpoints_implemented
    - blocked_ui_stories
    - test_status_baseline
```

Quando Claude Code comprime contexto, ele preserva essas regras e campos de estado para todos os agentes, garantindo continuidade mesmo após múltiplas rodadas de compactação de conversação.

### Distribuição prioritária do agente

- **Crítico** (2 agentes): mentor, API
- **Alto** (9 agentes): interface do usuário, banco de dados, ci, devops, testes, segurança, documentação, desempenho, consulta à base de código
- **Médio** (15 agentes): migração de dados, controle de qualidade, acessibilidade, monitoramento, pesquisa, redator de anúncios, análises, integrações, dispositivos móveis, design, conformidade, refatoração, produto, atualizador readme
- **Baixo** (2 agentes): planejador épico, orquestrador

Essa abordagem em camadas garante que os agentes mais críticos sempre mantenham o foco, enquanto outros degradam o contexto quando necessário.

## Agentes de Geração

### Via /babysit (recomendado)

O mentor __NOTRASLATE_39__ detecta automaticamente qual agente é necessário:

```bash
/agileflow:babysit
# Then describe your task - babysit will spawn the right agent
```

### Desova Direta

Você também pode spawn agentes diretamente usando a ferramenta Tarefa:

```text
Task(
  description: "Design user sessions schema",
  prompt: "Create a sessions table to track user logins with proper indexes",
  subagent_type: "agileflow-database"
)
```

## Melhores práticas

1. **Deixe a babá escolher**: O mentor sabe qual agente se adapta melhor
2. **Forneça contexto**: informe aos agentes sobre os padrões existentes em sua base de código
3. **Revisar atualizações**: verifique as alterações do status.json que os agentes fazem
4. **Use vários especialistas para incertezas**: quando você precisar de alta confiança, implante vários especialistas
