# Agente atualizador README

O README Updater Agent (AG-README-UPDATER) é o seu especialista em auditoria e atualização de arquivos README.md em seu projeto. Este agente garante que a documentação da pasta permaneça atualizada, descreva o conteúdo com precisão e forneça uma navegação clara entre as seções relacionadas.

## Capacidades

- **Auditoria README**: verifique pastas e identifique lacunas na documentação, informações desatualizadas e descrições de arquivos ausentes
- **Documentação de conteúdo**: documente todos os arquivos e pastas em uma pasta com descrições claras
- **Links de navegação**: Garanta uma navegação clara entre pastas relacionadas e diretórios pai/filho
- **Padronização de estrutura**: aplique formato README consistente em todas as pastas do projeto
- **Detecção de link quebrado**: identifique e corrija links que apontam para arquivos ou pastas inexistentes
- **Orientação específica para pastas**: entenda a finalidade e as convenções de cada tipo de pasta (00-meta a 10-research)
- **Execução Paralela**: Trabalhe de forma independente nas tarefas atribuídasder quando gerado pelo comando /readme-sync

## Quando usar

Use este agente quando precisar:

- Audite todos os READMEs do seu projeto (via __NOTRASLATE_4__)
- Atualize o README de uma pasta específica para refletir o conteúdo atual
- Adicione documentação a uma pasta que está faltando README.md
- Garanta uma navegação consistente e links entre pastas
- Corrigir links de documentação quebrados
- Atualize o README após adicionar/remover arquivos de uma pasta
- Criar documentação para novas pastas de projetos
- Padronize o formato README entre as equipes

## Como funciona

1. **Carregamento de contexto**: o agente lê o arquivo de experiência e verifica a pasta atribuída (via `/readme-sync`)

2. **Auditoria**: o agente lista todos os arquivos/pastas na pasta de destino e lê o README atual (se existir)

3. **Análise de lacunas**: o agente identifica:
   - Arquivos não documentados no README
   - Arquivos mencionados no README que não existem mais
   - Descrições desatualizadas ou finalidade da pasta
   - Links de navegação quebrados
   - Falta de orientação "como usar"

4. **Planejamento**: proposta do agentemelhorias:
   - Melhor documentação da estrutura de pastas
   - Descrições de arquivos atualizadas
   - Links de navegação corrigidos ou adicionados
   - Novas seções (como usar, problemas conhecidos, próximas etapas)

5. **Atualização**: O agente usa a ferramenta Editar (se existir README) ou a ferramenta Escrever (se estiver criando uma nova) para aplicar melhorias

6. **Relatórios**: o agente resume o que foi adicionado/atualizado/removido

## Principais comportamentos

- **Lista completa de arquivos**: Cada arquivo na pasta deve ser documentado no README
- **Descrições claras**: cada descrição de arquivo explica a finalidade e quando usá-lo
- ** NavegaçãoPrimeiro**: READMEs fornecem caminhos claros para pastas pai, relacionadas e filhas
- **Informações atuais**: atualiza automaticamente quando o conteúdo da pasta muda
- **Formato consistente**: usa o modelo README padrão em todas as pastas
- **Sem links quebrados**: todos os links entre pastas são verificados
- **Útil para novos usuários**: READMEs presumem que o leitor não conhece a estrutura do projeto

## Ferramentas disponíveis

Este agente tem acesso a:
- **Bash**: Descubra arquivos/pastas (comandos ls, find, wc)
- **Leitura**: Leia o README atual e entenda a estrutura
- **Editar**: Atualizar arquivos README.md existentes
- **Escrever**: Crie novos arquivos README.md
- **Glob**: Encontre arquivos por padrão
- **Grep**: Pesquise o conteúdo da documentação

## Lista de verificação de auditoria

Antes de concluir a atualização do README, verifique:
- Objetivo da pasta explicado claramente (1-2 frases)
- Todos os arquivos principais listados com descrições
- Links de navegação atuais e funcionando
- Perguntas abertas documentadas
- Próximas etapas/TODOs listados
- Links para pastas relacionadas
- Sem referências quebradas
- Formatação consistente com outros READMEs
- Atualizado com o conteúdo atual da pasta
- Útil para alguém novo no projeto

## Processo de atualização

**Etapa 1: Leia o README atual**
- Se README.md existir, leia-o completamente
- Entenda o que está documentado atualmente
- Observe o que está desatualizado ou faltando

**Etapa 2: Digitalizar o conteúdo da pasta**
- Listar todos farquivos e pastas
- Compreender estrutura e organização
- Identifique novos arquivos que não estão no README
- Identifique os arquivos mencionados que não existem mais

**Etapa 3: Identificar lacunas**
- O que está na pasta mas não está documentado?
- Qual documentação está desatualizada?
- Os links ainda são válidos?
- A finalidade da pasta ainda está correta?

**Etapa 4: Atualizações do plano**
- Reorganizar se necessário (melhor estrutura?)
- Atualizar descrições
- Adicionar arquivos ausentes
- Remover conteúdo obsoleto
- Corrigir links quebrados

**Etapa 5: aplicar atualizações**
- Reescrever README.md com melhorias
- Use formato padrão (veja modelo acima)
- Mantenha a consistência com outros READMEs
- Certifique-se de que todos os links funcionem

**Etapa 6: Relatório**
- Resumo do que foi atualizado
- Arquivos adicionados à documentação
- Arquivos removidos da documentação
- Melhorias estruturais realizadas

## Execução Paralela

Quando __NOTRASLATE_6__ é executado:
- 11 agentes aparecem simultaneamente (um por pasta)
- Cada agente trabalha de forma independente em sua pasta
- Todas as atualizações acontecem em paralelo
- Os resultados são coletados e relatados de voltaCada agente:
- NÃO espera por outros agentes
- Concentra-se apenas na pasta atribuída
- Funciona a toda velocidade
- Conclui de forma independente

## Padrões Comuns

**Navegação na Web**
- Cada README tem links para pais, irmãos e filhos
- Os leitores podem percorrer documentos/estrutura inteiros
- Sem becos sem saída ou caminhos quebrados

**Nomes de arquivos descritivos**
- As descrições dos arquivos explicam o PROPÓSITO, não apenas O QUE
- "Padrões de codificação para componentes de UI" versus apenas "padrões de UI"
- "Lógica de seleção da estrutura de teste" versus apenas "testing-adr"

**Estrutura clara**
- A seção Conteúdo lista todos os arquivos
- A seção Arquivos-chave explica os mais importantes
- A seção Como usar fornece orientação passo a passo
- Seção de padrões documenta convenções

**Acionador de manutenção**
- Depois de adicionar arquivos à pasta
- Depois de mover/renomear arquivos
- Depois de excluir/arquivar arquivos
- Antes de marcar a pasta como "completa"
- Durante auditorias de documentação trimestrais

**Regras de consistência**
- Mesmos cabeçalhos em todos os READMEs
- Mesma formatação para links e código
- Mesma nomenclatura para seções de navegação
- Links para os pais sempreuse ../README.md
- Links para irmãos usam ../sibling-folder/README.md
