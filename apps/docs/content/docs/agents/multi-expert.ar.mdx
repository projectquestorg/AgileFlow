# وكيل متعدد الخبراء

الوكيل المتعدد الخبراء هو منسق يقوم بتحليل الأسئلة المعقدة من خلال نشر 3-5 خبراء متخصصين في المجال بالتوازي، ثم يقوم بتجميع النتائج التي توصلوا إليها في تحليل شامل وحائز على الثقة.

## القدرات

- يولد العديد من خبراء المجال المتخصصين بالتوازي لإجراء تحليل شامل
- يكتشف المجالات ذات الصلة من الأسئلة (الأمان، API، واجهة المستخدم، قاعدة البيانات، الاختبار، الأداء، CI، DevOps)
- تجميع نتائج الخبراء مع تسجيل نقاط الثقة (عالية/متوسطة/منخفضة)
- يحدد الاتفاقات بين الخبراء (ثقة عالية) والخلافات (يحتاج إلى مراجعة)
- مقتطفات رؤى فريدة من الخبراء الأفراد
- يوفر الإجراءات الموصى بها ذات الأولوية

## متى يستخدم

استخدم هذا الوكيل عندما:

- **أسئلة معقدة تشمل مجالات متعددة** - "هل تنفيذ المصادقة لدينا آمن؟" يحتاج إلى الأمان + API + الاختبار + منظورات قاعدة البيانات
- **مراجعات الأمان** - تحليل التعليمات البرمجية من زوايا متعددة (الأمان، والهندسة المعمارية، والاختبار)
- **مهندس معماريالمراجعات الحقيقية ** - احصل على تعليقات من خبراء المجال بشأن قرارات التصميم
- **مراجعات العلاقات العامة** - مراجعة التعليمات البرمجية متعددة وجهات النظر للتأكد من صحتها
- **تحليل التدفق الشامل** - تتبع الطلبات من خلال طبقات متعددة مع التحقق من صحة الخبراء
- **تصحيح المشكلات الصعبة** - أسباب محتملة متعددة تحتاج إلى تحليل متخصص
- **أسئلة "أفضل الممارسات"** - تحتاج إلى التحقق عبر نطاقات متعددة

## كيف يعمل

يتبع الوكيل سير عمل منظم:

__لا تترجم_0__

### الخطوة 1: تحليل المجال

يقوم الوكيل بتحليل السؤال الخاص بالكلمات الرئيسية للمجال ويختار 3-5 خبراء:
- **خبير أساسي واحد** - المجال الأكثر صلة
- ** 2-4 خبراء داعمين ** - تقديم وجهات نظر إضافية

### الخطوة الثانية: النشر بالتوازي

يتم نشر جميع الخبراء في وقت واحد باستخدام التنفيذ في الخلفية لتحقيق الكفاءة:

__لا تترجم_1__

### الخطوة 3: جمع النتائج

يستخدم `TaskOutput` مع `block: true` لانتظار جميع ردود الخبراء.

### الخطوة 4: التوليف مع تسجيل النقاط بثقة

يحلل آلل الردود ونتائج المجموعات:

- **اتفاقية (أكثر من خبيرين)** → ثقة عالية
- **رؤية فريدة (خبير واحد)** → نتيجة ملحوظة
- **الخلاف** → تم وضع علامة عليه للمراجعة البشرية

### الخطوة 5: التقرير

مخرجات التحليل المنظم مع الأقسام:
- **النتائج الرئيسية (ثقة عالية)** - اتفاقيات بين أكثر من خبيرين
- **رؤى فريدة** - نتائج ملحوظة من خبراء فرديين
- **الخلافات (تحتاج إلى مراجعة)** - آراء متضاربة
- **الإجراءات الموصى بها** - الخطوات التالية ذات الأولوية

## تسجيل الثقة

| المستوى | المعايير |
|-------|----------|
| **عالي** | يتفق أكثر من 3 خبراء مع الأدلة المحددة المذكورة |
| **متوسط** | يتفق الخبراء مع بعض الأدلة |
| **منخفض** | خبير واحد فقط، يحتاج إلى التحقق |

## كشف المجال

يقوم الوكيل بتعيين الكلمات الرئيسية للسؤال لخبراء المجال:

| الكلمات المفتاحية | خبير |
|----------|--------|
| قاعدة البيانات، المخطط، الجدول، SQL، الاستعلام | قاعدة بيانات Agileflow |
| واجهة برمجة التطبيقات (API)، نقطة النهاية، REST، المسار، وحدة التحكم | Agileflow-api |
| المكون، واجهة المستخدم، الواجهة الأمامية، النمط | Agileflow واجهة المستخدم |
| teالأول، المواصفات، التغطية، وهمية | اختبار التدفق الرشيق |
| الأمان، المصادقة، JWT، الضعف | Agileflow-الأمن |
| الأداء، ذاكرة التخزين المؤقت، التحسين، بطيء | أداء رشيق |
| CI، سير العمل، خط الأنابيب، بناء | Agileflow-ci |
| نشر، البنية التحتية، دوكر | Agileflow-devops |

## السلوكيات الرئيسية

- **نشر الخبراء دائمًا بالتوازي** - لا يتم ذلك بشكل تسلسلي (الكفاءة)
- **ينشر دائمًا جميع الخبراء في رسالة واحدة** - تستدعي المهام المجمعة معًا
- **لا تقدم أبدًا إجابة نهائية دون تجميع جميع ردود الخبراء** - جميع وجهات النظر مهمة
- **يتضمن دائمًا قراءة الخبرة في مطالبات الخبراء** - `"FIRST: Read packages/cli/src/core/experts/{domain}/expertise.yaml"`
- **نشر إجمالي 3-5 خبراء** - ليس أقل، وليس أكثر (التوليف الأمثل)
- **يستخدم دائمًا TaskOutput مع الكتلة: true** - لجمع كل النتائج

## الأدوات المتاحة

هذا الوكيل لديه حق الوصول إلى:
- **قراءة** - الوصول إلى ملفات خبرة الوكيل
- **اكتب** - أنشئ ملفوثائق التحليل
- **تحرير** - تحديث التحليلات الموجودة
- **باش** - تشغيل أوامر التحقق
- **Glob** - ابحث عن الملفات ذات الصلة
- **Grep** - كود البحث والوثائق
- **المهمة** - نشر وكلاء خبراء بالتوازي
- **TaskOutput** - جمع نتائج الخبراء
