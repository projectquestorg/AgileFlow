# Prise en charge des langues RTL

La documentation AgileFlow prend en charge les langues s'écrivant de droite à gauche (RTL), notamment l'arabe, l'hébreu, le persan et l'ourdou. Ce guide explique comment utiliser le support RTL et comment contribuer aux traductions.

## Utiliser le mode RTL

### Sélecteur de langue

Cliquez sur l'icône langue/globe dans l'en-tête du site pour basculer entre les modes LTR et RTL. Le site ajustera automatiquement :

- Sens du texte
- Mise en miroir de la mise en page
- Alignement de la navigation
- Positionnement des composants

### Détection automatique

Si votre navigateur est réglé sur une langue RTL, le site démarrera automatiquement en mode RTL. Langues RTL prises en charge :

- arabe (ar)
- Hébreu (il)
- persan/farsi (fa)
- Ourdou (ur)

## CSS compatible RTL

Le site utilise Tailwind CSS avec des variantes RTL. Utilisez le `rtl:` préfixe pour les styles spécifiques à RTL :

```css
/* Example RTL styles */
.my-element {
  margin-left: 1rem;   /* LTR default */
}

[dir="rtl"] .my-element {
  margin-left: 0;
  margin-right: 1rem;  /* RTL override */
}

/* Using Tailwind rtl: variant */
<div className="ml-4 rtl:ml-0 rtl:mr-4">
  Content
</div>
```

### Propriétés logiques

Dans la plupart des cas, préférez les propriétés logiques CSS qui gèrent automatiquement la direction :

```css
/* Instead of this */
margin-left: 1rem;
padding-right: 2rem;
text-align: left;

/* Use this */
margin-inline-start: 1rem;
padding-inline-end: 2rem;
text-align: start;
```

Equivalents de vent arrière :
- `ms-*` = marge-inline-start
- `me-*` = marge-en-ligne-fin
- `ps-*` = padding-inline-start
- `pe-*` = padding-inline-end
- `start-*` = encart-inline-start
- `end-*` = encart-en-ligne-fin

## Flux de travail de traduction

### Ajout d'une nouvelle langue

1. Créer des fichiers de traduction dans `/locales/{lang}/`:

```
locales/
├── en/
│   ├── common.json
│   └── docs.json
├── ar/
│   ├── common.json
│   └── docs.json
└── he/
    ├── common.json
    └── docs.json
```

2. Ajouter la langue à `LANGUAGES` dans `components/language-switcher.tsx`:

```typescript
const LANGUAGES: Language[] = [
  { code: "en", name: "English", nativeName: "English", direction: "ltr" },
  { code: "ar", name: "Arabic", nativeName: "العربية", direction: "rtl" },
  // Add new language here
]
```

3. Créez des fichiers MDX traduits :

```
content/docs/
├── index.mdx           # English (default)
└── ar/
    └── index.mdx       # Arabic translation
```

### Directives de traduction

1. **Preserve formatting**: Conservez les blocs de code, les liens et les composants intacts
2. **Translate alt text**: Les attributs alt de l'image doivent être traduits
3. **Keep URLs**: Ne traduisez pas les slugs d'URL sauf si cela est absolument nécessaire
4. **Review technical terms**: Certains termes techniques peuvent rester en anglais

### Exemple de contenu RTL

Voici un exemple de texte arabe pour tester la mise en page RTL :

```arabic
مرحبًا بكم في توثيق AgileFlow. هذا النص باللغة العربية لاختبار دعم اللغات من اليمين إلى اليسار.
```

```hebrew
ברוכים הבאים לתיעוד AgileFlow. זהו טקסט בעברית לבדיקת תמיכה בשפות מימין לשמאל.
```

## Test de la mise en page RTL

1. Passer en mode RTL à l'aide du sélecteur de langue
2. Vérifiez ces zones pour une mise en miroir correcte :
   - Barre latérale de navigation (doit être à droite)
   - Alignement des icônes de recherche
   - Sens de défilement du bloc de code
   - Barres de progression
   - Icônes (certaines devraient s'inverser, d'autres non)

### Icônes qui ne doivent PAS basculer

- Icône de recherche
- Icône de lien externe
- Coche
- Plus/moins
- Fermer (X)

### Icônes qui DEVRAIENT retourner

- Flèche gauche/droite
- Chevron gauche/droite
- Retrait/retrait
- Répondre/transférer

## Mise en œuvre technique

### Fournisseur de direction

Le `DirectionProvider` le composant gère l'état de direction :

```tsx
import { useDirection } from "@/components/direction-provider"

function MyComponent() {
  const { direction, setDirection, isRTL } = useDirection()

  return (
    <div className={isRTL ? "text-right" : "text-left"}>
      Current direction: {direction}
    </div>
  )
}
```

### Accéder à la direction en CSS

Le `dir` l'attribut est défini sur le `<html>` élément:

```css
/* CSS selector */
[dir="rtl"] .my-element {
  /* RTL styles */
}

/* Tailwind */
<div className="rtl:flex-row-reverse">
```
