# Modelos de especificação de componentes

Este guia aborda como documentar componentes de UI usando os modelos de especificação padronizados fornecidos pelo AgileFlow.

## Visão geral

Cada componente deverá ter:
- **Tabela de adereços** - Gerada automaticamente a partir de tipos TypeScript
- **Instrução de importação** - Como importar o componente
- **Exemplo de uso** - Exemplo de código básico
- **Lista de verificação de acessibilidade** - Status de conformidade com as WCAG
- **Botão Copiar** - Cópia de componente com um clique

## Usando os modelos

### Em arquivos MDX

Importe o componente __NOTRASLATE_4__:

```tsx
import { ComponentSpec, CopyComponent } from "@/components/component-spec"

<ComponentSpec
  name="MyComponent"
  description="Component description"
  importStatement={'import { MyComponent } from "@/components/my-component"'}
  props={[
    {
      name: "propName",
      type: "string",
      description: "What the prop does",
      required: true,
    },
  ]}
  usageExample={'<MyComponent propName="value" />'}
  accessibility={[
    { label: "Check 1", status: "pass" },
    { label: "Check 2", status: "warn", description: "Details" },
  ]}
/>
```

### Esquema da tabela de adereços

```typescript
interface PropDefinition {
  name: string       // Prop name
  type: string       // TypeScript type
  description: string // What the prop does
  required?: boolean // Is it required?
  default?: string   // Default value
}
```

### Esquema da lista de verificação de acessibilidade

```typescript
interface AccessibilityItem {
  label: string              // Check name
  status: "pass" | "warn" | "info"
  description?: string       // Additional details
}
```

## Verificações de acessibilidade padrão

Use estas verificações comuns de consistência:

### Componentes interativos
- Teclado acessível (Enter/Espaço)
- Foco visível
- Ordem de foco lógica
- Alvo de toque >= 44x44px

### Componentes do formulário
- Etiquetas associadas
- Mensagens de erro anunciadas
- Campos obrigatórios indicados
- Estado inválido transmitido

### Componentes de conteúdo
- Contraste de cor >= 4,5:1
- Texto redimensionável
-Motíon reduzível
- Texto alternativo presente

## Melhores práticas

1. **Mantenha o mínimo de adereços** - Documente apenas adereços de API públicos
2. **Use tipos precisos** - Copie tipos do código-fonte
3. **Teste a acessibilidade** - Verifique as reivindicações antes de marcar a aprovação
4. **Atualize regularmente** - Mantenha as especificações sincronizadas com o código
5. **Link para a fonte** - Consulte a fonte do GitHub quando possível

## Guia de migração

Para aplicar este modelo a documentos existentes:

1. Importe __NOTRASLATE_5__ no topo do arquivo MDX
2. Extraia adereços do componente TypeScript
3. Adicione um exemplo de uso do código existente
4. Corraauditoria de acessibilidade
5. Substitua a documentação antiga por __NOTRASLATE_6__
