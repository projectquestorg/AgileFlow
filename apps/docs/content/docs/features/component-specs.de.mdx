# Komponentenspezifikationsvorlagen

In diesem Leitfaden erfahren Sie, wie Sie UI-Komponenten mithilfe der von AgileFlow bereitgestellten standardisierten Spezifikationsvorlagen dokumentieren.

## Übersicht

Jede Komponente sollte Folgendes haben:
- **Props-Tabelle** – Automatisch aus TypeScript-Typen generiert
- **Import-Anweisung** – So importieren Sie die Komponente
- **Verwendungsbeispiel** – Einfaches Codebeispiel
- **Checkliste zur Barrierefreiheit** – WCAG-Konformitätsstatus
- **Schaltfläche „Kopieren“** – Kopieren von Komponenten mit einem Klick

## Verwendung der Vorlagen

### In MDX-Dateien

Importieren Sie die Komponente `ComponentSpec`:

```tsx
import { ComponentSpec, CopyComponent } from "@/components/component-spec"

<ComponentSpec
  name="MyComponent"
  description="Component description"
  importStatement={'import { MyComponent } from "@/components/my-component"'}
  props={[
    {
      name: "propName",
      type: "string",
      description: "What the prop does",
      required: true,
    },
  ]}
  usageExample={'<MyComponent propName="value" />'}
  accessibility={[
    { label: "Check 1", status: "pass" },
    { label: "Check 2", status: "warn", description: "Details" },
  ]}
/>
```

### Props-Tabellenschema

```typescript
interface PropDefinition {
  name: string       // Prop name
  type: string       // TypeScript type
  description: string // What the prop does
  required?: boolean // Is it required?
  default?: string   // Default value
}
```

### Checklistenschema für Barrierefreiheit

```typescript
interface AccessibilityItem {
  label: string              // Check name
  status: "pass" | "warn" | "info"
  description?: string       // Additional details
}
```

## Standard-Barrierefreiheitsprüfungen

Verwenden Sie diese allgemeinen Konsistenzprüfungen:

### Interaktive Komponenten
- Tastatur zugänglich (Enter/Leertaste)
- Fokus sichtbar
- Fokusreihenfolge logisch
- Berühren Sie das Ziel >= 44x44px

### Formularkomponenten
- Zugehörige Etiketten
- Fehlermeldungen angekündigt
- Erforderliche Felder angegeben
- Ungültiger Status übermittelt

### Inhaltskomponenten
- Farbkontrast >= 4,5:1
- Textgröße veränderbar
- Motionenreduzierbar
- Alt-Text vorhanden

## Ein-Klick-Kopierfunktion

Fügen Sie eine Kopierschaltfläche für den vollständigen Komponentencode hinzu:

## Best Practices

1. **Requisiten minimal halten** – Nur öffentliche API-Requisiten dokumentieren
2. **Verwenden Sie genaue Typen** – Kopieren Sie Typen aus dem Quellcode
3. **Testen Sie die Zugänglichkeit** – Überprüfen Sie die Ansprüche, bevor Sie sie als bestanden bewerten
4. **Regelmäßig aktualisieren** – Halten Sie die Spezifikationen mit dem Code synchron
5. **Link zur Quelle** – Verweisen Sie nach Möglichkeit auf die GitHub-Quelle

## Migrationsleitfaden

So wenden Sie diese Vorlage auf vorhandene Dokumente an:

1. Importieren Sie `ComponentSpec` oben in der MDX-Datei
2. Extrahieren Sie Requisiten aus der TypeScript-Komponente
3. Fügen Sie ein Anwendungsbeispiel aus vorhandenem Code hinzu
4. LaufenPrüfung der Barrierefreiheit
5. Ersetzen Sie die alte Dokumentation durch `<ComponentSpec />`
