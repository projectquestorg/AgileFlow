# سمات

يتضمن AgileFlow ميزات قوية لحماية قاعدة التعليمات البرمجية الخاصة بك، وأتمتة سير العمل، والحفاظ على إنتاجية فريقك.

## الميزات الأساسية

### نظام المهارات

قم بإنشاء وإدارة وتحسين مطالبات الذكاء الاصطناعي المخصصة التي تتعلم من تعليقاتك.

- **Research-backed creation** - تم إنشاؤها من مستندات الويب والوثائق المحلية وقاعدة التعليمات البرمجية الخاصة بك
- **Cookbook workflows** - أدلة محددة لحالة الاستخدام المتعددة في مهارة واحدة
- **Self-improving** - المهارات تتعلم من تصحيحاتك وصقلها تلقائيا
- **MCP integration** - الاتصال بخوادم بروتوكول السياق النموذجي للوصول إلى الأداة
- **Team shareable** - Commit to git and share institutional knowledge

[تعرف على المهارات](/docs/features/skills)

### نظام الخبرة الوكيل

تمكين وكلاء التحسين الذاتي الذين يحافظون على معرفة المجال وتسريع تنفيذ المهام.

- **Expertise files** - يحتفظ الوكلاء بالنماذج الذهنية لمجالهم
- **Three-step workflow** - التخطيط (بالخبرة) ← البناء ← التحسين الذاتي
- **Automatic learning** - تحديثات الخبرة أثناء عمل الوكلاء وإجراء التغييرات
- **Validation & metrics** - أدوات لضمان بقاء الخبرة دقيقة وحديثة
- **Focused domains** - معرفة متخصصة بقاعدة البيانات وواجهة برمجة التطبيقات (API) وواجهة المستخدم (UI) والاختبار وCI/CD والمزيد

[تعرف على خبرة الوكيل](/docs/features/agent-expertise-system)

### الجلسات الموازية

اعمل على ميزات أو أخطاء أو تجارب متعددة في وقت واحد باستخدام مساحات عمل معزولة وحماية الحدود.

- **Git worktrees** - لكل جلسة دليلها وفرعها الخاص
- **Boundary protection** - تمنع خطافات التحرير/الكتابة عمليات التحرير غير المقصودة عبر الجلسات
- **Easy switching** - يستخدم `/add-dir` لتبديل الجلسات في نفس المحطة
- **Session tracking** - يتتبع AgileFlow جميع الجلسات بالحالة والنشاط
- **Independent context** - تحافظ كل جلسة على قصتها وتقدمها

[تعرف على الجلسات الموازية](/docs/features/parallel-sessions)

## الأتمتة

### وضع الحلقة

قم بتشغيل Claude بشكل مستقل مع التنفيذ المستمر للمهمة حتى الانتهاء.

- **Autonomous execution** - يعمل كلود من خلال المهام دون انقطاع
- **Configurable limits** - تعيين حدود التكرار والمهلات
- **Progress tracking** - انظر ما يحدث في كل خطوة
- **Safe fallbacks** - فحوصات السلامة المضمنة تمنع الحلقات الهاربة

[تعرف على وضع الحلقة](/docs/features/loop-mode)

### الوضع المرئي

اسمح لـ Claude برؤية الواجهات المرئية والتفاعل معها من خلال لقطات الشاشة وأتمتة المتصفح.

- **Screenshot analysis** - يستطيع كلود تحليل المحتوى المرئي
- **Browser automation** - التكامل المسرحي لاختبار الويب
- **UI verification** - التحقق من صحة التغييرات المرئية تلقائيًا
- **Cross-platform** - يعمل على أي نظام تشغيل مع دعم المتصفح

[Learn about Visual Mode](/docs/features/visual-mode)

### الحفاظ على السياق المضغوط

الحفاظ على معرفة الوكيل الهامة وحالته أثناء ضغط السياق التلقائي.

- **Context-aware compaction** - يتم الحفاظ على القواعد المهمة عندما يقوم كلود كود بضغط السياق
- **Priority-based preservation** - العوامل الحرجة محفوظة دائمًا
- **State continuity** - تم الحفاظ على القصة الحالية والحاصرات والتقدم
- **Long conversation support** - تركيز الوكيل المستمر على مدار أيام العمل
- **Zero-overhead** - تلقائي، لا حاجة إلى التكوين اليدوي

[تعرف على السياق المضغوط](/docs/features/compact-context)

## الإعداد والسلامة

### نظام التكوين

9 وكلاء متخصصين لإعداد وتخصيص سير عمل التطوير الخاص بك.

- **Git setup** - تهيئة المستودع والاتصالات عن بعد
- **Hooks system** - أتمتة تعتمد على الأحداث (SessionStart، PreCompact، Stop)
- **Status line** - شريط الحالة المخصص في كلود كود
- **Archival** - الإدارة التلقائية لحجم ملف Status.json
- **CI/CD** - الاختبار الآلي باستخدام GitHub Actions وGitLab وCircleCI
- **Attribution** - التحكم في الكشف عن الذكاء الاصطناعي في التزامات git
- **And more** - سياق PreCompact والتحقق وإدارة الرمز المميز

[تعرف على التكوين](/docs/features/configuration)

### تكاملات IDE

قم بتثبيت AgileFlow وتكوينه لبيئة التطوير المتكاملة (IDE) المفضلة لديك باستخدام الأوامر التلقائية والوكيل وتثبيت المهارات.

- **Multiple IDEs** - كلود كود، المؤشر، ركوب الأمواج، OpenAI Codex CLI
- **Auto-detection** - كشف أي IDEs تم تكوينها في مشروعك
- **Damage Control** - نظام أمان اختياري لكلود كود
- **Dynamic content** - يتم إدخال الأوامر والوكلاء تلقائيًا عند التثبيت
- **Multi-IDE support** - استخدم نفس المشروع عبر IDEs متعددة

[تعرف على تكاملات IDE](/docs/features/ide-integrations)

### السيطرة على الأضرار

قم بحماية قاعدة التعليمات البرمجية الخاصة بك من الأوامر المدمرة باستخدام خطافات PreToolUse.

- **Bash command validation** - منع الأنماط الخطيرة مثل `rm -rf /`, `git push --force`
- **File access control** - حماية الأسرار والمفاتيح والملفات الهامة
- **Customizable rules** - إضافة أنماط الحماية الخاصة بالمشروع
- **Ask-first mode** - طلب تأكيد للعمليات المحفوفة بالمخاطر
- **Three protection layers** - التحقق من صحة أداة Bash والتحرير والكتابة

[تعرف على التحكم في الأضرار](/docs/features/damage-control)

## مرجع سريع

| Feature | Purpose | Get Started |
|---------|---------|-------------|
| **Skills** | Custom AI prompts that learn | `/agileflow:skill:create` |
| **Agent Expertise** | Self-improving domain knowledge | Built-in |
| **Parallel Sessions** | Work on multiple features | `/agileflow:session:new` |
| **Loop Mode** | Autonomous task execution | `/agileflow:loop` |
| **Visual Mode** | Screenshot and browser automation | `/agileflow:visual` |
| **Compact Context** | Preserve state through compression | Built-in |
| **Configuration** | Setup and customize workflow | `/agileflow:configure` |
| **IDE Integrations** | Multi-IDE support | `npx agileflow setup` |
| **Damage Control** | Block dangerous commands | `/agileflow:configure damagecontrol` |

## الخطوات التالية

1. **Start with Setup** - يجري `/agileflow:configure` لإعداد مشروعك
2. **Enable Safety** - قم بتكوين التحكم في الأضرار لحماية قاعدة التعليمات البرمجية الخاصة بك
3. **Learn Skills** - إنشاء مطالبات مخصصة لسير العمل الخاص بك
4. **Try Parallel Sessions** - العمل على ميزات متعددة في وقت واحد
