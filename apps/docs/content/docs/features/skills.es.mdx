# Sistema de Habilidades

El sistema Habilidades le permite crear y administrar indicaciones de IA personalizadas que aprenden y mejoran con el tiempo. Las habilidades son plantillas reutilizables respaldadas por investigaciones que combinan las mejores prácticas de su proyecto, documentación externa y patrones de base de código en guías enfocadas y prácticas para Claude.

## Inicio rápido

Descubre habilidades disponibles en el marketplace:

```bash
/agileflow:skill:list
```

Explore las habilidades instaladas y obtenga recomendaciones del marketplace basadas en su stack tecnológico.

Obtenga recomendaciones personalizadas:

```bash
/agileflow:skill:recommend
```

## ¿Qué son las habilidades?

A **skill** es una plantilla de indicaciones estructurada que le enseña a Claude cómo manejar tareas específicas en el contexto de su proyecto. Las habilidades son:

- **Research-backed** - Generado a partir de documentos web, documentación local y su código base.
- **Self-improving** - Aprenda de sus correcciones y refine automáticamente
- **Cookbook-based** - Múltiples flujos de trabajo de casos de uso en una sola habilidad
- **Marketplace-powered** - Explore e instale habilidades desde el marketplace skills.sh

### Habilidades de ejemplo

Podrías crear habilidades para:

- **Database operations**: "Supabase Swift" - Cómo trabajar con Supabase en su aplicación iOS
- **API patterns**: "Clientes API REST": mejores prácticas para el patrón de integración API de su proyecto
- **Component development**: "Reaccionar componentes con accesibilidad" - Cómo construir componentes siguiendo sus estándares
- **Deployment**: "GitHub Actions CI/CD": el flujo de trabajo de implementación y prueba de su equipo
- **Domain expertise**: "Aprendizaje automático con TensorFlow": mejores prácticas específicas para sus proyectos de aprendizaje automático

## Conceptos básicos

### Estructura de habilidades

Cada habilidad contiene:

```
.claude/skills/<skill-name>/
├── SKILL.md              # Main skill file with activation triggers and instructions
├── references.md         # Links to web docs, local docs, and code patterns
├── cookbook/             # Per-use-case workflows
│   ├── use-case-1.md
│   └── use-case-2.md
└── .mcp.json            # Optional MCP server configuration
```

### Cómo funcionan las habilidades

1. **Activation**: Cuando le preguntas a Claude sobre un tema que coincide con los activadores de una habilidad, la habilidad se activa.
2. **Execution**: Claude lee SKILL.md y sigue las instrucciones
3. **Learning**: Cuando corriges a Claude, la habilidad aprende tus preferencias.
4. **Evolution**: Cada corrección actualiza el archivo de aprendizaje, haciendo que la habilidad sea más inteligente

### Sistema de aprendizajes

Las habilidades pueden mejorar automáticamente a través de un sistema de aprendizaje:

```
.agileflow/skills/_learnings/
├── supabase-swift.yaml
├── api-client.yaml
└── react-components.yaml
```

Cada archivo de aprendizaje rastrea:
- **Preferences** - Qué prefieren los usuarios (formato, estilo, enfoque)
- **Conventions** - Patrones y estándares específicos del proyecto.
- **Anti-patterns** - Qué evitar
- **Examples** - Buenos y malos ejemplos de correcciones.

## El sistema de aprendizajes

### Qué habilidades se aprenden

Cuando usas una habilidad y corriges la salida de Claude, la habilidad aprende:

1. **Explicit corrections** - Aprendizajes de alta confianza a partir de comentarios directos.
2. **Approved responses** - Aprendizajes de confianza media a partir de las aprobaciones.
3. **Patterns** - Observaciones de baja confianza para revisión.

### Formato de archivo de aprendizaje

Cada habilidad tiene un archivo de aprendizaje opcional en `.agileflow/skills/_learnings/<skill>.yaml`:

```yaml
skill: supabase-swift
version: 1
last_updated: 2025-01-08T10:30:00Z

preferences:
  code_style: "swift-concurrency"
  error_handling: "Result<T, Error>"
  async_patterns: "async/await"

conventions:
  naming: "camelCase for variables, PascalCase for types"
  structure: "Separate data, service, and UI layers"
  testing: "Unit tests required for all public APIs"

anti_patterns:
  - "Avoid callback patterns (use async/await instead)"
  - "Never hardcode API keys in code"

examples:
  good:
    - "let user = try await supabase.auth.user()"
  bad:
    - "var apiKey = 'sk-...' // Never do this"

metadata:
  corrections_count: 12
  approvals_count: 8
  last_correction: 2025-01-08T10:30:00Z
```

### Cómo funciona la superación personal

**On invocation:**
1. Claude comprueba si existe un archivo de aprendizaje
2. Lee preferencias, convenciones y antipatrones.
3. Los aplica mientras ejecuta la habilidad.

**On correction:**
1. Usted proporciona comentarios o corrección
2. Claude extrae la señal (qué cambiar)
3. Determina el nivel de confianza (alto/medio/bajo)
4. Actualiza el archivo de aprendizajes.
5. La próxima invocación utiliza las preferencias actualizadas.

## Mejores prácticas

### Creando habilidades efectivas

1. **Use specific names** - `supabase-swift` no solo `database`
2. **Include multiple use cases** - Agregar entradas de libros de cocina para diferentes escenarios
3. **Link to project patterns** - Haga referencia a las convenciones de su código base
4. **Add troubleshooting** - Anticipar y abordar problemas comunes.
5. **Keep it focused** - Un dominio por habilidad, no demasiado amplio

### Mantener habilidades

1. **Test regularly** - Verifique regularmente la calidad de sus habilidades
2. **Review learnings** - Verifique lo que la habilidad aprendió de las correcciones.
3. **Update references** - Mantener los enlaces de documentación actualizados
4. **Archive unused skills** - Limpiar habilidades que ya no usas
5. **Share improvements** - Comprometer lo aprendido con Git para lograr coherencia en el equipo.

### Habilidades de organización

**Suggested skill categories:**

- **Data layer** - Operaciones de bases de datos, migraciones, consultas.
- **API layer** - Clientes REST, GraphQL, webhooks, autenticación
- **UI layer** - Componentes, estilo, accesibilidad, animaciones.
- **Infrastructure** - Despliegue, CI/CD, monitorización, seguridad.
- **Workflows** - Pruebas, revisión de código, depuración, creación de perfiles.
- **Domain** - Modelos ML, procesamiento de imágenes, procesamiento de pagos.

## Flujos de trabajo de libros de cocina

### ¿Qué es un libro de cocina?

La entrada de un libro de cocina es una guía paso a paso para un caso de uso específico dentro de una habilidad.

### Plantilla de libro de cocina

```markdown
# Use Case Title

Brief description of this workflow.

## Prerequisites

- Prerequisite 1
- Prerequisite 2

## Instructions

1. **Step 1**: Description with example
   ```language
   ejemplo de código
   ```

2. **Step 2**: Next step
   ```language
   ejemplo de código
   ```

## Example

**Input**: Sample input or scenario
**Output**: Expected output

## Troubleshooting

- **Problem**: Solution
```

### Cuándo agregar libros de cocina

Agregue entradas de libros de cocina cuando:
- Existen múltiples casos de uso comunes
- Se aplican diferentes flujos de trabajo
- La configuración difiere entre escenarios
- Los usuarios solicitan con frecuencia orientación paso a paso.

## Ciclo de vida de las habilidades

### Descubrimiento

1. Ejecutar `/agileflow:skill:list` para explorar el marketplace
2. Ejecutar `/agileflow:skill:recommend` para obtener sugerencias personalizadas
3. Instalar habilidades desde el marketplace skills.sh
4. Revisar la habilidad

### Uso

1. Trabajar en tareas que coincidan con el dominio de la habilidad.
2. Claude activa la habilidad automáticamente.
3. La habilidad proporciona orientación y ejemplos.
4. Ejecutas las recomendaciones de la habilidad.

### Mejora

1. Cuando al resultado de Claude le falta algo, corríjalo
2. La habilidad aprende de la corrección.
3. La próxima vez, la habilidad proporciona una mejor orientación.

### Evolución

1. Con el tiempo, las habilidades se vuelven más personalizadas
2. El archivo de aprendizaje captura patrones específicos del proyecto
3. Los nuevos miembros del equipo heredan el conocimiento.
4. Las habilidades se convierten en memoria institucional

## Compartir habilidades

### Exportar una habilidad

Copie un directorio de habilidades para compartir:

```bash
cp -r .claude/skills/my-skill /path/to/share/
```

### Importar una habilidad

Copie a su directorio de habilidades:

```bash
cp -r /path/to/skill .claude/skills/
```

### Control de versiones

Comprometer habilidades para git (incluir aprendizajes):

```bash
git add .claude/skills/
git add .agileflow/skills/_learnings/
git commit -m "Add supabase-swift skill with learnings"
```

Los miembros del equipo obtienen automáticamente las habilidades y los aprendizajes.

## Solución de problemas

### Habilidad no activada

**Problem**: Preguntas sobre el tema de la habilidad pero Claude no la activa.

**Solution**:
1. Consulte la sección "Cuándo utilizar" en SKILL.md
2. Asegúrese de que su pregunta coincida con los factores desencadenantes de activación
3. Edite la habilidad para agregar más palabras clave si es necesario

### Falta el archivo de aprendizajes

**Problem**: La habilidad no mejora incluso después de las correcciones.

**Solution**:
1. Cree manualmente un archivo de aprendizajes en `.agileflow/skills/_learnings/<skill>.yaml`
2. Se realizará un seguimiento de las correcciones futuras.
