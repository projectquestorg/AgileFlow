# Modèles de spécifications de composants

Ce guide explique comment documenter les composants de l'interface utilisateur à l'aide des modèles de spécifications standardisés fournis par AgileFlow.

## Aperçu

Chaque composant doit avoir :
- **Tableau Props** - Généré automatiquement à partir de types TypeScript
- **Instruction d'importation** - Comment importer le composant
- **Exemple d'utilisation** - Exemple de code de base
- **Liste de contrôle d'accessibilité** - Statut de conformité WCAG
- **Bouton Copier** - Copie de composants en un clic

## Exemple de spécification de composant

Voici un exemple de composant entièrement documenté :

## Utilisation des modèles

### Dans les fichiers MDX

Importez le composant `ComponentSpec` :

```tsx
import { ComponentSpec, CopyComponent } from "@/components/component-spec"

<ComponentSpec
  name="MyComponent"
  description="Component description"
  importStatement={'import { MyComponent } from "@/components/my-component"'}
  props={[
    {
      name: "propName",
      type: "string",
      description: "What the prop does",
      required: true,
    },
  ]}
  usageExample={'<MyComponent propName="value" />'}
  accessibility={[
    { label: "Check 1", status: "pass" },
    { label: "Check 2", status: "warn", description: "Details" },
  ]}
/>
```

### Schéma de la table des accessoires

```typescript
interface PropDefinition {
  name: string       // Prop name
  type: string       // TypeScript type
  description: string // What the prop does
  required?: boolean // Is it required?
  default?: string   // Default value
}
```

### Schéma de la liste de contrôle d'accessibilité

```typescript
interface AccessibilityItem {
  label: string              // Check name
  status: "pass" | "warn" | "info"
  description?: string       // Additional details
}
```

## Vérifications d'accessibilité standard

Utilisez ces contrôles courants de cohérence :

### Composants interactifs
- Clavier accessible (Entrée/Espace)
- Mise au point visible
- Ordre de mise au point logique
- Cible tactile >= 44x44px

### Composants du formulaire
- Etiquettes associées
- Messages d'erreur annoncés
- Champs obligatoires indiqués
- Etat invalide transmis

### Composants de contenu
- Contraste des couleurs >= 4,5:1
- Texte redimensionnable
-Motion réductible
- Texte alternatif présent

## Fonctionnalité de copie en un clic

Ajoutez un bouton de copie pour le code complet du composant :

<CopyComponent
  name="Button"
  code={`import * as React from "react"
import { Slot } from "@radix-ui/react-slot"
import { cva, type VariantProps } from "class-variance-authority"
import { cn } from "@/lib/utils"

const buttonVariants = cva(
  "inline-flex items-center justify-center...",
  {
    variants: {
      variant: { default: "...", destructive: "..." },
      size: { default: "...", sm: "...", lg: "..." },
    },
  }
)

export interface ButtonProps
  extends React.ButtonHTMLAttributes<HTMLButtonElement>,
    VariantProps<type de boutonVariants> {
  asChild? : booléen
}

const Bouton = React.forwardRef<HTMLButtonElement, ButtonProps>(
  ({ className, variante, taille, asChild = false, ...props }, ref) => {
    const Comp = asChild ? Emplacement : "bouton"
    retour (
      <Comp
        className={cn(buttonVariants({ variant, size, className }))}
        ref={ref}
        {...props}
      />
    )
  }
)
Bouton.displayName = "Bouton"

exporter { Bouton, boutonVariantes }`}
/>

## meilleures pratiques

1. **Gardez les accessoires au minimum** - Documentez uniquement les accessoires de l'API publique
2. **Utilisez des types précis** - Copiez les types à partir du code source
3. **Test d'accessibilité** - Vérifiez les affirmations avant de réussir
4. **Mettre à jour régulièrement** - Gardez les spécifications synchronisées avec le code
5. **Lien vers la source** - Référencez la source GitHub lorsque cela est possible

## Guide de migration

Pour appliquer ce modèle à des documents existants :

1. Importez `ComponentSpec` en haut du fichier MDX
2. Extraire les accessoires du composant TypeScript
3. Ajouter un exemple d'utilisation à partir du code existant
4. Courezaudit d'accessibilité
5. Remplacez l'ancienne documentation par `<ComponentSpec />`

Voir [Button](/components/button), [Card](/components/card), [Input](/components/input), [Dialog](/components/dialog) et [Select](/components/select)] pour des exemples en direct.
