# تكامل IDE

يدعم AgileFlow العديد من IDEs من خلال نظام تثبيت موحد. يحتوي كل IDE على دليل التكوين الخاص به ويدعم ميزات مختلفة. يغطي هذا المستند التثبيت والإعداد والاستخدام لجميع بيئات التطوير المتكاملة المدعومة.

#### نموذج التثبيت

يستخدم Codex CLI نموذجًا مختلفًا عن IDEs الأخرى:

1. **المهارات** (لكل مستودع): يصبح وكلاء AgileFlow مهارات Codex
   - يقع في `.codex/skills/agileflow-*/`
   - كل مهارة لها ملف `SKILL.md`
   - تم استدعاؤه باستخدام صيغة `$skill-name`

2. **المطالبات** (على مستوى المستخدم): تصبح أوامر AgileFlow بمثابة مطالبات Codex
   - يقع في `~/.codex/prompts/agileflow-*.md`
   - مشتركة عبر كافة المشاريع على الجهاز
   - تم استدعاؤه باستخدام صيغة `/prompt-name`

3. **تعليمات المشروع**: ملف AGENTS.md في جذر الريبو
   - السقالات لاتفاقيات المشروع
   - يوفر السياق حول ما هو متاحيقتل
   - هيكل المشروع المراجع

## استكشاف أخطاء إعداد IDE وإصلاحها

### عدم ظهور الأوامر

**المشكلة**: تم تثبيت IDE لكن الأوامر لا تظهر.

**الحل**:
1. تحقق من التثبيت: `ls .claude/commands/agileflow/` (لكلود كود)
2. أعد تشغيل IDE بالكامل (قم بإنهاء الأمر وإعادة فتحه)
3. إعادة التثبيت: `npx agileflow setup --ide claude-code`

### التحكم في الأضرار لا يعمل (كود كلود)

**المشكلة**: لا يتم حظر الأوامر الخطيرة.

**الحل**:
1. التحقق من تكوين الخطافات: `ls .claude/hooks/damage-control/`
2. تحقق من أن settings.json يحتوي على خطافات PreToolUse
3. أعد تشغيل Claude Code بالكامل
4. راجع [استكشاف أخطاء التحكم في الأضرار وإصلاحها](/features/damage-control#troubleshooting)]

### Codex CLI لا يجد المهارات

**المشكلة**: المهارات غير متوفرة في الدستور الغذائي.

**الحل**:
1. تحقق `.codex/skills/` موجود ولديه أدلة مهارات
2. تحقق من `~/.codex/prompts/` بحثًا عن الملفات السريعة
3. أعد تشغيل Codex CLI
4. تحقق من صفحة الدستور الغذائي: `echo $CODEX_HOME`

### تعارضات IDE المختلطة

**المشكلة**: يؤدي استخدام نفس المشروع في عدة بيئات تطوير متكاملة إلى حدوث مشكلات.

**الحل**:
- كل IDE له دليل التكوين الخاص به (بدون تعارضات)
- مشاركة جوهر `.agileflow/` عبر بيئات تطوير متكاملة
- استخدم التحكم في الإصدار لمشاركة التكوينات

## أفضل الممارسات

### اختيار IDE

- **Claude Code**: موصى به لمعظم المستخدمين، بميزات كاملة
- **المؤشر**: بديل جيد، يعمل مع امتدادات VSCode
- **ركوب الأمواج**: الأفضل لإدارة سير العمل
- **Codex CLI**: جيد للتطوير الأول لـ CLI والبيئات السحابية

### إدارة التكوين

1. **التحكم في الإصدار** إعداد AgileFlow:
   __لا تترجم_39__

2. **المشاركة عبر الفريق** - يحصل أعضاء الفريق على نفس الإعداد
3. **الحفاظ على الأنماط** - لا تحذف `.agileflow/` بين الإعدادات

### الأمن

- **التحكم في الأضرار** يحمي Claude Code بشكل افتراضي
- **وكلاء.md** في COdex يوثق ممارسات السلامة
- مراجعة الأنماط قبل مشاركتها مع الفريق

### بيئة تطوير متكاملة متعددة

- **التثبيت مرة واحدة**: نواة واحدة `.agileflow/` مشتركة عبر بيئات التطوير المتكاملة
- **التبديل بحرية**: استخدم `code .`، `windsurf .`، `cursor .`
- **لا توجد تعارضات**: يحتوي كل IDE على دليل تكوين منفصل
