# Soporte de lenguaje RTL

La documentación de AgileFlow admite idiomas de derecha a izquierda (RTL), incluidos árabe, hebreo, persa y urdu. Esta guía cubre cómo utilizar el soporte RTL y cómo contribuir con traducciones.

## Usando el modo RTL

### Cambiador de idiomas

Haga clic en el ícono de idioma/globo en el encabezado del sitio para cambiar entre los modos LTR y RTL. El sitio se ajustará automáticamente:

- Dirección del texto
- Duplicación de diseño
- Alineación de navegación
- Posicionamiento de componentes

### Detección automática

Si su navegador está configurado en un idioma RTL, el sitio se iniciará automáticamente en modo RTL. Idiomas RTL soportados:

- Árabe (ar)
- hebreo (él)
- Persa/farsi (fa)
- Urdu (ur)

## CSS compatible con RTL

El sitio utiliza Tailwind CSS con variantes RTL. Utilice el `rtl:` prefijo para estilos específicos de RTL:

```css
/* Example RTL styles */
.my-element {
  margin-left: 1rem;   /* LTR default */
}

[dir="rtl"] .my-element {
  margin-left: 0;
  margin-right: 1rem;  /* RTL override */
}

/* Using Tailwind rtl: variant */
<div className="ml-4 rtl:ml-0 rtl:mr-4">
  Content
</div>
```

### Propiedades lógicas

En la mayoría de los casos, prefiera las propiedades lógicas de CSS que manejan automáticamente la dirección:

```css
/* Instead of this */
margin-left: 1rem;
padding-right: 2rem;
text-align: left;

/* Use this */
margin-inline-start: 1rem;
padding-inline-end: 2rem;
text-align: start;
```

Equivalentes de viento de cola:
- `ms-*` = margen-inicio-en línea
- `me-*` = margen-en-línea-final
- `ps-*` = relleno-inicio-en línea
- `pe-*` = relleno-en-línea-final
- `start-*` = inserto-en-línea-inicio
- `end-*` = inserto-en-línea-fin

## Flujo de trabajo de traducción

### Agregar un nuevo idioma

1. Crear archivos de traducción en `/locales/{lang}/`:

```
locales/
├── en/
│   ├── common.json
│   └── docs.json
├── ar/
│   ├── common.json
│   └── docs.json
└── he/
    ├── common.json
    └── docs.json
```

2. Agregar el idioma a `LANGUAGES` en `components/language-switcher.tsx`:

```typescript
const LANGUAGES: Language[] = [
  { code: "en", name: "English", nativeName: "English", direction: "ltr" },
  { code: "ar", name: "Arabic", nativeName: "العربية", direction: "rtl" },
  // Add new language here
]
```

3. Cree archivos MDX traducidos:

```
content/docs/
├── index.mdx           # English (default)
└── ar/
    └── index.mdx       # Arabic translation
```

### Directrices de traducción

1. **Preserve formatting**: Mantenga intactos los bloques de código, enlaces y componentes
2. **Translate alt text**: Los atributos alternativos de la imagen deben traducirse
3. **Keep URLs**: No traduzca slugs de URL a menos que sea absolutamente necesario
4. **Review technical terms**: Algunos términos técnicos pueden permanecer en inglés

## Prueba del diseño RTL

1. Cambie al modo RTL usando el selector de idiomas
2. Verifique estas áreas para ver si el reflejo es correcto:
   - Barra lateral de navegación (debe estar a la derecha)
   - Alineación del icono de búsqueda
   - Dirección de desplazamiento del bloque de código
   - Barras de progreso
   - Íconos (algunos deberían voltear, otros no)

### Iconos que NO deberían voltear

- Icono de búsqueda
- Icono de enlace externo
- Marca de verificación
- Más/menos
- Cerrar (X)

### Iconos que DEBEN voltear

- Flecha izquierda/derecha
- Cheurón izquierda/derecha
- Sangría/quitar sangría
- Responder/reenviar

## Implementación técnica

### DirecciónProveedor

El `DirectionProvider` El componente gestiona el estado de dirección:

```tsx
import { useDirection } from "@/components/direction-provider"

function MyComponent() {
  const { direction, setDirection, isRTL } = useDirection()

  return (
    <div className={isRTL ? "text-right" : "text-left"}>
      Current direction: {direction}
    </div>
  )
}
```

### Accediendo a la dirección en CSS

El `dir` El atributo se establece en el `<html>` elemento:

```css
/* CSS selector */
[dir="rtl"] .my-element {
  /* RTL styles */
}

/* Tailwind */
<div className="rtl:flex-row-reverse">
```
