# Kompetenzsystem

Mit dem Skills-System können Sie benutzerdefinierte KI-Eingabeaufforderungen erstellen und verwalten, die mit der Zeit lernen und sich verbessern. Bei Fertigkeiten handelt es sich um wiederverwendbare, forschungsgestützte Vorlagen, die die Best Practices Ihres Projekts, externe Dokumentation und Codebasismuster in gezielten, umsetzbaren Leitfäden für Claude kombinieren.

## Schnellstart

Entdecken Sie verfügbare Skills im Marketplace:

```bash
/agileflow:skill:list
```

Durchsuchen Sie installierte Skills und erhalten Sie Marketplace-Empfehlungen basierend auf Ihrem Tech-Stack.

Erhalten Sie personalisierte Empfehlungen:

```bash
/agileflow:skill:recommend
```

## Was sind Fähigkeiten?

A **skill** ist eine strukturierte Eingabeaufforderungsvorlage, die Claude beibringt, wie er bestimmte Aufgaben in Ihrem Projektkontext handhabt. Fähigkeiten sind:

- **Research-backed** – Generiert aus Webdokumenten, lokaler Dokumentation und Ihrer Codebasis
- **Self-improving** - Lernen Sie aus Ihren Korrekturen und verfeinern Sie sie automatisch
- **Cookbook-based** - Mehrere Anwendungsfall-Workflows in einem einzigen Skill
- **Marketplace-powered** - Durchsuchen und installieren Sie Skills aus dem skills.sh-Marktplatz

### Beispielfähigkeiten

Sie können Fähigkeiten entwickeln für:

- **Database operations**: „Supabase Swift“ – So arbeiten Sie mit Supabase in Ihrer iOS-App
- **API patterns**: „REST-API-Clients“ – Best Practices für das API-Integrationsmuster Ihres Projekts
- **Component development**: „Reagieren Sie auf Komponenten mit Zugänglichkeit“ – So erstellen Sie Komponenten nach Ihren Standards
- **Deployment**: „GitHub Actions CI/CD“ – Der Bereitstellungs- und Testworkflow Ihres Teams
- **Domain expertise**: „Maschinelles Lernen mit TensorFlow“ – Best Practices speziell für Ihre ML-Projekte

## Kernkonzepte

### Fähigkeitsstruktur

Jede Fertigkeit enthält:

```
.claude/skills/<skill-name>/
├── SKILL.md              # Main skill file with activation triggers and instructions
├── references.md         # Links to web docs, local docs, and code patterns
├── cookbook/             # Per-use-case workflows
│   ├── use-case-1.md
│   └── use-case-2.md
└── .mcp.json            # Optional MCP server configuration
```

### Wie Fähigkeiten funktionieren

1. **Activation**: Wenn Sie Claude nach einem Thema fragen, das zu den Auslösern einer Fertigkeit passt, wird die Fertigkeit aktiviert
2. **Execution**: Claude liest SKILL.md und folgt den Anweisungen
3. **Learning**: Wenn Sie Claude korrigieren, lernt der Skill Ihre Vorlieben
4. **Evolution**: Bei jeder Korrektur wird die Lerndatei aktualisiert, wodurch der Skill intelligenter wird

### Lernsystem

Fähigkeiten können durch ein Lernsystem automatisch verbessert werden:

```
.agileflow/skills/_learnings/
├── supabase-swift.yaml
├── api-client.yaml
└── react-components.yaml
```

Jede Lerndatei verfolgt Folgendes:
- **Preferences** - Was Benutzer bevorzugen (Formatierung, Stil, Ansatz)
- **Conventions** - Projektspezifische Muster und Standards
- **Anti-patterns** - Was Sie vermeiden sollten
- **Examples** - Gute und schlechte Beispiele aus Korrekturen

## Das Lernsystem

### Welche Fähigkeiten lernen

Wenn Sie eine Fertigkeit verwenden und Claudes Ausgabe korrigieren, lernt die Fertigkeit Folgendes:

1. **Explicit corrections** - Hohes Vertrauen beim Lernen durch direktes Feedback
2. **Approved responses** - Mittleres Vertrauen in die Erkenntnisse aus Genehmigungen
3. **Patterns** - Beobachtungen mit geringem Vertrauen zur Überprüfung

### Dateiformat lernen

Für jede Fertigkeit gibt es eine optionale Lerndatei unter `.agileflow/skills/_learnings/<skill>.yaml`:

```yaml
skill: supabase-swift
version: 1
last_updated: 2025-01-08T10:30:00Z

preferences:
  code_style: "swift-concurrency"
  error_handling: "Result<T, Error>"
  async_patterns: "async/await"

conventions:
  naming: "camelCase for variables, PascalCase for types"
  structure: "Separate data, service, and UI layers"
  testing: "Unit tests required for all public APIs"

anti_patterns:
  - "Avoid callback patterns (use async/await instead)"
  - "Never hardcode API keys in code"

examples:
  good:
    - "let user = try await supabase.auth.user()"
  bad:
    - "var apiKey = 'sk-...' // Never do this"

metadata:
  corrections_count: 12
  approvals_count: 8
  last_correction: 2025-01-08T10:30:00Z
```

### Wie Selbstverbesserung funktioniert

**On invocation:**
1. Claude prüft, ob eine Lerndatei vorhanden ist
2. Liest Präferenzen, Konventionen und Anti-Muster
3. Wendet sie beim Ausführen der Fertigkeit an

**On correction:**
1. Sie geben Feedback oder Korrekturen
2. Claude extrahiert das Signal (was zu ändern ist)
3. Bestimmt das Konfidenzniveau (hoch/mittel/niedrig)
4. Aktualisiert die Lerndatei
5. Beim nächsten Aufruf werden die aktualisierten Einstellungen verwendet

## Best Practices

### Effektive Fähigkeiten schaffen

1. **Use specific names** - `supabase-swift` nicht nur `database`
2. **Include multiple use cases** - Fügen Sie Kochbucheinträge für verschiedene Szenarien hinzu
3. **Link to project patterns** - Verweisen Sie auf Ihre Codebasiskonventionen
4. **Add troubleshooting** - Häufige Probleme antizipieren und angehen
5. **Keep it focused** - Eine Domäne pro Fähigkeit, nicht zu breit

### Fähigkeiten erhalten

1. **Test regularly** - Überprüfen Sie regelmäßig die Qualität Ihrer Fähigkeiten
2. **Review learnings** - Überprüfen Sie, was der Skill aus Korrekturen gelernt hat
3. **Update references** - Halten Sie die Dokumentationslinks aktuell
4. **Archive unused skills** - Bereinigen Sie Fähigkeiten, die Sie nicht mehr verwenden
5. **Share improvements** - Übertragen Sie die Erkenntnisse auf Git, um die Teamkonsistenz zu gewährleisten

### Organisationsfähigkeiten

**Suggested skill categories:**

- **Data layer** - Datenbankoperationen, Migrationen, Abfragen
- **API layer** - REST-Clients, GraphQL, Webhooks, Authentifizierung
- **UI layer** - Komponenten, Styling, Zugänglichkeit, Animationen
- **Infrastructure** - Bereitstellung, CI/CD, Überwachung, Sicherheit
- **Workflows** - Testen, Codeüberprüfung, Debugging, Profilerstellung
- **Domain** - ML-Modelle, Bildverarbeitung, Zahlungsabwicklung

## Kochbuch-Workflows

### Was ist ein Kochbuch?

Ein Kochbucheintrag ist eine Schritt-für-Schritt-Anleitung für einen bestimmten Anwendungsfall innerhalb eines Skills.

### Kochbuchvorlage

```markdown
# Use Case Title

Brief description of this workflow.

## Prerequisites

- Prerequisite 1
- Prerequisite 2

## Instructions

1. **Step 1**: Description with example
   ```language
   Codebeispiel
   ```

2. **Step 2**: Next step
   ```language
   Codebeispiel
   ```

## Example

**Input**: Sample input or scenario
**Output**: Expected output

## Troubleshooting

- **Problem**: Solution
```

### Wann Kochbücher hinzugefügt werden sollten

Fügen Sie Kochbucheinträge hinzu, wenn:
- Es gibt mehrere häufige Anwendungsfälle
- Es gelten unterschiedliche Arbeitsabläufe
- Die Einrichtung unterscheidet sich je nach Szenario
- Benutzer fragen häufig nach einer Schritt-für-Schritt-Anleitung

## Skill-Lebenszyklus

### Entdeckung

1. Ausführen `/agileflow:skill:list` um den Marketplace zu durchsuchen
2. Ausführen `/agileflow:skill:recommend` für personalisierte Vorschläge
3. Skills aus dem skills.sh-Marktplatz installieren
4. Die Fertigkeit überprüfen

### Verwendung

1. Arbeiten Sie an Aufgaben, die zur Domäne der Fertigkeit passen
2. Claude aktiviert die Fertigkeit automatisch
3. Skill provides guidance and examples
4. Sie führen die Empfehlungen des Skills aus

### Verbesserung

1. Wenn Claudes Ausgabe etwas übersieht, korrigieren Sie es
2. Geschicklichkeit lernt aus der Korrektur
3. Das nächste Mal bietet Geschicklichkeit eine bessere Orientierung

### Evolution

1. Mit der Zeit werden die Fähigkeiten individueller
2. Die Lerndatei erfasst projektspezifische Muster
3. Neue Teammitglieder erben das Wissen
4. Fähigkeiten werden zum institutionellen Gedächtnis

## Fähigkeiten teilen

### Exportieren Sie einen Skill

Kopieren Sie ein Skill-Verzeichnis zum Teilen:

```bash
cp -r .claude/skills/my-skill /path/to/share/
```

### Importieren Sie eine Fertigkeit

Kopieren Sie in Ihr Kompetenzverzeichnis:

```bash
cp -r /path/to/skill .claude/skills/
```

### Versionskontrolle

Übertragen Sie Fähigkeiten auf Git (einschließlich Erkenntnisse):

```bash
git add .claude/skills/
git add .agileflow/skills/_learnings/
git commit -m "Add supabase-swift skill with learnings"
```

Team members automatically get the skill and learnings.

## Fehlerbehebung

### Fertigkeit wird nicht aktiviert

**Problem**: Du fragst nach dem Thema des Skills, aber Claude aktiviert ihn nicht.

**Solution**:
1. Sehen Sie sich den Abschnitt „Wann zu verwenden” in SKILL.md an
2. Stellen Sie sicher, dass Ihre Frage mit den Aktivierungsauslösern übereinstimmt
3. Bearbeiten Sie den Skill, um bei Bedarf weitere Schlüsselwörter hinzuzufügen

### Lerndatei fehlt

**Problem**: Die Fähigkeit verbessert sich auch nach Korrekturen nicht.

**Solution**:
1. Erstellen Sie manuell eine Lerndatei unter `.agileflow/skills/_learnings/<skill>.yaml`
2. Zukünftige Korrekturen werden nachverfolgt

## Erweiterte Nutzung

### Domänenspezifische Skills finden

Durchsuchen Sie den Marketplace nach spezialisierten Skills für komplexe Domänen:

```bash
# Marketplace-Empfehlungen für Ihr Projekt anzeigen
/agileflow:skill:recommend
```

### Fertigkeitskomposition

Verweisen Sie auf andere Fertigkeiten innerhalb einer Fertigkeit:

In SKILL.md:
```markdown
## Related Skills

This skill works best with:
- [REST API clients](/skills/api-client)
- [Error handling patterns](/skills/error-handling)
```

### Anpassen von Erkenntnissen

Bearbeiten Sie die Lerndatei manuell für eine erweiterte Anpassung:

```bash
.agileflow/skills/_learnings/my-skill.yaml
```

Fügen Sie projektspezifische Konventionen hinzu, die keine Korrekturen erfordern.

1. **Browse the marketplace** - Verwenden Sie `/agileflow:skill:list` um verfügbare Skills zu entdecken
2. **Get recommendations** - Verwenden Sie `/agileflow:skill:recommend` für personalisierte Vorschläge
3. **Use it** - Arbeiten Sie an Aufgaben und lassen Sie die Fähigkeit aktivieren
4. **Improve it** - Korrigieren Sie Claudes Ausgabe, um die Fertigkeit zu erlernen

Fähigkeiten sind das institutionelle Gedächtnis Ihres Projekts. Im Laufe der Zeit verschlüsseln sie Ihre Best Practices und Ihr projektspezifisches Wissen, sodass Claude Sie effektiver beim Aufbau unterstützen kann.
