# Caractéristiques

AgileFlow inclut des fonctionnalités puissantes pour protéger votre base de code, automatiser les flux de travail et maintenir la productivité de votre équipe.

## Fonctionnalités principales

### Système de compétences

Créez, gérez et améliorez continuellement les invites d'IA personnalisées qui tirent parti de vos commentaires.

- **Création basée sur la recherche** - Généré à partir de documents Web, de documentation locale et de votre base de code
- **Flux de travail des livres de recettes** - Plusieurs guides spécifiques à des cas d'utilisation dans une seule compétence
- **Auto-amélioration** - Les compétences apprennent de vos corrections et s'affinent automatiquement
- **Intégration MCP** - Connectez-vous aux serveurs Model Context Protocol pour accéder aux outils
- **Partageable en équipe** - S'engager à git et partager les connaissances institutionnelles

[En savoir plus sur les compétences](/docs/features/skills)

### Système d'expertise des agents

Activez des agents auto-améliorés qui maintiennent la connaissance du domaine et accélèrent l’exécution des tâches.

- **Fichiers d'expertise** - Les agents maintiennent des modèles mentaux de leur domaine
- **Flux de travail en trois étapes** - Planifier (avec expertise) → Construire → Auto-amélioration
- **Apprentissage automatique** - EMises à jour de l'expertise au fur et à mesure que les agents travaillent et apportent des modifications
- **Validation et mesures** - Outils pour garantir que l'expertise reste précise et à jour
- **Domaines ciblés** - Connaissances spécialisées en matière de bases de données, d'API, d'interface utilisateur, de tests, de CI/CD, etc.

[En savoir plus sur l'expertise des agents](/docs/features/agent-expertise-system)

### Sessions parallèles

Travaillez simultanément sur plusieurs fonctionnalités, bugs ou expériences avec des espaces de travail isolés et une protection des limites.

- **Git worktrees** - Chaque session obtient son propre répertoire et sa propre branche
- **Protection des limites** - Les hooks d'édition/écriture empêchent les modifications accidentelles entre sessions
- **Changement facile** - Utilisez `/add-dir` pour changer de session dans le même terminal
- **Suivi de session** - AgileFlow suit toutes les sessions avec leur statut et leur activité
- **Contexte indépendant** - Chaque session conserve sa propre histoire et sa propre progression

[En savoir plus sur les sessions parallèles](/docs/features/parallel-sessions)

## Automatisation

### Mode boucle

Exécutez Claude de manière autonome avec une exécution continue des tâches jusqu'à leur achèvement.

- **Exécution autonome** - Claude travaille à travers des tâches sans interruption
- **Limites configurables** - Définir les limites d'itération et les délais d'attente
- **Suivi des progrès** - Découvrez ce qui se passe à chaque étape
- **Replis sécurisés** - Les contrôles de sécurité intégrés empêchent les boucles incontrôlables

[En savoir plus sur le mode boucle](/docs/features/loop-mode)

### Mode visuel

Laissez Claude voir et interagir avec les interfaces visuelles via des captures d'écran et l'automatisation du navigateur.

- **Analyse de capture d'écran** - Claude peut analyser le contenu visuel
- **Automatisation du navigateur** - Intégration de Playwright pour les tests Web
- **Vérification de l'interface utilisateur** - Valider automatiquement les modifications visuelles
- **Multiplateforme** - Fonctionne sur n'importe quel système d'exploitation avec prise en charge du navigateur

[En savoir plus sur le mode visuel](/docs/features/visual-mode)

### Préservation du contexte compact

Préservez les connaissances et l’état critiques des agents pendant le compactage automatique du contexte.

- **Compactage contextuel** - Règles critiques préservées lorsque Claude Code compresse le contexte
- **Préservation basée sur les priorités** - Agents critiques toujours préservés
- **Continuité de l'état** - État actuelry, bloqueurs et progrès maintenus
- **Support pour les conversations longues** - Concentration soutenue de l'agent pendant les journées de travail
- **Zéro surcharge** - Automatique, aucune configuration manuelle nécessaire

[En savoir plus sur le contexte compact](/docs/features/compact-context)

## Configuration et sécurité

### Système de configuration

9 agents spécialisés pour mettre en place et personnaliser votre workflow de développement.

- **Configuration Git** - Initialiser le référentiel et les connexions à distance
- **Système Hooks** - Automatisation basée sur les événements (SessionStart, PreCompact, Stop)
- **Ligne d'état** - Barre d'état personnalisée dans Claude Code
- **Archivage** - Gérer automatiquement la taille du fichier status.json
- **CI/CD** - Tests automatisés avec GitHub Actions, GitLab, CircleCI
- **Attribution** - Contrôler la divulgation de l'IA dans les commits git
- **Et plus** - Contexte PreCompact, vérification, gestion des jetons

[En savoir plus sur la configuration](/docs/features/configuration)

### Intégrations IDE

Installez et configurez AgileFlow pour votre IDE préféré avec l'installation automatique des commandes, des agents et des compétences.

- **Plusieurs IDE** - Claude Code, Curseur, Planche à voile, OpenAI Codex CLI
- **Détection automatique** - Détectez quels IDE sont configurés dans votre projet
- **Damage Control** - Système de sécurité en option pour Claude Code
- **Contenu dynamique** - Commandes et agents injectés automatiquement lors de l'installation
- **Prise en charge multi-IDE** - Utilisez le même projet sur plusieurs IDE

[En savoir plus sur les intégrations IDE](/docs/features/ide-integrations)

### Contrôle des dégâts

Protégez votre base de code des commandes destructrices à l'aide des hooks PreToolUse.

- **Validation de la commande Bash** - Bloque les modèles dangereux comme `rm -rf /`, `git push --force`
- **Contrôle d'accès aux fichiers** - Protégez les secrets, les clés et les fichiers critiques
- **Règles personnalisables** - Ajoutez des modèles de protection spécifiques au projet
- **Mode Demander en premier** - Exiger une confirmation pour les opérations à risque
- **Trois couches de protection** - Validation des outils Bash, Edit et Write

[En savoir plus sur le contrôle des dommages](/docs/features/damage-control)

## Référence rapide

| Fonctionnalité | Objectif | Commencer |
|---------|---------|-------------|
| **Compétences** | Cdes invites d'IA personnalisées qui apprennent | `/agileflow:skill:create` |
| **Expertise des agents** | Connaissance du domaine auto-améliorée | Intégré |
| **Séances parallèles** | Travailler sur plusieurs fonctionnalités | `/agileflow:session:new` |
| **Mode boucle** | Exécution autonome des tâches | `/agileflow:loop` |
| **Mode visuel** | Capture d'écran et automatisation du navigateur | `/agileflow:visual` |
| **Contexte compact** | Préserver l'état grâce à la compression | Intégré |
| **Configuration** | Configurer et personnaliser le flux de travail | `/agileflow:configure` |
| **Intégrations IDE** | Prise en charge multi-IDE | `npx agileflow setup` |
| **Contrôle des dommages** | Bloquer les commandes dangereuses | `/agileflow:configure damagecontrol` |

## Prochaines étapes

1. **Commencez par la configuration** - Exécutez `/agileflow:configure` pour configurer votre projet
2. **Activer la sécurité** - Configurez Damage Control pour protéger votre base de code
3. **Apprendre des compétences** - Créez des invites personnalisées pour votre flux de travail
4. **Essayez les sessions parallèles** - Travaillez sur plusieurs fonctionnalités simultanément
