# دعم لغة RTL

تدعم وثائق AgileFlow اللغات ذات الاتجاه من اليمين إلى اليسار (RTL) بما في ذلك العربية والعبرية والفارسية والأردية. يغطي هذا الدليل كيفية استخدام دعم RTL وكيفية المساهمة في الترجمات.

## استخدام وضع RTL

### محول اللغة

انقر على أيقونة اللغة/الكرة الأرضية في رأس الموقع للتبديل بين الوضعين LTR وRTL. سيتم تعديل الموقع تلقائيًا:

- اتجاه النص
- النسخ المتطابق للتخطيط
- محاذاة الملاحة
- تحديد موضع المكونات

### الكشف التلقائي

إذا تم ضبط متصفحك على لغة RTL، فسيبدأ الموقع تلقائيًا في وضع RTL. لغات RTL المدعومة:

- العربية (ar)
- العبرية (هو)
- الفارسي/الفارسية (fa)
- الأردية (اور)

## RTL-Aware CSS

يستخدم الموقع Tailwind CSS مع متغيرات RTL. استخدم `rtl:` بادئة للأنماط الخاصة بـ RTL:

```css
/* Example RTL styles */
.my-element {
  margin-left: 1rem;   /* LTR default */
}

[dir="rtl"] .my-element {
  margin-left: 0;
  margin-right: 1rem;  /* RTL override */
}

/* Using Tailwind rtl: variant */
<div className="ml-4 rtl:ml-0 rtl:mr-4">
  Content
</div>
```

### الخصائص المنطقية

في معظم الحالات، تفضل خصائص CSS المنطقية التي تتعامل مع الاتجاه تلقائيًا:

```css
/* Instead of this */
margin-left: 1rem;
padding-right: 2rem;
text-align: left;

/* Use this */
margin-inline-start: 1rem;
padding-inline-end: 2rem;
text-align: start;
```

معادلات الرياح الخلفية:
- `ms-*` = هامش-مضمن-بداية
- `me-*` = هامش مضمن نهاية
- `ps-*` = الحشو مضمنة البداية
- `pe-*` = الحشو مضمنة النهاية
- `start-*` = أقحم مضمنة البداية
- `end-*` = أقحم مضمنة نهاية

## سير عمل الترجمة

### إضافة لغة جديدة

1. إنشاء ملفات الترجمة في `/locales/{lang}/`:

```
locales/
├── en/
│   ├── common.json
│   └── docs.json
├── ar/
│   ├── common.json
│   └── docs.json
└── he/
    ├── common.json
    └── docs.json
```

2. أضف اللغة إلى `LANGUAGES` في `components/language-switcher.tsx`:

```typescript
const LANGUAGES: Language[] = [
  { code: "en", name: "English", nativeName: "English", direction: "ltr" },
  { code: "ar", name: "Arabic", nativeName: "العربية", direction: "rtl" },
  // Add new language here
]
```

3. إنشاء ملفات MDX مترجمة:

```
content/docs/
├── index.mdx           # English (default)
└── ar/
    └── index.mdx       # Arabic translation
```

### إرشادات الترجمة

1. **Preserve formatting**: حافظ على كتل التعليمات البرمجية والروابط والمكونات سليمة
2. **Translate alt text**: يجب ترجمة السمات البديلة للصورة
3. **Keep URLs**: لا تترجم الارتباطات الثابتة لعناوين URL إلا في حالة الضرورة القصوى
4. **Review technical terms**: قد تبقى بعض المصطلحات الفنية باللغة الإنجليزية

### عينة من محتوى RTL

فيما يلي نموذج للنص العربي لاختبار تخطيط RTL:

```arabic
مرحبًا بكم في توثيق AgileFlow. هذا النص باللغة العربية لاختبار دعم اللغات من اليمين إلى اليسار.
```

```hebrew
ברוכים הבאים לתיעוד AgileFlow. זהו טקסט בעברית לבדיקת תמיכה בשפות מימין לשמאל.
```

## اختبار تخطيط RTL

1. قم بالتبديل إلى وضع RTL باستخدام محوّل اللغة
2. تحقق من هذه المناطق للحصول على النسخ المتطابق الصحيح:
   - الشريط الجانبي للتنقل (يجب أن يكون على اليمين)
   - محاذاة أيقونة البحث
   - اتجاه تمرير كتلة التعليمات البرمجية
   - أشرطة التقدم
   - الرموز (بعضها يجب أن ينقلب والبعض الآخر لا ينبغي)

### الرموز التي لا ينبغي أن تقلب

- أيقونة البحث
- أيقونة الرابط الخارجي
- علامة اختيار
- زائد/ناقص
- إغلاق (X)

### الرموز التي يجب أن تقلب

- السهم لليسار / لليمين
- شيفرون يسار / يمين
- مسافة بادئة/مسافة بادئة
- الرد / الأمام

## التنفيذ الفني

### DirectionProvider

ال `DirectionProvider` يدير المكون حالة الاتجاه:

```tsx
import { useDirection } from "@/components/direction-provider"

function MyComponent() {
  const { direction, setDirection, isRTL } = useDirection()

  return (
    <div className={isRTL ? "text-right" : "text-left"}>
      Current direction: {direction}
    </div>
  )
}
```

### الوصول إلى الاتجاه في CSS

ال `dir` تم تعيين السمة على `<html>` عنصر:

```css
/* CSS selector */
[dir="rtl"] .my-element {
  /* RTL styles */
}

/* Tailwind */
<div className="rtl:flex-row-reverse">
```
