## نظرة عامة

يجب أن يحتوي كل مكون على:
- **جدول الدعائم** - يتم إنشاؤه تلقائيًا من أنواع TypeScript
- **بيان الاستيراد** - كيفية استيراد المكون
- **مثال الاستخدام** - مثال التعليمات البرمجية الأساسية
- **قائمة التحقق من إمكانية الوصول** - حالة الامتثال لـ WCAG
- **زر النسخ** - نسخ المكونات بنقرة واحدة

## اختبارات إمكانية الوصول القياسية

استخدم هذه الفحوصات الشائعة للتأكد من الاتساق:

### المكونات التفاعلية
- يمكن الوصول إلى لوحة المفاتيح (إدخال / مسافة)
- التركيز مرئيا
- التركيز بترتيب منطقي
- المس الهدف >= 44x44 بكسل

### مكونات النموذج
- التسميات المرتبطة
- أعلنت رسائل الخطأ
- الحقول المطلوبة موضحة
- تم نقل حالة غير صالحة

### مكونات المحتوى
- تباين الألوان >= 4.5:1
- يمكن تغيير حجم النص
- موتأيون قابل للتخفيض
- النص البديل موجود

## أفضل الممارسات

1. **حافظ على الحد الأدنى من الدعائم** - قم بتوثيق دعائم واجهة برمجة التطبيقات العامة فقط
2. **استخدم أنواعًا دقيقة** - انسخ الأنواع من كود المصدر
3. **اختبار إمكانية الوصول** - تحقق من المطالبات قبل وضع علامة المرور
4. **التحديث بانتظام** - حافظ على مزامنة المواصفات مع التعليمات البرمجية
5. **رابط إلى المصدر** - قم بالإشارة إلى مصدر GitHub عندما يكون ذلك ممكنًا

## دليل الهجرة

لتطبيق هذا القالب على المستندات الموجودة:

1. قم باستيراد `ComponentSpec` أعلى ملف MDX
2. استخراج الدعائم من مكون TypeScript
3. أضف مثال استخدام من التعليمات البرمجية الموجودة
4. اركضتدقيق إمكانية الوصول
5. استبدل الوثائق القديمة بـ `<ComponentSpec />`

راجع [Button](/components/button)، [Card](/components/card)، [Input](/components/input)، [Dialog](/components/dialog)، و[Select](/components/select) للحصول على أمثلة مباشرة.
