# Suporte ao idioma RTL

A documentação do AgileFlow oferece suporte a idiomas da direita para a esquerda (RTL), incluindo árabe, hebraico, persa e urdu. Este guia aborda como usar o suporte RTL e como contribuir com traduções.

## Usando o modo RTL

### Alternador de idioma

Clique no ícone de idioma/globo no cabeçalho do site para alternar entre os modos LTR e RTL. O site ajustará automaticamente:

- Direção do texto
- Espelhamento de layout
- Alinhamento de navegação
- Posicionamento de componentes

### Detecção Automática

Se o seu navegador estiver configurado para um idioma RTL, o site iniciará automaticamente no modo RTL. Idiomas RTL suportados:

- Árabe (ar)
- Hebraico (ele)
- Persa/farsi (fa)
- Urdu (você)

## CSS compatível com RTL

O site usa Tailwind CSS com variantes RTL. Use o `rtl:` prefixo para estilos específicos de RTL:

```css
/* Example RTL styles */
.my-element {
  margin-left: 1rem;   /* LTR default */
}

[dir="rtl"] .my-element {
  margin-left: 0;
  margin-right: 1rem;  /* RTL override */
}

/* Using Tailwind rtl: variant */
<div className="ml-4 rtl:ml-0 rtl:mr-4">
  Content
</div>
```

### Propriedades Lógicas

Na maioria dos casos, prefira propriedades lógicas CSS que controlam automaticamente a direção:

```css
/* Instead of this */
margin-left: 1rem;
padding-right: 2rem;
text-align: left;

/* Use this */
margin-inline-start: 1rem;
padding-inline-end: 2rem;
text-align: start;
```

Equivalentes do vento favorável:
- `ms-*` = margem-inline-início
- `me-*` = margem-inline-end
- `ps-*` = preenchimento-inline-start
- `pe-*` = preenchimento-inline-end
- `start-*` = inserção-inline-start
- `end-*` = inserção-inline-end

## Fluxo de trabalho de tradução

### Adicionando um novo idioma

1. Crie arquivos de tradução em `/locales/{lang}/`:

```
locales/
├── en/
│   ├── common.json
│   └── docs.json
├── ar/
│   ├── common.json
│   └── docs.json
└── he/
    ├── common.json
    └── docs.json
```

2. Adicione o idioma ao `LANGUAGES` em `components/language-switcher.tsx`:

```typescript
const LANGUAGES: Language[] = [
  { code: "en", name: "English", nativeName: "English", direction: "ltr" },
  { code: "ar", name: "Arabic", nativeName: "العربية", direction: "rtl" },
  // Add new language here
]
```

3. Crie arquivos MDX traduzidos:

```
content/docs/
├── index.mdx           # English (default)
└── ar/
    └── index.mdx       # Arabic translation
```

### Diretrizes de Tradução

1. **Preserve formatting**: Mantenha blocos de código, links e componentes intactos
2. **Translate alt text**: Os atributos alternativos da imagem devem ser traduzidos
3. **Keep URLs**: Não traduza slugs de URL, a menos que seja absolutamente necessário
4. **Review technical terms**: Alguns termos técnicos podem permanecer em inglês

## Testando layout RTL

1. Alterne para o modo RTL usando o seletor de idioma
2. Verifique estas áreas para espelhamento correto:
   - Barra lateral de navegação (deve estar à direita)
   - Alinhamento do ícone de pesquisa
   - Direção de rolagem do bloco de código
   - Barras de progresso
   - Ícones (alguns deveriam virar, outros não)

### Ícones que NÃO devem virar

- Ícone de pesquisa
- Ícone de link externo
- Marca de seleção
- Mais/menos
- Fechar (X)

### Ícones que DEVEM virar

- Seta para esquerda/direita
- Chevron esquerda/direita
- Recuar/recuar
- Responder/encaminhar

## Implementação Técnica

### Provedor de direção

O `DirectionProvider` componente gerencia o estado de direção:

```tsx
import { useDirection } from "@/components/direction-provider"

function MyComponent() {
  const { direction, setDirection, isRTL } = useDirection()

  return (
    <div className={isRTL ? "text-right" : "text-left"}>
      Current direction: {direction}
    </div>
  )
}
```

### Acessando a direção em CSS

O `dir` atributo está definido no `<html>` elemento:

```css
/* CSS selector */
[dir="rtl"] .my-element {
  /* RTL styles */
}

/* Tailwind */
<div className="rtl:flex-row-reverse">
```
